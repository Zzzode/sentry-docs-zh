认证令牌（_authentication tokens_ 的缩写）是与 Sentry 进行身份验证的一种方式。它们类似于密码，但专为与 Sentry 的程序化交互而设计。以下是一些使用认证令牌的示例：

- 在 CI 构建期间上传 Source Maps
- 使用 [Sentry CLI](/cli/) 与 Sentry 交互
- 使用 [Sentry API](/api/auth/)

每个认证令牌都创建时都会分配一组特定的权限和作用域，这些权限和作用域映射到 Sentry API 的[权限和作用域](/api/permissions/)。某些类型的认证令牌具有无法编辑的固定权限，而其他类型则可以在创建令牌时自定义权限，并在之后进行编辑。

我们建议为每个用例使用单独的认证令牌。例如，您应该使用与 Sentry CLI 不同的认证令牌来上传 source maps。这样做的好处是，如果某个认证令牌被泄露，您可以撤销该令牌而不会影响其余的工作流程。

## 认证令牌的类型

Sentry 中有三种主要的认证令牌类型：

- [组织令牌](#organization-tokens)：
  这些令牌绑定到组织，可以访问该组织内的所有项目。它们具有有限的权限集，专为在 CI 环境和 Sentry CLI 中使用而设计。

- [内部集成](#internal-integrations)：
  这些令牌绑定到组织，可以访问该组织内的所有项目。它们可以使用自定义权限集创建，专为在组织令牌权限不足的情况下使用而设计。

- [个人令牌](#personal-tokens)：
  这些令牌绑定到用户，可以访问该用户有权访问的所有组织和项目。

### 我应该使用哪种令牌？

对于大多数场景，我们建议使用[组织令牌](#organization-tokens)。它们专为在 CI 环境中使用而设计，具有有限的权限集。这意味着如果您存储认证令牌的地方被泄露，攻击者只能造成有限的损害。

组织令牌的权限不可自定义。它们被设置为允许大多数与 CI 相关的任务，而没有任何不必要的权限。

当您需要完整的 API 访问权限（组织令牌无法授予）并且希望代表组织与 Sentry API 交互时，应使用[内部集成](#internal-integrations)。例如，要以编程方式创建新项目，您将使用内部集成。

作为内部集成一部分创建的认证令牌的权限是可自定义和可编辑的。

[个人令牌](#personal-tokens)应用于代表用户与 Sentry API 交互。例如，要获取用户的所有问题，您将使用个人令牌。我们不建议将个人令牌用于 CI 任务，因为如果创建令牌的用户从组织中移除，该令牌将停止工作。

个人令牌的权限是可自定义的，但之后无法编辑。

### 组织令牌

[组织令牌](https://sentry.io/orgredirect/organizations/:orgslug/settings/auth-tokens/)可以在 [sentry.io](https://sentry.io) 的 **设置 > 开发者设置 > 组织令牌** 页面上的 **认证令牌** 页面创建。

![](./img/org-auth-tokens-overview.png)

<Alert>

如果您已登录，它们也可以在 Sentry 文档的某些页面上生成，并通过使用 Sentry Wizard 配置上传 source maps 来生成。

</Alert>

除非您通过 Sentry UI 创建令牌，否则组织令牌的名称是自动生成的。此名称仅用于显示目的 - 它有助于识别认证令牌，以防您稍后想要撤销它。您可以在 [sentry.io](https://sentry.io) 的 **设置 > 开发者设置 > 组织令牌** 页面上的 **编辑认证令牌** 页面更改组织令牌的名称。

出于安全原因，组织令牌仅在创建后立即可见_一次_。如果您丢失了认证令牌，您将不得不创建一个新的。这意味着您无法在概览页面或令牌详情页面上看到完整的令牌，您只能看到令牌的最后几个字符以帮助识别它。

任何用户都可以为其任何组织创建组织令牌。这允许任何用户（不仅仅是组织所有者）通过创建和使用组织令牌来配置 Sentry SDK 和设置 CI 流程。由于组织令牌具有有限的访问权限，因此滥用的可能性有限。

![](./img/org-auth-token-create.png)

组织的所有所有者将在创建新的组织令牌时收到安全电子邮件，并可以随时撤销这些令牌。_只有_组织所有者和管理员可以撤销组织令牌。

### 内部集成

我们仅建议在您希望代表组织与 Sentry API 交互时使用内部集成。有关如何执行此操作的更多信息，请参阅 Sentry 的 API [认证](/api/auth/) 文档。

要开始使用，请参阅我们关于[内部集成](/organization/integrations/integration-platform/)的文档。

### 个人令牌

[个人令牌](https://sentry.io/settings/account/api/auth-tokens/)可以在 [sentry.io](https://sentry.io) 的左上角帐户下拉菜单中的 **个人令牌** 页面创建。

![](./img/user-auth-tokens-menu.png)

个人令牌可以由任何用户创建，并绑定到该用户。令牌可以被授予该用户有权访问的所有组织和项目的权限。这意味着个人令牌的_最大_作用域是用户对于给定组织具有访问权限的所有作用域。个人令牌不能超过创建它的用户的权限。有关 Sentry 中权限如何工作的更多详细信息，请参阅[组织和用户管理](/organization/membership/)。

创建新的个人令牌时，您可以在 UI 中选择令牌应具有的[作用域](/api/permissions/)：

![](./img/user-auth-token-create.png)

目前，您可以在创建后在 UI 中查看个人令牌。这是一种可能在未来更改的遗留行为。我们建议每个个人令牌仅使用一次，并为每个用例创建一个新的令牌。任何用户始终可以撤销其任何个人令牌。

![](./img/user-auth-tokens-overview.png)

我们建议仅使用个人令牌来代表用户与 Sentry API 交互。有关如何执行此操作的更多信息，请参阅 Sentry 的 API [认证](/api/auth/) 文档。

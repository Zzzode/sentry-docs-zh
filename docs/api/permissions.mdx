---
title: 权限和作用域
sidebar_order: 2
---

如果您正在基于 Sentry 的 API 构建（即使用[认证令牌](/api/auth/)），您将需要某些作用域来访问不同的 API 端点。

要为[集成令牌](/organization/integrations/integration-platform/#permissions)设置作用域，请从下拉菜单中选择作用域。这些可以在以后编辑。

要为个人令牌设置作用域，请在[创建个人令牌](https://sentry.io/api/)时从下拉菜单中选择作用域。这些以后无法编辑。

![](./img/user-auth-token-create.png)

如果您正在寻找有关成员角色的信息，请访问[成员资格](/organization/membership/)文档。

### 组织

|              |             |
| :----------: | :---------: |
|   **GET**    | `org:read`  |
| **PUT/POST** | `org:write` |
|  **DELETE**  | `org:admin` |

### 项目

|              |                 |
| :----------: | :-------------: |
|   **GET**    | `project:read`  |
| **PUT/POST** | `project:write` |
|  **DELETE**  | `project:admin` |

<Alert level="warning" title="注意">

`project:releases` 作用域将使您可以访问**项目**和**组织**发布端点。可用端点在 API 文档的[发布](/api/releases/)部分中列出。

</Alert>

### 团队

|              |              |
| :----------: | :----------: |
|   **GET**    | `team:read`  |
| **PUT/POST** | `team:write` |
|  **DELETE**  | `team:admin` |

### 成员

|              |                |
| :----------: | :------------: |
|   **GET**    | `member:read`  |
| **PUT/POST** | `member:write` |
|  **DELETE**  | `member:admin` |

### 问题和事件

|            |               |
| :--------: | :-----------: |
|  **GET**   | `event:read`  |
|  **PUT**   | `event:write` |
| **DELETE** | `event:admin` |

<Alert level="warning" title="注意">

**PUT/DELETE** 方法仅适用于更新/删除问题。
Sentry 中的事件是不可变的，只能通过删除整个问题来删除。

</Alert>

### 发布

|                         |                    |
| :---------------------: | :----------------: |
| **GET/PUT/POST/DELETE** | `project:releases` |

<Alert level="warning" title="注意">

请注意，如果您使用 `sentry-cli` 来[管理您的发布](/cli/releases/)，您还需要一个具有 `org:read` 作用域的令牌。

</Alert>

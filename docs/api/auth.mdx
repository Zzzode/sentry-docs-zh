## 认证令牌

认证令牌通过 auth 头传递，用于作为用户或组织帐户与 API 进行身份验证。在我们的文档中，我们有一些出现在花括号或尖括号中的占位符，例如 `{API_KEY}` 或 `<auth_token>`，您需要将其替换为您的某个认证令牌才能有效使用 API 调用。

例如，当文档说：

```bash
curl -H 'Authorization: Bearer {TOKEN}' https://sentry.io/api/0/organizations/{organization_slug}/projects/
```

如果您的认证令牌是 `1a2b3c`，并且您的组织 slug 是 `acme`，那么命令应该是：

```bash
curl -H 'Authorization: Bearer 1a2b3c' https://sentry.io/api/0/organizations/acme/projects/
```

您可以通过[创建内部集成](/organization/integrations/integration-platform/#internal-integrations)在 Sentry 中创建认证令牌。这也适用于自托管的 Sentry。

### 用户认证令牌

某些 API 端点需要与您的用户帐户关联的认证令牌，而不是来自内部集成的认证令牌。这些认证令牌可以在 Sentry 的"用户设置"页面（**用户设置 > 个人令牌**）中创建，并分配特定的作用域。

需要用户认证令牌的端点特定于您的用户，例如[检索组织](/api/organizations/retrieve-an-organization/)。

## DSN 认证

某些 API 端点可能允许基于 DSN 的身份验证。这通常非常有限，端点会描述是否支持。这与 Bearer 令牌身份验证类似，但使用您的 DSN（客户端密钥）。

```bash
curl -H 'Authorization: DSN {DSN}' https://sentry.io/api/0/{organization_slug}/{project_slug}/user-reports/
```

## API 密钥

<Alert level="warning" title="注意">

API 密钥是一种传统的身份验证方式。它们仍将被支持，但新帐户已禁用。您应尽可能使用**认证令牌**。

</Alert>

API 密钥使用 HTTP 基本身份验证传递，其中用户名是您的 api 密钥，密码为空值。

例如，要获取有关您的密钥绑定到的项目的信息，您可以发出如下请求：

```bash
curl -u {API_KEY}: https://sentry.io/api/0/organizations/{organization_slug}/projects/
```

<Alert>

您**必须**为密码传递一个值，这就是为什么我们的示例中出现了 `:` 的原因。

</Alert>

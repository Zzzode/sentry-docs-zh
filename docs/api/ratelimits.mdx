---
title: 速率限制
sidebar_order: 3
---

Sentry 对每个发送到 API 的请求进行速率限制，以防止滥用和资源过度使用。该限制应用于调用者和端点的每个唯一组合。

我们限制请求的频率（每秒请求速率限制）和调用者可以同时发出的并发请求数量（并发速率限制）。

每秒请求速率限制遵循固定窗口方法。请求被计数到由窗口大小确定的时间桶中。如果来自调用者的请求数量在窗口内超过允许的数量，则该请求将被拒绝。每个端点都有自己的最大请求数量和窗口大小。

同时，如果调用者同时进行的请求过多，并发速率限制器将拒绝请求。

您可以通过查看响应中的特殊头来跟踪您的速率限制使用情况。

## 头信息

每个 API 请求响应都包括以下头信息：

- `X-Sentry-Rate-Limit-Limit`
  - 窗口内允许的最大请求数
- `X-Sentry-Rate-Limit-Remaining`
  - 此调用者在当前窗口内在此端点上剩余的请求数
- `X-Sentry-Rate-Limit-Reset`
  - 下一个速率限制窗口开始和计数重置的时间，以 UTC 秒为单位从纪元开始测量
- `X-Sentry-Rate-Limit-ConcurrentLimit`
  - 允许的最大并发请求数
- `X-Sentry-Rate-Limit-ConcurrentRemaining`
  - 此调用者剩余的并发请求数

## 附加信息

速率限制器查看调用者的身份，而不是承载令牌或 cookie。因此，无法通过生成多个令牌来绕过速率限制。

<Alert>

轮询 API 进行更新可能会很快触发速率限制。我们建议尽可能使用我们的[webhooks](/organization/integrations/integration-platform/webhooks/)。

</Alert>

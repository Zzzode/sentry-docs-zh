本指南将指导您使用 Sentry 的 API 列出组织中的团队并创建新团队的基础知识。

本教程中使用的 API：

- [列出组织的团队](/api/teams/list-an-organizations-teams/)
- [创建新团队](/api/teams/create-a-new-team/)

## 先决条件

- 具有 team:write 作用域或更高作用域以及 project:read 作用域或更高作用域的 Sentry 认证令牌。

  > 如果您没有认证令牌，请按照[创建 Sentry 认证令牌](/api/guides/create-auth-token)教程创建具有以下内容或更高内容的组织令牌：
  >
  > - 组织：读取
  > - 团队：写入

我们建议为此教程使用免费的 [Sentry 开发者帐户](https://sentry.io/pricing/)。

## 列出组织的团队

首先，您将使用[列出组织的团队](/api/teams/list-an-organizations-teams/) API 列出组织中的所有团队。

### 查找您的组织 ID

列出团队 API 要求您传入组织 ID 以列出团队。

您可以在 Sentry 实例的浏览器 URL 中找到您的组织 ID。例如，https://test-org.sentry.io/。

### 进行 cURL 调用

1. 打开您的终端。

1. 将您的认证令牌和组织 ID 保存为环境变量以便于使用。将以下命令粘贴到您的终端中，将 `<organization_id>` 替换为您的组织 ID，将 `<auth_token>` 替换为您之前复制的认证令牌：

   ```bash
   export SENTRY_ORG_ID=<organization_id>
   export SENTRY_AUTH_TOKEN=<auth_token>
   ```

1. 将以下 cURL 命令粘贴到您的终端中：

   ```bash
   curl "https://sentry.io/api/0/organizations/$SENTRY_ORG_ID/teams/?detailed=0" \
       -H 'Authorization: Bearer $SENTRY_AUTH_TOKEN'
   ```

1. 以下是命令输出的示例，对于具有名为"test-team"的团队的组织：


   输出为您提供有关指定组织内团队的详细信息。

   如果您的组织有足够的团队，API 将返回分页结果。请参阅 Sentry 的[分页](/api/pagination/)文档，了解如何处理分页结果。

1. [可选] 要列出与每个团队关联的项目的详细信息，您可以将 `detailed` 查询参数设置为 `1`：

   ```bash

       -H 'Authorization: Bearer $SENTRY_AUTH_TOKEN'
   ```

## 创建团队

现在您知道了组织中已存在的团队，请使用[创建新团队](/api/teams/create-a-new-team/) API 创建另一个团队。

1. 确保您的认证令牌和组织 ID 仍作为环境变量存储在您的 shell 中。

1. 将以下 cURL 命令粘贴到您的终端中：

   ```bash
   curl -d "name=tutorial-team" -X POST "https://sentry.io/api/0/organizations/$SENTRY_ORG_ID/teams/" \
       -H 'Authorization: Bearer $SENTRY_AUTH_TOKEN'
   ```

   `name` 查询参数意味着创建的团队的名称应为"tutorial-team"。

1. 以下是命令输出的示例：

   ```json
   {
     "id": "4505524553711616",
     "slug": "tutorial-team",
     "name": "tutorial-team",
     "dateCreated": "2023-07-13T22:54:05.074990Z",
     "isMember": false,
     "teamRole": null,
     "flags": {
       "idp:provisioned": false
     },
     "access": [
       "event:write",
       "member:read",
       "alerts:read",
       "project:releases",
       "project:read",
       "event:read",
       "team:read",
       "org:read"
     ],
     "hasAccess": true,
     "isPending": false,
     "memberCount": 0,
     "avatar": {
       "avatarType": "letter_avatar",
       "avatarUuid": null
     },
     "orgRole": null
   }
   ```

1. [可选] 要创建 `slug` 值与 `name` 值不同的团队，您可以在 POST 调用中同时指定两者，如下所示：

   ```bash
   curl -d "name=tutorial-team&slug=tutorial-team-slug" -X POST "https://sentry.io/api/0/organizations/$SENTRY_ORG_ID/teams/" \
       -H 'Authorization: Bearer $SENTRY_AUTH_TOKEN'
   ```

所有 API 请求都应发送到 `/api/0/` 前缀，并将返回 JSON 作为响应：

```bash
curl -i https://sentry.io/api/0/
```

```http
HTTP/1.0 200 OK
Date: Sat, 14 Feb 2015 18:47:20 GMT
Content-Type: application/json
Content-Language: en
Allow: GET, HEAD, OPTIONS

{"version": "0"}
```

## HTTP 动词

Sentry 尝试坚持适当的 HTTP 动词，但我们始终优先考虑可用性而非正确性。

| 方法      | 描述                                                           |
| --------- | --------------------------------------------------------------------- |
| `DELETE`  | 用于删除资源。                                          |
| `GET`     | 用于检索资源。                                        |
| `OPTIONS` | 描述给定的端点。                                         |
| `POST`    | 用于创建资源。                                          |
| `PUT`     | 用于更新资源。尽可能接受部分数据。 |

## 参数和数据

未包含在 URL 中的任何参数都应编码为 JSON，Content-Type 为 `'application/json'`：

```bash
curl -i https://sentry.io/api/0/organizations/acme/projects/1/groups/ \
    -d '{"status": "resolved"}' \
    -H 'Content-Type: application/json'
```

附加参数有时通过查询字符串指定，即使对于 `POST`、`PUT` 和 `DELETE` 请求也是如此：

```bash
curl -i https://sentry.io/api/0/organizations/acme/projects/1/groups/?status=unresolved \
    -d '{"status": "resolved"}' \
    -H 'Content-Type: application/json'
```

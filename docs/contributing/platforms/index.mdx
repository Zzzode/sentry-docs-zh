平台内容位于`docs/platforms`中，并遵循一些特定规则来生成内容。内容使用与所有其他[页面](../)相同的功能进行渲染，但具有一些额外功能来克隆内容并将平台封装为门户。

目录结构在平台目录内具有意义，看起来类似于：

```
[platformName]/
  child.mdx
  childTwo/
    index.mdx
  common/
    aSharedPage.mdx
  guides/
    [guideName]/
      uniqueChild.mdx
      childTwo/
        index.mdx
```

平台将生成一个"指南"列表，该列表继承common中的所有内容。鉴于上述示例，我们最终得到各种半重复的URL：

```
/platforms/platformName/
/platforms/platformName/config.yml
/platforms/platformName/child/
/platforms/platformName/childTwo/
/platforms/platformName/aSharedPage/
/platforms/platformName/guides/guideName/
/platforms/platformName/guides/guideName/config.yml
/platforms/platformName/guides/guideName/child/
/platforms/platformName/guides/guideName/uniqueChild/
/platforms/platformName/guides/guideName/childTwo/
/platforms/platformName/guides/guideName/aSharedPage/
```

这是通过继承`common/`目录中的所有内容，然后添加（或覆盖）兄弟节点（或我们称之为子节点）的内容来生成的。在上面的示例中，您将看到`aSharedPage`在两个不同的URL中加载，而`childTwo`被`guideName`覆盖。

平台页面上的侧边栏（由`<PlatformSidebar>`处理）将生成指南，或基础平台作为主要部分，以及其下方可用的其他指南列表。这意味着所有内容都集中在他们当前所在的指南（通常是框架）上，确保导航的便利性。

## 配置

配置按顺序从`[namespace]/index.mdx`和`[namespace]/config.yml`中读取。这意味着平台的索引可以通过公共页面完全渲染。

有效的配置属性包括：

`title`

平台的显示名称。

`supportLevel`

支持级别。`production`（默认）或`community`

`caseStyle`

代码示例的大小写样式。`canonical`（默认）、`camelCase`、`snake_case`或`PascalCase`

`categories`

未来支持的类别列表。已知值为`browser`、`mobile`、`desktop`和`server`。

`fallbackPlatform`

用于默认值的键。可以使用`platformName`或`platformName.guideName`的完整键语法。

`sdk`

SDK的名称（如果可用）。用于在页面上嵌入SDK信息。

`aliases`

此平台的别名列表。例如，"Cocoa"可能适用于Apple，或"C#"可能适用于.NET。

`showIntegrationsInSearch`

是否在主页的搜索结果中显示集成。

**注意：** 这仅适用于没有指南的平台，如Python。

## 共享内容

SDK（平台）内的共享（重复）内容在`src/platforms/[platformName]/common/`文件夹中可用。

所有这些内容将自动在每个指南中复制。这利用了`PlatformContent`等组件，可以自动替换内容。

### 页面可见性

有时页面在给定指南的上下文中可能没有意义。要控制这一点，您可以在平台的公共页面上使用`supported`和`notSupported`前置事项。

<Alert>

页面可见性的工作方式类似于其他平台组件中的supported/notSupported属性（如`PlatformSection`）。

</Alert>

例如，如果您想默认隐藏平台的内容，但为指南启用它，您可以这样做：

```markdown
---
supported:
  - native.wasm
notSupported:
  - native
---
```

如果您要条件隐藏的页面有子页面，您还需要将`supported`/`notSupported`注释添加到子页面。

## 划分

我们通过语言和开发者交互来区分平台。顶级SDK被视为平台。平台可能具有其支持的框架指南，这些指南通常从父平台继承大部分内容。

一般来说，使用此决策树：

- 它是新的编程语言吗？-> 新平台
- 它是否需要单独的SDK或将数据发送到Sentry的方式？-> 新平台
- 它是否算作"平台"-> 新平台
- 否则 -> 作为指南添加到现有平台

### 示例

JavaScript是顶级SDK，因为它支持的框架共享大部分相同内容。

另一方面，Android和Java的重叠很少。因此，它们都是顶级平台。

## 警报

渲染警报（有时称为标注）以突出显示重要信息。

<Alert title="注意">
  这是一个信息警报。使用它来强调用户应该阅读的信息。
</Alert>

<Alert level="warning" title="重要">
  这是一个警告警报。使用它来提醒用户注意他们应该意识到的潜在风险。
</Alert>

<Alert level="success" title="提示">
  这是一个成功警报。使用它来提供可选信息，可以帮助用户更成功。
</Alert>

<Alert>
  这是一个没有标题的警报。`warning` 警报应该始终有标题以强调其重要性。
</Alert>

<Alert>
  这是一个没有标题的多行警报。保持信息简洁明了。如果这具有挑战性，请考虑该主题是否需要在另一页上记录，或者使用 [可展开](#可展开) 组件是否更合适。
</Alert>

<Alert title="列表">
  <ul>
    <li>您可以创建列表</li>
    <li>并 **格式化** _您的_ `文本`</li>
  </ul>
</Alert>

```markdown {tabTitle:信息}
<Alert title="注意">
这是一个信息警报。
</Alert>
```

```markdown {tabTitle:警告}
<Alert level="warning" title="重要">
这是一个警告警报。
</Alert>
```

```markdown {tabTitle:成功}
<Alert level="success" title="提示">
这是一个成功警报。
</Alert>
```

```markdown {tabTitle:无标题}
<Alert>
这是一个没有标题的信息警报。
</Alert>
```

属性：

- `title` (字符串) - 可选
- `level` (字符串: `'info' | 'warning' | 'success'`) - 可选，默认为 `'info'`

**放置警报的位置：**

确保您的警报不会破坏内容的流程。例如，不要在解释单个概念的两个段落之间放置警报。谨慎使用警报；单个页面上的太多警报可能会让人不知所措。

**何时使用哪种警报级别：**

- **信息**：使用它来强调用户应该阅读的信息。例如：
  - 用户成功所需的信息
  - 您通常会包装在注释标注中的信息
- **警告**：使用它来提醒用户注意他们应该意识到的潜在风险。例如：

  - 关于潜在错误的警告，例如由项目配置中的常见疏忽引起的错误
  - 关于破坏性更改的警告

  最佳实践：

  - 始终使用此警报级别的标题

- **成功**：使用它来提供可选信息，可以帮助用户更成功。例如：
  - 有助于改进已成功任务的信息，例如指向额外阅读材料的链接
  - 最佳实践的提示

</Expandable>

<Expandable title="想要一些改进的提示？" level="success">

</Expandable>


```markdown {tabTitle:永久链接}

  这是一个带有永久链接的可展开部分。
</Expandable>
```

```markdown {tabTitle:警告}

  这是一个 `'warning'` 警报样式的可展开部分。
</Expandable>
```

```markdown {tabTitle:成功}
<Expandable level="success" title="想要一些改进的提示？">
  这是一个 `'success'` 警报样式的可展开部分。
</Expandable>
```

属性：

- `title` (字符串)
- `permalink` (布尔值) - 可选：将标题包装在链接中并在目录中显示。

**何时使用可展开：**

可展开帮助保持我们的文档整洁，允许用户专注于重要信息，同时为他们提供探索额外相关细节的选项。

例如，使用可展开来：

- 提供对某些人有用但不是所有人的信息，如故障排除提示
- 提供背景信息或对相关概念的见解

**最佳实践**

- 将可展开标题写成问题或陈述，以澄清用户可以从内容中期待什么：
  - 您想了解更多关于这些功能的信息吗？
  - 您是否使用Node.js版本15或更低版本？
  - XY用例的高级配置选项。
- 当您需要分享用户应该阅读的重要信息时，请考虑使用 [警报](#警报) 代替，因为其内容默认可见。
- 避免过度使用此组件，并确保提供与当前主题相关的有价值信息。

## 代码块

连续的代码块将自动折叠到选项卡容器中。如果您想用多种语言定义示例，这种行为通常很有用：

````markdown {tabTitle:示例}
```javascript
function foo() {
  return "bar";
}
```

```python
def foo():
  return 'bar'
```
````

有时您可能不希望这种行为。为了解决这个问题，您可以用一些额外的文本分隔代码块，或使用 `<Break />` 组件。

此外，代码块还支持 `tabTitle` 和 `filename` 属性：

````markdown {tabTitle:示例}
```javascript {tabTitle: Hello} {filename: index.js}
var foo = "bar";
```
````

您可以使用 `{fromLineA-toLineB}` 表示范围或 `{lineA,lineB}` 表示特定行（或两者的组合）来突出显示代码块中的特定行：

```javascript {2} {tabTitle:示例}
function foo() {
  let lookat = "me";
  return "bar";
}
```

````markdown {tabTitle:源码}
```javascript {2}
function foo() {
  let lookat = "me";
  return "bar";
}
```
````

您还可以使用 `diff` 语言突出显示差异：

````markdown {tabTitle:示例}
```diff
- minus one
+ plus one
```
````

如果您想保留语法高亮，您可以将 `diff` 元数据添加到任何代码块，然后用 `+` 和 `-` 注释差异：

```javascript {diff} {tabTitle:示例}
function foo() {
-  return "bar";
+  return "baz";
}
```

````markdown {tabTitle:源码}
```javascript {diff}
function foo() {
-  return "bar";
+  return "baz";
}
```
````

## ConfigKey

以给定平台的正确大小写格式渲染带有配置键的标题。

<Alert>

如果指定了内容，当给定 `platform` 未在上下文中选择时，它将自动隐藏内容。

</Alert>

```markdown {tabTitle:示例}
<ConfigKey name="send-default-pii" notSupported={["javascript"]}>

send-default-pii 的描述

</ConfigKey>
```

属性：

- `name` (字符串)
- `platform` (字符串) - 默认为页面上下文中的 `platform` 值
- `supported` (字符串[])
- `notSupported` (字符串[])

## PageGrid

渲染本文档的所有 `next_steps` 或默认子页面，包括其 `description`（如果可用）。

您可以在页面的前端内容中指定 `next_steps` 以将它们包含在网格中。它支持相对路径并将自动解析它们。

```markdown {tabTitle:示例}
---
# 在页面的前端内容中：
next_steps:
  - ./child-one
  - ./child-two
  - ../parent/child-three
---

<PageGrid />
```

属性：

- `header` (字符串) - 可选的标题值，渲染为H2

- `nextPages` (布尔值) - 仅渲染基于侧边栏排序的下一页

- `exclude` (字符串[]) - 要从网格中排除的页面数组。指定页面的文件名，例如，对于 `index.mdx` 的 `"index"`。

## PlatformContent

基于上下文中当前选择的 `platform` 渲染包含。

```markdown {tabTitle:示例}
<PlatformContent includePath="sdk-init" />
```

属性：

- `includePath` (字符串) - 映射到的 `/includes` 中的子文件夹
- `platform` (字符串) - 默认为页面上下文中的 `platform` 值

一些注意事项：

- 当当前平台来自页面上下文且找不到匹配的包含时，内容将被隐藏。

- 类似于 `PlatformSection`，您可以在块中嵌入内容，该内容将在给定包含可用时 _之前_ 渲染。

- 如果没有其他内容匹配，将使用名为 `_default` 的文件。

注意：这目前会导致目录生成问题，因此建议在使用它时禁用TOC。

## PlatformIdentifier

在页面的正文文本（不在代码示例中）中以正确的案例渲染术语，基于平台 case_style 设置：

```markdown {tabTitle:示例}
<PlatformIdentifier name="before-send" />
```

例如，如果您使用 `<PlatformIdentifier name="before-send" />`，它将渲染为：

- `beforeSend` 如果 case_style=camelCase
- `before_send` 如果 case_style=snake_case
- `BeforeSend` 如果 case_style=PascalCase

此组件仅在平台页面中正常工作。

## PlatformLink

用于链接到平台特定内容，当没有特定平台已选择时很有用。

```markdown {tabTitle:示例}
<PlatformLink to="/enriching-events/" />
```

这将引导用户到一个页面，在那里他们可以选择平台，然后到适当的链接。如果他们已经在具有活动平台的页面中，它将简单地链接到适当的页面并跳过重定向。

## PlatformSection

基于上下文中当前选择的 `platform` 渲染部分。当平台无效时，内容将被隐藏。

```markdown {tabTitle:示例}
<PlatformSection notSupported={["javascript"]}>

适用于所有平台但不适用于javascript或node的内容。

</PlatformSection>
```

属性：

- `platform` (字符串) - 默认为页面上下文中的 `platform` 值
- `supported` (字符串[])
- `notSupported` (字符串[])
- `noGuides` (布尔值) - 在所有指南上隐藏此内容（优先于 `supported`/`notSupported`）

## 入门选项

如果您正在编写产品功能特定文档，您可以在代码块中指定标记，启用或禁用代码片段的某些部分：

<Alert title="警告">
  不要复制以下代码片段，`___PRODUCT_OPTION_START___` 和
  `___PRODUCT_OPTION_END___` 标记有一个额外字符（零宽空格）
  作为在示例中正确显示它们的解决方法。
</Alert>

````markdown
```go
  // _​__PRODUCT_OPTION_START___ performance
  // your code here
  // _​__PRODUCT_OPTION_END___ performance
```
````

语法使用您正在记录的编程语言的标准注释样式。例如：

- TypeScript/JavaScript: `// ___PRODUCT_OPTION_START___ feature`
- Python: `# ___PRODUCT_OPTION_START___ feature`

其中 `feature` 是功能id（例如 `performance`、`profiling` 或 `session-replay`）。

范围可见性将由 `OnboardingOptionButtons` 组件控制：

```jsx diff
<OnboardingOptionButtons
  options={["error-monitoring", "profiling", "performance"]}
/>
```

- `options` 可以是以下形状的对象：

```typescript
{
  id: 'error-monitoring' | 'performance' | 'profiling' | 'session-replay',
  disabled: boolean,
  checked: boolean
}
```

或一个字符串（这些 `id` 之一 👆）以方便使用默认值。

<Alert level="warning" title="重要">
  底层实现依赖于代码块中的 `onboardingOptions` 元数据是有效的JSON语法。
</Alert>

- 默认值： `checked: false` 和 `disabled: false`（对于 `error-monitoring` 为 `true`）。

示例（上述输出）：

<OnboardingOptionButtons
  options={["error-monitoring", "performance", "profiling"]}
  dontStick
/>

```go
import (
	"fmt"
	"net/http"

	"github.com/getsentry/sentry-go"
	sentrygin "github.com/getsentry/sentry-go/gin"
	"github.com/gin-gonic/gin"
)

// To initialize Sentry's handler, you need to initialize Sentry itself beforehand
if err := sentry.Init(sentry.ClientOptions{
	Dsn: "___PUBLIC_DSN___",
  // ___PRODUCT_OPTION_START___ performance
	EnableTracing: true,
	// Set TracesSampleRate to 1.0 to capture 100%
	// of transactions for performance monitoring.
	// We recommend adjusting this value in production,
	TracesSampleRate: 1.0,
  // ___PRODUCT_OPTION_END___ performance
  // Adds request headers and IP for users,
  // visit: https://docs.sentry.io/platforms/go/data-management/data-collected/ for more info
  SendDefaultPII: true,
}); err != nil {
	fmt.Printf("Sentry initialization failed: %v\n", err)
}

// Then create your app
app := gin.Default()

// Once it's done, you can attach the handler as one of your middleware

// Set up routes
app.GET("/", func(ctx *gin.Context) {
	ctx.String(http.StatusOK, "Hello world!")
})

// And run it
app.Run(":3000")
```

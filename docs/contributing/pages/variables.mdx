对Markdown和MDX文件都公开了一个转换，支持以Django/Jekyll风格的方式处理变量。
可用的变量是全局范围的，并在 `remark-variables.js` 中配置。

例如：

```markdown {tabTitle:替换}
JavaScript SDK: {{@inject packages.version("sentry.javascript.browser") }}
```

```markdown {tabTitle:源码}
JavaScript SDK: {{\@inject packages.version("sentry.javascript.browser") }}
```

在这种情况下，我们公开 `packages` 作为包注册表中的访问器，这就是为什么有 `packages.version` 函数可用。此外，我们公开了一个默认的上下文变量 `page`，它包含给定markdown节点的前端内容。例如，`{{\@inject page.title }}`。

当变量调用无效（或出错）或不匹配已知范围中的某些内容时，它将在开发模式的输出中产生错误消息。它将强制生产中的构建失败。

要转义表达式，请在 `@inject` 前添加前导 `\`。

## `packages`

`packages` 助手是 `PackageRegistry` 的一个实例，公开了几个方法。

### `packages.version`

返回给定包的最新版本。

```text {tabTitle:源码}
{{\@inject packages.version("sentry.javascript.browser") }}
```

您还可以选择指定一个后备，以防找不到包（或上游有错误）：

```text {tabTitle:源码}
{{\@inject packages.version("sentry.javascript.browser", "1.0.0") }}
```

### `packages.checksum`

返回包中给定文件的校验和。

```text {tabTitle:源码}
{{\@inject packages.checksum("sentry.javascript.browser", "bundle.min.js", "sha384") }}
```

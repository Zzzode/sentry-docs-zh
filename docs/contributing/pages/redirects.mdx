重定向允许您自动将传入的请求路径重定向到新的目标路径。当您移动或重命名文件时，您应该确保设置从旧路径到新路径的重定向，以便旧URL仍能将用户带到正确的地方。

## 添加新重定向

在Sentry文档中有两种添加重定向的方法：

- [在 `middleware.ts` 中添加新重定向（推荐）](#在-middlewarets-中添加新重定向推荐)
- [在 `redirects.js` 中添加新重定向](#在-redirectsjs-中的重定向示例)

<Alert>
  这两个文件都区分了SDK文档（即最终用户文档）和开发者文档。
</Alert>

由于Sentry的 `next.config.js` 重定向数量有限，您应该尽可能在 `middleware.ts` 中配置重定向。只有在重定向需要使用正则表达式和通配符路径段时，才应使用 `redirects.js`（即 `next.config.js`）。

<Alert level="warning">
  由于限制（在localhost上不工作，需要静态文件...），不再推荐使用 `vercel.json` 进行重定向。
</Alert>

### 在 `middleware.ts` 中添加新重定向（推荐）

在 `src/middleware.ts` 中设置所有简单的一对一重定向。

要在 [`src/middleware.ts`](https://github.com/getsentry/sentry-docs/blob/master/src/middleware.ts) 中添加新重定向，请向 `REDIRECTS` 添加一个新对象，该对象具有以下属性：

- `from`: 传入的请求路径。
- `to`: 您想要路由到的新目标路径。

#### 在 `middleware.ts` 中的重定向示例

下面的示例将 `https://docs.sentry.io/performance/` 重定向到 `https://docs.sentry.io/product/performance/`：

```typescript {filename:middleware.ts} {2-5}
const REDIRECTS: {from: PathWithTrailingSlash; to: string}[] = [
  {
    from: '/performance/',
    to: '/product/performance/',
  },
];
```

### 在 `redirects.js` 中添加新重定向

Sentry的Vercel重定向数量有限，因此只有在重定向需要使用正则表达式时，才应在 `vercel.json` 中配置重定向。否则，请使用 [`middleware.ts`](#在-middlewarets-中添加新重定向推荐)。

要在 `redirects.js`（由 `next.config.js` 的 [`redirects`](https://nextjs.org/docs/app/api-reference/next-config-js/redirects) 使用）中添加新重定向，请向相应的重定向添加一个新对象，该对象具有以下属性：

- `source`: 传入的请求路径模式。
- `destination`: 您想要路由到的新目标路径。

#### 在 `redirects.js` 中的重定向示例

下面的示例使用 [路径模式](https://nextjs.org/docs/app/api-reference/next-config-js/redirects#path-matching) 或 [正则表达式](https://nextjs.org/docs/app/api-reference/next-config-js/redirects#regex-path-matching) 将类似 `https://docs.sentry.io/cli/:path*` 的URL重定向到 `https://docs.sentry.io/product/cli/:path*`。

例如，`https://docs.sentry.io/cli/installation/` 将被重定向到 `https://docs.sentry.io/product/cli/installation/`。

```json {filename:redirects.js}
 "redirects": [
    {
      "source": "/cli/:path*",
      "destination": "/product/cli/:path*",
    },
 ]
```

如果您想向文档中添加图片，可以将它们添加到与您的 `.mdx` 文件相同目录的 `./img` 文件夹中。
您不必遵循严格的文件夹结构，只要图片的相对路径（以 `./` 开头）正确即可。参见 [本地图片](#本地图片)。

如果图片在多个地方使用，最好将其放在 `includes/common-imgs/imgs` 中，并在本地 [`include` 文件](/contributing/approach/product-docs/write-index/#includes) 中引用它，以便在目标 `mdx` 文件中使用。参见 [通用图片](#通用图片)。

或者，图片也可以添加到 `public` 文件夹中，并使用文件夹内的相对路径或完整路径引用。参见 [公共图片](#公共图片)

<Alert>
  每个以 `/` 开头（而不是 `./`）的图片路径都被认为在 `public` 文件夹中。
</Alert>

### 示例

#### 本地图片

文件 `docs/product/alerts/alert-types.mdx` 可以像下面这样引用本地 `docs/product/alerts/img/alert-details-example.png` 图片：

```markdown {filename:docs/product/alerts/alert-types.mdx}
![Alert details ...](./img/alert-details-example.png)
```

#### 通用图片

在多个文件中重用的图片应作为 `include` 文件进行单一来源。例如，通用图片 `includes/common-imgs/imgs/tags.png` 在 `include` 文件 `includes/common-imgs/tags.mdx` 中引用，如下所示：

```markdown {filename:includes/common-imgs/tags.mdx}
![Tags](./img/tags.png)
```

然后可以通过以下语法在多个地方（如 `docs/platforms/ruby/common/enriching-events/tags/index.mdx`）使用图片，通过引用 `include` 文件：

```markdown {filename:docs/platforms/ruby/common/enriching-events/tags/index.mdx}
<Include name="common-imgs/tags" />
```

#### 公共图片

虽然我们建议将图片本地存储在引用它们的文件中，但您也可以将它们放在 `public` 文件夹中。默认情况下，文档平台假设以 `/` 开头的路径引用的图片位于 `public` 文件夹中，因此您不需要在图片路径中包含 `public`。

`public` 的文件夹结构模仿 `docs` 的结构，以便您可以使用相对路径引用这些图片。例如，文件 `docs/product/alerts/alert-types.mdx` 可以像下面这样引用图片 `public/product/alerts/issue-alert.png`：

```markdown {filename:docs/product/alerts/alert-types.mdx}
![Issue alert](issue-alert.png)
```

位于其他位置的内容文件也可以通过使用它们的完整路径（不包括 `public`）引用公共图片。例如，文件 `docs/platform/apple/index.mdx` 也可以像下面这样引用相同的图片 `public/product/alerts/issue-alert.png`：

```markdown {filename:docs/platform/apple/index.mdx}
![Issue alert](/product/alerts/issue-alert.png)
```

## 图片优化

为了节省空间并确保一致性，我们要求所有图片文件都进行压缩和裁剪到下面列出的尺寸。添加图片时，请确保它是高分辨率且清晰的，并且文本足够大以便阅读。

### 推荐尺寸
- 如果您希望图片占据页面的整个宽度，请使其 **1484像素** 宽。

- 如果您希望图片缩进一次（例如，如果它在列表项下），请使其 **1400像素** 宽。

### 图片压缩

出于性能原因，图片应该被压缩。我们推荐使用ImageOptim。以下是遵循的步骤：

1. 下载并安装 https://imageoptim.com/。
2. 将您的图片拖放到工具中。文件将被就地压缩。
3. 等待绿色复选标记出现，然后将压缩的图片添加到文档repo的 `public` 文件夹中的适当路径。

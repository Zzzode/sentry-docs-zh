Sentry在文档中使用极简主义方法。我们的文档有一个目标：帮助客户**快速**掌握。文档中的极简主义将读者置于内容的中心。我们的受众阅读文档是为了解决问题或完成设置或理解。因此，**非常批判性地**思考所提供的内容，尤其是在主页面。

## 使用此测试

在开发内容时询问这些问题：

- 这些信息对于帮助开发者启动和运行是否至关重要？如果不是，请考虑将其移动到子页面，并从主页面链接到它（或者，正如我们的一位工程经理建议的那样，"很好，我可以稍后收藏这个"）。

- 这在UI中是否显而易见？我们的UI在很大程度上是自我记录的，因此我们不需要解释已经显而易见的内容。

## 内容放在哪里？

我们的内容有一些相当明确的分工：

- 平台/SDK内容 - 检测和配置您的SDK
- 产品内容 - 查看所有摄取到Sentry中的数据

在这些主要类别中：

- 参考：客户需要知道的内容
- 操作：客户可以做什么（修改、过滤等）

如果您不确定，[询问](https://github.com/getsentry/sentry-docs/issues)。

## 为Sentry写作

在为文档做出贡献时牢记这些概念：

1. 技术准确性是我们的主要考虑因素。我们的内容帮助每个开发者诊断、修复和优化他们的代码。
2. 使用包容性语言，我们[在这里](https://develop.sentry.dev/getting-started/inclusive-language/)更全面地讨论了这一点。
3. 反馈是一份礼物 - 分享您的PR，以便我们可以改进我们的内容。

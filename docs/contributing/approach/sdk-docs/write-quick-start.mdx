新的SDK应使用本页概述的快速入门指南格式。您可能仍会看到现有SDK的入门指南。

<Alert type="info" title="注意">
  如果您想更新现有的入门指南而不将其重构为快速入门指南，请参阅 [如何编写 - 入门指南](/contributing/approach/sdk-docs/write-getting-started/)。如果您不确定正在查看哪种指南，请通过 [在GitHub上提出问题](https://github.com/getsentry/sentry-docs/issues/new/choose) 来提问。
</Alert>

<Expandable type="info" title="为什么改变？">
这一改变源于我们对遵循最佳实践并使我们的文档尽可能易于访问的承诺。

通过新的快速入门指南，我们希望提供一个更有结构和更集中的方法，帮助作者自信地创建有价值的内容，最终帮助用户使用Sentry实现他们的目标。

**快速入门与入门指南：**
| | 快速入门 | 入门指南 |
| -------- | ----------------------------------------------------------------------------------- | --------------------------------------------------------------------- |
| **重点** | 目标导向，立即行动，最少步骤 | 学习导向，更广泛的概念，背景信息 |
| **内容** | 重点介绍关键步骤，较少关注如何和为什么 | 更具解释性并提供背景信息 |
| **受众** | 想要快速开始的开发者 | 寻求更深入理解的开发者 |
| **格式** | 单页；简洁 | 通常跨越多页；全面 |

</Expandable>

我们的快速入门指南旨在：

- 使开发者能够快速使用错误监控、跟踪和会话重放等功能上线Sentry
- 通过避免不必要的解释或背景信息来保持说明简洁
- 通过包含代码示例和命令来专注于行动
- 在用户的Sentry项目中实现功能设置并获得可见结果
- 作为进一步定制的基础

## 构建config.yml文件

config.yml文件提供SDK级别的详细信息，如其标题、支持级别和设备类别。此文件位于 `/docs/platform/<SDK_name>/` 目录中。示例：

```yaml
title: JavaScript
caseStyle: camelCase
supportLevel: production
sdk: "sentry.javascript.browser"
categories:
  - browser
```

## 结构模板

我们创建了两个快速入门指南模板：一个用于向导设置，一个用于手动设置。将它们作为基础，并根据需要添加新的部分（步骤）。

如果SDK同时具有快速入门指南，请确保两个指南都达到相同的结果，这意味着开发者项目中的修改应该保持一致（在合理范围内）。

### 向导设置

向导是设置Sentry的最快方式 - 您的指南应通过依赖向导进行核心设置来反映这一点，同时在有意义且帮助开发者时提供额外信息。您可以将这些附加内容包装在 `Expandable` 组件中，以默认隐藏它们并在视觉上整理您的页面。

我们建议您基于类似的现有SDK页面，例如：

- [Next.js快速入门指南向导设置](/platforms/javascript/guides/nextjs/)

**最小结构：**

- **标题和摘要**
  - 标题自动从config.yml文件中获取
  - 描述开发者通过遵循此指南将实现什么
- **先决条件**
- **步骤1：安装**
  - 确保"安装"始终是第一步，以快速让开发者采取行动
  - 提供此SDK的主要安装方法（我们在单独的页面上记录替代方法，如 [此示例](/platforms/javascript/install/)）
  - 简要介绍向导可以启用的功能；使用 `Expandable` 组件默认隐藏内容
  - 列出向导在开发者应用程序中应用的更改，如更新配置文件和创建用于稍后测试设置的示例代码；使用 `Expandable` 组件默认隐藏内容
- **步骤2：验证您的设置**
  - 解释如何测试设置。例如，指导开发者使用向导创建的示例
  - **小节：在Sentry中查看捕获的数据**
    - 指导开发者查看其Sentry项目中捕获的错误
    - 可选：使用 `<Expandable>` 提供详细说明
- **后续步骤**
  - 总结并总结开发者实现的内容
  - 提供指向推荐后续主题的链接列表

### 手动设置

为您的手动设置指南使用此模板。保持说明简洁，并问自己："开发者需要知道这一点才能成功完成指南，还是这是他们可以在文档中稍后阅读的主题？"

我们建议您基于类似的现有SDK页面，例如：

- [Next.js快速入门指南手动设置](/platforms/javascript/guides/nextjs/manual-setup/)

**最小结构：**

- **标题和摘要**
  - 标题自动从config.yml文件中获取
  - 描述开发者通过遵循此指南将实现什么
- **先决条件**
- **步骤1：安装**
  - 确保"安装"始终是第一步，以快速让开发者采取行动
  - 在安装说明之前在此处包含入门选项（功能设置）（使用 [`OnboardingOption`](/contributing/pages/components/#onboarding-options) 组件）
    - **重要**：使用 `OnboardingOption` 组件在指南中切换功能特定说明的可见性。例如，如果开发者想要设置跟踪但不是会话重放，他们不应该获得如何配置会话重放的说明。
  - 简要解释这些功能；使用 `Expandable` 组件默认隐藏内容
  - 提供此SDK的主要安装方法（我们在单独的页面上记录替代方法，如 [此示例](/platforms/javascript/install/)）
- **步骤2：配置**
  - 提供配置示例并在代码示例中添加注释，以限制您放入周围散文中的内容
  - 展示如何配置Sentry和所选功能
  - 使用小节进行逻辑组织和分割长内容
- **步骤3 - 步骤X：SDK特定说明**
  - 为源映射、隧道或其他必要的工具创建步骤
  - 每个主要主题应该是一个单独的步骤
- **步骤Y：验证您的设置**
  - 为此SDK提供验证代码示例
  - 使此部分适应所选入门选项的功能。例如，如果启用了跟踪，您需要描述如何验证其设置正确
  - **小节：在Sentry中查看捕获的数据**
    - 指导开发者查看其Sentry项目中捕获的错误
    - 可选：使用 `<Expandable>` 提供详细说明
- **后续步骤**
  - 总结并总结开发者实现的内容
  - 提供指向推荐后续主题的链接列表

## 内容放置和可重用性

如果您想要包含的内容尚不存在，请在 [`/src/platform-includes/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes) 中的相应文件夹中为您的SDK创建一个新文件。

**示例：**

- **安装** [`/platform-includes/getting-started-install/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-install)
- **配置** [`/platform-includes/getting-started-config/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-config)
- **验证** [`/src/platform-includes/getting-started-verify/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-verify)

使用 `PlatformSection` 组件将此内容集成到您的指南中。

<Alert type="info">
  如果SDK支持性能监控，请将其添加到从产品中链接回SDK内容的列表中，存储在
  [`/docs/product/insights/getting-started.mdx`](https://github.com/getsentry/sentry-docs/blob/master/docs/product/insights/getting-started.mdx)。
</Alert>

## 我们向导的额外要求

快速入门内容也被我们的应用内入门向导用来让用户使用Sentry上线。保持所有内容最新至关重要。

重要：

- 如果您更新SDK的代码示例，也请更新向导
- 如果您添加新内容，如额外配置，到向导，也请更新手动设置指南

### 更新向导

应用内向导是开发者首次集成我们SDK的地方。由于两个原因，它不是从快速入门简单复制/粘贴：

1. 新用户缺乏我们Docs提供的上下文（这样想，没有左侧边栏可供查看）。
2. 有经验的用户可能已经关闭了警报等，所以不要期望一个清晰的工作流程。也不要回顾工作流程。将用户指向产品进行游览。

因此，简单复制/粘贴快速入门是不够的。以 [Next.js向导](https://github.com/getsentry/sentry-wizard/tree/master/src/nextjs) 为例。

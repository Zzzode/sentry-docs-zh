我们的SDK内容结构专注于让用户快速上手，然后了解可以在子页面中配置或启用的所有功能。这些子页面提供参考材料，带有特定于SDK的代码示例。

## 内容部分

SDK提供专注于10个部分的内容（其中两个是自动生成的），这些部分依赖于通用内容。每个部分在自己的页面上有详细解释。快速概述：

1. **快速入门/入门指南（传统）**

   一个单一的、关键的页面。

   对于新的SDK和更新的指南，使用：
   **[如何编写 - 快速入门](/contributing/approach/sdk-docs/write-quick-start/)**

   要更新现有的入门指南，使用：
    **[如何编写 - 入门指南](/contributing/approach/write-getting-started)**

   此内容用于Sentry客户的两个不同区域：

   1. [docs.sentry.io](https://docs.sentry.io)
   2. 为客户创建或添加项目的应用内向导

2) **配置**

   涵盖客户管理其SDK实现的所有方式。它由子页面组成（所有这些子页面都可以根据SDK的适当性打开或关闭）。

   **[如何更新：配置](/contributing/approach/write-configuration)**

3) **使用**

   涵盖如何捕获错误和消息，以及设置级别的子页面。

   **[如何更新：使用](/contributing/approach/write-usage)**

4) **故障排除**

   涵盖我们知道的角落和边缘情况。此内容在主要SDK级别开发，因此由主要SDK及其相关指南共享。它不适用于所有SDK/平台。

   内容位于此目录 `/docs/platforms/<SDK>/common/troubleshooting/`

5) (可选) **迁移指南**

   为需要从SDK的Raven版本升级的客户提供。较新的SDK不需要。

6) **跟踪** 部分

   涵盖如何启用跟踪（针对以前仅使用错误监控的客户），如每个SDK所启用，以及关于采样和事务的子页面。

   _此部分是正在进行中的工作。_

7) **丰富事件** 部分

   涵盖发送到Sentry的数据以及如何自定义它。

   **[如何更新：丰富事件](/contributing/approach/write-enriching-events)**

8) **数据管理** 部分

   涵盖我们分组算法以及如何修改它，以及如何使用SDK清理敏感数据。

   **[如何更新：数据管理](/contributing/approach/write-data-management)**

9) (自动生成) **其他指南**

   列出此SDK的其他相关平台/集成/SDK。

10) (自动生成) **其他平台**

    列出其他主要SDK/平台。

这个模型非常灵活 - 一些SDK将包含更多信息，其他SDK将需要更少的信息。

我们使用这种格式有几个目标：

- 确保客户有一个单一的页面可以快速开始。
- 更新SDK内容，使其既**技术上准确**又**完整**。
- 确保我们已经架构以支持我们文档的结构，该结构将平台/SDK内容与产品内容分开。将此视为"平台/SDK文档帮助客户将数据发送到Sentry"和"产品文档帮助客户理解 [sentry.io](http://sentry.io) 中的内容，然后确定适当的响应"

## 从哪里开始

1. 阅读 [贡献者指南](/contributing/)。指南将帮助您设置文档环境（迁移到我们的新工具非常重要）并提供有关SDK内容、搜索、重定向和向导页面的关键技术细节。如果需要帮助，请通过在GitHub上提出问题来寻求帮助。
2. 使用"如何"页面来了解内容涵盖的位置。
3. 使用JavaScript中的文件和文件夹结构作为模型。
4. 创建PR，并请请求 getsentry/docs 作为审阅者。

## 不要做什么

- 思考和沉思。该模型并不意味着100%的时间都有效。预计某些内容不会整齐地工作。如果您不确定内容应该显示在哪里，请通过在GitHub上提出问题来提问。
- 不要忘记更新使用入门指南内容来帮助指导 sentry.io 上的客户的向导页面
- 不要忘记更新支持性能监控的SDK列表，如适当。

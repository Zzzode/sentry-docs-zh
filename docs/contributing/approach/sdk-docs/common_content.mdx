我们的文档力求在重复与复杂性之间实现贡献者友好的平衡。这意味着我们通常不会尝试在SDK之间共享内容，即使它们具有相似的信息架构、部分甚至措辞。
我们不是通过共享内容来强制执行一致性，而是鼓励贡献者参考其他SDK文档以尽可能保持一致。

对于具有框架特定文档的SDK（我们称为指南或子项），该SDK框架的通用内容位于 `/docs/platforms/<platform_name>/common/` 中。

## 层次结构

为用户显示的内容依赖于我们内容中详细讨论的 [平台与指南](/contributing/platforms/) 的层次结构。简而言之，如果提供了指南的内容，则显示指南的内容。如果未提供，则显示平台内容。

## 调整通用内容

为了确保通用内容可以随着我们支持更多平台和框架而扩展，我们使用 `include` 文件。这些文件的范围可以从最常见的情况（特定于平台或框架的代码示例）到增强特定平台或框架的核心内容的信息。这些特定于平台的 `include` 文件位于 [`plaftorm-includes/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes) 中。我们还有 [平台无关的 `include` 文件](/contributing/approach/product-docs/write-index/#includes)，这些文件在我们的产品文档中讨论。

内容的丰富事件部分涵盖了客户如何为发送到Sentry的数据添加上下文。新框架可以采用通用内容，这些内容存储在 `/docs/platforms/<SDK>/common/enriching-events` 中 - 请随意浏览该目录中的文件以回答任何问题。

<Alert >

不要更改通用内容。该更改将波及所有依赖通用内容的指南。相反，请在GitHub上提出问题并附带问题/建议。

</Alert >

这些页面中的大多数都是相当不言自明的，因此只提供基本信息。但请通过在GitHub上提出问题来询问任何问题。

## 从这里开始

确定页面是否适用：

- 如果内容**不适用**，请将指南添加到文件的前端内容中的 `notSupported` 列表中。这将关闭此指南的内容显示。
- 如果内容**适用**，请将 `include` 文件添加到每个页面中注明的正确目录中。如果未提供代码示例，页面将显示一个大灰框，要求客户为缺失的示例提出问题。
- JS系列内容需要导入SDK，以便代码示例在每个页面上都有意义。将导入语句添加到 `/src/platform-includes/enriching-events/enriching-events/import`。

### 添加上下文

此文件是 `context.mdx`。它解释了如何启用自定义上下文。将代码示例添加到此目录：

- `/src/platform-includes/enriching-events/set-context/`

如果指南可以直接传递上下文，请将其添加到"直接传递上下文"上方支持的指南列表中。

### 识别用户

此文件是 `identify-user.mdx`。它解释了如何捕获用户。将代码示例添加到这些目录：

- `/src/platform-includes/enriching-events/set-user/`
- `/src/platform-includes/enriching-events/unset-user/`

### 设置事务名称

此文件是 `transaction-name.mdx`。它解释了如何覆盖事务名称。将代码示例添加到此目录：

- `/src/platform-includes/enriching-events/set-transaction-name/`

如果指南可以控制事务的开始和停止，请将其添加到指向我们跟踪文档的支持指南列表中。

### 自定义标签

此文件是 `tags.mdx`。它解释了如何为事件自定义标签。将代码示例添加到此目录：

- `/src/platform-includes/enriching-events/set-tag/`

如果指南不将标签绑定到当前作用域，请将其添加到不支持此选项的指南列表中。

### 附件

此文件是 `/attachments/index.mdx`。它解释随事件附加文件。将代码示例添加到此目录：

- `/src/platform-includes/enriching-events/add-attachment/`

如果指南是本机系列的一部分，请将其添加到支持以下本机内容的指南列表中：

- 调试信息文件和对本机崩溃的内置支持
- 大型崩溃报告
- "崩溃报告和隐私"部分
- 关于限制每个问题的崩溃报告的段落

### 面包屑

此文件是 `breadcrumbs.mdx`。它解释手动面包屑。将手动添加记录面包屑的代码示例添加到此目录：

- `/src/platform-includes/enriching-events/breadcrumbs/breadcrumbs-example/`
- `/src/platform-includes/enriching-events/breadcrumbs/before-breadcrumb/`

一个细微差别：

- 根据需要更新 `PlatformSection supported` 中支持的指南列表，该列表位于"可用的面包屑键..."之前

### 用户反馈

此文件是 `user-feedback.mdx`。它解释了如何使用可嵌入的JS小部件或API。

- 根据需要更新 `PlatformSection supported` 中支持的SDK列表，该列表位于"用户反馈API"之前。
  然后将代码示例添加到 `src/platform-includes/user-feedback/sdk-api-example` 目录
- 对于.NET系列，如果用户反馈API有特定集成，请将其添加到"使用.NET SDK"之前的 `PlatformSection supported` 中，并添加适当的链接。
- 根据需要更新不支持JS小部件的SDK列表，在列出的 `PlatformSection notSupported` 中，该列表位于"可嵌入JavaScript小部件"之前。
- 对于使用JS小部件的SDK，将代码示例添加到 `/src/platform-includes/user-feedback/user-feedback-example-widget/`

### 作用域

此文件是 `scopes.mdx`。它解释管理作用域。将代码示例添加到这些目录：

- `/src/platform-includes/enriching-events/scopes/configure-scope/`
- `/src/platform-includes/enriching-events/unset-user/`
- `/src/platform-includes/enriching-events/scopes/with-scope/`

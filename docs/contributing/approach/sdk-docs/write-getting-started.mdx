<Alert type="info">
  本页面描述了传统的"入门指南"格式。新的SDK和已更新的指南应使用 [快速入门指南格式](/contributing/approach/sdk-docs/write-quick-start/)。
</Alert>

新的平台/SDK应基于类似的现有SDK页面来创建其入门指南页面。

以下是入门指南页面的一般组件概述。

## config.yml

此文件位于 `/docs/platform/<SDK_name>/` 目录中。示例：

```yaml
title: JavaScript
caseStyle: camelCase
supportLevel: production
sdk: "sentry.javascript.browser"
categories:
  - browser
```

## 入门内容

在这两个部分中高层次解释SDK：

1. 一个斜体句子，解释SDK启用的功能（错误，或错误和事务）
2. 一个简要段落，提供技术见解和支持的版本

   <Alert>

   如果入门内容未开发，此回退将显示"_Sentry的SDK支持错误和异常的自动报告_"。

   </Alert>

将入门内容添加到此目录：

- [`/platform-includes/getting-started-primer/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-primer)

## 安装方法

提供此SDK的主要安装方法示例。虽然我们记录了替代方法，但我们将其保留在单独的页面上（如 [此示例](/platforms/javascript/install/)），以保持入门内容专注于最常见、最快上手的方式。

将安装方法添加到此目录：

- [`/platform-includes/getting-started-install/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-install)

## 配置代码示例

提供此SDK配置的示例，在代码示例中注释，而不是周围的散文中。

将配置代码示例添加到此目录：

- [`/platform-includes/getting-started-config/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-config)

<Alert>

如果SDK支持性能监控，请将其添加到从产品中链接回SDK内容的列表中，存储在 [`/docs/product/insights/getting-started.mdx`](https://github.com/getsentry/sentry-docs/blob/master/docs/product/insights/getting-started.mdx)。

</Alert>

## 验证代码示例

为此SDK提供验证代码示例。它可以简单到一行，尽管SDK可能需要更复杂的代码示例。

将验证代码示例添加到此目录：

- [`/src/platform-includes/getting-started-verify/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-verify)

## 但这还不是全部

入门指南内容也被我们的应用内入门向导用来让客户快速上手Sentry。保持所有内容最新至关重要。

**如果更新SDK的代码示例，也请更新向导。**

### 更新向导

应用内向导是客户首次实现我们SDK的地方。由于两个原因，它不是从入门指南简单复制/粘贴：

1. 新客户缺乏我们Docs提供的上下文（这样想，没有左侧边栏可供查看）。
2. 有经验的客户可能已经关闭了警报等，所以不要期望一个清晰的工作流程。也不要回顾工作流程。将客户指向产品进行游览。

因此，简单复制/粘贴入门指南是不够的。以JavaScript向导为例。

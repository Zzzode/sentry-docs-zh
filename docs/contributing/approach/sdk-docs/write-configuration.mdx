不出所料，内容的配置部分涵盖了客户如何配置SDK。新框架可以采用通用内容，这些内容存储在 `/docs/platforms/<SDK>/common/configuration/` 中 - 请随意浏览该目录中的文件以帮助回答任何问题。

<Alert >

不要更改通用内容。该更改将波及所有依赖通用内容的指南。相反，请在GitHub上提出问题并附带问题/建议。

</Alert >

这些页面中的大多数都是相当不言自明的，因此只提供基本信息。但如果有任何问题，请通过在GitHub上提出问题来询问。

## 从这里开始

确定页面是否适用：

- 如果内容**不适用**，请将指南添加到文件的前端内容中的 `notSupported` 列表中。这将关闭此指南的内容显示。
- 如果内容**适用**，请将 `include` 文件添加到每个页面中注明的正确目录中。如果未提供代码示例，页面将显示一个大灰框，要求客户为缺失的示例提出问题。

### 基本选项

此文件是 `options.mdx`。它解释了所有配置选项。

为此文件开发内容：

1. 创建 `configuration/config-intro/` 语句，涵盖如何设置选项。
2. 对于指南**不支持**的单个选项，将指南名称添加到该选项的 `ConfigKey` 语句中。这将阻止该选项为此SDK显示。例如：

   `<ConfigKey name="debug" notSupported={["javascript.react"]}>`

   或者，对于指南**支持**的单个选项，将指南名称添加到该选项的 `ConfigKey` 语句中，确保选项显示。例如：

   `<ConfigKey name="send-default-pii" supported={["javascript.electron", "javascript.ember"]}>`

3. 对于指南**支持**的选项组，将指南名称添加到这些选项的 `PlatformSection` 语句中，确保选项显示。例如：

   `<PlatformSection supported={["javascript.react"]}>`

### 发布和健康

此文件是 `releases.mdx`。它解释了如何配置SDK以告诉Sentry有关发布的信息。为此SDK提供设置发布的代码示例，然后将代码示例添加到此目录：

- `/platform-includes/set-release/`

一些细微差别：

- 如果添加与JavaScript相关的SDK，请根据需要更新 `PlatformSection supported` 语句中的源映射相关段落。
- 对于支持发布健康的SDK，我们使用 `PlatformSection supported` 语句控制内容的显示。对于这些SDK，提供以下代码示例：选择退出自动会话跟踪，存储在 `/platform-includes/configuration/auto-session-tracking` 中。

### 环境

此文件是 `environments.mdx`。它解释了如何配置SDK以设置环境。

将代码示例添加到此目录：

- `/platform-includes/set-environment/`

### 过滤事件

此文件是 `filtering.mdx`。它解释了如何配置SDK以过滤发送到Sentry的事件。将代码示例添加到这些目录：

- `/platform-includes/configuration/before-send/`
- `/platform-includes/configuration/before-send-hint/`

通过将SDK添加到 `supported` 或 `notSupported` 列表中来启用或禁用"使用提示"的内容，然后将代码示例添加到这些目录：

- `/platform-includes/configuration/before-send-fingerprint/`
- `/platform-includes/configuration/decluttering/`

通过将SDK添加到 `supported` 或 `notSupported` 列表中来启用或禁用"使用采样过滤事务事件"的内容，然后将代码示例添加到此目录：

- `/platform-includes/performance/traces-sampler-as-filter/`

### 集成

此内容在主要SDK/平台级别开发。它位于 `/docs/platforms/<SDK>/integrations/` 或 `/docs/platforms/<SDK>/common/integrations/` 中，通常由以下内容组成：

1. 默认集成 - `default.mdx` - 系统集成默认启用。我们记录它们，以便客户既可以了解它们的作用，又可以在它们引起问题时禁用它们。
2. 可插拔集成 - `plugin.mdx` - 可插拔集成是增强特定应用程序和/或框架功能的代码片段。我们记录它们，以便客户可以看到它们的作用以及它们可以被启用。
3. 自定义集成 - `custom.mdx` - 解释如何启用自定义集成。

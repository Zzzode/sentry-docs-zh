内容的数据管理部分涵盖了客户如何使用SDK管理发送到Sentry的数据。我们在产品中提供了一个并行部分，重点介绍用户可以在 [sentry.io](http://sentry.io) 中使用的设置来控制数据。新框架可以采用通用内容，这些内容存储在 `/docs/platforms/<SDK>/common/data-management` 中 - 请随意浏览该目录中的文件以回答任何问题。

<Alert >

不要更改通用内容。该更改将波及所有依赖通用内容的指南。相反，请在GitHub上提出问题并附带问题/建议。

</Alert >

这些页面中的大多数都是相当不言自明的，因此只提供基本信息。但如果有任何问题，请通过在GitHub上提出问题来询问。

## 从这里开始

确定页面是否适用：

- 如果内容**不适用**，请将指南添加到文件的前端内容中的 `notSupported` 列表中。这将关闭此指南的内容显示。
- 如果内容**适用**，请将 `include` 文件添加到每个页面中注明的正确目录中。如果未提供代码示例，页面将显示一个大灰框，要求客户为缺失的示例提出问题。

### 敏感数据

此文件是 `/sensitive-data/index.mdx`。它解释了如何在SDK内清理或过滤数据，使其不会与事件一起发送。

对于指南**不支持**的数据清理，将指南名称添加到 `PlatformSection notSupported` 语句中，确保内容不会显示。例如：

`<PlatformSection notSupported={["javascript.react"]}>`

将代码示例添加到这些目录：

- `/src/platform-includes/configuration/before-send`
- `/src/platform-includes/sensitive-data/set-tag`
- `/src/platform-includes/sensitive-data/set-user`

### 调试文件

此文件是 `/sensitive-data/debug-files.mdx`。它解释了调试信息文件如何为Sentry工作。

对于指南支持的特定控件，将指南名称添加到 `PlatformSection supported` 语句中，确保内容显示。

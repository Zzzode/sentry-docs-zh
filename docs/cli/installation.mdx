根据您的平台，有不同的方法可用于安装 `sentry-cli`。

## 手动下载

您可以在 [GitHub 发布页面](https://github.com/getsentry/sentry-cli/releases/) 上找到发布列表。我们为 Linux、OS X 和 Windows 提供可执行文件。这是一个单个文件下载，收到文件后，您可以将其重命名为 `sentry-cli` 或 `sentry-cli.exe` 来使用它。

## 自动安装

如果您使用的是 macOS 或 Linux，您可以使用自动下载器，它将为您获取最新的发布版本并安装它：

```bash
curl -sL https://sentry.io/get-cli/ | sh
```

但是，我们鼓励您固定 CLI 的特定版本，以便您的构建始终可重现。
为此，您可以使用完全相同的方法，并附加版本说明符：

```bash
curl -sL https://sentry.io/get-cli/ | SENTRY_CLI_VERSION="{{@inject apps.version('sentry-cli') }}" sh
```

这将自动为您的操作系统下载正确版本的 `sentry-cli` 并安装它。如有必要，它将提示您输入管理员密码以使用 `sudo`。对于不同的安装位置或没有 `sudo` 的系统（如 Windows），您可以在运行此命令之前 `export INSTALL_DIR=/custom/installation/path`。

要验证其是否正确安装，您可以调出帮助：

```bash
sentry-cli --help
```

## 通过 NPM 安装

在某些特殊情况下，还可以通过 npm 安装 `sentry-cli`。例如，这对于构建服务器很有用。该包称为 `@sentry/cli`，在安装后它将下载适当的发布二进制文件：

```bash {tabTitle:npm}
npm install @sentry/cli
```

```bash {tabTitle:yarn}
yarn add @sentry/cli
```

```bash {tabTitle:pnpm}
pnpm add @sentry/cli
```

然后您可以在 `.bin` 文件夹中找到它：

```bash
./node_modules/.bin/sentry-cli --help
```

如果您想使用 sudo 全局安装此 npm，您需要传递 `--unsafe-perm`：

```bash
sudo npm install -g @sentry/cli --unsafe-perm
```

<Alert>

但是，不建议这种安装方式。

</Alert>

### 从自定义源下载

默认情况下，此包将从 [Fastly](https://www.fastly.com/) 管理的 CDN 下载 sentry-cli。要使用自定义 CDN，请设置 npm 配置属性 `sentrycli_cdnurl`。下载器将附加 `"/<version>/sentry-cli-<dist>"`。

```bash
npm install @sentry/cli --sentrycli_cdnurl=https://mymirror.local/path
```

或将属性添加到您的 `.npmrc` 文件 ([https://docs.npmjs.com/files/npmrc](https://docs.npmjs.com/files/npmrc))

```bash
sentrycli_cdnurl=https://mymirror.local/path
```

另一个选项是使用环境变量 `SENTRYCLI_CDNURL`。

```bash
SENTRYCLI_CDNURL=https://mymirror.local/path npm install @sentry/cli
```

### 可用的安装选项

下面列出的选项控制通过 `npm` 安装 `sentry-cli` 时安装脚本的行为。

`SENTRYCLI_CDNURL`:

如果设置，脚本将使用给定的 URL 获取二进制文件。默认为 `https://downloads.sentry-cdn.com/sentry-cli`。

`SENTRYCLI_USE_LOCAL`:

如果设置为 `1`，则 `sentry-cli` 二进制文件将从您的 `$PATH` 中发现并本地复制，而不是从外部服务器下载。它仍将验证版本号，该版本号必须匹配。

`SENTRYCLI_SKIP_DOWNLOAD`:

如果设置为 `1`，则脚本将完全跳过下载二进制文件。

`SENTRYCLI_SKIP_CHECKSUM_VALIDATION`:

如果设置为 `1`，则脚本将跳过校验和验证阶段。您可以通过访问[构建校验和](/cli/installation/#build-checksums)页面手动验证校验和。

`SENTRYCLI_NO_PROGRESS_BAR`:

如果设置为 `1`，则脚本将不显示下载进度条。这是 CI 环境的默认行为。

`SENTRYCLI_LOG_STREAM`:

如果设置，脚本将更改其输出写入的位置。可能的值为 `stdout` 和 `stderr`。默认为 `stdout`。

## 通过 Homebrew 安装

如果您使用的是 OS X，您可以通过 homebrew 安装 `sentry-cli`：

```bash
brew install getsentry/tools/sentry-cli
```

## 通过 Scoop 安装

如果您使用的是 Windows，您可以通过 [Scoop](https://scoop.sh) 安装 `sentry-cli`：

```powershell
> scoop install sentry-cli
```

## Docker 镜像

对于不受支持的发行版和 CI 系统，我们提供了一个预装了 `sentry-cli` 的 Docker 镜像。建议使用 `latest` 标签，但您也可以固定到特定版本。默认情况下，该命令在 `/work` 目录内运行。挂载相关的项目文件夹和构建输出，以允许 `sentry-cli` 扫描资源：

```bash
docker pull getsentry/sentry-cli
docker run --rm -v $(pwd):/work getsentry/sentry-cli --help
```

## 更新和卸载

您可以使用 `sentry-cli update` 和 `sentry-cli uninstall` 来更新或卸载 `sentry-cli` 二进制文件。在某些情况下，这些命令可能不可用，通常是当 `sentry-cli` 已由 homebrew 或 yarn 等工具安装时，无论是直接安装还是作为另一个包的依赖项。在这些情况下，需要使用相同的工具进行更新和删除。如果您发现 `sentry-cli update` 和 `sentry-cli uninstall` 不起作用，并且您不知道包是如何安装的，运行 `which sentry-cli` 通常会提供有关使用哪个工具的线索。

`sentry-cli` 工具也可用于发送事件。如果您想使用它，您需要导出 `SENTRY_DSN` 环境变量并将其指向您项目的 DSN：

```bash
export SENTRY_DSN='___PUBLIC_DSN___'
```

完成后，您可以开始使用 `sentry-cli send-event` 命令。

## 基本事件

对于基本消息事件，您只需提供 `--message` 或 `-m` 参数来发送消息：

```bash
sentry-cli send-event -m "来自 Sentry 的问候"
```

这将向 Sentry 发送一条消息并将其记录为事件。与该事件一起，它发送有关您运行 `sentry-cli` 的机器的基本信息。您可以多次提供 `-m` 来发送多行：

```bash
sentry-cli send-event -m "来自 Sentry 的问候" -m "这是更多文本"
```

## 带参数的事件

此外，您可以在消息中使用 `%s` 作为占位符，并使用 `-a` 参数填充它。这有助于查看它们，因为所有消息将自动分组在一起：

```bash
sentry-cli send-event -m "你好 %s!" -a "Joe"
sentry-cli send-event -m "你好 %s!" -a "Peter"
```

## 发送面包屑

您还可以将日志文件传递给 `send-event` 命令，该命令将被解析并作为面包屑发送。最后 100 个项目将被发送：

```bash
sentry-cli send-event -m "任务失败" --logfile error.log
```

日志文件可以是各种格式。如果您想自己创建一个，您可以执行以下操作：

```bash
echo "$(date +%c) 这是一个日志记录" >> output.log
echo "$(date +%c) 这是另一条记录" >> output.log
sentry-cli send-event -m "演示事件" --logfile output.log
rm output.log
```

## 额外数据

可以使用 `-e` 参数将额外数据附加为 `KEY:VALUE`。例如，您可以像这样发送一些键值对：

```bash
sentry-cli send-event -m "失败" -e task:create-user -e object:42
```

同样，标签可以使用 `-t` 以相同的格式发送：

```bash
sentry-cli send-event -m "失败" -t task:create-user
```

## 存储的事件

从版本 `1.71` 开始，`send-event` 命令可以接受一个可选参数，该参数指定存储的 [事件负载](https://develop.sentry.dev/sdk/data-model/event-payloads/) 的 JSON 表示的文件路径。使用时，它将加载文件，验证事件并将其发送到 Sentry。

```bash
sentry-cli send-event ./events/20211029150006.json
```

此参数也可以以 glob 的形式提供，这将导致它处理所有匹配的事件。

```bash
sentry-cli send-event "./events/*.json"
```

如果使用 `--raw` 标志调用 `send-event`，则事件在发送前不会被验证。

```bash
sentry-cli send-event --raw ./events/20211029150006.json
```

## 指定发布

可以使用 `--release` 参数发送发布。如果您从 git 仓库中使用 sentry-cli，则会自动选择默认发布。

## Bash 钩子

对于 bash 脚本，您还可以通过使用 sentry-cli bash 钩子启用自动错误发送。这启用 `set -e` 并将为未处理的错误发送 Sentry 事件。

此限制为：

- sentry-cli 仅在启用 `set -e` 时才真正工作（默认情况下它会为您启用）。
- sentry-cli 注册 `EXIT` 和 `ERR` 陷阱。

用法：

```bash
#!/bin/bash
export SENTRY_DSN='___PUBLIC_DSN___'
eval "$(sentry-cli bash-hook)"
# 脚本的其余部分放在这里
```

或者，您可以使用其他机制，如 [`.sentryclirc` 文件](/cli/configuration/#configuration-file) 来配置 DSN。

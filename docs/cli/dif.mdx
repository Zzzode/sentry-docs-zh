调试信息文件允许 Sentry 提取堆栈跟踪并为大多数编译平台提供有关崩溃报告的更多信息。`sentry-cli` 可用于验证和上传调试信息文件。有关更一般的信息，请参阅[调试信息文件](/platforms/native/data-management/debug-files/)。

## 权限

`sentry-cli` 需要[组织令牌](https://sentry.io/orgredirect/organizations/:orgslug/settings/auth-tokens/)才能上传调试信息文件。

<Alert>

源映射虽然是调试信息文件，但在 Sentry 中的处理方式不同。有关更多信息，请参阅 [sentry-cli 中的源映射](/cli/releases/#sentry-cli-sourcemaps)。

</Alert>

## 检查文件

并非所有调试信息文件都可以被 Sentry 使用。要查看它们是否可用，您可以使用 `sentry-cli debug-files check` 命令：

```bash
sentry-cli debug-files check mylibrary.so.debug

调试信息文件检查
  类型：elf 调试伴侣
  包含的调试标识符：
    > 924e148f-3bb7-06a0-74c1-36f42f08b40e (x86_64)
  包含的调试信息：
    > symtab, debug
  可用：是
```

这将报告调试信息文件的调试标识符以及它是否通过 Sentry 的基本要求。

## 查找文件

如果您在 Sentry 的 UI 中看到调试信息文件缺失，但不确定如何找到它们，您可以使用 `sentry-cli debug-files find` 命令来查找它们：

```bash
sentry-cli debug-files find <identifier>
```

此外，`sentry-cli debug-files upload` 可以自动在文件夹或 ZIP 存档中搜索文件。

## 创建源包

为了在 Sentry UI 中的堆栈跟踪中获得内联源上下文，`sentry-cli` 可以扫描调试文件以查找源代码文件的引用，在本地文件系统中解析它们并将它们打包。生成的源包是一个包含特定调试信息文件引用的所有源文件的存档。

这在构建和上传调试信息文件分离时特别有用。在这种情况下，可以在构建时创建源包，并可以在以后任何时候使用 `sentry-cli debug-files upload` 上传。

要创建源包，请在调试信息文件列表上使用 `debug-files bundle-sources` 命令：

```bash
# 在构建机器上：
sentry-cli debug-files bundle-sources /path/to/files...

# 在任何时候：
sentry-cli debug-files upload --type sourcebundle /path/to/bundles...
```

要为所有调试信息文件创建多个源包，请对每个文件单独使用该命令。

或者，将 `--include-sources` 选项添加到 `debug-files upload` 命令中，这将在上传过程中即时生成源包。这要求上传在与应用程序构建相同的机器上执行：

```bash
sentry-cli debug-files upload --include-sources /path/to/files...
```

<Alert>

此功能由 Sentry 支持的构建工具支持，这些工具生成 Sentry 支持的调试信息文件，如 DWARF 和 PDB。这适用于 C/C++/C#/Swift/Rust/Zig 等语言。
对于 Java/Kotlin 和其他 JVM 语言，请使用其中一个插件：[Maven](/platforms/java/maven/) 或 [Gradle](/platforms/java/gradle/)。

</Alert>

## 上传文件

使用 `sentry-cli debug-files upload` 命令将调试信息文件上传到 Sentry。该命令将递归扫描提供的文件夹或 ZIP 存档。已上传的文件将自动跳过。

我们建议在上传或发布应用程序时上传调试信息文件。或者，可以在构建过程中上传文件。请参阅[调试信息文件](/platforms/native/data-management/debug-files/)了解更多信息。

<Alert>

您需要指定您正在处理的组织和项目，因为调试信息文件在项目级别工作。有关此的更多信息，请参阅[使用项目](/cli/configuration/#sentry-cli-working-with-projects)。

</Alert>

基本的调试文件上传可以这样开始：

```bash
sentry-cli debug-files upload -o <org> -p <project> /path/to/files...

> 找到 2 个调试信息文件
> 准备调试信息文件进行上传
> 已上传 2 个缺失的调试信息文件
> 文件处理完成：

  PENDING 1ddb3423-950a-3646-b17b-d4360e6acfc9 (MyApp; x86_64 可执行文件)
  PENDING 1ddb3423-950a-3646-b17b-d4360e6acfc9 (MyApp; x86_64 调试伴侣)
```

上传后，Sentry 分析文件以符号化未来事件。如果您想向 Sentry 发送原生崩溃以验证正确操作，请确保调试文件列在_项目设置 > 调试文件_中。或者，在 CLI 中指定 `--wait`，这将阻塞直到服务器端分析完成：

```bash
sentry-cli debug-files upload -o <org> -p <project> --wait /path/to/files...

> 找到 2 个调试信息文件
> 准备调试信息文件进行上传
> 已上传 2 个缺失的调试信息文件
> 文件处理完成：

      OK 1ddb3423-950a-3646-b17b-d4360e6acfc9 (MyApp; x86_64 可执行文件)
      OK 1ddb3423-950a-3646-b17b-d4360e6acfc9 (MyApp; x86_64 调试伴侣)
```

### 上传选项

该命令的 `--help` 输出列出并解释了所有选项：

```bash
sentry-cli debug-files upload --help
```

### 符号映射

如果您正在隐藏来自 Apple 的调试符号，调试文件将不包含许多有用的符号。在这种情况下，sentry-cli 上传将警告您它需要 BCSymbolMaps：

```bash
sentry-cli debug-files upload ...
> 找到 34 个调试信息文件
> 警告：找到 10 个符号文件，符号被隐藏（需要 BCSymbolMaps）
```

在这种情况下，您需要与您的文件匹配的 BCSymbolMaps。通常，这些由 Xcode 构建过程生成。提供 `--symbol-maps` 参数并将其指向包含符号映射的文件夹：

```bash
sentry-cli debug-files upload --symbol-maps path/to/symbolmaps path/to/debug/symbols
```

### Breakpad 文件

与原生调试文件相比，Breakpad 符号丢弃了大量不需要处理 minidump 的信息。最值得注意的是，内联函数未被声明，因此 Sentry 无法在堆栈跟踪中显示内联帧。

如果可能，请上传原生调试文件，如 dSYM、PDB 或 ELF 文件，而不是 Breakpad 符号。

## ProGuard 映射上传

`sentry-cli` 可用于将 ProGuard 文件上传到 Sentry；然而，在大多数情况下，您会使用 [Gradle 插件](https://github.com/getsentry/sentry-android-gradle-plugin) 来完成此操作。尽管如此，在某些情况下，您可能会手动上传 ProGuard 文件。例如，当您只发布您正在创建的部分构建时。

<Alert>

您需要指定您正在处理的组织和项目，因为 ProGuard 文件在项目级别工作。有关此的更多信息，请参阅[使用项目](/cli/configuration/#sentry-cli-working-with-projects)。

</Alert>

`upload-proguard` 命令是用于上传 ProGuard 文件的命令。它接受一个或多个 ProGuard 映射文件的路径，并将它们上传到 Sentry。

```bash
sentry-cli upload-proguard \
    app/build/outputs/mapping/{BuildVariant}/mapping.txt
```

由于 Android Sentry SDK 需要知道映射文件的 UUID，因此您需要将其与上传关联。但是，您首先需要将该 UUID 放入 `AndroidManifest.xml` 文件中：

```xml
<application>
  <meta-data android:name="io.sentry.proguard-uuid" android:value="A_VALID_UUID" />
</application>
```

相同的 UUID 需要用于上传映射文件：

```bash
sentry-cli upload-proguard \
    --uuid A_VALID_UUID \
    app/build/outputs/mapping/{BuildVariant}/mapping.txt
```

此外，您可以将 ProGuard 映射文件关联到特定版本：

<Alert>

此步骤不是反混淆工作所必需的，但当存在多个相同应用程序的映射文件时，可以帮助您识别映射文件属于哪个版本。

</Alert>

```bash {5-7}
# 将映射文件关联到版本 my.app.id@1.0.0+1
sentry-cli upload-proguard \
    --uuid A_VALID_UUID \
    app/build/outputs/mapping/{BuildVariant}/mapping.txt \
    --app-id my.app.id \
    --version 1.0.0 \
    --version-code 1
```

上传后，Sentry 反混淆未来事件。
为确保其有效，您可以检查_项目设置 > ProGuard_，查看上传的映射文件是否已列出。

### 上传选项

该命令的 `--help` 输出列出并解释了所有选项：

```bash
sentry-cli upload-proguard --help
```

## JVM 源包

如果您使用的是我们不支持的构建工具，或者您更喜欢不使用 Sentry 的构建工具插件，您将需要手动上传源包文件。

在此处查看我们当前支持的构建工具插件：

- [Android](/platforms/android/enhance-errors/source-context/)
- [Java](/platforms/java/source-context/)

您还可以使用 `sentry-cli` 手动上传 ProGuard 文件，用于任何基于 JVM 的语言，如 Java 或 Kotlin。

<Alert>

在您能够上传源文件之前，您必须配置 Sentry CLI 以指定您正在为其上传包文件的组织和项目。请参阅 CLI 文档中的[使用项目](/cli/configuration/#sentry-cli-working-with-projects)以了解如何执行此操作。

</Alert>

### 创建源包

运行 `debug-files bundle-jvm` 命令为源目录创建源包。

```bash
sentry-cli debug-files bundle-jvm \
    --output some/dir \
    --debug-id A_VALID_UUID \
    path/to/source/dir
```

您必须提供源包的 UUID 供 Java/Android SDK 发送。
更多详细信息可以在 Java 的[源上下文](/platforms/java/source-context/#manually-uploading-source-context)文档和 Android 的[源上下文](/platforms/android/enhance-errors/source-context/#manually-uploading-source-context)文档中找到。

### 上传源包

`debug-files upload` 命令允许您将创建的源包上传到 Sentry。

```bash
sentry-cli debug-files upload \
    --type jvm \
    output/path/of/bundle-jvm/command
```

上传后，Sentry 将源上下文附加到未来事件。

为确保这些步骤有效，请检查 Sentry.io 中的**项目设置 > 调试文件**，查看上传的源包文件是否已列出。

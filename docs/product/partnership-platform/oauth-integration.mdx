# OAuth 集成

Sentry 的 OAuth 集成允许您将现有 Sentry 客户连接到您的平台。这使得用户能够将其 Sentry 组织与您的产품关联，提供对错误监控和性能数据的无缝访问。

## 概述

OAuth 流程支持：

- **账户关联** 在您的平台和特定 Sentry 组织之间
- **基于权限的访问** 具有可配置的 API 范围
- **用户控制的权限** 具有简单的撤销功能
- **无缝身份验证** 无需用户共享凭据

## 用户体验流程

### 1. 账户关联

用户登录您的平台，并看到一个"连接到 Sentry"按钮，其中包含关于您的产将拥有的权限的上下文信息。

### 2. Sentry 身份验证

点击按钮会打开一个 Sentry 登录模态框，用户在其中使用其 Sentry 账户进行身份验证。

### 3. 权限确认

登录后，用户会看到以下详细信息：
- 您的平台所需的权限
- 正在连接的特定 Sentry 组织
- 您的平台可以执行的操作

### 4. 连接完成

确认后，您的平台会收到一个授权码，可以将其交换为 API 令牌。

## 实现步骤

### 1. 合作伙伴注册

在实施 OAuth 之前，Sentry 必须注册您的应用程序。请联系我们的合作伙伴团队，提供以下信息：

- **客户端名称**：您的平台名称
- **重定向 URI**：您的 OAuth 回调 URL
- **允许的来源**：您的域名
- **主页 URL**：您平台的主页
- **隐私政策 URL**：您的隐私政策
- **条款和条件 URL**：您的服务条款
- **所需范围**：您的平台需要访问的 API。请参阅[权限和范围](/api/permissions/)获取范围列表。

### 2. 授权请求

将用户定向到 Sentry 的授权端点：

```
https://sentry.io/oauth/authorize/?client_id=YOUR_CLIENT_ID&response_type=code&scope=SCOPE1%20SCOPE2
```

**参数：**
- `client_id`：您注册的客户端 ID
- `response_type`：始终为 `code`
- `scope`：以空格分隔的所需权限列表

**示例：**
```
https://sentry.io/oauth/authorize/?client_id=0895306f2d310a075e82dff03cae9604c07e2c9465d4b990cc3f4c3010bb31d6&response_type=code&scope=project:read%20project:write
```

### 3. 处理授权码

用户批准后，Sentry 会重定向到您的回调 URL，并附带授权码：

```
https://your-platform.com/oauth/sentry/callback?code=2767ac2e734ce35fe648a5f7b831bbb82776e384bb172fbcea42c292dc195e20
```

### 4. 交换代码获取令牌

发起 POST 请求，将授权码交换为访问令牌：

```python
import requests

TOKEN_URL = "https://sentry.io/oauth/token/"

token_data = {
    "client_id": "YOUR_CLIENT_ID",
    "client_secret": "YOUR_CLIENT_SECRET",
    "grant_type": "authorization_code",
    "code": "AUTHORIZATION_CODE"
}

response = requests.post(TOKEN_URL, data=token_data)
tokens = response.json()
```

**响应：**
```json
{
  "access_token": "8923f2eb3ec0fb9b328ccba6280e48debe119f5b5220273ffa13fb659b2d014a",
  "refresh_token": "74dbc60782aed64814513804e75e5efa6aaa67e3163744c083c0eb77478edb8b",
  "expires_in": 2591999,
  "expires_at": "2024-11-27T23:20:21.054320Z",
  "token_type": "bearer",
  "scope": "org:read project:read",
  "user": {
    "id": "2",
    "name": "user 1",
    "email": "user1@test.com"
  }
}
```

### 5. 刷新令牌

访问令牌每 8 小时过期一次。使用刷新令牌获取新令牌：

```python
refresh_data = {
    "client_id": "YOUR_CLIENT_ID",
    "client_secret": "YOUR_CLIENT_SECRET",
    "grant_type": "refresh_token",
    "refresh_token": "YOUR_REFRESH_TOKEN"
}

response = requests.post(TOKEN_URL, data=refresh_data)
new_tokens = response.json()
```

## API 范围

[范围](/api/permissions/)定义了您的应用程序拥有的权限。常见范围包括：

- `org:read` - 读取组织信息
- `org:write` - 修改组织设置
- `project:read` - 读取项目数据
- `project:write` - 创建和修改项目
- `member:read` - 读取组织成员
- `member:write` - 管理组织成员

所需范围取决于您的用例，并在我们的 [API 文档](/api/) 中有详细说明。

## 使用访问令牌

在 API 请求中使用 Authorization 头包含访问令牌：

```python
headers = {
    "Authorization": f"Bearer {access_token}",
    "Content-Type": "application/json"
}

response = requests.get(
    "https://sentry.io/api/0/organizations/",
    headers=headers
)
```

## 组织范围

Sentry 用户可以属于多个组织。访问令牌仅提供对在 OAuth 流程期间用户选择的特定组织的访问权限。

要获取连接的组织：

```python
# 列出此令牌可访问的组织
response = requests.get(
    "https://sentry.io/api/0/organizations/",
    headers=headers
)
organizations = response.json()

# 响应将仅包括用户连接的组织
```

## 错误处理

### 权限错误 (403)

如果您的应用程序请求用户没有的权限：

```python
if response.status_code == 403:
    # 处理权限不足的情况
    # 您可能需要请求额外的范围或优雅地处理
```

### 令牌过期

通过刷新来处理过期的令牌：

```python
if response.status_code == 401:
    # 令牌已过期，刷新它
    new_tokens = refresh_access_token(refresh_token)
    # 使用新令牌重试原始请求
```

## 用户管理

### 撤销访问权限

用户可以随时通过其 Sentry 账户设置撤销您应用程序的访问权限（设置 → 账户 → 授权应用程序）。

### 处理撤销的访问权限

监控 401 响应并优雅地处理：

```python
if response.status_code == 401:
    # 令牌可能已被撤销
    # 提示用户重新连接或适当处理
```

## 安全最佳实践

1. **安全存储令牌**：在数据库中加密令牌
2. **使用 HTTPS**：始终对 OAuth 流程使用 HTTPS
3. **验证重定向 URI**：确保重定向 URI 与您注册的 URL 匹配
4. **处理令牌过期**：实施自动令牌刷新
5. **监控撤销**：处理用户撤销访问权限的情况

## 示例实现

以下是处理 OAuth 流程的完整示例：

```python
from flask import Flask, request, redirect, session
import requests

app = Flask(__name__)
app.secret_key = 'your-secret-key'

CLIENT_ID = 'your-client-id'
CLIENT_SECRET = 'your-client-secret'
REDIRECT_URI = 'https://your-platform.com/oauth/sentry/callback'

@app.route('/connect-sentry')
def connect_sentry():
    auth_url = (
        f"https://sentry.io/oauth/authorize/"
        f"?client_id={CLIENT_ID}"
        f"&response_type=code"
        f"&scope=org:read%20project:read"
        f"&redirect_uri={REDIRECT_URI}"
    )
    return redirect(auth_url)

@app.route('/oauth/sentry/callback')
def oauth_callback():
    code = request.args.get('code')
    
    # 交换代码获取令牌
    token_data = {
        "client_id": CLIENT_ID,
        "client_secret": CLIENT_SECRET,
        "grant_type": "authorization_code",
        "code": code
    }
    
    response = requests.post("https://sentry.io/oauth/token/", data=token_data)
    tokens = response.json()
    
    # 安全存储令牌
    session['sentry_access_token'] = tokens['access_token']
    session['sentry_refresh_token'] = tokens['refresh_token']
    
    return "成功连接到 Sentry！"

@app.route('/sentry-data')
def get_sentry_data():
    access_token = session.get('sentry_access_token')
    
    headers = {
        "Authorization": f"Bearer {access_token}",
        "Content-Type": "application/json"
    }
    
    # 获取组织数据
    response = requests.get(
        "https://sentry.io/api/0/organizations/",
        headers=headers
    )
    
    if response.status_code == 401:
        # 令牌已过期，刷新它
        refresh_token = session.get('sentry_refresh_token')
        new_tokens = refresh_access_token(refresh_token)
        session['sentry_access_token'] = new_tokens['access_token']
        
        # 重试请求
        response = requests.get(
            "https://sentry.io/api/0/organizations/",
            headers={"Authorization": f"Bearer {new_tokens['access_token']}"}
        )
    
    return response.json()

def refresh_access_token(refresh_token):
    refresh_data = {
        "client_id": CLIENT_ID,
        "client_secret": CLIENT_SECRET,
        "grant_type": "refresh_token",
        "refresh_token": refresh_token
    }
    
    response = requests.post("https://sentry.io/oauth/token/", data=refresh_data)
    return response.json()
```

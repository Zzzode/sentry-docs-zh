# 账户配置 API

Sentry 配置 API 允许合作伙伴平台自动为其客户创建 Sentry 组织。这使您的用户能够无缝地开始使用 Sentry 的错误监控和性能洞察。

## 概述

当您首次代表客户调用 Sentry 配置 API 时，客户将收到：

- **一个 Sentry 组织**，名称与其在您平台上的组织相同
- **一个默认团队**，用于组织内的协作
- **组织所有权**分配给提供的电子邮件地址
- **订阅计划**基于您的合作伙伴协议
- **预配置项目**（可选）您指定的项目
- **集成令牌**（可选）用于以编程方式管理组织

## API 端点

```
POST https://sentry.io/remote/channel-provision/account/
```

## 身份验证

API 需要带有 SHA256 HMAC 签名的自定义 `X-Request-Signature` 标头。签名使用您的 `API_SECRET_KEY` 和请求正文构建。

### 签名生成

```python
import hmac
import hashlib
import json

def generate_signature(data, secret_key):
    secret_key_bytes = secret_key.encode("utf-8")
    json_data = json.dumps(data)
    data_bytes = json_data.encode("utf-8")
    signature = hmac.new(
        key=secret_key_bytes, 
        msg=data_bytes, 
        digestmod=hashlib.sha256
    ).hexdigest()
    return signature
```

## 请求参数

| 参数 | 类型 | 必需 | 描述 |
|-----------|------|----------|-------------|
| `channel` | 字符串 | 是 | 您的合作伙伴名称（不区分大小写） |
| `email` | 字符串 | 是 | 客户的电子邮件地址（成为组织所有者） |
| `organizationName` | 字符串 | 是 | 客户的组织名称（最多50个字符） |
| `organizationID` | 字符串 | 是 | 您平台上客户组织的唯一ID |
| `hasAgreedTerms` | 布尔值 | 是 | 客户对 Sentry 条款的同意 |
| `timestamp` | 整数 | 是 | 当前时间戳用于请求过期 |
| `url` | 字符串 | 是 | 必须是 `https://sentry.io/remote/channel-provision/account/` |
| `projects` | 数组 | 否 | 要创建的项目列表 |
| `region` | 字符串 | 否 | 数据驻留（`us` 或 `de`，默认值：`us`） |
| `isTest` | 布尔值 | 否 | 设置为 `true` 进行测试（默认值：`false`） |

### 项目对象结构

```json
{
  "name": "project-name",
  "platform": "java"
}
```

### 支持的平台

`platform` 字段支持所有 [Sentry 平台标识符](/platforms/)，包括：
- `javascript`、`python`、`java`、`csharp`、`php`、`ruby`、`go`、`rust`、`swift`、`kotlin`、`dart` 等等

## 示例请求

```python
import requests
import json
import time

URL = "https://sentry.io/remote/channel-provision/account/"
API_SECRET_KEY = "您的合作伙伴特定密钥"

DATA = {
    "channel": "您的平台名称",
    "organizationName": "客户公司",
    "organizationID": "唯一客户ID",
    "email": "customer@example.com",
    "projects": [
        {"name": "web-app", "platform": "javascript"},
        {"name": "api-service", "platform": "python"}
    ],
    "hasAgreedTerms": True,
    "timestamp": int(time.time()),
    "url": URL,
    "isTest": False
}

# 生成签名
signature = generate_signature(DATA, API_SECRET_KEY)

# 发出请求
response = requests.post(
    URL, 
    data=json.dumps(DATA), 
    headers={"X-Request-Signature": signature}
)

print(response.json())
```

## 响应

### 成功 (200)

```json
{
  "email": "customer@example.com",
  "organization": {
    "id": "123",
    "slug": "customer-corp",
    "name": "客户公司"
  },
  "projects": [
    {
      "dsn": "https://...@sentry.io/123",
      "project": {
        "id": "456",
        "slug": "web-app",
        "name": "web-app",
        "platform": "javascript"
      }
    }
  ],
  "integration_api_token": "您的集成令牌"
}
```

### 错误响应

- **400 错误请求**：无效参数（检查响应正文以获取详细信息）
- **401 未授权**：无效签名（检查您的 API_SECRET_KEY）
- **500 服务器错误**：内部服务器错误（检查响应正文以获取详细信息）

## 额外操作

### 添加项目

再次调用 API 并添加新项目到现有组织：

```python
DATA = {
    # ... 现有参数 ...
    "projects": [
        {"name": "new-service", "platform": "go"}
    ]
}
```

### 添加管理员

将其他用户添加为组织的管理员：

```python
DATA = {
    # ... 现有参数 ...
    "email": "manager@example.com"  # 新管理员电子邮件
}
```

## 客户体验

### 欢迎邮件

当您配置组织时，客户将收到：
- 包含组织详细信息的欢迎邮件
- 管理其 Sentry 组织的说明
- 关于您的合作伙伴集成的信息
- 支持联系信息

### 合作伙伴特定功能

根据您的合作伙伴协议，您可以启用：

- **合作伙伴存在**：将您的平台显示为组织成员
- **持久计划**：允许客户独立升级计划
- **配额可见性**：控制使用配额的可见性

## 集成令牌

响应包括一个 `integration_api_token`，允许您：
- 创建和管理项目
- 管理组织成员
- 代表客户访问 Sentry API

令牌权限在您的合作伙伴协议中定义。

## 最佳实践

1. **存储组织映射**：跟踪您的客户 ID 和 Sentry 组织 ID 之间的关系
2. **优雅地处理错误**：为失败的配置实施适当的错误处理
3. **使用测试模式**：在开发和测试期间设置 `isTest: true`
4. **尊重速率限制**：在请求之间实施适当的延迟
5. **保护您的密钥**：安全地存储您的 API_SECRET_KEY，切勿在客户端代码中暴露它

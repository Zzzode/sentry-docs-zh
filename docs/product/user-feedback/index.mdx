Sentry 会自动检测应用程序抛出的错误，例如性能问题和用户体验问题，如愤怒点击。但您的用户可能还会遇到其他挫折（权限流程损坏、链接损坏、拼写错误、误导性 UX、业务逻辑缺陷等）。

这就是 Sentry 的用户反馈小组件和崩溃报告模态功能发挥作用的地方。设置其中一个或两个可以让您从最终用户那里收集他们在应用程序中遇到错误时的反馈。然后，Sentry 将该反馈与可用的丰富调试上下文（包括错误、重播和标签，如 URL）链接起来，让您更快地找到问题的根本原因。

例如，将 [会话重播](/product/explore/session-replay/) 与用户反馈小组件结合使用可以节省时间，通过准确查看用户在提交反馈之前所经历的内容，帮助您快速理解用户问题的上下文。

## 用户反馈小组件

用户反馈小组件是一个开箱即用的持久小组件，您可以将其添加到 Web 应用程序的任何位置。它允许用户在遇到任何不按预期工作的情况时快速轻松地提交反馈。了解如何安装和自定义它 [此处](/product/user-feedback/setup/)。

![用户反馈小组件。](./img/user-feedback-widget.png)

通过反馈小组件收集的反馈可能包含以下部分或全部信息（取决于您标记为必需的内容以及错误上下文中可用的内容）：

- 来自用户的描述
- 来自用户的屏幕截图
- 用户电子邮件
- 关联的重播事件（用户提交反馈前最多 60 秒的重播录制）
    - 了解如何设置会话重播 [此处](/product/explore/session-replay/getting-started/)。
- 用户提交反馈时所在页面的 URL
- 标签

## 崩溃报告模态

收集反馈的另一种方法是安装崩溃报告模态。这允许用户在通过错误后出现的自动模态提交反馈。了解如何安装和自定义它 [此处](/platforms/javascript/user-feedback/#crash-report-modal)。

![崩溃报告模态。](./img/crash-report-user-feedback.png)

通过崩溃报告模态收集的反馈将包含以下信息：

- 来自用户的描述
- Sentry 问题预览和问题详情页面的链接
- 用户提交反馈时所在页面的 URL
- 标签

## 用户反馈 API

我们还提供了一个 API 以编程方式发送用户反馈。这使您可以将自己的用户界面连接到 Sentry。有关更多详细信息，请参见 [设置指南](/product/user-feedback/setup/#supported-sdks-for-user-feedback-api)。

## 在哪里查看用户反馈

无论您选择通过反馈小组件、崩溃报告模态还是两者收集用户反馈，您都可以从 **用户反馈** 页面查看它。该页面按时间顺序组织，最新的未解决反馈位于顶部。您可以点击单个反馈以在一个视图中查看更多详细信息，类似于您的电子邮件收件箱。右侧显示 **用户反馈详情** 视图，其中包括用户的描述和任何可用的调试上下文。

您可以使用项目、环境和日期范围过滤器缩小反馈列表中的结果范围。

![用户反馈列表。](./img/user-feedback-list.png)

## AI 驱动的用户反馈摘要

Sentry 现在提供 **AI 驱动的用户反馈摘要**，自动分析您的反馈，让您一目了然地获得可操作的洞察。这种一目了然的摘要可以帮助您理解大量用户反馈，而无需手动分类每条反馈，将压倒性的数量转化为可操作的洞察，并帮助您优先考虑用户体验改进。

### 工作原理

在用户反馈索引的顶部，您将看到一个摘要，突出显示您在所选项目（s）和日期范围内用户的最常见情绪。摘要提供了用户对您产品的评价的简明概述：

![AI 用户反馈摘要。](./img/ai-feedback-summary.png)

## 分类用户反馈提交

您可以通过点击 **用户反馈详情** 视图右上角的"解决"按钮手动解决用户反馈提交，或通过多选反馈进行批量解决：

![用户反馈列表分类。](./img/user-feedback-triaging.png)

您还可以在 **用户反馈详情** 视图中将团队成员分配给特定的用户反馈提交。如果您遇到认为是垃圾邮件的用户反馈，请选择反馈并点击"标记为垃圾邮件"。然后，此反馈将被过滤掉并移至"垃圾邮件"标题下。

## 从用户反馈创建或链接外部问题

如果您已将 GitHub、Jira 或任何其他 [问题跟踪集成](https://sentry.io/integrations/) 与 Sentry 设置，您可以从用户反馈页面创建外部问题或链接现有问题。在 **用户反馈详情** 视图中，您应该在项目名下的标题中看到所有集成。

![用户反馈标题。](./img/user-feedback-header.png)

点击集成名称将打开一个模态，允许您创建新的外部问题或链接现有问题。

![用户反馈链接问题模态。](./img/user-feedback-linked-issue-modal.png)

模态将填充默认标题（"用户反馈"）和描述，其中包含反馈的基本详细信息，包括 Sentry 中反馈的链接和用户的消息。您可以编辑标题和描述，以及配置问题的仓库、标签和受让人。

要创建问题，只需点击模态底部的"创建问题"按钮。要链接现有问题，请点击模态中的"链接"选项卡，然后点击"链接问题"。



## 搜索用户反馈

您可以使用搜索框按反馈进行搜索。例如，它可以让您搜索在特定页面上发送的反馈。

搜索示例：`url:*/payments/*`

有关搜索字段的列表，请参阅 [搜索文档](/concepts/search/searchable-properties/user-feedback/)。

## 用户反馈的垃圾邮件检测

默认情况下，Sentry 自动检测可能是垃圾邮件的用户反馈提交，并将它们放在 **用户反馈** 页面的"垃圾邮件"文件夹中。这提高了主列表中反馈的质量和可操作性。为了提高警报的质量，被标记为垃圾邮件的反馈将绕过您可能配置的任何用户反馈警报规则，您不会收到垃圾邮件的警报。

为了使自动垃圾邮件检测成为可能，我们使用对消息进行分类的大型语言模型。此语言模型在 Google Cloud Platform 内部，不存储或保留任何数据。这是我们现有 [子处理器协议](https://sentry.io/legal/dpa/#subprocessors) 的一部分。

如果您想选择退出垃圾邮件检测，请在 [设置](https://sentry.io/orgredirect/organizations/:orgslug/settings/) > 项目 > [项目名称] > 用户反馈（在"项目"下）中关闭"启用垃圾邮件检测"。



## 获取用户反馈警报

如果您已为设置用户反馈的项目打开 Sentry 的默认问题警报（"在每个新问题上提醒我"），那么您应该自动在每次通过用户反馈小组件传入新用户反馈时收到警报。

如果您没有打开 Sentry 的默认问题警报，请按照以下步骤设置每次新反馈的警报：

1. 在 Sentry 中创建 [新警报规则](https://sentry.io/alerts/new/issue/)。
2. 滚动到"设置条件"部分，并将"IF"过滤器设置为 `问题的类别等于... "反馈"`。
3. 在"THEN"过滤器中选择要执行的操作。
4. 添加警报名称和所有者。

![用户反馈警报配置。](./img/user-feedback-alert-configuration.png)

要在收到崩溃报告反馈时获取通知，请确保在设置 > 项目 > [项目名称] > 用户反馈中打开"启用崩溃报告通知"。

## 用户反馈的自动问题创建

如果您想在每次收到用户反馈时自动创建外部问题（如 GitHub 或 Jira 问题），您可以通过 Sentry 中的警报页面执行此操作，类似于如何为用户反馈创建问题警报（如上节所示）。

请按照以下步骤设置每次新反馈的自动 GitHub 或 Jira 问题创建：

1. 在 Sentry 中创建 [新警报规则](https://sentry.io/alerts/new/issue/)。
2. 滚动到"设置条件"部分，并将"IF"过滤器设置为 `问题的类别等于... "反馈"`。
3. 在"THEN"过滤器操作下，滚动到"创建新..."部分并选择您要使用的集成。
4. 可以通过点击"问题链接设置"按钮配置默认问题链接设置，如仓库、标签和受让人。
5. 添加警报名称和所有者。

配置此警报后，您应该在每次提交用户反馈时看到自动创建的 GitHub 或 Jira 问题。

![用户反馈自动问题警报配置。](./img/user-feedback-auto-issue-creation-configuration.png)

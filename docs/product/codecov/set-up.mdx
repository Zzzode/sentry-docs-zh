如果您同时是 Codecov 和 Sentry 的用户，您可以在 Sentry 应用中的**组织设置**页面中通过 1 次点击启用 Codecov 集成。您将能够直接在 Sentry 的堆栈跟踪中看到导致错误的代码行是否具有部分或完整测试覆盖率。

## 先决条件

为了使 Codecov 在 Sentry 中工作，必须满足一些先决条件。您必须拥有：

- Sentry.io 中的所有者或管理员级别权限。
- 在 Sentry 中安装 [GitHub 集成](/organization/integrations/source-code-mgmt/github/)并启用[代码映射](/organization/integrations/source-code-mgmt/github/#stack-trace-linking)。

  <Alert>
    如果您正确设置了 GitHub 集成，您的堆栈跟踪将
    显示"在 GitHub 中打开此行"提示，将您带到您的应用程序的
    代码。
  </Alert>

- 启用了[源代码覆盖率](https://about.codecov.io/product/feature/source-code-coverage/)的 [Codecov](https://about.codecov.io/) 帐户。

## 启用 Codecov 见解

满足先决条件后，您可以通过转到 Sentry 中的**组织设置**页面并切换"启用代码覆盖率见解"来启用 Codecov。

![组织设置中的 Codecov](./img/codecov-setup-in-new-ui.png)

<Alert>
  Sentry 每 24 小时自动检查一次，看看您在 Codecov 中是否有您的 GitHub 组织。
  如果尚未启用 Codecov 并且我们发现您已满足所有
  上面列出的先决条件，我们将为您打开它。
</Alert>

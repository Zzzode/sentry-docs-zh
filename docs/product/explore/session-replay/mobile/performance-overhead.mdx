移动端会话重放每秒捕获一次视图层次结构快照以及同一帧内的屏幕截图。这些被压缩成代表用户会话部分的视频文件，然后与追踪标识符、面包屑和其他调试信息一起流式传输到 Sentry，以重建完整会话。

为了避免影响主机应用程序性能，Sentry 通过仅在必要时非侵入式地录制屏幕截图和视图层次结构，仅发送必要的字节，并使用地理上接近最终用户的低延迟接收端点，来最小化 CPU 和内存使用。

**对于大多数移动应用程序，我们的会话重放 SDK 的性能开销对最终用户来说是不可察觉的。**

## 会话重放如何优化？

Sentry 的会话重放 SDK 采取了几项措施来避免对运行它的屏幕性能产生负面影响。

- 会话重放 SDK 通过捕获屏幕截图，将它们编码成小的视频段并将内容传输到 Sentry 的服务器来工作。为了减少传输的数据量，采用了多种优化策略：
  - SDK 最多每秒捕获一次屏幕截图，像素密度为 1:1（在 iOS 上通常称为 `@1x`，在 Android 上称为 `mdpi`）。这显著加快了屏幕截图捕获过程，同时保持了可接受的质量。
  - 屏幕截图被压缩成有损 `jpeg` 文件以减少磁盘空间使用，并在崩溃或 ANR 事件发生时能够恢复重放的最后几帧。
  - 当发送重放段时，屏幕截图被进一步压缩成使用 `h.264` 压缩的 `mp4` 视频文件，比特率为 75 kbps（默认行为）。这确保了小的有效负载大小，同时保持视频质量。我们观察到将五个屏幕截图压缩成视频段时大小减少了近 5 倍。
  - 视频段以及其他调试信息（如面包屑）在通过 HTTP 传输之前在客户端上进行 gzip 压缩。此压缩在后台线程上执行，确保应用程序的主线程不受影响。
  - 为了最小化磁盘空间使用，屏幕截图和视频段文件在上传成功后从磁盘中删除。
- Sentry 的事件接收基础设施使用[分布式存在点（PoPs）](https://blog.sentry.io/2022/03/31/sentry-points-of-presence-how-we-built-a-distributed-ingestion/)，将接收服务器放置在世界各地并靠近您的用户。当会话重放事件被传输时，用户的设备连接到并传输事件有效负载到其区域可用的最近 PoP。这大大减少了端到端延迟，并最小化了放置在设备上的网络开销。
- SDK 被设计为在需要时优雅地降级。如果发送太多重放事件并达到 Sentry 的速率限制，SDK 将禁用录制以确保遵守速率限制。一旦速率限制解除，SDK 将继续录制。

## 性能开销的因素

虽然我们的会话重放 SDK 的性能开销在大多数时候对最终用户来说是不可察觉的，但它可能基于应用程序的复杂性而变化。如果应用程序具有深度嵌套视图树的屏幕，与更简单的屏幕（例如，设置）相比，将有更高的开销。

如果启用了默认掩码行为，网络开销将进一步减少，因为屏幕截图具有更少的像素，从而产生更小的文件大小。

移动应用程序有生命周期的不同阶段。每个阶段都有不同的性能特征。以下是生命周期阶段的列表以及您应该注意的最重要的性能指标。

### 应用程序启动

在应用程序启动期间，SDK 几乎没有开销。它仅执行几个轻量级任务，如注册系统事件的必要监听器和调度后台线程进行捕获。

### 主线程

SDK 仅在主线程上执行最低限度的工作来为重放捕获新帧（屏幕截图和视图层次结构快照），然后立即切换到后台线程进行后处理任务。这些任务包括掩码屏幕截图、将它们存储在磁盘缓存中或编码视频段。此过程最多每秒发生一次，根据帧率设置，它甚至可以更不频繁。

SDK 还监视视图树变化。如果自上次帧捕获以来没有发生任何变化，它会重用之前捕获的数据，完全避免主线程上的任何工作。

### 捕获触摸事件

- 点击/触摸、滑动和滚动事件的记录非常小且快速，使开销可以忽略不计。
  - 我们通过去抖动（限制事件频率）和平滑移动事件来优化触摸事件记录。这确保了清晰、详细的调试体验，而不会捕获过多或冗余的数据。

## 测量性能开销

获得准确指标的唯一方法是自行测量性能开销。您可以对您的应用程序应用现实的访问模式，并将其与您自己的顶级业务指标相关联。

我们在生产环境中拥有数百万用户的各种开源应用程序上测量了重放 SDK 的开销。例如，具有大量不同组件和功能的现实世界应用程序，如 [Bluesky](https://github.com/bluesky-social/social-app) 和 [Pocket Casts](https://github.com/Automattic/pocket-casts-android)。您可以通过访问下面的 SDK 文档来检查结果：

- [iOS](/platforms/apple/guides/ios/session-replay/performance-overhead/)
- [Android](/platforms/android/session-replay/performance-overhead/)
- [React Native](/platforms/react-native/session-replay/performance-overhead/)

会话重放允许您查看用户会话的重现，这可以帮助您了解错误发生之前、期间和之后发生了什么。当您回放每个会话时，您将能够看到每个用户交互与网络请求、前端和后端错误、后端跨度等的关系。

重放帮助您准确了解用户如何受到错误的影响。因为它们与我们的问题产品集成，您将能够在 Sentry 的[问题详情](/product/issues/issue-details/)页面上看到连接到错误事件的会话重放。为确保后端错误也包含在重放中，请参见我们的[后端设置说明](/product/explore/session-replay/web/getting-started/#replays-for-backend-errors)。

## 什么是会话重放？

移动端会话重放通过每秒拍摄一次视图层次结构快照以及同一帧内的屏幕截图来工作。这些屏幕截图被压缩成代表用户会话片段的视频文件。所有这些小段然后与额外的追踪标识符、面包屑和其他有用的调试信息一起流式传输到 Sentry，以组成完整的用户会话。

用户会话（重放）持续到用户关闭应用程序、应用程序崩溃、用户将应用程序置于后台超过 30 秒，或会话达到 60 分钟持续时间。结果是视频回放，可以帮助您重现移动应用程序中的错误，包括与后端 API 和微服务相关的难以调试的问题。

为了编辑任何潜在的敏感内容并保护最大用户隐私，视图层次结构用于查找控件（如文本框、图像、标签和按钮）的位置。编辑使用块完成，该块使用控件的最主要颜色绘制在这些控件上。

## 重放详情

每个重放都有一个包含嵌入式视频播放器和丰富调试上下文的详细视图。回放视频将允许您看到每个用户交互与网络请求、前端和后端错误、后端跨度等的关系。此页面上的几乎每个组件都通过时间戳连接。

以下列表显示了每个组件的细分及其价值：

- **面包屑：** 重放面包屑显示关键用户交互发生的时间，具体包括：用户点击相关组件、导航、用户将应用程序置于后台或前台的时间，以及您的组织设置的[自定义面包屑](/product/sentry-basics/integrate-backend/configuration-options/#breadcrumbs)。面包屑还提供对特定时间戳给定用户会话设备的洞察：
      - `设备电池`（当电池电量或充电状态改变时）
      - `方向`（当用户旋转设备时）
      - `连接`（当此状态在 wifi、蜂窝和离线之间改变时）。

这些面包屑与重放播放器同步，并将在视频播放时自动滚动。[问题详情](/product/issues/issue-details/breadcrumbs/)中可见的某些[面包屑类型](/product/issues/issue-details/breadcrumbs/)不是重放面包屑列表的 1:1。在重放详情页面上，通常在问题详情页面中看到的事件轨迹改为显示在网络和控制台组件中。


- **时间线：** 这是重放详情页面底部的部分，说明重要事件（如错误、设备电池和用户交互）在重放过程中的发生位置。这允许用户通过拖动时间线轻松擦除到关键事件。它还直观地传达了事件之间经过的时间量，并具有缩放功能，因此您可以轻松放大以区分紧密发生的事件。

- **网络：** 这是应用程序在重放录制处于活动状态时发起的所有网络请求的列表。当视频播放时，有一个视觉指示器跟踪网络请求表，突出显示在此点之前或之后发生的请求。当请求失败时，它以红色突出显示。您还可以点击每个请求最右侧的时间戳，将您带到重放播放器中的该点。

- **控制台：** 一些不属于面包屑列表的调试消息将显示在这里。例如，React Native 中的自定义 `console.log`。[Logcat](/platforms/android/integrations/logcat/) 和 [Timber](/platforms/android/integrations/timber/) 的日志也得到支持，并将显示在这里。

- **错误：** 重放中发生的所有错误（包括您的后端），带有到相应事件和[问题](/product/issues/issue-details/error-issues/)的链接，以及当您悬停在问题 ID 上时，这些问题对您的应用程序所有用户的整体影响。

- **标签：** 与重放关联的内置字段和自定义标签的完整列表，如操作系统版本和名称、设备规格、发布和用户详细信息。

- **追踪：** 连接重放期间发生的所有[追踪](/product/sentry-basics/tracing/distributed-tracing/#traces-transactions-and-spans)的视图。

## 支持的 SDK

移动端会话重放目前可用于 Android 和 iOS 的原生 SDK，以及 React Native 和 Flutter。

我们建议您更新到最新版本，但支持的最低版本是：

- [iOS](/platforms/apple/guides/ios/session-replay/), [8.43.0](https://github.com/getsentry/sentry-cocoa/releases)
- [Android](/platforms/android/session-replay/), [7.20.0](https://github.com/getsentry/sentry-java/releases)
- [React Native](/platforms/react-native/session-replay/), [6.5.0](https://github.com/getsentry/sentry-react-native/releases)
- [Flutter](/platforms/dart/guides/flutter/session-replay/), [8.12.0](https://github.com/getsentry/sentry-dart/releases)

确保您已在后端项目中设置[追踪传播](/product/sentry-basics/concepts/tracing/#trace-propagation)。例如：

```swift {tabTitle:Swift}
import Sentry

SentrySDK.start { options in
    options.dsn = "___PUBLIC_DSN___"
    options.tracePropagationTargets = ["MyAppDomain.com"]
}
```

```objc {tabTitle:Objective-C}
@import Sentry;

[SentrySDK startWithConfigureOptions:^(SentryOptions *options) {
    options.dsn = @"___PUBLIC_DSN___";
    options.tracePropagationTargets = @[@"MyAppDomain.com"]
}];
```

```java {tabTitle:Java}
SentryAndroid.init(context, options -> {
    final ArrayList<String> tracePropagationTargets = new ArrayList<>();
    tracePropagationTargets.add("https://api.myproject.org/");

    options.setDsn("___PUBLIC_DSN___");
    options.setTracePropagationTargets(tracePropagationTargets);
});
```

```kotlin {tabTitle:Kotlin}
SentryAndroid.init(context, { options ->
    options.setDsn("___PUBLIC_DSN___")
    options.setTracePropagationTargets(listOf(
        "https://api.myproject.org/"
    ))
})
```

```javascript {tabTitle:JavaScript}
Sentry.init({
  dsn: "___PUBLIC_DSN___";
  tracePropagationTargets: ["https://myproject.org", /^\/api\//],
});
```

## 常见问题

**您如何保护用户数据？**

Sentry 以隐私为重点，因此保护用户数据始终是首要考虑。这就是为什么默认情况下，我们的隐私配置非常激进，并编辑所有文本和图像以确保不收集敏感用户信息。我们还提供服务器端清理调试上下文，以在服务器上额外过滤作为额外预防措施。

此外，我们在 UI 中提供自助删除单个重放的功能。

**重放包的性能开销是多少？**

对于大多数移动应用程序，我们的客户端 SDK 的性能开销对最终用户来说是不可察觉的。在我们自己的测试中，开销对最终用户来说并不明显。但是，此测试并不详尽，您可能会发现录制开销可能会对您的移动应用程序性能产生负面影响，具体取决于您的应用程序复杂性。

为了减少性能开销，我们只在屏幕上发生变化时拍摄屏幕截图。我们的默认帧率为每秒 1 帧。
如果您在安装会话重放后遇到任何性能下降，请在 GitHub 上为您的相应 SDK 打开一个问题：[Android](https://github.com/getsentry/sentry-java/issues/new/choose)、[iOS](https://github.com/getsentry/sentry-cocoa/issues/new/choose)、[React Native](https://github.com/getsentry/sentry-react-native/issues/new/choose) 和 [Flutter](https://github.com/getsentry/sentry-dart/issues/new/choose)。

**使用移动端会话重放的费用是多少？**

移动端会话重放目前可用于所有计划。查看我们的[定价页面](https://sentry.io/pricing/)了解详情。

**未处理的异常（例如崩溃）是否作为移动端会话重放的一部分可用？**

是的，移动端会话重放涵盖处理过的和未处理的异常以及 ANR/应用程序挂起和原生（NDK）崩溃。确保使用最新 SDK 版本，如[支持的 SDK](#sdks-supported)中所述。

**如果我的应用程序离线，移动端会话重放如何工作？**

移动端会话重放目前仅在指定 `onErrorSampleRate` 时支持离线模式。当发生错误时，SDK 将捕获它录制到该时间的段（最多 30 秒）并将其存储到磁盘。当应用程序重新上线时，该段将与错误一起发送到 Sentry。

目前 `sessionSampleRate` 在离线模式下不受支持。请订阅此 [GitHub 问题](https://github.com/getsentry/sentry/issues/68287)以获取更新。

Sentry 中的 [**追踪浏览器**](https://sentry.io/orgredirect/organizations/:orgslug/traces/) 旨在使性能调查更容易和更直观。您现在可以探索跨度样本、可视化跨度属性，并使用灵活的查询和过滤器聚合数据。本指南将引导您了解追踪浏览器的关键概念和功能。

![追踪浏览器截图](./img/Trace_Explorer_home.png)

## 关键概念

### 追踪

- 追踪表示通过单个系统的一系列事务或请求。这包括用户浏览器会话、HTTP 请求、数据库查询、中间件、缓存等内容。
- 它捕获一系列操作（_跨度_），显示应用程序的不同部分在每个事务期间如何交互。
- 每个追踪都由一个 **追踪 ID** 标识，您可以使用它来跨服务或项目跟踪请求。
- [分布式追踪](/platforms/javascript/tracing/trace-propagation/#what-is-distributed-tracing) 由遍历多个服务器、客户端、边缘工作器和系统其他部分的事务和跨度组成。它通过浏览器标头传递追踪 ID 来组装。

### 跨度

- _跨度_ 表示追踪中的单个操作。这可能是数据库查询、HTTP 请求或 UI 渲染任务。
- 每个跨度都有：
  - **属性**：键值对，如 `http.method`、`db.query`、`span.description` 或自定义属性，如 `cart.value`，提供可能对调试和调查模式有用的额外上下文。这些是数字或字符串。注意：数字跨度属性可用于计算 **跨度指标**，如下所示。
  - **持续时间 (`span.duration`)**：操作所花费的时间，用于衡量性能。

### 跨度指标

- [跨度指标](/concepts/key-terms/tracing/span-metrics/) 是通过对您的跨度属性（默认或自定义）应用函数得出的，如 `p50(cart.value)` 或 `sum(ai.token_use)`，在细粒度时间范围内。此计算推断出指标，然后填充仪表板和警报，基于该滚动时间窗口。
- 这些"指标"不是与您的跨度数据分开存储的。相反，它们是即时查询的。这减少了您管理、接收和存储自定义指标的需求。

## 实际示例

- **查看您的更改**：搜索 `span.domain is localhost:XXXX` 并按时间戳排序以查看来自您的开发环境的追踪。
- **诊断慢页面**：搜索 `span.op:navigation` 并可视化 `p90(span.duration)` 以查明慢页面加载。
- **查找有问题的 API 调用**：按 `http.url` 聚合并过滤 `avg.(span.duration) > 2s` 以识别慢外部 API 调用。
- **数据库查询分析**：按 `db.query` 聚合并按 `avg(span.duration)` 排序以查找低效查询。

使用这些工具，新的追踪浏览器为您提供了强大的方式来了解应用程序的性能、识别瓶颈并进行明智的优化。

## 浏览跨度和追踪样本

追踪浏览器允许您浏览跨度和追踪样本并检查它们。方法如下：

1. **搜索样本**：
   - 使用搜索栏根据默认或自定义属性、标签和字符串（如 `http.request_method is "GET"`）过滤结果
   - **编辑和添加**跨度样本表中的列以查看任何属性
   - **排序**通过点击列标题在升序和降序之间切换
   - 根据您的焦点在 **跨度样本** 和 **追踪样本** 选项卡之间切换。
1. **检查样本**：

   - 点击 **跨度 ID** 或 **追踪 ID** 打开 **追踪瀑布视图**，显示操作序列及其持续时间。
   - 您可以详细探索单个跨度或查看追踪的整体结构。

## 可视化跨度属性

追踪浏览器使可视化从任何跨度属性聚合的指标变得容易。

1. **设置可视化**：
   - 使用 **可视化** 部分通过点击 **添加系列** 或 **+** 图标将 `avg(span.duration)` 或 `p90(span.duration)` 等指标添加到同一图表或不同图表。
   - 您还可以创建自定义可视化，如：
     - **跨度持续时间的 P90**：用于识别异常值。
     - **平均缓存大小**：分析特定于后端跨度的属性。
2. **过滤您的数据**：
   - 通过在搜索栏中应用过滤器（如 `span.op:db` 专注于数据库跨度或 `http.status_code:500` 用于错误）来细化您的可视化。

折线图在您应用过滤器时动态更新，帮助您跟踪趋势并识别异常。

## 聚合跨度数据

使用追踪浏览器中的聚合功能深入了解您的数据。

1. 点击进入 **聚合** 选项卡。

2. **按属性分组**以调查模式。

   - 尝试 `http.method` 比较 GET 与 POST 性能，`db.table` 查找慢数据库操作，`span.op` 调查追踪中的模式，或自定义标签，如购物车价值、CPU 或内存使用，或您能想到的任何其他内容。

3. **过滤和排序您的聚合**就像您对事务所做的那样，通过在搜索栏中查询。通过点击列标题（默认：`timestamp`）对聚合表进行排序。

4. **查看样本追踪**通过点击表格行左侧的堆栈图标来适合您的聚合。使用此功能查找异常值、调试问题并找到复杂的性能优化。

## 采样如何影响追踪浏览器中的查询

您可以使用 Sentry 的 SDK 对跨度进行采样并控制发送到 Sentry 的数据量。启用采样时，并非每个跨度都会被捕获，但 Sentry 仍将通过推断采样数据为您提供准确的聚合结果。

### 工作原理

发送到 Sentry 的每个跨度都包括其 **采样率**。我们还推导出 **采样权重**，这是速率的倒数，表示此样本代表的原始跨度数量。

例如：

- 在速率为 10% 时，权重为 10。
- 在速率为 50% 时，权重为 2。
- 在速率为 100% 时，权重为 1。

采样后发送给我们的所有跨度都会被存储。

当您在追踪浏览器中运行查询时，Sentry 使用采样权重来计算计数、平均值和百分位数。

### 示例

假设您查询 `span.op:db` 的跨度并在结果中获得 **2 个跨度**：

| span_id | span.op | span.duration | sampling_factor | sampling_weight |
| --- | --- | --- | --- | --- |
| a1b2c3 | db | 100ms | 0.1 | 10 |
| d4e5f6 | db | 200ms | 0.5 | 2 |

### 计数

由于每个跨度都以不同的权重采样，Sentry 推断出 **计数**：

```
10 + 2 = 12 估计跨度
```

追踪浏览器将显示 **计数：12**。

### 总和

类似地，Sentry 将权重纳入 **总和**：

```
10 * 100ms + 2 * 200ms = 1.4s
```

追踪浏览器将显示 **总和：1.4ms**。

### 平均值和百分位数

平均值和百分位数是 **加权的**。您不只是平均 2 个跨度——Sentry 考虑了它们代表更大数据集的事实。例如，加权平均值的工作方式如下：

```
(100ms * 10 + 200ms * 2) / (10 + 2) = 117ms（这是显示的平均持续时间）
```

Sentry 将其视为所有 12 个估计跨度的平均值表示。您可以相信 `avg()` 和 `p90()` 等指标反映了您服务的真实健康状况，已针对采样进行了调整。

### 采样警告

如果您的采样率 **太低** 而无法进行准确推断（例如，保留已经很少跨度的不到 5%），Sentry 可能会显示 **低采样率警告**。这让您知道推断可能不太可靠，特别是对于更具限制性的查询。

![追踪浏览器截图](./img/Trace_explorer_low_samples.png)

即使您对跨度进行采样，**追踪浏览器的推断也会为您提供相当准确的答案** 用于计数、平均值和百分位数——无需您做任何额外的事情。

使用这些工具，新的追踪浏览器为您提供了强大的方式来了解应用程序的性能、识别瓶颈并进行明智的优化。

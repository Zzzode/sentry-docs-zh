在**Discover**中，您可以基于查询列添加方程。每列就像一个变量，因为它代表查询的数值结果，这个数值在方程中使用。

例如，让我们看一个计算事务首次内容绘制（`measurements.fcp`）与事务最大内容绘制（`measurements.lcp`）比率的方程：

方程 = `measurements.fcp / measurements.lcp`

方程将使用这些查询列的结果来计算比率。在这种情况下，这些查询列的结果是：

- measurements.fcp = 50
- measurements.lcp = 100

方程变为：

`50 / 100`

方程的结果然后显示在更新的表格中，如下面的录制所示：

![在 discover 中添加方程](./img/add_equation.gif)

这些方程可以帮助您计算以下内容：

- [在阈值内完成的事务百分比](#calculate-the-percentage-of-transactions-completed)
- [自定义 Apdex 分数](#calculate-a-custom-apdex)
- [最大的平均非数据库时间](#find-largest-average-non-database-times)

## 添加方程

要向查询添加方程

1. 点击"列"按钮打开列编辑器。
2. 如果它们尚未包含在您的查询中，请添加方程所需的列。
3. 点击"添加方程"。
4. 通过选择列、输入数字（如果需要）和添加运算符来输入您的方程。
5. 点击"应用"。

## 方程指南

方程必须：

- 包含至少一个字段或函数
- 包含至少一个运算符
- 具有显式运算符。例如，`a(a+b)` 无效；相反，您应该输入 `a * (a + b)`

方程不能：

- 包含字段和函数的混合
- 包含不支持的指数

方程将遵循预期的运算顺序，首先解析括号，然后是除法和乘法，接着是加法和减法。方程还可以包括负数。

方程只能在数值列和函数上操作，包括：

- 函数：
  - count
  - count_unique
  - count_if
  - count_web_vitals
  - failure_count
  - avg
  - sum
  - 百分位数（p50、p75 等）
  - apdex
  - user_misery
  - eps() 和 epm()
- 字段
  - transaction.duration
  - measurements.fp
  - measurements.fcp
  - measurements.lcp
  - measurements.fid
  - measurements.cls
  - measurements.ttfb
  - spans.browser
  - spans.http
  - spans.db
  - spans.resource

## 方程示例

### 计算完成的事务百分比

获取在阈值内完成的事务的百分比

1. 创建具有以下三列的新查询：
   - 列 1：
     - 函数 `count_if`
     - 字段 `transaction.duration`
     - 条件 `小于或等于`
     - 将最终参数设置为阈值（如 300）
   - 列 2：
     - 函数 `count`
   - 列 3：
     - 字段 `transaction`
2. 添加方程并在方程字段中输入 `count_if(transaction.duration,lessOrEquals,300) / count() * 100`。

### 计算自定义 Apdex

[Apdex](/product/insights/overview/metrics/#apdex) 定义为 `(满意 + 可容忍*0.5 + 令人沮丧*0)/总数`。如果您想将可容忍的权重更改为 0.2，您将创建以下查询和方程。

1. 创建具有以下四列的新查询：
   - 列 1：
     - 函数 `count_if`
     - 字段 `transaction.duration`
     - 条件 `小于或等于`
     - 将最终参数设置为满意阈值（如 300）
   - 列 2：
     - 函数 `count_if`
     - 字段 `transaction.duration`
     - 条件 `小于或等于`
     - 将最终参数设置为可容忍阈值（如 1200）
   - 列 3：
     - 函数 `Count()`
   - 列 4：
     - 字段 `transaction`
2. 添加方程并在方程字段中输入以下内容：

   `(count_if(transaction.duration,lessOrEquals,300) + (count_if(transaction.duration,lessOrEquals,1200) - count_if(transaction.duration,lessOrEquals,300)) * 0.2)/count()`

### 查找最大的平均非数据库时间

查找事务中不是来自数据库调用的最大时间。

1. 创建具有以下两列的新查询：
   - 列 1：
     - 函数 `avg`
     - 字段 `transaction.duration`
   - 列 2：
     - 函数 `avg`
     - 字段 `spans.db`
2. 添加方程并在方程字段中输入 `avg(transaction.duration) - avg(spans.db)`。
3. 按此新方程排序。

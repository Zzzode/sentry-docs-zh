从**Discover**页面，您可以通过三种方式构建查询。

- 点击"构建新查询"
- 点击现有已保存查询卡片上的省略号以"复制"
- 进入任何现有查询
  1. 点击右上角的"另存为..."
  2. 输入新的显示名称
  3. 点击"保存"

有五个主要构建块会影响已保存查询的结果，可以组合使用这些构建块来缩小搜索范围：

1. 数据集选择
2. 项目、环境和日期范围过滤器
3. 搜索条件
4. 交互式图表
5. 表格列

## 数据集选择

数据集选择器允许您查询错误或事务。一次只能查询一个数据集。错误数据集返回与错误相关的事件，而事务数据集返回事务事件，帮助您查看应用程序性能。

## 按项目、环境和日期范围过滤

这些常用过滤器允许您过滤项目和环境，以及指定您想要缩放的日期范围。

## 按搜索条件过滤

所有事件都有内置的关键字段或自定义标签。使用搜索栏输入这些键并为其赋值。这将过滤您的事件列表。有关内置关键字段的列表，请查看我们的[可搜索属性文档](/concepts/search/searchable-properties/)。

### 语法

查询构建器语法与[Sentry 的搜索语法](/concepts/search/)相同。在您输入上述关键字段或自定义标签后，您可以使用任何引用的语法。例如，`count()` 给出事件发生的次数。这可以用以下方式编写：

- 精确匹配（等于）：`count():99`
- 上限（小于或等于）：`count():<99` 或 `count():<=99`
- 下限（大于或等于）：`count():>99` 或 `count():>=99`
- 多个界限（大于且小于）：`count():>10 count():<20`

在过滤器之间使用 `OR` 和 `AND` 搜索条件。但是 `OR` 不能在聚合和非聚合过滤器之间使用。有关这些条件的更多详细信息，请参阅[使用 `OR` 和 `AND`](/concepts/search/#using-or-and-and)。

您还可以通过将值放入列表中来搜索同一键的多个值。例如，"x:[value1, value2]" 将找到与 "x:value1 `OR` x:value2" 相同的结果。您可以在我们涵盖[同一键的多个值](/concepts/search/#multiple-values-on-the-same-key)的内容中了解更多信息。

### 标签摘要过滤器

每个事件都有一个标签值列表。标签摘要（或方面图）是按频率排序的前 10 个键的可视化。最常见的标签值直接列在条形图上方描述和百分比中。将鼠标悬停在条形图中的每个部分上，可以看到该标签的确切分布。

点击这些部分中的任何一个以进一步细化您的搜索。例如，点击代表浏览器条形图中 Chrome 的部分将自动更新标签摘要，然后将 Chrome 添加到搜索条件中。

![事务和已处理标签的方面图。图看起来像一个带有渐变颜色的条形图。](./img/discover-facet-map.png)

<Alert>

在事务事件上执行搜索时，根据您的搜索条件
和采样率，可用的事件可能会受到限制，因为 Discover 仅使用

## 按交互式图表过滤

每个查询都有一个交互式图表，反映下面表格中呈现的数据。要放大，只需在您想要调查的区域上单击并拖动。您还可以自定义显示和 Y 轴。这将与您的查询一起保存。

### 间隔选项

"间隔"选择器允许您更改图表中的日期分组。选择器有限制，这些限制基于当前选择的时间范围。这意味着，例如，如果您查询超过 90 天，您可以选择的最大间隔是 45 天，最小间隔是一小时。

### 显示选项

- 总期间
- 上一期间
- 发布标记
- 顶级期间
- 总每日
- 顶级每日
- 条形图

### 限制选项

当"显示"选择为"顶级期间"或"顶级每日"时，可以设置可选限制。该设置默认为五，显示图表中的前五个结果。

### Y 轴选项

- 计数
- 平均事务持续时间
- p75
- p95
- 唯一用户计数

当您的"显示"选择为"总期间"、"上一期间"、"总每日"或"条形图"时，您可以通过在"Y 轴"下拉列表中选择多个复选框来设置最多三个 y 轴。

## 使用表格列分组和堆叠

在表格上方，点击"列"打开显示结果表格中所有列列表的模态框。您可以添加、删除和移动结果分组所依据的基本关键字段列或自定义标签列。

### 堆叠函数

您还可以将以下函数作为列添加来堆叠事件，具体取决于所选数据集：

#### 错误
- `count()`
- `count_if(...)`
- `count_unique(...)`
- `eps()`
- `epm()`

#### 事务
- `any(...)`
- `apdex(...)`
- `avg(...)`
- `count()`
- `count_if(...)`
- `count_miserable(...)`
- `count_unique(...)`
- `count_web_vitals(...)`
- `epm()`
- `eps()`
- `failure_count()`
- `failure_rate()`
- `last_seen()`
- `max(...)`
- `min(...)`
- `percentile(...)`
- `p50(...)`
- `p75(...)`
- `p95(...)`
- `p99(...)`
- `p100(...)`
- `sum(...)`
- `user_misery(...)`

每个函数都会要求您分配一个参数。有些是必需的，有些是可选的。函数将根据相同的值堆叠事件。如果没有应用函数，则"查询结果"表格中的事件将保持单独列出。完成编辑列后，点击"应用"，结果将反映在查询结果中。请记住，如果添加了许多列，表格可能会水平滚动。

### 单元格过滤器

表格中的每个单元格在悬停时都会有一个省略号出现。这将打开一个上下文菜单，根据值类型提供额外的过滤功能。例如，您可以通过点击"添加到过滤器"选择性地将事务定位到搜索条件栏。

![显示添加到过滤器或从过滤器中排除的额外选项的菜单](./img/discover-table-cell.png)

### 添加方程

您还可以使用表格列作为方程变量将方程添加到查询中，结果将显示在查询结果表格中。在[添加查询方程](query-equations/)中了解更多信息。

### 排序列

点击您想要排序的列标题。向下箭头按降序对列进行排序，向上箭头按升序对列进行排序。这可能会刷新表格。

## 其他查询操作

### 编辑查询

如果您需要编辑这些查询中的任何一个，请进入查询，进行所需的更改，右上角将出现一个按钮询问您是否保存/更新查询。请记住，对查询条件的编辑**不会**自动保存。

要重命名已保存的查询，请点击标题旁边的铅笔图标并输入所需的显示名称。点击"回车"或区域外部以确认。

### 共享查询

根据需要共享您的查询。您可以与也具有访问同一组织权限的其他用户共享 URL。随着查询的每个部分被构建，结果会更新，URL 也会更新，以便可以通过电子邮件、聊天等方式共享进行中的搜索。

### 导出 CSV

如果您想将数据带到其他地方，请点击"导出"获取 CSV 文件。

对于大型导出，您将收到一封包含下载链接的电子邮件，一旦准备就绪。这将需要在 Sentry 下载页面上进行进一步验证。这包含文件的 `SHA1` 校验和，您可以使用以下命令验证其完整性：

```bash
echo "<SHA1 校验和> <下载的 CSV 名称>" | sha1sum -c -
```

<Alert>
  根据数据量，等待时间可能会有所不同。结果
  限制为 1000 万行或 1GB，以先到者为准。
</Alert>

### 删除查询

<Alert>删除已保存的查询是不可逆的。</Alert>

在**Discover**页面上，每个已保存的查询卡片都有一个省略号，将打开一个上下文菜单。从这里删除查询。您还可以通过点击右上角的垃圾桶在查询结果视图中删除查询。

有关如何查看已保存查询的更多详细信息，请参阅[Discover 的完整文档](/product/explore/discover-queries/)

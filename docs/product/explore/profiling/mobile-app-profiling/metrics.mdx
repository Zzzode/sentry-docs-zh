Sentry 性能分析器记录几个系统测量值来帮助分析它收集的回溯。一些，如 CPU 和堆使用情况，使用来自 `mach/mach.h` 的函数以常规采样间隔获取，尽管频率低于回溯采样。其他，如 GPU 信息，从 `CADisplayLink` 回调调用中获取，因为它们被接收。

## CPU 使用率

应用程序进程使用的 CPU 量（作为总 CPU 容量的缩放百分比，根据核心数量而变化）每 100 毫秒测量一次。该值使用带有 `THREAD_BASIC_INFO` 的 `thread_info` 计算，它返回每个线程的 CPU 使用值，并对所有当前线程的值求和。

## 堆使用情况

应用程序使用的堆内存量每 100 毫秒使用带有 `TASK_VM_INFO` 的 `task_info` 记录一次。

## GPU 信息

除了计算[慢速和冻结 UI 帧渲染](/product/insights/mobile/mobile-vitals/#slow-and-frozen-frames)的数量用于移动洞察之外，Sentry 现在还记录每个帧的时间戳，并将其叠加在性能分析火焰图之上：

![Sentry 在特定性能分析的火焰图上方显示慢速和冻结帧。](./img/flamechart-with-gpu-overlay.png)

性能分析器在启动时记录当前帧率，然后在它改变时记录（例如，如果用户将设备置于低功耗模式）。

## 能源使用

分配给应用程序进程的 CPU 工作所消耗的能源量每 100 毫秒使用带有 `TASK_POWER_INFO_V2` 的 `task_info` 计算一次。

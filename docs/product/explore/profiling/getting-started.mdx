连续性能分析和 UI 性能分析是 Sentry 性能分析功能的最新迭代，但目前仅在下面描述的选定 SDK 中受支持。

其他平台通过之前的基于事务的性能分析产品得到支持，但这些产品将无法受益于连续和 UI 性能分析引入的新功能（对性能分析生命周期的直接开始/停止控制和移除持续时间限制）。有关基于事务的性能分析与连续/UI 性能分析之间差异的更多信息，请阅读[此文档](/product/explore/profiling/transaction-vs-continuous-profiling)。

如果您当前正在使用基于事务的性能分析，并且想要在支持的 SDK 上迁移到连续性能分析或 UI 性能分析，请阅读[迁移指南](/product/explore/profiling/continuous-ui-profiling-migration-guide)。

所有当前支持基于事务的性能分析的 SDK 都将随着时间的推移迁移以支持连续性能分析和 UI 性能分析。

## 支持的 SDK

### 连续性能分析

<Alert>

连续性能分析既可以独立使用，也可以作为追踪产品的补充。

</Alert>

- 独立和服务器应用
  - [Node.js](/platforms/javascript/guides/node/profiling/#enable-continuous-profiling)
  - [Python](/platforms/python/profiling/#enable-continuous-profiling)

### UI 性能分析

<Alert>

UI 性能分析既可以独立使用，也可以作为追踪产品的补充。浏览器 JavaScript 支持即将推出。

</Alert>

- 移动端
  - [Android](/platforms/android/profiling/#continuous-profiling)
  - [iOS 和 macOS](/platforms/apple/guides/ios/profiling/#continuous-profiling)

### 基于事务的性能分析

<Alert>

如果您的 SDK 不支持连续性能分析或 UI 性能分析，您可以回退到下面平台的旧版基于事务的性能分析实现。

基于事务的性能分析需要事先启用 Sentry 的追踪产品。要在 SDK 中启用追踪和性能监控功能，请查看我们的[洞察指南](/product/insights/)。

</Alert>

- [浏览器 JavaScript](/platforms/javascript/profiling/) [测试版]
- 移动端
  - [React Native](/platforms/react-native/profiling/) [测试版]
  - [Flutter](/platforms/dart/guides/flutter/profiling/) [实验版，仅限 iOS 和 macOS]
  - [.NET](/platforms/dotnet/guides/maui/profiling/) [实验版，仅限 iOS]
- 独立和服务器应用
  - [PHP](/platforms/php/profiling/)
  - [Ruby](/platforms/ruby/profiling/) [测试版]
  - [.NET](/platforms/dotnet/profiling/) [实验版]

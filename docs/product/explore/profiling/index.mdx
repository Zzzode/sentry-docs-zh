Sentry 的性能分析产品能够在生产环境中为应用程序执行提供精确的代码级可见性。

## 何时应该使用性能分析？

性能分析帮助您了解应用程序在**函数级别**上花费时间的位置。虽然[追踪](/concepts/key-terms/tracing/)可以帮助您捕获有关消耗时间的更高级别操作（HTTP 请求、数据库查询、UI 渲染等）的信息，但性能分析通过可视化每个线程的调用堆栈来帮助您调试性能问题，贯穿整个应用程序执行过程。

这在您缺少明确插桩（跨度 span 和日志）来帮助您缩小问题发生位置的情况下特别有用。性能分析*不需要额外的插桩*，会自动捕获应用程序运行时最耗时的函数和代码行的信息。

性能分析帮助您快速识别性能瓶颈，使您能够从第一天开始就将[性能作为功能](https://blog.codinghorror.com/performance-is-a-feature/)构建进来。

## 性能分析产品

<Alert>

**当前支持的平台：**

- [Android（仅限 Java 和 Kotlin）](/platforms/android/profiling/)
- [iOS 和 macOS（仅限 Swift 和 Objective-C）](/platforms/apple/profiling/)
- [Python](/platforms/python/profiling/)
- [Node.js](/platforms/javascript/guides/node/profiling/)
- [PHP（包括 Laravel 和 Symfony）](/platforms/php/profiling/)
- [浏览器 JavaScript [测试版]](/platforms/javascript/profiling/)
- [Ruby [测试版]](/platforms/ruby/profiling/)
- [React Native [测试版]](/platforms/react-native/profiling/)
- [Flutter [实验版，仅限 iOS 和 macOS]](/platforms/dart/guides/flutter/profiling/)
- [.NET [实验版]](/platforms/dotnet/profiling/)

</Alert>

Sentry 支持两种产品的性能分析，适用于后端和前端用例：
* **连续性能分析**能够*持续*分析后端服务，找到导致过度资源使用和高基础设施成本的代码路径。
* **UI 性能分析**能够分析浏览器和[移动](/product/explore/profiling/mobile-app-profiling/)应用中的关键用户流程（或整个用户会话），找到导致滚动和动画卡顿、高交互延迟以及加载时间缓慢的代码路径，这些都会影响用户体验。

## 性能分析页面

在[**性能分析**](https://sentry.io/orgredirect/organizations/:orgslug/profiling/)页面上，您将找到所选项目中最慢函数的高级指标，以及在所选时间范围内性能回归或改进的任何函数。

![性能分析页面上的最慢函数和回归最多函数](./img/profiling-page-functions.png)

在**最慢函数**小组件上，您可以点击每个函数上的省略号按钮(...)来查看包含该函数的示例性能分析列表。在**回归最多/改进最多函数**小组件上，您可以点击变更前后的函数持续时间，查看包含该函数的变更前后示例性能分析。

在函数指标下方，您将找到两个选项卡：**事务**和**聚合火焰图**。

如果启用了[追踪](/concepts/key-terms/tracing/)，**事务**显示所选项目中所有已插桩事务的列表，允许您按端点、路由、用户流程等细分性能分析数据。它为您提供了应用程序*特定*部分内性能的更集中视图。

**聚合火焰图**选项卡*不需要*启用追踪，将为您显示所选项目内所有最频繁代码路径的高级视图。这为您提供了在服务或应用程序中总体上花费时间的更一般视图。

### 事务选项卡
<Alert>
如果未为至少一个所选项目启用追踪，事务列表将不会填充。
</Alert>

**事务**选项卡默认显示所选项目中所有事务的列表，按事务的频率（计数）排序。列表还可以通过点击表中的列标题按持续时间（p50/75/90/95）、项目、时间戳等进行排序。事务列表还可以通过使用搜索栏指定查询进行过滤。例如，您可以搜索来自特定用户的事务或标记有自定义属性的事务。

![性能分析页面上的事务选项卡](./img/profiling-page-transactions.png)

点击事务名称导航到**事务摘要**页面，其中包含限定于该事务的聚合性能数据。默认情况下，**性能分析**选项卡被选中，其中包含一个[聚合火焰图](/product/explore/profiling/flame-charts-graphs)，显示在该事务内性能分析的最频繁代码路径。此火焰图的行为与下面描述的**聚合火焰图**选项卡中的火焰图相同。

![事务摘要页面上的性能分析选项卡](./img/transaction-summary-profiles.png)

### 聚合火焰图选项卡

**聚合火焰图**选项卡显示一个[火焰图](/product/explore/profiling/flame-charts-graphs)，其中包含有关所选项目内代码路径频率的聚合信息。

![性能分析页面上的聚合火焰图选项卡](./img/profiling-page-aggregate-flamegraph.png)

较宽的框架表示更**频繁**的代码路径，这些可能是更相关的优化机会。您可以点击框架来选择它，这将在右侧边栏中填充包含所选函数的示例性能分析链接。

## 使用性能分析放大追踪

如果您已经在查看追踪，但没有足够的信息来调试问题的根本原因，性能分析数据可以帮助填补空白。如果有与追踪关联的性能分析数据，有两种方法可以从[**追踪视图**](/concepts/key-terms/tracing/trace-view/)中获取它。

一种方法是选择一个跨度并向下滚动到跨度详情的**性能分析**部分：

![追踪视图中跨度的性能分析数据](./img/trace-view-span-profile.png)

**此跨度中最频繁堆栈**小组件显示一个分页的调用堆栈列表，按频率降序排列，指示在所选跨度中大部分时间花费在代码的哪个位置。点击**>**按钮浏览其他堆栈，或点击**性能分析**按钮在火焰图查看器中查看整个性能分析。

另一种方法是点击追踪视图中的**性能分析**选项卡，这将显示追踪中包含的*所有*跨度的性能分析事件列表：

![追踪视图中的性能分析选项卡](./img/trace-view-profiles.png)

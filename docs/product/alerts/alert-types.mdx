您可以创建三种类型的警报：

1. **问题警报**：当问题符合特定条件时触发。
2. **指标警报**：当宏观级指标超过特定阈值时触发。
3. **正常运行时间警报**：当 HTTP 请求未返回成功响应时触发。

## 问题警报

问题警报在为项目中的任何问题接收到新事件时触发，只要该问题符合指定条件。这些条件可能是，例如，已解决的问题重新出现或影响许多用户的问题。

在"警报规则"选项卡中，这些警报由问题图标标识，默认情况下，它们显示在警报列表的底部。（如果您有多个指标警报，这可能会将您的问题警报推出列表的第一页。）

![警报列表底部的问题警报规则。](./img/issue-alert-rules.png)

在问题警报中，每次接收到新事件时，Sentry 都会评估配置的警报条件。警报条件有三个部分：

1. [触发器](/product/alerts/create-alerts/issue-alert-config/#when-conditions-triggers)指定您希望监控的活动类型，或**何时**应触发警报。
2. [过滤器](/product/alerts/create-alerts/issue-alert-config/#if-conditions-filters)通过仅在问题符合指定条件**时**触发警报来帮助控制噪音。
3. **然后**，[操作](/product/alerts/create-alerts/issue-alert-config/#then-conditions-actions)指定在满足触发条件且过滤器匹配时应执行的操作。

### 警报详情

**警报详情**页面向您显示问题警报规则随时间触发的次数，按一小时桶分组。单击"警报规则"选项卡中的警报规则名称，或您收到的通知将带您到此页面。该页面还包括警报规则条件、警报的当前状态（警告、严重或已解决）以及警报详细信息，例如创建时间、上次修改时间和拥有警报的团队。

![警报详情说明了一个图表，显示随时间触发的警报。](./img/issue-alert-status-page-example.png)

**警报详情**页面还包括触发警报的问题列表。您可以单击列表中的任何问题以转到该问题的详情页面以获取更多信息。

## 指标警报

<Include name="feature-available-for-plan-team.mdx" />

指标警报告诉您何时[指标](/product/insights/overview/metrics/)超过您设置的阈值，例如项目中错误数量的激增，或性能指标的变化，例如[事务持续时间](/product/insights/overview/metrics/#latency)、[Apdex](/product/insights/overview/metrics/#apdex)、[失败率](/product/insights/overview/metrics/#failure-rate)或[吞吐量](/product/insights/overview/metrics/#throughput-total-tpm-tps)。您可以使用[动态警报](/product/alerts/create-alerts/metric-alert-config/#dynamic-alerts)让 Sentry 为您定义阈值。

指标警报监控错误和事务事件的宏观级指标。指标获取一组事件并使用函数（如 `count()` 或 `avg()`）计算聚合值，该函数应用于事件属性在一段时间内。创建指标警报时，您可以按属性和<PlatformLink to="/enriching-events/tags/">标签</PlatformLink>过滤事件，这对于聚合未分组为单个问题的事件特别有用。Sentry 允许一个组织最多有 1000 个指标警报。

<Alert>

指标警报可能包括[已归档问题](/product/issues/states-triage/#archive)，如果这些事件来自符合指标警报规则过滤器的问题。来自已归档和已解决问题的事件可以通过在指标警报规则中使用 `is:unresolved` 过滤器来过滤掉。创建新指标警报时会默认添加此过滤器，但如果您希望它们排除已归档问题，您可能需要手动将其添加到较旧的指标警报。

</Alert>

这些警报使用_严重_和_警告_触发器来衡量严重性。警报的当前状态是活动的最高严重性触发器，可以是三个值之一：警告、严重或已解决。每当警报的状态发生变化时，Sentry 都会通知您。

创建警报时，所有显示的警报类型（"问题"除外）都可用于创建指标警报。您可以为错误、会话（崩溃率警报）和性能创建指标警报：

### 错误

错误警报对于监控项目中错误的总体水平、应用程序特定部分中发生的错误或影响用户的错误很有用。

- **错误数量：** <a name="number-of-errors"></a>
  当项目中符合您过滤器的错误数量超过您设置的阈值时发出警报。

- **遇到错误的用户：** <a name="users-experiencing-errors"></a>
  当项目中受错误影响的用户数量超过您设置的阈值时发出警报。

### 会话（崩溃率警报）

崩溃率警报可以让您更好地了解您的应用在每个项目基础上或在特定版本中的健康状况。当[会话或用户](/product/releases/health/#active-sessionsusers)的无崩溃百分比低于特定阈值时，它们会触发。这可能由于会话或用户[崩溃](/product/releases/health/#crash)数量的激增而发生。

- **无崩溃会话率：** <a name="crash-free-session-rate"></a>
  当崩溃会话数量超过您设置的阈值时发出警报。会话在用户启动应用程序时开始，在关闭或发送到后台时结束。崩溃是会话因错误而结束。

- **无崩溃用户率：** <a name="crash-free-user-rate"></a>
  当整体用户体验低于您设置的阈值时发出警报。无崩溃用户表示未经历崩溃的单个用户的百分比。

### 性能

应用程序性能警报可以帮助您查明和识别可能导致用户体验不佳的特定问题。

- **吞吐量：** <a name="throughput"></a>
  当吞吐量（项目中事务的总数）在您设置的指定时间段内达到您设置的阈值时发出警报。

- **事务持续时间：** <a name="transaction-duration"></a>
  当事务持续时间满足某些条件时发出警报。

**提示：** 条件可以使用百分位数、平均值和最小/最大值等灵活聚合进行自定义。

- **Apdex：** <a name="apdex"></a>
  对 Apdex 分数（特定事务或端点中令人满意、可容忍和令人沮丧的请求比率）发出警报。Apdex 是一个用于根据应用程序响应时间跟踪和衡量用户满意度的指标。

- **失败率：** <a name="failure-rate"></a>
  对失败率（不成功事务的百分比）发出警报。Sentry 将状态不是"正常"、"已取消"和"未知"的事务视为失败。

- **最大内容绘制：** <a name="largest-contentful-display"></a>
  当最大内容绘制（LCP）加载性能比预期慢时发出警报。LCP 标记最大图像或文本块在视口中可见的点。

**提示：** 快速的 LCP 有助于让用户确信页面有用。我们推荐小于 2.5 秒的 LCP。

- **首次输入延迟：** <a name="first-input-delay"></a>
  当首次输入延迟（FID）比预期长时发出警报，FID 衡量用户尝试与视口交互时的响应时间。

**提示：** 低 FID 有助于确保页面有用。我们推荐小于 100 毫秒的 FID。

- **累积布局偏移：** <a name="cumulative-layout-shift"></a>
  当累积布局偏移（CLS）增加时发出警报，CLS 通过量化页面整个生命周期中发生的意外布局偏移来衡量视觉稳定性。

**提示：** 小于 0.1 的 CLS 转化为良好的用户体验，而大于 0.25 的任何值都很差。

- **自定义指标：** <a name="custom-metric"></a>
  对上面未列出的指标发出警报，例如首次绘制（FP）、首次内容绘制（FCP）和首字节时间（TTFB）。

### 警报详情

**警报详情**页面默认显示指标警报规则过去 24 小时的历史记录，尽管可以使用"显示"下拉菜单修改时间段。当触发警报时，单击您收到的通知将带您到此页面，该页面显示警报处于活动状态的时期。该页面还包括警报规则条件、警报的当前状态以及警报在每个状态（严重、警告或已解决）中花费的时间摘要等详细信息。

![警报详情说明了一个图表，显示事务随时间的性能。](./img/alert-details-example.png)

**警报详情**页面还包括与指标相关的可疑问题或事务列表，以帮助更快地查明根本问题。您可以看到可能导致触发警报的原因，然后在 **[Discover](/product/explore/discover-queries)** 中打开指标以查找更多信息。

## 正常运行时间警报

正常运行时间警报在正常运行时间检查请求未能满足我们的[正常运行时间检查标准](/product/uptime-monitoring/#uptime-check-criteria)时触发。您将能够在"警报规则"选项卡中看到最新的正常运行时间检查请求状态（"正常"或"关闭"）。

### 警报详情

**警报详情**页面提供了与警报关联的正常运行时间检查事件的时间线。此时间线允许您跟踪故障发生的时间和地点。选择时间线的某个部分会自动更新日期范围过滤器，使您能够浏览历史正常运行时间检查数据。

**警报详情**页面还包括在检测到正常运行时间故障时由警报创建的[正常运行时间问题](/product/issues/issue-details/uptime-issues)列表。单击任何问题将带您到一个页面，其中包含可能帮助您调试该问题的额外信息。

## 用户反馈警报

您可以通过以下步骤在[问题警报](https://sentry.io/orgredirect/organizations/:orgslug/alerts/new/issue/)页面上为[用户反馈小组件](/product/user-feedback/#user-feedback-widget)启用警报：

1. 在 Sentry 中创建[新警报规则](https://sentry.io/orgredirect/organizations/:orgslug/alerts/new/issue/)。
2. 滚动到"设置条件"部分，并将"IF"过滤器设置为`问题的类别等于... "反馈"`。
3. 在"THEN"过滤器中选择要执行的操作。
4. 添加警报名称和所有者。

![用户反馈警报配置。](../user-feedback/img/user-feedback-alert-configuration.png)

要在[崩溃报告反馈](/product/user-feedback/#crash-report-modal)进入时获取警报，请确保在设置 > 项目 > [项目名称] > 用户反馈中打开"启用崩溃报告通知"。

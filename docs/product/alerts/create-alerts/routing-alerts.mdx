通过自定义[告警](/product/alerts/)规则并集成您已在使用的工具，您可以在需要时、需要的地方（以及是否需要时）接收告警，而不会造成中断。告警通知可以路由到 [Slack](/organization/integrations/notification-incidents/slack/)、多个[支持的集成](/organization/integrations/)，以及通过 [webhooks](/organization/integrations/integration-platform/webhooks/) 的自定义集成。创建告警规则时，您可以使用这些集成来配置通知谁以及如何通知。

## 集成

Sentry 的集成为您提供了通过常用应用程序（如 Slack、Discord、PagerDuty 和 Microsoft Teams）路由告警的选项。您可以在 **设置 > 集成** 中找到这些集成并为整个组织[安装](https://sentry.io/settings/integrations/)它们。

### Slack 告警

Sentry 组织所有者、管理员或具有管理权限的用户可以在其 Sentry 帐户中安装和配置 [Slack 集成](/organization/integrations/notification-incidents/slack/)。集成配置完成后，以下操作将在问题告警规则中可用：`向 {workspace} Slack 工作区的 {channel} 发送通知，并在通知中显示标签 {tags}`。在指标告警中，您的 Slack 团队将在其中一个操作下拉列表中可用。

此告警操作允许您将告警通知路由到选定频道（使用 `#` 前缀），或直接消息中的特定用户（使用 `@` 前缀）到您的 Slack 工作区。

![Slack 通知](./img/alerts_notifications_15.png)

然后，一旦您收到 Slack 通知，您可以使用"解决"、"归档"或"分配"按钮直接从 Slack 更新 [sentry.io](https://sentry.io) 中的问题。

### Discord 告警

Sentry 组织所有者、管理员或具有管理权限的用户可以在其 Sentry 帐户中安装和配置 [Discord 集成](/organization/integrations/notification-incidents/discord/)。集成配置完成后，`向 {server} Discord 服务器 ID: {channel-id} 发送通知，并在通知中显示标签 {tags}` 操作将在问题告警规则中可用。要通过 Discord 接收有关指标告警的通知，请在操作下拉列表中查找并添加您的 Discord 团队。

此告警操作将允许您将告警通知路由到 Discord 服务器中的选定频道。

![Discord 通知](./img/alerts-notifications-discord.png)

您将能够直接从 Discord 通知"解决"、"归档"或"分配" Sentry 问题。

### PagerDuty 告警

Sentry 组织所有者、管理员或具有管理权限的用户可以在其 Sentry 帐户中安装和配置 [PagerDuty 集成](/organization/integrations/notification-incidents/pagerduty/)。集成配置完成后，以下操作将在问题告警规则中可用：`向 PagerDuty 帐户 {account} 和服务 {service} 发送通知`。在指标告警中，您的 PagerDuty 帐户将在其中一个操作下拉列表中可用。

### Microsoft Teams 告警

Sentry 组织所有者、管理员或具有管理权限的用户可以在其 Sentry 帐户中安装和配置 [Microsoft Teams](/organization/integrations/notification-incidents/msteams/)。集成配置完成后，以下操作将在问题告警规则中可用：`向 {team} 团队发送到 {channel(s)} 的通知`。在指标告警中，您的 Microsoft 团队将在其中一个操作下拉列表中可用。

## 构建您自己的集成

如果您希望将告警通知路由到 Sentry 没有现成集成的其他解决方案，您可以使用我们的[集成平台](/organization/integrations/integration-platform/#alerts)。集成平台为外部服务提供了一种使用 REST API 和 webhooks 与 Sentry SaaS 服务交互的方式。

将告警发送到 webhooks 也很有帮助，如果您想要执行诸如聚合来自不同监控系统的告警或编写自定义规则以更智能地路由告警等操作。

当您创建新集成并在其上启用"告警规则操作"选项时，您的集成将在问题告警规则创建期间选择`通过集成发送通知`操作时显示为服务。在指标告警中，您的集成在其中一个操作下拉列表中可用。

## 旧版集成

旧版集成（也称为插件）是作为 Python 库打包的 Sentry 扩展，并在项目级别配置。要将告警发送到旧版集成，请在问题告警中选择`通过集成发送通知`或`向所有旧版集成发送通知`操作。您无法将指标告警路由到旧版集成。

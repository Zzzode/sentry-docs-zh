Sentry 提供了多种配置选项，可根据您组织的需求创建指标告警。

## 指标（类型 + 函数 + 时间间隔）

要创建指标告警，您首先需要选择指标类型。对于某些告警类型，函数内置于告警中，而对于其他类型，您可以选择要应用的函数和参数。例如，如果您选择"用户遇到错误"，这将转换为函数 `count_unique(user.id)`。由于编辑此函数会改变告警的性质，因此它是不可编辑的，因此被隐藏。另一方面，如果您选择"最大内容绘制时间"，使用的测量值是 `measurement.lcp`，您还需要选择一个函数，例如 `p75()`，组合后的指标函数为 `p75(measurement.lcp)`。

<Alert>
创建新的指标告警时，默认会添加 `is:unresolved` 过滤器。此过滤器会排除已归档和已解决问题的相关事件。如果您希望指标告警包含已归档的问题，可以手动从[过滤器](#filters)中移除 `is:unresolved`。
</Alert>

### 用于告警的指标类型

#### 错误

- 错误数量
- 遇到错误的用户

#### 会话

- 无崩溃会话率
- 无崩溃用户率

#### 性能

- 吞吐量
- 事务持续时间
- Apdex
- 失败率
- 最大内容绘制时间
- 首次输入延迟
- 累积布局偏移
- 自定义测量

#### 指标
- 自定义指标

- `count()`

- `apdex(...)`
- `count()`

### 时间间隔

选择评估指标的时间段。您可以选择从一分钟到一天的时间范围。Sentry 会每分钟评估指定的窗口。例如，如果您指定一小时的时间窗口，Sentry 会评估：

- 下午3:00：下午2:00 - 下午3:00
- 下午3:01：下午2:01 - 下午3:01
- 下午3:02：下午2:02 - 下午3:02
- ...

## 过滤器

以下一组过滤器会转换为在告警配置页面顶部图表中显示的[Discover 查询](/product/explore/discover-queries/)。

### 项目

指定哪个项目将使用此特定告警规则。创建的告警规则将仅处理来自此项目的事件。

### 环境

指定哪个<PlatformLink to="/configuration/environments/">环境</PlatformLink>将使用此特定告警规则。此控制会过滤事件中的 `environment` 标签。此过滤器很有用，因为您应用于生产告警的紧急程度和工作流程可能与应用于来自 QA 环境的告警不同。

此处的"环境"下拉列表与常见"环境"过滤器下拉列表中选定项目的可用环境相同（这不包括隐藏的环境）。选择"所有环境"等同于没有环境过滤器。

### 事件类型

对于某些指标告警，您可以在"事件"下拉菜单中设置要接收告警的事件类型：

- `event.type:error` 或 `event.type:default`
- `event.type:default`
- `event.type:error`

### 标签和属性

在提供的输入中添加过滤器以缩小您将接收告警的范围，例如标签或其他事件属性。可用属性取决于您的告警类型：

- 对于**错误**事件，所有错误属性都可用。查看[可搜索属性](/concepts/search/searchable-properties/events/#searchable-properties)获取完整列表。
- 对于**事务**事件，通常提供以下基本属性：`release`、`transaction`、`transaction.status`、`transaction.op`、`http.method`、`http.status_code`、`os.name`、`browser.name` 和 `geo.country_code`。

#### 事务的高级过滤器

<Alert>

此功能仅适用于拥有商业或企业计划的组织。

</Alert>

除了上面提到的基本属性外，还有更广泛的字段或标签可用，因此您可以创建更高级的过滤条件。但是，Sentry 仅在告警保存后才开始收集这些指标。如果使用此类过滤器，Sentry 会在告警创建期间提供前一时期的指标估计。在某些旧版计划中，每个项目最多允许使用50个带有高级过滤器的告警。

![事务的高级过滤器](./img/alerts-advanced-filters.png)

#### 无效过滤器

虽然 Sentry 不允许您使用无效或不可用的属性创建新告警，但任何具有不可用字段的现有告警都不会受到影响。但是，如果您需要编辑或复制它们，则需要移除不可用的属性。

## 阈值

有三种阈值类型：

- **静态**：由您设置的固定数值阈值。（例如，如果在设定的时间段内有100个错误。）
- **百分比变化**：基于百分比的阈值，例如当时间段内的错误比前一时期多10%时。这些也称为[变化告警](#change-alerts-percent-change)。
- **异常**：由 Sentry 维护的动态阈值，当值超出预期范围时检测异常。

默认情况下，指标告警使用固定阈值。

### 变化告警（百分比变化）

<Include name="feature-available-for-plan-trial-business.mdx" />

变化告警，或使用百分比变化阈值的告警，在您想知道指标与正常情况是否有显著差异时很有用。为此，您需要选择一个指标间隔（选择指标类型时）和一个进行比较的时间。一个例子是将最后一小时的错误数量与一周前的同一时间段进行比较。如果最后一小时的错误比一周前的同一时期多25%，则会触发告警。

![选择百分比变化选项时。](./img/percent-change.png)

### 异常告警

<Include name="feature-available-for-plan-trial-business.mdx" />

异常告警会自动检测超出预期值的趋势。它们对于静态或基于百分比的阈值过于嘈杂的尖峰或季节性数据特别有帮助。Sentry 将查看给定指标的历史数据，并确定当前数据是否异常。某些指标（如 Apdex）目前不支持异常告警。

在幕后，异常告警使用两种开源算法的组合——[Matrix Profile](https://stumpy.readthedocs.io/en/latest/index.html) 和 [Prophet 预测](https://facebook.github.io/prophet/)——来检测异常模式并预测预期趋势。通过结合这些方法，Sentry 可以更好地区分真实异常和正常波动，减少误报和漏报。

告警响应性有三个选项：低、中和高。如果您选择低响应性，则告警触发频率较低，但置信度较高。如果您选择高响应性，则告警触发频率较高，误报的可能性更大。我们的建议是从中等响应性（我们的默认值）开始，并根据您看到的结果调整响应性。

另一个设置是设置告警的方向：高于和/或低于预期范围。此设置可以帮助增加告警规则的信号。

![选择动态阈值时。](./img/dynamic-threshold.png)

### 设置触发告警的阈值

您可以使用标签设置满足阈值时告警规则的状态：

- 严重
- 警告
- 已解决

您必须设置"警告"阈值，使其在"严重"阈值之前触发。当 Sentry 评估告警时，告警的状态会更新为匹配的最高严重性触发器。如果您不设置"已解决"阈值，当不再违反"严重"或"警告"条件时，告警会自动解决。您也可以手动解决告警。

<Alert>

注意：异常告警阈值由 Sentry 控制，无法手动配置。

</Alert>

### 自动解决

默认情况下，当指定指标不再违反"严重"或"警告"条件时，指标告警会自动解决。但是，您可以设置不同的解决阈值。例如，假设您应用的正常错误级别小于2000/分钟，并且当超过5000/分钟时您希望收到告警。您可能希望仅当错误级别回到2000/分钟以下时才解决告警，而不是5000/分钟。通过这种方式设置"已解决"阈值，如果错误级别仅回落到4000/分钟，虽然低于您的告警阈值，但您仍会认为有问题，告警不会解决。

## 操作

操作定义您和您的团队将如何接收告警：

- 向成员或团队发送电子邮件。如果发送给成员，成员的个人项目告警选择退出设置将被覆盖。
- 发送 [Slack](/organization/integrations/notification-incidents/slack/) 通知。
- 发送 [Discord](/organization/integrations/notification-incidents/discord/) 通知。
- 触发 [PagerDuty](/organization/integrations/notification-incidents/pagerduty/) 事件。
- 发送 [Microsoft Teams](/organization/integrations/notification-incidents/msteams/) 通知。
- 使用 [Sentry 集成](/organization/integrations/integration-platform/#alerts) 发送请求。

了解更多关于[使用集成路由告警](/product/alerts/create-alerts/routing-alerts/)的信息。

## 规则名称

为您的告警提供一个描述性名称，例如受影响的团队和告警主题。例如，"前端延迟"、"后端失败率"或"账单 Apdex"。此名称必须在**组织级别唯一**；也就是说，您的组织中创建的任何其他指标告警都不能具有相同的规则名称。

## 团队

您可以选择一个团队与告警关联，以便该团队的成员可以编辑此告警。请注意，只有团队成员才能进行此关联。如果未选择团队，任何人都可以编辑告警。

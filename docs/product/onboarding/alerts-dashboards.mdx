在本指南结束时，您将能够创建和自定义 Sentry 警报和仪表板，以监控关键指标、跟踪问题趋势，并实时了解应用程序性能。如果您在任何地方遇到困难，请尝试[询问 Sentry AI](/?askAI=true)或加入我们的 [Discord](https://discord.gg/sentry) 社区。

## 1. 警报策略

在与 [Slack 集成](https://sentry-docs-git-ob-alerts.sentry.dev/organization/integrations/notification-incidents/slack/)并添加您的成员和团队后，是时候创建 [Sentry 警报](/product/alerts/best-practices/)以确保实时问题检测，从而实现更快的调试并减少停机时间。使用特定阈值或条件配置警报有助于确定问题的优先级，提高系统可靠性和事件响应时间。

### 警报建议

- **错误警报（高频）** - 当新的或现有的生产问题突然激增时触发。

<Arcade src="https://demo.arcade.software/YaLyaOnchFhnNgnnegdx?embed" />

- **性能警报** - 检测并警报特定性能指标何时超过指定阈值。

<Arcade src="https://demo.arcade.software/OKhoLV2GUeP1xHQIDAbM?embed" />

- **崩溃率警报** - 通过跟踪在给定时间范围内遇到崩溃的用户百分比来监控应用程序稳定性。

<Arcade src="https://demo.arcade.software/r2YOeIh3sj9Lzmc3vBmj?embed" />

- **异常检测** - Sentry 可以通过自动检测异常并主动提醒您来简化您的警报需求。

<Arcade src="https://demo.arcade.software/lkvlGb3Cxh4ZwvEbfM0X?embed" />

您还可以根据关键性能指标（如[吞吐量](/product/alerts/alert-types/#performance)）为您的所有自定义检测事务/跨度创建指标警报。

<Arcade src="https://demo.arcade.software/Ij5ofpPdcu6i71HPdKX2?embed" />

## 2. Sentry 仪表板和发现

设置 Sentry 仪表板对于加快分类并提供应用程序中关键问题、趋势和性能的实时视图至关重要。如果观察到异常，Sentry 仪表板允许用户通过几次点击直接跳入故障排除。

### 推荐仪表板

1. **错误概览仪表板** - 提供应用程序健康状况的高级视图，向您显示错误趋势、最频繁的问题和错误发生次数的激增。

2. **性能监控仪表板** - 跟踪事务持续时间、慢速 API 调用和数据库查询性能以识别瓶颈。

3. **移动崩溃率和发布健康仪表板** - 通过跟踪不同版本之间的崩溃率来监控应用程序稳定性和发布影响。

4. **关键体验仪表板** - 跟踪应用程序关键体验中的错误或性能瓶颈。[查看示例](https://sandbox.sentry.io/dashboard/126679/)。

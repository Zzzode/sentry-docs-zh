## 日志记录

Relay 将日志输出到标准错误流 (`stderr`)，默认情况下为 `INFO` 日志级别。
例如，启动 Relay 后，您可能会看到如下输出：

```
INFO  relay::setup > 从配置文件夹 .relay 启动 relay
INFO  relay::setup >   relay 模式: managed
INFO  relay::setup >   relay id: cde0d72e-0c4e-4550-a934-c1867d8a177c
INFO  relay::setup >   日志级别: INFO
```

此示例显示默认日志级别 (`INFO`) 的消息，您可以修改它以显示更多或更少的信息。有关配置日志记录的详细信息，请参阅选项页面上的[日志记录](/product/relay/options/#logging)部分。

## 错误报告

默认情况下，Relay 将错误记录到配置的日志记录器。您可以在 Relay 配置文件中启用到您在 Sentry 中的项目的错误报告：

```yaml
sentry:
  enabled: true
  dsn: <您的_dsn>
```

有关可用选项及其含义的更多信息可以在[选项页面](/product/relay/options/#internal-error-reporting)上找到。

## 健康检查

Relay 提供两个 URL 用于检查系统状态：

- `GET /api/relay/healthcheck/live/`：测试 Relay 是否正在运行并监听 HTTP 请求。
- `GET /api/relay/healthcheck/ready/`：测试 Relay 是否与上游进行身份验证并正常运行。

两个端点在成功时都返回 _200 OK_ 响应：

```json
{
  "is_healthy": true
}
```

## 指标

您可以通过将 `metrics.statsd` 键配置为 `ip:port` 元组来将统计信息提交到 StatsD 服务器。

### 示例配置

```yaml
metrics:
  # 您的 StatsD 服务器的端点
  statsd: 127.0.0.1:8126
  # 使用此前缀所有指标名称
  prefix: mycompany.relay
```

配置指标报告的选项记录在[选项页面](/product/relay/options/#statsd-metrics)上。

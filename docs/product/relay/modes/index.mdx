Relay 可以在几种主要模式之一中运行，如果您正在配置 Relay 服务器而不是使用默认设置，则理解这些模式至关重要。

模式存储在配置文件中，该文件包含 `relay.mode` 字段。此字段指定 Relay 将运行的模式：`managed` 或 `proxy`。Relay 模式控制 Relay 是否为事件获取项目设置。

在 Sentry 中，事件处理根据项目和组织设置进行配置。某些设置（如隐私控制）在组织级别设置，然后由该组织中的所有项目继承；其他设置按项目指定。对于 Relay，事件根据事件发送到的继承项目设置进行处理。

Relay 的配置通过轮询 Sentry 定期刷新。Sentry 不需要对您的 Relays 进行视线。有关间隔、超时和重试的配置，请参阅[配置选项](/product/relay/options/)。

## 托管模式

托管是 Relay 的默认模式。由于设置是从 Sentry 获取的，因此在此模式下需要身份验证。如果身份验证失败，Relay 将不接受任何事件。

当 Relay 从您的应用程序接收事件时，它将从 Sentry 请求项目设置以处理事件。如果 Sentry 无法为特定项目提供设置，则该项目的所有数据都将被丢弃。

要激活托管模式，请设置此配置：

```yaml
relay:
  mode: managed
```

## 代理模式

代理模式以最小处理转发所有事件，并且不会从 Sentry 接收任何项目设置。

<Alert title="速率限制">

在 `proxy` 模式下，所有项目仍然应用速率限制。

</Alert>

要激活代理模式，请设置此配置：

```yaml
relay:
  mode: proxy
```

## 静态模式

从 Relay 版本 25.9.0 开始，静态模式已被弃用。请改用 `managed` 或 `proxy` 模式。切换到 `managed` 模式时，不要忘记通过[在 Sentry UI 中配置它们](/security-legal-pii/scrubbing/)来迁移配置目录中的任何数据清理规则。

开始使用 [Relay](/product/relay/) 就像使用默认设置一样简单。您还可以配置 Relay 以满足您组织的需求。查看[配置选项](../options/)页面以获取操作场景的详细讨论。

<Alert>

Relay 服务器称为 `relay`。从 [GitHub
Releases](https://github.com/getsentry/relay/releases) 下载二进制文件。Docker 镜像在 [DockerHub](https://hub.docker.com/r/getsentry/relay/) 上提供。

</Alert>

## 初始化配置

要创建初始配置，请使用 Relay 的 `relay config init` 命令，该命令将配置文件放在当前工作目录下的 `.relay` 文件夹中。

<Alert>

在 Docker 中，使用 Docker 提供的标准机制提供配置目录，通过挂载 [Docker
卷](https://docs.docker.com/storage/volumes/) 或通过构建新容器并复制文件。

</Alert>

```shell {tabTitle:在 Docker 中运行}
# 调整配置目录的权限
# :z 选项仅在 selinux 上必要。
# 参见 https://docs.docker.com/storage/bind-mounts/#configure-the-selinux-label。
docker run --rm -it                  \
  -v $(pwd)/config/:/work/.relay/:z  \
  --entrypoint bash                  \
  getsentry/relay                    \
  -c 'chown -R relay:relay /work/.relay'

# 生成配置
docker run --rm -it                  \
  -v $(pwd)/config/:/work/.relay/:z  \
  getsentry/relay                    \
  config init
```

```shell {tabTitle:运行可执行文件}
./relay config init
```

选择默认配置以创建最小配置文件。您可以选择通过选择
_"创建自定义配置"_ 并自定义这些参数来覆盖默认设置：

- `mode` 设置，它配置 Relay 运行的主要模式。有关可用 Relay 模式的更多信息，请参阅 [Relay 模式](../modes/)。

- `upstream` 设置配置 Relay 将转发事件到的服务器（默认情况下是主要的 `sentry.io` URL）。

- `port` 和 `host` 设置配置 Relay 将监听的 TCP 端口。这是 SDK 发送事件的地址。

设置记录在 `.relay/config.yml` 中。请注意，所有配置值
都是可选的，默认为这些设置：

```yaml {tabTitle:在 Docker 中运行}
relay:
  mode: managed
  # 上游主机名取自您的任何 DSN。
  # 转到您的项目设置，然后转到"客户端密钥 (DSN)"以查看它们。
  upstream: https://___ORG_INGEST_DOMAIN___
  host: 0.0.0.0
  port: 3000
```

```yaml {tabTitle:运行可执行文件}
relay:
  mode: managed
  # 上游主机名取自您的任何 DSN。
  # 转到您的项目设置，然后转到"客户端密钥 (DSN)"以查看它们。
  upstream: https://___ORG_INGEST_DOMAIN___
  host: 127.0.0.1
  port: 3000
```

配置在[配置选项](../options/)中完全记录。

## 创建凭据

<Alert>

在 `proxy` 模式下不适用。

</Alert>

除了 `config.yml` 之外，`init` 命令还会创建一个凭据文件 `credentials.json`，该文件放在相同的 `.relay` 目录中。此文件包含 Relay 用于与上游服务器进行身份验证的公钥和私钥。

**因此，此文件必须受到保护，防止未经授权的实体修改或查看。**

典型的凭据文件类似于以下示例：

```json
{
  "secret_key": "5gkTAfwOrJ0lMy9aOAOmHKO1k6gd8ApYkAInmg5VfWk",
  "public_key": "nDJI79SbEYH9-8NEJAI7ezrgYfoIPW3Bnkg00k1z0fA",
  "id": "cde0d72e-0c4e-4550-a934-c1867d8a177c"
}
```

使用 `public_key` 在 `managed` 模式下运行时将您的 Relay 注册到上游服务器。

## 将 Relay 注册到 Sentry

<Alert>

在 `proxy` 模式下不适用。

</Alert>

要在 `managed` 模式下运行，Relay 从您的组织和项目设置中拉取 PII 清理、
过滤和速率限制的配置
在 Sentry 中。由于这些设置可能包含敏感信息，它们的访问受到 Sentry 的限制，并且需要授权。

要将 Relay 注册到 Sentry：

1. 复制公钥的内容，通过检查 `credentials.json` 文件或运行：

   ```shell {tabTitle:在 Docker 中运行}
   docker run --rm -it                \
     -v $(pwd)/config/:/work/.relay/  \
     getsentry/relay                  \
     credentials show
   ```

   ```shell {tabTitle:运行可执行文件}
   ./relay credentials show
   ```

2. 点击 Sentry 主导航中的 _设置_，然后选择 _Relays_。

![](./img/add-relay-key.png)

3. 点击 _新建 Relay 密钥_ 以添加密钥并保存：

![](./img/edit-relay-key.png)

此过程将 Relay 注册到 Sentry，使其准备好发送消息。请参阅
[配置选项](../options/) 以了解有关 Relay 配置选项的更多信息。

## 运行 Relay

将 Relay 注册到 Sentry 后，它就可以运行了：

```shell {tabTitle:在 Docker 中运行}
docker run --rm -it                \
  -v $(pwd)/config/:/work/.relay/  \
  -p 3000:3000                     \
  getsentry/relay                  \
  run
```

```shell {tabTitle:运行可执行文件}
./relay run
```

在默认配置下，Relay 发出类似于以下内容的日志输出：

```
 INFO  relay::setup > 从配置文件夹 .relay 启动 relay
 INFO  relay::setup >   relay 模式: managed
 INFO  relay::setup >   relay id: cde0d72e-0c4e-4550-a934-c1867d8a177c
 INFO  relay::setup >   公钥: nDJI79SbEYH9-8NEJAI7ezrgYfoIPW3Bnkg00k1z0fA
 ...
 INFO  relay_server::actors::upstream > relay 成功注册到上游
```

有关如何配置 Relay 日志的更多信息，请参阅[日志记录](../monitoring/#logging)页面。

如果您移动了配置文件夹（例如，出于安全原因），请使用 `--config` 选项指定位置：

```shell {tabTitle:在 Docker 中运行}
docker run --rm -it                \
  -v $(pwd)/config/:/etc/relay/    \
  -p 3000:3000                     \
  getsentry/relay                  \
  run --config /etc/relay/
```

```shell {tabTitle:运行可执行文件}
./relay run --config /etc/relay/
```

## 发送测试事件

一旦 Relay 运行并与 Sentry 进行身份验证，就向您组织中的一个项目发送测试事件。

通过导航到您的项目设置 _设置 > 项目 > {您的项目名称} > 客户端密钥 (DSN)_ 获取您项目的 DSN，并选择现有的 DSN 之一，它看起来类似于：

`https://12345abcdef10111213141516171819@o1.ingest.sentry.io/2345`

接下来，替换 DSN 的部分以匹配 Relay 可访问的地址。
例如，如果 Relay 在 `http://localhost:3000` 监听，请将 DSN 的协议
和主机更改为：

<p>
  <code>
    <strong>
      <i>http</i>
    </strong>
    ://12345abcdef10111213141516171819@
    <strong>
      <i>localhost:3000</i>
    </strong>
    /2345
  </code>
</p>

在您的 SDK 配置中使用新的 DSN。要测试此功能，您可以使用 `sentry-cli` 发送消息
（[安装说明](/cli/installation/)）：

```shell
export SENTRY_DSN='http://12345abcdef10111213141516171819@127.0.0.1:3000/2345'
sentry-cli send-event -m '测试事件'
```

几秒钟后，事件应该出现在您项目的问题流中。

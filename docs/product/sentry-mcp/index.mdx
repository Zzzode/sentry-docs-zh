## 概述

[Sentry MCP 服务器](https://mcp.sentry.dev) 提供了一种安全的方式，将 Sentry 的完整问题上下文带入能够利用[模型上下文协议 (MCP)](https://modelcontextprotocol.io/introduction) 的系统中。Sentry 托管和管理一个远程 MCP 服务器，您可以连接并集中利用。使用 MCP，您可以：

- 访问 Sentry 问题和错误
- 在特定文件中搜索错误
- 查询项目和组织
- 列出并为项目创建 Sentry DSN
- 调用 Seer 自动修复问题并检索问题修复的状态和详细信息

Sentry MCP 服务器包括对现代 MCP 功能的支持：

- **远程托管（首选）或本地 STDIO 模式** - 托管版本提供更低摩擦，始终包含最新功能和稳定性
- **OAuth 支持** - 使用您现有的 Sentry 组织进行身份验证，无缝访问您的项目
- **支持流式 HTTP 和优雅的 SSE 回退** - 自动回退确保与不同客户端的兼容性
- **16+ 个工具调用和提示** - 将 Sentry 上下文带入 LLM 交互的综合工具集

## 开始使用

### OAuth 配置（推荐）

对于支持 OAuth 的客户端，您可以使用提供自动身份验证的简化配置：

```json
{
  "mcpServers": {
    "Sentry": {
      "url": "https://mcp.sentry.dev/mcp"
    }
  }
}
```

此配置：
- 启用与您的 Sentry 组织的 OAuth 身份验证
- 使用支持自动 SSE 回退的流式 HTTP 传输
- 提供对所有 16+ 个可用工具的访问
- 自动处理身份验证和会话管理

使用 OAuth 配置时，系统将提示您：
1. 接受 OAuth 授权
2. 通过您现有的 Sentry 组织登录
3. 授予必要的权限

<Alert>
  如果您加入了新的 Sentry 组织，请退出 LLM 的 MCP 集成并重新登录以刷新访问权限。
</Alert>

身份验证完成后，您将在 MCP 客户端中看到工具变得可用。

![可用的 Sentry MCP 工具](./img/mcp-tools.png)

### 远程 MCP 配置（传统）

对于不支持 OAuth 的客户端，您可以继续使用现有的远程 MCP 端点配置：

```json
{
  "mcpServers": {
    "Sentry": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote@latest",
        "https://mcp.sentry.dev/mcp"
      ]
    }
  }
}
```

### 本地 STDIO 模式

或者，如果您不想要 Remote-MCP 服务器，您可以通过遵循 [README](https://github.com/getsentry/sentry-mcp) 中的说明在 STDIO 模式下本地运行 MCP 服务器。这对于自托管的 Sentry 安装特别有用。

要使用 STDIO 模式，您需要一个具有以下范围的 Sentry 用户身份验证令牌：
- `org:read`
- `project:read`
- `project:write`
- `team:read`
- `team:write`
- `event:write`

启动 STDIO 传输：

```bash
npx @sentry/mcp-server@latest --access-token=sentry-user-token --host=sentry.example.com
```

您也可以使用环境变量：
```bash
SENTRY_ACCESS_TOKEN=your-token
SENTRY_HOST=your-sentry-host
```

### mcp.sentry.dev（实时演示）

[Sentry MCP 页面](https://mcp.sentry.dev) 提供了一种简单的方法来配置和测试 MCP 服务器。从这个 Web 客户端，您可以使用您的 Sentry 组织进行身份验证，并访问用于测试的托管 MCP 服务器。

## 可用工具

Sentry MCP 服务器提供用于与 Sentry 数据交互的综合工具：

### 核心工具
- **组织**：列出和查询组织信息
- **项目**：查找、列出和创建项目
- **团队**：管理和查询团队信息
- **问题**：访问问题详细信息、搜索和分析问题
- **DSN**：列出并为项目创建数据源名称

### 分析工具
- **错误搜索**：在特定文件或跨项目中查找错误
- **问题分析**：带有上下文的详细问题调查
- **Seer 集成**：调用 Sentry 的 AI 代理进行根本原因分析和自动修复

### 高级功能
- **发布管理**：查询和分析发布信息
- **性能监控**：访问事务和性能数据
- **自定义查询**：跨 Sentry 数据执行复杂搜索

## 已验证的客户端

Sentry MCP 服务器已验证可在以下环境中工作：

### Claude 桌面版

通过 `CMD + ,` → `开发者` → `编辑配置` → 编辑 `claude_desktop_config.json` 文件访问开发者工具

### Claude.ai

导航到 `设置` → `个人资料` → 滚动到 `集成`，选择 `添加更多`，并添加 Sentry MCP 服务器 URL `https://mcp.sentry.dev/mcp`

### Claude Code

自从最近达到 1.0 以来，Claude Code 原生支持远程托管的 MCP 服务器。从您的 CLI 输入 `claude mcp add --transport http sentry https://mcp.sentry.dev/mcp`，然后使用 `claude` 访问 Claude Code。

进入后，系统将提示您使用 OAuth 对 Sentry 进行身份验证。

### Cursor

通过 `Cursor` → `设置` → `Cursor 设置` → `MCP` 按照提示配置 Sentry MCP。Cursor 1.0+ 包括增强的 MCP 支持，具有 OAuth 和流式 HTTP。

如果您愿意，您仍然可以手动编辑 `mcp.json` 文件。

### VS Code 和 GitHub Copilot

使用 `CMD+Shift+P` 并选择 `MCP: 添加服务器` 来添加服务器

### Windsurf

通过 Cascade 中的 `配置 MCP` 选项进行配置（CMD + L）

### 其他客户端

Sentry MCP 服务器遵循标准 MCP 协议，应该适用于支持以下功能的任何客户端：
- OAuth 身份验证（推荐）
- 远程 MCP 服务器
- SSE 或流式 HTTP 传输

## 与 Seer 集成

Sentry MCP 服务器提供与 [Seer](/product/ai-in-sentry/seer/)（Sentry 的 AI 代理）的无缝集成。您可以：

- **触发 Seer 分析**：启动问题的自动根本原因分析
- **获取修复建议**：接收针对错误和性能问题的 AI 生成解决方案
- **监控修复状态**：跟踪 Seer 分析和实施的进度

**注意**：MCP 和 Seer 是互补工具。MCP 将 Sentry 上下文带入您的 LLM，而 Seer 专为深度问题分析和自动调试而构建。您可以使用 MCP 调用 Seer 进行复杂的调试工作流。

## 故障排除

### 常见问题

**OAuth 身份验证问题**
- 确保您的客户端支持 OAuth 身份验证
- 如果 OAuth 不起作用，请尝试传统远程 MCP 配置
- 检查您在 Sentry 组织中是否具有必要的权限

**连接问题**
- 验证 MCP 服务器 URL 是否正确：`https://mcp.sentry.dev/mcp`
- 对于传统设置，请使用：`https://mcp.sentry.dev/sse`
- 检查您客户端的 MCP 配置语法

**缺少工具**
- 确保身份验证成功完成
- 验证您的 Sentry 组织访问权限
- 检查客户端控制台中是否有任何错误消息

如需其他支持，请访问 [GitHub 仓库](https://github.com/getsentry/sentry-mcp) 或联系 Sentry 支持。

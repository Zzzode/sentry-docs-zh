Sentry提供了多种方式来定义问题的"所有权"。通过定义所有权，我们可以自动分配问题并向所有者发送警报。Sentry使用_代码所有者_和_所有权规则_来定义所有权。代码所有者功能让您导入您的[GitHub](https://docs.github.com/en/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/about-code-owners)或[GitLab](https://docs.gitlab.com/ee/user/project/code_owners.html) CODEOWNERS文件，然后我们根据这些文件路径分配问题。所有权规则允许您基于代码所有者覆盖分配，并提供高级匹配器类型（例如，url和标签）。这些规则还可以匹配堆栈跟踪中文件的路径、请求的URL或事件标签。

如果用户在Sentry自动分配问题后手动分配问题，则该问题的未来自动分配将被关闭。

## 方法

所有权规则和代码所有者与问题中的单个事件匹配。此匹配依赖于[sentry.io](https://sentry.io)的其他区域，如以下部分所述。

### 所有权规则

您可以通过为每个项目定义所有权规则来设置所有权。要配置所有权规则，请导航到**[项目] > 设置 > 所有权规则**，或点击**问题详情**页面上的"创建所有权规则"按钮。

可用的匹配类型：

1. 路径：匹配事件堆栈跟踪中的所有文件路径
2. 模块：匹配事件堆栈跟踪中的模块
3. URL：匹配事件的`url`
4. 标签：匹配事件标签

规则的一般格式为：`type:pattern owners`

- **`type`** - 可以是`path`、`module`、`url`或`tags.TAG_NAME`，匹配指定的标签。
- **`pattern`** - 您要匹配的模式，支持unix风格的[glob语法](<https://en.wikipedia.org/wiki/Glob_(programming)>)。例如，添加`*`匹配任何内容，`?`匹配单个字符。_这不是正则表达式。_例如：
  - `src/javascript/*.[tj]sx`用于`path`类型
  - `com.example.app`用于`module`类型
  - `https://www.example.io/v/[0-9]/checkout`用于`url`类型
  - `Chrome 81.0.?`用于`tags.browser`类型
- **`owners`** - 所有者或所有者列表，用空格分隔。所有者可以通过Sentry用户的电子邮件或团队名称（前缀为`#`）来识别。例如：
  - `example@company.com`
  - `example1@company.com example2@company.com`
  - `#backend-team`

团队_必须_具有对项目的访问权限才能成为所有者。要授予团队对项目的访问权限，请导航到**[项目] > 设置 > 项目团队**，然后点击"添加团队"。

要授予用户对项目的访问权限，用户必须是具有项目访问权限的团队的成员。要将用户添加到团队，请导航到**设置 > 团队**，选择一个团队，然后点击"添加成员"。

您可以从**[项目] > 设置 > 所有权规则**添加所有权规则并点击"编辑规则"：

![在项目设置中定义所有权规则的配置选项。](./img/project_settings_edit_ownership_rules.png)

您还可以通过点击侧边栏"分配给"部分中的齿轮图标从**问题详情**页面添加所有权规则：

![分配给带有齿轮](./img/assigned_to_cog.png)

![从问题页面定义所有权规则的配置选项。](./img/issue_details_edit_ownership_rules.png)

当您从**问题详情**页面创建规则时，您将看到基于问题中事件的一些建议路径和URL。请注意[sentry.io](https://sentry.io)不会建议标签。

### 代码所有者

<Include name="feature-available-for-plan-business.mdx" />

![UI中的代码所有者](./img/codeowners.png)

导入您的CODEOWNERS文件并将其与您的所有权规则一起使用来分配Sentry问题。此功能的前提条件是[堆栈跟踪链接](/organization/integrations/source-code-mgmt/github/#stack-trace-linking)中的**代码映射**。

要快速为项目设置代码映射：

1. 转到您项目中的问题。您应该看到以下选项来链接到您的源代码中的堆栈跟踪：
    ![链接堆栈跟踪](./img/link_stack_trace.png)
2. 当您点击链接时，将打开一个模态框。输入与堆栈跟踪文件名对应的GitHub/GitLab源代码URL，我们将自动为项目创建代码映射并设置堆栈跟踪链接。

现在我们有了代码映射，我们能够转换CODEOWNERS文件中的路径到您预期的构建输出。例如，您可能在`dist/`文件夹中构建您的程序，但这不会在您的CODEOWNERS中。然而`dist/`前缀在堆栈跟踪的文件路径中被捕获。代码映射允许Sentry找出如何在您的构建输出和CODEOWNERS路径之间规范化文件路径。

您可以使用[外部团队/用户映射](/product/issues/ownership-rules/#external-teamuser-mappings)导入并增量添加"源代码控制团队/用户"和"Sentry团队/用户"之间的映射。Sentry使用"外部团队/用户映射"将CODEOWNERS文件中的所有者转换为他们在Sentry中的等效项。Sentry将自动忽略缺少团队/用户映射的规则。

此功能支持GitHub和GitLab CODEOWNERS文件语法，但有以下例外：

- 使用`\`转义以`#`开头的模式，使其被视为模式而不是注释
- 使用`!`来否定模式
- 使用`[ ]`来定义字符范围
- [GitLab CODEOWNERS语法](https://docs.gitlab.com/user/project/codeowners/reference/)
- [GitHub CODEOWNERS语法例外](https://docs.github.com/en/github/creating-cloning-and-archiving-repositories/creating-a-repository-on-github/about-code-owners#syntax-exceptions)

<Alert>

当Sentry分配问题时，所有权规则优先于CODEOWNERS文件提供的信息。

</Alert>

#### 限制

我们支持最大大小为3MB的CODEOWNERS文件。如果您的文件超过此限制，我们建议使用通配符规则将多个条目合并为一个。

通过导航到**设置 > 集成 > GitHub/GitLab > [配置] > 团队映射/用户映射**为您的GitHub/GitLab团队和用户创建外部团队/用户映射。建议将来自组织的任何CODEOWNERS文件的项目。

## 评估流程

如果您同时具有所有权规则和代码所有者，Sentry按以下顺序评估事件的in-app帧与规则：

1. 代码所有者，从上到下

   a. 如果规则中有多个所有者，从左到右

2. 所有权规则，从上到下

   a. 如果规则中有多个所有者，从左到右

评估后，最后一个匹配规则返回分配。事件堆栈跟踪文件路径的顺序在确定规则分配时无关紧要。

### 示例

**代码所有者：**

```
codeowners:*.js                    #ecosystem
```

**所有权规则：**

```
path:*.js                          #ecosystem #frontend
path:sentry/api/*                  #api
url:*subscription*                 #billing
path:dist/frontend/components/*    #frontend
```

当Sentry接收到具有堆栈跟踪文件路径的事件：`dist/frontend/components/sidebar.js`时，我们：

1. 从上到下评估代码所有者；我们得到一个匹配，规则中有两个所有者。

   a. 从左到右评估规则；我们选择第一个所有者。

2. 从上到下评估所有权规则；我们得到两个匹配。

按顺序的匹配是：

```
[
  "codeowners:*.js                    #ecosystem",
  "path:*.js                          #ecosystem",
  "path:dist/frontend/components/*    #frontend"
]
```

Sentry将返回最后一个匹配规则，在这种情况下是`path:dist/frontend/components/* #frontend`

## 应用位置

配置所有权规则后，它们以以下方式应用：

### 建议的受让人

在**问题详情**页面上，您将看到基于与事件匹配的所有权规则的建议受让人，以及[可疑提交](/product/releases/suspect-commits/)。您可以通过从下拉菜单中选择一个来手动将问题分配给建议的受让人。如果事件匹配多个所有权规则，它可以有多个建议的受让人。

![建议受让人的头像图标和点击它们进行分配的能力。](./img/suggested_assignees.png)

### 自动分配

您可以通过在**[项目] > 设置 > 所有权规则**中启用"自动分配给问题所有者"设置来自动将问题分配给其所有者。

![从下拉菜单中选择以自动将问题分配给其所有者。](./img/auto-assign-issue-owner.png)

如果问题过去已被手动分配，新事件将不会更改当前受让人。如果问题尚未被分配且新事件有多个所有者，Sentry将其分配给最后一个匹配规则的所有者（无论规则`type`如何）。

请记住，由于速率限制，如果项目在给定时间创建太多新问题，可能会跳过自动分配。我们将尝试在下次为这些跳过的问题传入事件时自动分配它们。如果您遇到此限制且问题不重复，您可能需要查看[问题分组](/concepts/data-management/event-grouping/)策略。

## 故障排除

确保所有团队和用户都具有对项目的正确访问权限。如果没有，所有权规则将无法保存。要授予团队对项目的访问权限，请导航到**[项目] > 设置 > 项目团队**，然后点击"添加团队"。用户需要具有与项目关联的团队的至少成员级别访问权限。要授予用户对与项目关联的团队的访问权限，请导航到**设置 > 团队**，选择一个团队，然后点击"添加成员"。

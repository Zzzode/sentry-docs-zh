Sentry中的[**问题**](https://sentry.io/orgredirect/organizations/:orgslug/issues/)页面显示有关应用程序中问题的信息。此页面允许您按浏览器、设备、受影响用户或错误是否未处理等属性进行过滤。然后您可以检查问题详情以更好地理解问题并有效地[分类](/product/issues/states-triage/)。

![问题主页](./img/issue_page.png)

典型的应用程序会向Sentry发送大量事件。您可以将<SandboxLink scenario="oneIssue" projectSlug="react">问题</SandboxLink>视为应用程序中的单个错误或问题。为了使它们易于管理，我们根据指纹[将相似事件分组为问题](/concepts/data-management/event-grouping/)。这种将事件分组为问题的方式允许您查看问题发生的频率以及它影响了多少用户。

<a name="quota"></a>

<Alert>

**问题和事件配额**

您的配额由_事件_消耗，而不是_问题_。问题由您接受的错误或事务事件生成。生成问题不会导致Sentry为您接受更多事件，也**不会**直接影响您的配额。然而，Sentry确实提供了工具来控制接受的错误和事务事件的类型和数量。在[配额管理](/pricing/quotas/)中了解更多信息。

</Alert>

对于每个问题，页面显示：

- 问题类型和描述
- 关联项目
- 问题时间，如最后和首次出现时间

对于错误问题，要查看最新事件的堆栈跟踪，您可以将鼠标悬停在问题标题上。您还可以将鼠标悬停在问题信息左侧的彩色方块上，以查看最新事件的错误级别（错误、信息、致命、警告、调试或示例）：

## 问题类别

有两类问题：[_错误问题_](/product/issues/issue-details/error-issues/)和[_性能问题_](/product/issues/issue-details/performance-issues/)。错误问题是错误事件的分组；性能问题是性能不佳事务的分组。要了解有关每个问题类别及其详细视图中捕获的信息类型的更多信息，请查看我们的完整[错误问题](/product/issues/issue-details/error-issues/)和[性能问题](/product/issues/issue-details/performance-issues/)文档。

## 问题分类

从**问题**页面，您可以开始分类。页面被组织成选项卡，每个选项卡对应过滤后的问题列表，这些不同的列表可帮助您进行分类：

- 所有未解决 (`is:unresolved`)：所有未解决的问题，包括需要审核的问题。
- 待审核 (`is:unresolved is:for_review`)。也称为**审核列表**，待审核问题是所有未解决问题的子集，可以包括新问题或尚未审核的回归。
- 回归 (`is:regressed`)：所有回归问题；已解决但再次出现的问题。
- 已归档 (`is:archived`)：所有已归档的问题。
- 升级中 (`is:escalating`)：所有升级中的问题；先前已归档但已超过其预测事件量的问题。

在[问题状态和分类](/product/issues/states-triage/)中了解更多关于分类问题及其不同状态的信息。

## 如何排序问题

通过从排序下拉菜单中选择来更改问题流中的排序方式：

![问题排序](./img/issue_sort.png)

- **最后出现**：最新的事件首先显示。
- **首次出现**：最新问题首先显示。
- **趋势**：新问题和升级中的问题（事件量呈上升趋势）首先显示。较旧的问题和最近事件较少的问题排序较低。排名目前依赖于三个主要因素：
  - 相对量：相对于基线具有更高最近量的升级问题排名更高。
  - 绝对量：事件量更多的问题权重更高。最近事件比旧事件权重更高。
  - 问题年龄：新问题优先 - 指数衰减因子每12小时减半权重。
- **事件**：问题按总事件量排序。
- **用户**：问题按受影响用户数量排序。

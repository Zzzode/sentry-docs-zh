使用Sentry[**问题**页面](https://sentry.io/issues/)上附加到问题的状态标签来帮助您分类和优先考虑对您重要的应用程序问题。请记住，问题一次只能有**一个状态**。

以下是所有状态的列表、它们如何分配给问题以及它们的自定义搜索词：

| 状态         | 条件                                                                                                                                                                                                            | 自定义搜索词 |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------ |
| **新**        | 在过去7天内创建的问题。                                                                                                                                                                        | is:new       |
| **进行中**    | 超过7天前创建的问题或已手动标记为已审核的问题。                                                                                                                                      | is:ongoing   |
| **升级中** | 超过其预测事件量的问题。有关更多详细信息，请参见[升级问题算法](escalating-issues)。请注意，升级问题目前不适用于合并/取消合并的问题。 | is:escalating |
| **回归**  | 已解决但再次出现的问题。                                                                                                                                                                       | is:regressed |
| **已归档**   | 已被标记为已归档的问题。                                                                                                                                                                         | is:archived  |
| **已解决**   | 已被标记为已修复的问题。                                                                                                                                                                         | is:resolved  |

限制您看到的问题的一种方法是选择**问题**页面顶部的选项卡。在"未解决"选项卡上，您会找到`新`、`进行中`、`升级中`和`回归`问题。您还可以通过选择"待审核"、"回归"、"升级中"或"已归档"选项卡进一步缩小范围。

## 手动分类问题

虽然某些问题状态会自动添加和更新，但您可以手动`归档`或`解决`问题，这也会更改其状态并将其从`is:for_review`列表中删除。`is:for_review`列表中的项目是新问题、回归问题或尚未审核的未解决问题。

### 归档

归档问题以将其移出问题流并暂停其警报，直到问题变得更糟。归档适用于嘈杂的问题，这些问题不太紧急或不适用于您或您的团队。归档将问题状态更改为`已归档`并将问题从"未解决"移动到"已归档"选项卡。

如果该问题的事件在短时间内显著增加，Sentry将自动将问题返回到列表顶部并将其状态更改为`升级中`。要了解有关此工作原理的更多信息，请参见[升级问题算法](escalating-issues)。默认情况下，问题在升级前被归档。还有将问题标记为`已归档`的选项：

- 永久
- 设定时间段
- 直到发生设定次数
- 直到影响设定数量的用户

如果您将问题归档"永久"，与该问题相关的事件将继续被记录，但即使满足升级条件，问题也永远不会被标记为升级。您仍然可以取消归档任何已归档的问题，包括那些已被"永久"归档的问题。所有取消归档的问题将显示在"未解决"选项卡中。

### 解决

当问题已修复时，您可以手动将其标记为`已解决`。您还可以通过以下方式进一步指定解决：

- 您的下一个版本
- 当前版本
- 另一个版本

或

- 通过[在提交中包含问题ID](/product/releases/associate-commits/?original_referrer=https%3A%2F%2Fdocs.sentry.io%2Fproduct%2Fissues%2Fstates-triage%2F#resolve-issues-by-commit)

如果相同的问题再次出现，其状态将自动更改为`回归`。

### 删除

您可以通过删除问题将其从问题列表中删除，但如果它再次发生，它将作为新问题重新出现。

还有`删除并永久丢弃`的选项，这将使问题永远不会再次出现，即使它再次发生。与永久删除的问题相关的任何未来事件将自动被丢弃，并且不会计入[您的配额](/pricing/quotas/)。

<Alert>

您只能删除[错误问题](/product/issues/issue-details/error-issues/?original_referrer=https%3A%2F%2Fdocs.sentry.io%2Fproduct%2Fissues%2Fstates-triage%2F)。其他问题类别，如[性能问题](/product/issues/issue-details/performance-issues/?original_referrer=https%3A%2F%2Fdocs.sentry.io%2Fproduct%2Fissues%2Fstates-triage%2F)，不支持此功能。

</Alert>

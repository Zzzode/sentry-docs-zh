如果您启用了[会话重放](/product/explore/session-replay/)，您将能够在Sentry的[**问题**](https://sentry.io/orgredirect/organizations/:orgslug/issues/)页面上看到新的问题类型。

## 配额消耗

<Alert>
重放问题不消耗错误配额。
</Alert>

当创建重放时，它会消耗[重放配额](/pricing/quotas/manage-replay-quota/)（不要与[错误配额](/pricing/quotas/manage-event-stream-guide/)混淆）。当重放被服务器处理时，Sentry会检测下面列出的重放问题类型并将它们分组为问题。这些错误是在摄取后在后端创建的，不消耗错误配额。它们不会出现在您的[统计](https://sentry.io/orgredirect/organizations/:orgslug/stats/)页面上的"总错误"或"已接受错误"图表中。您不会被收取它们的费用。

## 重放问题类型

- [愤怒点击](/product/issues/issue-details/replay-issues/rage-clicks/)是在同一无响应页面元素上的一系列连续点击。它们是用户沮丧的强烈信号，很可能值得您关注。

- [水合错误](/product/issues/issue-details/replay-issues/hydration-error/)是React特有的问题，当初始UI与服务器渲染的内容不匹配时发生。它们可能导致用户页面加载体验变慢。

## 搜索重放问题

如果您在问题详情搜索中使用术语`issue.category:replay`，您将能够过滤愤怒点击和水合错误问题。

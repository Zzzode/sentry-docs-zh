未压缩资源问题发生在需要下载以加载浏览器页面的文件在传输过程中未压缩时。这可能表明提供资产文件的服务器或CDN（内容交付网络）配置错误。

## 检测标准

此性能问题的检测器会查找具有以下条件的特定资源（资源）跨度：

- 跨度必须是：`resource.link`或`resource.script`跨度之一，实际上这意味着它是一个`.css`或`.js`文件。
- 它必须超过`512kb`大小（以过滤掉不太可能从压缩中受益的较小文件）。
- 跨度持续时间必须超过`500ms`（以过滤掉更快的传输或内部网络中提供的资产）。
- 事务中必须有另一个具有压缩文件的资产跨度。这有助于过滤掉用户浏览器未发送`Accept-Encoding`标头的情况。

如果Sentry没有在您期望的地方检测到未压缩资源问题，可能是因为事务未满足上述任一标准。

您可以在**项目设置 > 性能**中配置未压缩资源问题的检测器阈值：

![未压缩资源检测器阈值设置](./img/uncompressed-asset-detector-settings.png)

## 跨度证据

您可以通过"跨度证据"部分中的四个主要方面来识别未压缩资源问题：

- 事务名称
- 慢资源跨度 - 未压缩的慢跨度，通常包含资产url
- 资产大小
- 持续时间影响 - 如果此跨度在关键路径中，可能添加到总事务时间的分数

![未压缩资源跨度证据](./img/span-evidence.png)

通过转到Sentry中的**问题**页面，选择您的项目，点击您要检查的未压缩资源错误，然后在"详情"选项卡中向下滚动到"跨度证据"部分来查看它。

## 建议

检查托管资产的服务器或CDN（内容交付网络），并确保它已设置为gzip或brotli压缩。

端点回归问题是一类通用问题，其中事务持续时间随时间增加并降低应用程序性能。Sentry主动监控开箱即用的常见端点，并报告任何可能的回归，将它们分组为端点回归问题类型。

## 检测标准

性能问题的检测器会定期检查项目中最常见端点的第95百分位事务持续时间。当检测到p95值的显著增加并已持续一段时间时，会创建回归问题。

要被归类为端点，事务操作必须是以下之一：

- `function.aws`
- `function.aws.lambda`
- `http.server`
- `serverless.function`
- `asgi.server`
- `rails.request`

## 回归证据

要查找额外信息，请转到问题页面并点击您感兴趣的端点回归问题。在"详情"选项卡的顶部部分，您将找到以下内容：

- **端点名称：** 已回归的事务名称
- **持续时间变化：** 持续时间的绝对和相对变化值
- **近似开始时间：** 回归发生的大致时间

## 潜在原因

"潜在原因"部分显示了可能最导致减速的跨度列表。在每个跨度旁边，您将找到以下内容：

- 跨度描述，指向[跨度摘要](/product/insights/overview/transaction-summary/)页面，您可以在其中找到有关该特定跨度的更多信息
- 回归前跨度自身时间的p95
- 回归后跨度自身时间的p95
- 回归前后p95跨度自身时间的百分比变化

## 比较事件

"比较事件"部分允许您比较回归发生前后的示例事件。我们将接近原始p95基线持续时间的事件称为"基线事件"，将接近新p95基线的事件称为"回归事件"。

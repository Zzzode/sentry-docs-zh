移动应用程序中的主UI线程处理用户界面事件，如按钮按下和页面滚动。为了防止应用程序挂起和应用程序无响应错误，主UI线程不应用于执行长时间运行的操作，如数据库查询。这类操作会阻塞整个UI，直到它们完成运行，并妨碍用户与应用程序交互。

## 检测标准

此性能问题的检测器会查看主线程上数据库跨度的总非重叠持续时间。如果超过16ms，则会创建性能问题。

您可以在**项目设置 > 性能**中配置主线程数据库操作问题的检测器阈值：

![主线程数据库操作检测器阈值设置](./img/db-on-main-thread-detector-settings.png)

## 跨度证据

跨度证据通过向您显示三个主要方面来识别主线程数据库操作问题的根本原因：

- **事务名称**
- **父跨度：** 数据库跨度发生的位置
- **违规跨度：** 在主线程中执行数据库查询的实际跨度

![数据库跨度证据](./img/db-io.png)

通过转到Sentry中的**问题**页面，选择您的Android项目，点击您要检查的数据库错误，然后在"详情"选项卡中向下滚动到"跨度证据"部分来查看它。

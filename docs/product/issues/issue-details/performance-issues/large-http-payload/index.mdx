当http跨度的编码主体大小超过阈值时，会创建大型HTTP负载问题。

## 检测标准

此性能问题的检测器会查找满足以下条件的单个http跨度：

- HTTP url不能以\_next/static/或\_next/data/开头
- HTTP url不能以扩展名结尾（例如.js、.css、.png、.jpg、.jpeg、.mp3），除非扩展名是.json
- HTTP跨度必须具有`http.response_content_length`（由`@sentry/browser` SDK版本`7.53.0`添加），且超过`300kb`
- HTTP跨度持续时间必须超过100ms

如果Sentry没有在您期望的地方检测到大型HTTP负载问题，可能是因为事务未满足上述任一标准。

您可以在**项目设置 > 性能**中配置大型HTTP负载问题的检测器阈值：

![大型HTTP负载检测器阈值设置](./img/large-http-payload-detector-settings.png)

## 跨度证据

您可以通过"跨度证据"部分中的四个主要方面来找到有关大型HTTP负载问题的额外信息：

- **事务名称**
- **大型HTTP负载跨度：** 大型HTTP跨度，通常包含资产url。
- **负载大小：** 负载的大小（以字节为单位）。

![大型HTTP负载跨度证据](./img/span-evidence.png)

通过转到Sentry中的**问题**页面，选择您的项目，点击您要检查的大型HTTP负载问题，然后在"详情"选项卡中向下滚动到"跨度证据"部分来查看它。

## 修复大型HTTP负载问题

出现连续大型HTTP负载问题的原因各不相同，但以下是一些帮助您解决它们的建议：

- 如果您可以控制后端，请检查您是否发送了比所需更多的数据，或者数据是否可以在发送到客户端之前进一步处理。
- 如果需要所有端点数据，请考虑使用分页等方法在视图之间分割数据。
- 检查是否对序列化程序进行了任何更改，这些更改可能导致主体大小比必要的大。

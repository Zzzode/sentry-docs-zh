移动应用程序中的主（或UI）线程负责处理所有用户交互，需要能够实时响应手势和点击。如果长时间运行的操作阻塞了主线程，应用程序会变得无响应，影响用户体验质量。

将JSON blob解码为数据结构的过程，是可能影响应用程序响应性的长时间运行操作的一个示例。如果在主线程上检测到长时间运行的JSON解码操作，Sentry会将其标记为问题。

将JSON blob解码为数据结构可能是一个长时间运行的操作，特别是对于大型blob。当我们检测到主线程上的长时间运行JSON解码操作时，会创建此问题类型。

## 检测标准

必须启用[性能分析](/product/explore/profiling/)，Sentry才能检测主线程JSON解码问题。Sentry会查找在主线程上运行至少40ms的与JSON解码相关的函数。此阈值确保函数在性能分析器收集的足够数量的样本（~10ms每个样本x 4 = ~40ms总计）中出现，然后才会被检测为问题。

## 函数证据

要查找有关主线程JSON解码问题的额外信息，请转到其**问题详情**页面并向下滚动到"函数证据"部分，该部分显示以下内容：

- **事务名称：** 检测到问题的事务名称
- **可疑函数：** 触发问题检测的函数（在这种情况下，是JSON解码函数）
- **持续时间：** 函数执行所需的时间以及性能分析器收集的包含该函数的连续样本数量

## 示例

### iOS

以下代码通过调用`JSONDecoder.decode<T>`在主线程上解码JSON来更新视图：

```swift
func updatePersonView(json: Data) throws {
  personView.person = try JSONDecoder().decode(Person.self, from: json)
}
```

性能可以通过将JSON解码移动到后台队列来改善：

```swift
func updatePersonView(json: Data) {
	DispatchQueue.global(qos: .userInitiated).async {
    do {
			let person = try JSONDecoder().decode(Person.self, from: json)
			DispatchQueue.main.async {
				personView.person = person
			}
    catch let error {
			print(error) // 处理错误
    }
	}
}
```

### Android

以下代码通过调用`Gson.fromJson()`在UI线程上解码JSON来更新视图：

```kotlin
fun updatePersonView(json: String) {
	personView.person = Gson().fromJson(json, Person::class.java)
}
```

性能可以通过将JSON解码移动到协程来改善：

```kotlin
fun updatePersonView(json: String) {
	lifecycleScope.launch(Dispatchers.IO) {
	  val person = Gson().fromJson(json, Person::class.java)

    withContext(Dispatchers.Main) {
      personView.person = person
    }
  }
}
```

移动应用程序中的主（或UI）线程负责处理所有用户交互，需要能够实时响应手势和点击。如果长时间运行的操作阻塞了主线程，应用程序会变得无响应，影响用户体验质量。

评估字符串和正则表达式（regex）之间的匹配可能是可能影响应用程序响应性的长时间运行操作。如果在主线程上检测到可以卸载到后台的长时间运行正则表达式操作，Sentry会将其标记为问题。

## 检测标准

必须启用[性能分析](/product/explore/profiling/)，Sentry才能检测主线程正则表达式问题。Sentry会查找在主线程上运行至少40ms的与正则表达式相关的函数。此阈值确保函数在性能分析器收集的足够数量的样本（~10ms每个样本x 4 = ~40ms总计）中出现，然后才会被检测为问题。

## 函数证据

要查找有关主线程正则表达式问题的额外信息，请转到其**问题详情**页面并向下滚动到"函数证据"部分，该部分显示以下内容：

- **事务名称：** 检测到问题的事务名称
- **可疑函数：** 触发问题检测的函数（在这种情况下，是正则表达式函数）
- **持续时间：** 函数执行所需的时间以及性能分析器收集的包含该函数的连续样本数量

## 示例

### iOS

以下代码在大型字符串中搜索与正则表达式的匹配：

```swift
let url = Bundle.main.url(forResource: "mobydick", withExtension: "txt")!
let text = try! String(contentsOf: url)
let regex = try! NSRegularExpression(pattern: #"([tT]he)?.*([wW]hale)"#, options: [.dotMatchesLineSeparators])
let results = regex.matches(in: text, range: NSMakeRange(0, text.count))
```

性能可以通过将创建和使用正则表达式的调用移动到较低优先级队列（以及在此示例中加载被搜索文本字段的调用）来改善：

```swift
DispatchQueue.global(qos: .userInitiated).async {
    let url = Bundle.main.url(forResource: "mobydick", withExtension: "txt")!
    let text = try! String(contentsOf: url)
	let regex = try! NSRegularExpression(pattern: #"([tT]he)?.*([wW]hale)"#, options: [.dotMatchesLineSeparators])
	let results = regex.matches(in: text, range: NSMakeRange(0, text.count))
	DispatchQueue.main.async {
		// 显示结果...
	}
}
```

### Android

以下代码在UI线程上调用`String.matches`以在大型文本中搜索与正则表达式的匹配：

```kotlin
fun matchRegex(text: String, regexPattern: String) {
  val matches = text.matches(regexPattern.toRegex())
  if (matches) {
    // 执行某些操作
  }
}
```

性能可以通过将匹配移动到协程来改善：

```kotlin
fun matchRegex(text: String, regexPattern: String) {
  lifecycleScope.launch(Dispatchers.Default) {
    val pattern = Pattern.compile(regexPattern)
    val matches = pattern.matcher(text).matches()

    withContext(Dispatchers.Main) {
      if (matches) {
        // 执行某些操作
      }
    }
  }
}
```

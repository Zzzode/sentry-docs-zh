移动应用程序中的主UI线程处理用户界面事件，如按钮按下和页面滚动。为了防止应用程序挂起和应用程序无响应错误，主UI线程不应用于执行长时间运行的操作，如文件I/O。这类操作会阻塞整个UI，直到它们完成运行，并妨碍用户与应用程序交互。

## 检测标准

此性能问题的检测器会查看文件I/O跨度的总非重叠持续时间。如果超过16ms，则会创建性能问题。

您可以在**项目设置 > 性能**中配置主线程文件I/O问题的检测器阈值：

![主线程文件I/O检测器阈值设置](./img/file-main-thread-io-detector-settings.png)

## 跨度证据

跨度证据通过向您显示三个主要方面来识别主线程文件I/O问题的根本原因：

- 事务名称
- 父跨度 - 文件I/O跨度发生的位置
- 违规跨度 - 在主线程中执行文件I/O的实际跨度

![文件I/O跨度证据](./img/file-io.png)

通过转到Sentry中的**问题**页面，选择您的Android项目，点击您要检查的文件I/O错误，然后在"详情"选项卡中向下滚动到"跨度证据"部分来查看它。

## 示例

以下是如果您使用Swift或Kotlin，文件I/O错误可能如何在您的代码中出现以及如何解决的示例。

如果您使用Swift，当您打开文件以获取其内容时，您可能会看到以下内容：

```swift
if let path = Bundle.main.path(forResource: "LoremIpsum", ofType: "txt") {
    let content = String(contentsOfFile: path)
    label.text = content
}
```

为了修复此性能问题，您可以使用调度队列：

```swift
let dispatchQueue = DispatchQueue(label: "ViewController", attributes: .concurrent) {
    dispatchQueue.async {
        if let path = Bundle.main.path(forResource: "LoremIpsum", ofType: "txt") {
            let content = String(contentsOfFile: path)
                DispatchQueue.main.async {
                    label.text = content
                }
        }
    }
}
```

如果您使用Kotlin，您可能会看到以下内容：

```kotlin
val file = File(context.cacheDir, "LoremIpsum.txt")
val content = file.readText()
```

为了修复此性能问题，您可以初始化另一个线程来读取文件。

```kotlin
suspend fun readFile(): String = withContext(Dispatchers.IO) {
    val file = File(context.cacheDir, "LoremIpsum.txt")
    val content = file.readText()
    return content
}
```

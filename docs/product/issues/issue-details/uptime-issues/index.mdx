正常运行时间问题是对特定URL检测到的停机事件的分组。停机事件由活动正常运行时间警报在HTTP请求未能满足我们的[正常运行时间检查标准](/product/uptime-monitoring/#uptime-check-criteria)时生成。

![正常运行时间问题详情](./img/uptime-issue-details.png)

## 追踪错误

针对配置了Sentry支持的SDK之一的Web服务进行的正常运行时间检查包含一个[追踪](/concepts/key-terms/tracing/)，可用于跟踪由失败的HTTP正常运行时间检查导致的检测到的错误。追踪导航器允许您浏览停机的潜在根本原因，是快速识别和解决问题的强大工具。

## 问题生命周期

正常运行时间问题按监控的URL分组，并在首次检测到停机时创建。当监控的URL返回到健康状态并满足我们的[正常运行时间检查标准](/product/uptime-monitoring/#uptime-check-criteria)时，Sentry会自动解决正在进行的正常运行时间问题。如果URL随后经历停机，问题的状态将更改为回归。

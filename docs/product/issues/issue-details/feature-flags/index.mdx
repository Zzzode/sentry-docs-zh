<Alert title="当前处于测试版">

功能标志更改跟踪和功能标志评估跟踪当前处于开放测试版。

</Alert>

启用功能标志集成可提供对错误前应用程序状态的深入洞察。每个错误事件上都会显示标志及其评估结果的列表，按评估顺序排列。

将 Sentry 与您的功能标志提供商集成使 Sentry 能够将功能标志更改与新的错误事件相关联，并将某些更改标记为可疑。

## 评估跟踪

标志评估将显示在问题详情页面的"功能标志"部分中作为表格，其中"可疑"标志预测以黄色突出显示。

![标志更改可疑](./img/SuspectFlag.png)

对于每个事件，我们跟踪错误前最近评估的 100 个标志。通过阅读 [问题详情页面文档](/product/issues/issue-details/#feature-flags) 了解有关如何在 Sentry UI 中与功能标志洞察进行交互的更多信息。

设置评估跟踪还允许您将问题搜索与功能标志结合使用。在问题搜索中，使用 `flags` 关键字将允许您过滤包含错误的问题，其中功能标志评估值为 true 或 false。

![标志更改搜索](./img/FlagsSearch.png)

这使您可以快速找到特定标志评估及其感兴趣值存在的所有错误。有关搜索语法的更多详细信息，请参见 [可搜索属性](/concepts/search/searchable-properties/issues/#flags)。

### 设置评估跟踪

要设置评估跟踪，请访问您平台的 SDK 集成文档：
* [JavaScript](/platforms/javascript/feature-flags/)
* [Python](/platforms/python/feature-flags/)

### 标志分布视图

标志分布视图允许您查看接收特定功能标志值的用户百分比。您可以通过从问题详情视图中选择"查看所有标签和功能标志"，然后在右上角选择"所有功能标志"来访问此视图。

![标志分布](./img/FlagDistribution.png)

此视图对于诊断功能标志更改的影响受众很有用。例如，如果发生新问题，并且标志分布视图显示 100% 受影响的用户正在接收最近更改的功能标志值，这可能表明功能标志更改导致了新错误。

## 更改跟踪

更改跟踪使 Sentry 能够监听您的功能标志的添加、删除和修改。更改时，我们将在审计日志中记录更改事件。

![标志更改跟踪](./img/FlagChangeTracking.png)

在问题中，审计日志在"事件"图表中表示，并呈现为"发布"线。

![标志更改问题](./img/FlagChangeIssue.png)

如果更改导致新的错误事件，我们将通过在问题详情页面上将功能标志标记为"可疑"来通知您。

![标志更改可疑](./img/SuspectFlag.png)

通过阅读 [问题详情文档](/product/issues/issue-details/#feature-flags) 了解有关如何在 Sentry UI 中与功能标志洞察进行交互的更多信息。

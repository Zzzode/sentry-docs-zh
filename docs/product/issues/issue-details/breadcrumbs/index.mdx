Sentry 使用 _面包屑_ 来创建问题发生前的事件轨迹。这些事件与传统日志非常相似，但可以记录更丰富的结构化数据。

SDK 通过启用集成自动开始记录面包屑。要了解如何为您的平台执行此操作，请从左侧导航顶部的下拉菜单中选择它。您将被带到您使用的平台特定的面包屑文档。

如果错误事件提供了轨迹，您将在 [Sentry.io](https://sentry.io/orgredirect/organizations/:orgslug/issues/) 中的 **问题详情** 页面上看到面包屑。

![面包屑部分](../img/issue-breadcrumbs.png)

## 面包屑显示

您将在问题的"详情"选项卡中看到事件可能可用的一些面包屑。要访问其余的，请点击部分底部的"查看全部"按钮。这将打开一个滑出抽屉，您可以在其中搜索、过滤、排序和更改所有可用面包屑的时间显示。

以下是您可以缩小所见内容的所有方法：
- **搜索**：仅显示与查询匹配的面包屑。匹配可以发生在 `type`、`category`、`message` 和 `data` 上。
- **过滤**：按关联的 `type` 和 `level` 过滤。
- **排序**：确定面包屑是按最新还是最旧优先显示。（您的排序偏好将保存到浏览器。）
- **时间显示**：确定面包屑时间戳的显示方式，可以是绝对时间戳（尊重用户对 24 小时时钟/时区的偏好），或相对于列表中第一个面包屑。如果相对时间戳显示为负数，则意味着它发生在列表中顶部面包屑之前。（您的时间显示偏好将保存到浏览器。）

![面包屑抽屉](./img/breadcrumbs-drawer.png)


## 面包屑属性

每个面包屑由以下内容组成：

- **类型**：`type` 是一个半内部属性，控制面包屑的分类方式。如果保持不变，所有面包屑都记录为 `default`，但 Sentry 提供其他 [类型](https://develop.sentry.dev/sdk/data-model/event-payloads/breadcrumbs/#breadcrumb-types)，每种类型在 UI 中都有自己的颜色/图标。

- **类别**：事件类别。此数据类似于记录器名称，可帮助您了解事件发生的地点，例如 `auth`。

- **消息**：描述事件的字符串，呈现为保留所有空格的文本。它通常用作传统日志消息的替代品。

- **数据**：面包屑任意数据的键值映射。这对于附加与面包屑相关的结构化信息很有用，如 ID 或变量值。

- **级别**：事件的严重性。级别设置为五个值之一：fatal、error、warning、info 和 debug，按严重性顺序排列。

- **时间**：显示面包屑发生时间的时间戳。格式是 [RFC 3339](https://tools.ietf.org/html/rfc3339) 中定义的字符串，或表示自 Unix 纪元以来经过的秒数的数值。


在 [面包屑接口开发者文档](https://develop.sentry.dev/sdk/data-model/event-payloads/breadcrumbs/) 中了解有关面包屑数据的更详细信息。

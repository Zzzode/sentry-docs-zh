小部件库包含一系列预构建的小部件，您可以将其添加到[自定义仪表板](/product/dashboards/customize-dashboards/)中。您可以通过点击"添加小部件"按钮并在仪表板上选择"从小部件库"来访问小部件库。

该库包括以下小部件：

- **持续时间分布**：多数据系列图表，可视化事务持续时间在各种百分位数（p50、p75 和 p95）上的分布
- **高吞吐量事务**：前五个具有最多事件的事务，以面积图表示
- **按国家/地区的LCP**：显示每个国家/地区页面加载时间 p75 的表格
- **痛苦用户**：经历过慢事务（事务持续时间大于 1200 毫秒）的唯一用户总数
- **慢速与快速事务**：条形图比较超过 300 毫秒（慢速）和低于 300 毫秒（快速）的事务百分比
- **待审查问题**：待审查未解决问题的表格，按最近看到的问题排序
- **顶级未处理错误类型**：最常遇到的未处理错误
- **受错误影响的用户**：错误总数与受错误影响的唯一用户数量的比较
- **最近发布的崩溃率**：最近发布的会话崩溃百分比
- **会话健康**：异常、崩溃、错误和健康会话的总数

您可以通过点击小部件上的上下文菜单并选择"编辑小部件"来更改这些预构建小部件的标题、查询、字段、可视化类型、排序顺序和其他字段，以适合您的用例。

## 示例用例

### 跨发布版本的性能

要监控您的应用程序在某个发布版本上的表现，您可以通过将发布版本添加到搜索条件来修改"持续时间分布"小部件：

- 搜索条件 1：`release.version:{version}`

要与另一个发布版本的性能进行比较，您可以点击"添加查询"按钮添加另一个查询，并查询不同的发布版本：

- 搜索条件 2：`release.version:{another_version}`

您还可以使用以下条件比较某个版本之前和之后的性能：

- 搜索条件 1：`release.version:<{version}`
- 搜索条件 2：`release.version:>={version}`

### 响应阈值

"持续时间分布"小部件显示事务持续时间时间的分布。此可视化可以修改为显示事务持续时间是否在某些目标阈值内，并将它们呈现为"令人满意"、"可容忍"或"令人沮丧"的事务。要创建此类可视化，请将"可视化显示"更改为"面积图"。面积图堆叠结果，更适合累积结果：

- 可视化显示：`面积图`

将其中一个 y 轴值设置为 `count()` 并移除其他两个轴：

- Y 轴：`count()`

对于第一个查询，添加令人满意事务持续时间的搜索条件（此示例使用 300 毫秒作为令人满意的响应阈值）：

- 搜索条件 1：`transaction.duration:<300`

点击"添加查询"按钮添加另一个可容忍条件的查询。在此示例中，我们使用 [Apdex](/product/insights/overview/metrics/#apdex) 定义，其中可容忍响应时间在令人满意阈值和令人满意阈值的四倍之间：

- 搜索条件 2：`transaction.duration:<1200 AND transaction.duration:>=300`

最后，添加第三个令人沮丧事务持续时间的查询：

- 搜索条件 3：`transaction.duration:>=1200`

图表现在显示不同响应时间阈值的累积计数。

### 我的首要问题

您可以通过计算与每个唯一问题关联的错误事件来跟踪分配给您或您团队的问题的影响。通过更新搜索条件以过滤问题分配来修改"待审查问题"小部件：

- 搜索条件：`is:unresolved is:for_review assigned_or_suggested:me`

更新"列"以添加 `project`，这样您可以看到每个问题关联的项目，并将"排序方式"更改为 `Events`，这样小部件按降序显示具有最多错误事件的问题：

- 列：`issue, assignee, events, title, project`
- 排序方式：`Events`

### 优先问题

[问题数据集](/product/dashboards/widget-builder/#issues)的独特之处在于，您可以在搜索条件中添加一个自由形式[标记](/concepts/search/#syntax)，该标记将按包含该标记的问题标题进行过滤。例如，您可以搜索 `integrity` 以按所有包含 `integrity` 的问题标题进行过滤。使用您选择的标记更新搜索条件：

- 搜索条件：`is:unresolved is:for_review {token}`

更新"列"以添加 `links`，这样您可以看到与问题相关的任何外部链接，例如 GitHub 或 Jira 问题。将"排序方式"更新为 `Priority`，这样小部件显示最近呈上升趋势的问题：

- 列：`issue, assignee, events, title, links`
- 排序方式：`Priority`

### 发布健康

要随时间监控发布的健康状况，您可以按如下方式修改"会话健康"小部件：

- 可视化显示：`面积图`

添加搜索过滤器以将数据缩小到特定发布：

- 搜索条件 1：`release:{version}`

## 什么是 Sentry？

Sentry 是一个软件监控工具，帮助开发者识别和调试性能问题和错误。从端到端分布式跟踪到性能监控，Sentry 提供代码级可观察性，使诊断问题和持续了解应用程序代码在系统和服务间的健康状况变得容易。

## Sentry 能帮助解决什么？

作为全栈应用程序监控解决方案，Sentry 提供以下功能来帮助您保持应用程序平稳运行和用户满意：

### 错误监控

Sentry 的错误监控功能使您能够快速找到问题的根本原因，以便了解并修复损坏的内容。

Sentry 自动捕获未处理的异常（错误）。然后它将类似的错误分组为 [问题](/product/issues/)，使分类和管理它们更加高效。

- 如果您正在监控 Web 项目，您可以 [**重播**](/product/explore/session-replay/) 用户在您的应用上的体验，并通过 [**源映射**](/product/sentry-basics/integrate-frontend/upload-source-maps/) 将该错误连接到有问题的代码行。
- 如果您同时为前端和后端设置 Sentry，您将能够使用 [**分布式跟踪**](/product/sentry-basics/concepts/tracing/distributed-tracing/) 查看导致感兴趣事件的（前端和后端）事件序列。
- 此外，您可以设置 [**集成**](/organization/integrations/source-code-mgmt/) 来链接您的源代码，以便 Sentry 可以显示 [**可疑提交**](/product/issues/suspect-commits/) 和 [**堆栈跟踪**](/organization/integrations/source-code-mgmt/github/#stack-trace-linking) 以显示有问题的代码行，以及基于提交作者 [**建议受让人**](/organization/integrations/source-code-mgmt/github/#suspect-commits-and-suggested-assignees) 的问题。

### 跟踪

跟踪通过查看堆栈各个组件之间的交互，为您提供应用程序中发生的情况从前端到后端以及系统之间的完整视图。如果您依赖微服务或试图找到困难用户投诉的根本原因，这尤其有用。通过 [分布式跟踪](/concepts/key-terms/tracing/distributed-tracing/)，您可以快速识别性能瓶颈。

您还可以通过查看 [跟踪视图](/concepts/key-terms/tracing/trace-view/) 页面获取有关错误或性能问题发生时发生的情况的完整上下文。这将使您能够除其他事项外，将加载缓慢的页面一直追溯到性能不佳的 API 调用。最后，您可以使用 [跟踪浏览器](/product/explore/traces/) 页面搜索跨度数据，包括与跟踪本身结构相关的条件。

### 性能分析

Sentry [**性能分析**](/product/profiling/) 提供对生产环境中应用程序性能的代码级洞察，以便您可以更快地找到并修复有问题的代码行。您可以在应用程序运行的任何环境中分析代码，以便快速查看哪些函数导致了缓慢的跨度。

### 性能监控

我们的性能监控功能帮助您跟踪应用程序性能，测量吞吐量和延迟等指标，并查看跨多个服务的错误影响。

- Sentry 提供洞察来帮助您监控应用程序的性能。这包括 Web 指标、移动指标、查询、HTTP 请求等。您可以深入研究事件样本以调查性能变化。

### 发布健康监控

让 Sentry 了解您的 [**发布**](/product/releases/) 使您能够看到发布开始退化的那一刻，以便您可以快速采取行动。您可以实时了解所有发布的情况，让您查看核心指标，如无崩溃会话、版本采用率和失败率。

发布与 Sentry 的其余部分集成，因此您可以直接查看错误或性能问题如何受到发布的影响，以及能够解决发布特定的问题。

### AI 代理监控

我们的 [**AI 代理监控**](/product/insights/ai/agents/) 功能让您在应用程序的更广泛背景下了解您的 AI 代理工作流程。当您在使用 AI 代理的项目中 `pip install sentry` 时，Sentry 将自动获取有用的指标，如代理调用、工具执行、代理之间的交接和令牌使用情况，将它们发送到我们的 AI 代理洞察仪表板。

### 正常运行时间监控

Sentry 的 [**正常运行时间监控**](/product/uptime-monitoring/) 通过监控相关 URL 帮助您保持 Web 服务的正常运行时间。它持续跟踪配置的 URL，提供警报和洞察，以快速识别停机时间并解决问题。通过利用 [分布式跟踪](/product/uptime-monitoring/uptime-tracing/)，Sentry 使您能够精确定位正常运行时间检查期间发生的任何错误，简化分类并加速根本原因分析。

### 周期性作业监控

[**Cron 监控器**](/product/crons/) 允许您在 Sentry 中监控任何计划、周期性作业的正常运行时间和性能。一旦实施，它将允许您获得警报和指标，帮助您解决错误、检测超时并防止服务中断。

### 跨环境的数据可见性

[**发现**](/product/explore/discover-queries/) 通过构建和丰富您的错误数据，提供跨环境的数据可见性。您可以 [**查询**](/product/explore/discover-queries/query-builder/) 并解锁对整个系统健康状况的洞察，并在一个地方获得关键业务问题的答案。您还可以构建自己的 [**仪表板**](/product/dashboards/) 来帮助您监控对您的应用最重要的内容。

### 代码覆盖率

[Codecov](/product/codecov/) 为开发者提供 **测试代码覆盖率**。它帮助改进测试、覆盖率和代码可靠性，让您在监控错误时直接在 Sentry 中获得测试覆盖率洞察。

### 无缝开发者工作流程

Sentry 提供许多 [**集成**](/organization/integrations/) 来支持无缝开发者工作流程，包括：

- [源代码管理](/organization/integrations/source-code-mgmt/)、代码映射、代码所有者和与 GitHub、GitLab、Azure DevOps 和 Bitbucket 的可疑提交。
- [问题跟踪](/organization/integrations/issue-tracking/) 与 Jira、GitHub 和许多其他工具。
- [警报和通知](/organization/integrations/notification-incidents/) 与 Slack、Discord、PagerDuty 等。

### 支持 100 多个平台的 SDK

Sentry 支持 [100 多个平台和框架](/platforms/)，包括 30 多种编程语言，因此无论您的应用程序构建在什么之上，您都可以跟踪错误和性能问题。我们的 SDK 为您提供保障，因此您只需几行代码即可开始使用 Sentry。

您可以通过搜索或使用页面级显示筛选器来筛选**性能**页面上显示的信息。

## 事务视图

该页面显示事务的视图。您可以筛选和设置事务显示的不同方式在此页面上完整描述。

### 所有事务

当您选择"所有事务"时，表格包括您选择的任何项目和时间段的所有事务。

### Web Vitals

[Web Vitals](/product/insights/web-vitals/)包括前端页面加载事务。事务会自动筛选为仅包括`transaction.op:pageload`。这通常对应于所有浏览器页面转换，即页面的首次加载。它还包括从加载本身开始的性能信息，以及正在加载的SPA框架（如React或Vue）的性能（如果可用）。

### 前端

"前端"包括所有前端事务，而不特别关注Web Vitals。这通常对应于SPA框架内部处理的页面转换，如React或Vue。请注意，如果您选择了后端项目，此模式也会显示后端事务。

### 后端

"后端"强调持续时间、吞吐量、失败率和Apdex。请注意，如果您选择了前端项目，此模式也会显示所有前端事务。

### 移动

"移动"强调与[移动应用程序开发](/product/insights/mobile/)相关的性能指标。请注意，如果您选择了非移动项目，此模式也会显示所有这些事务。

## 筛选事务

### 事务名称搜索

**性能**页面上的搜索栏不是通过匹配事件属性来搜索事务，而是仅按事务名称筛选。您可以搜索特定的事务名称或使用通配符，通配符可以在搜索词中代替一个或多个字符（例如，`/api/*/organization`）。

![更新的事务搜索](./img/transaction-name-search.png)

当您开始输入时，会出现与您的搜索匹配的事务名称列表。有两种方式与搜索结果交互：

1. 点击事务名称将在**性能**页面上应用筛选器，仅显示所选事务的事务数据。
2. 点击搜索结果上的向上箭头图标将带您到该事务的摘要页面。

### 搜索条件筛选器

在事务摘要中，搜索栏的操作类似于Discover[查询构建器](/product/explore/discover-queries/query-builder/#filter-by-search-conditions)中的搜索栏。例如，您可以使用`release:`作为键字段并分配版本作为值来将事务搜索细化为特定版本。

## 小组件

[小组件](/product/insights/overview/filters-display/widgets/)提供您可以更改以最适合您工作流的可视化。

![带有Web Vitals小组件的性能主页。](./img/01_half_widget_frontend.png)

在第一行，有所选项目类型（前端或后端）的常见指标的时间序列和分布图，如用户痛苦或失败率。您可以并排比较这些以找到任何相关性。

在第二行，有可操作的小组件，可以显示您应用程序中关注的区域，如最回归的事务或最重复的问题。在这些小组件下方，有事务列表，您可以点击查看其他上下文。这些列表中显示的事务可以通过点击事务名称旁边的'X'暂时从视图中删除。

每个小组件在右上角都有一个菜单，您可以在其中更改显示的小组件。您的更改将本地保存会话或直到应用程序存储被清除。之后，显示默认的小组件集。

## 事务表

事务表提供了一个基于当前页面筛选器和显示选项的最频繁事务列表。默认情况下，表格按每分钟事务数排序，最频繁的事务首先显示。表格可能会根据页面和搜索条件筛选器进一步筛选。
[显示筛选器](#display-filter)控制表格列中显示的内容，如：

- 事务名称
- 项目
- 操作
- [TPM](/product/insights/overview/metrics/#throughput-total-tpm-tps)
- [P50](/product/insights/overview/metrics/#average-transaction-duration)
- [P95](/product/insights/overview/metrics/#p95-threshold)
- [失败率](/product/insights/overview/metrics/#failure-rate)
- [Apdex](/product/insights/overview/metrics/#apdex)
- 唯一用户
- [用户痛苦](/product/insights/overview/metrics/#user-misery)

### 标记关键事务

如果您有经常返回的事务，您可以通过点击相应行中的星号将其标记为您任何团队的关键事务。每个团队最多可以将100个事务标记为关键事务。这些关键事务与团队的所有成员共享，确保最重要的信息被提升到列表顶部。您还可以在相应[摘要视图](/product/insights/overview/transaction-summary/#starring-key-transactions)中将事务标记为关键事务。

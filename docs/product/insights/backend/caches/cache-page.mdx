**缓存**页面为当前所选后端项目的所有端点提供缓存性能概览，包括**未命中率**和**每分钟请求数**（吞吐量）的摘要图表。您可以将这些作为起点，看看是否存在任何潜在的缓存性能问题，例如高于预期的未命中率百分比。

如果您发现异常或想进一步调查某个时间范围，请直接在图表中点击并拖动以选择特定时间范围，数据将仅针对该特定时间范围进行筛选。

事务表格显示了包含至少一个`cache.get`span的端点列表，以及：

- 其平均值大小（从缓存获取的字节数）
- 吞吐量
- 平均事务持续时间
- 未命中率百分比（查找未返回值的频率）
- 花费时间（应用程序在给定事务上花费的总时间）

默认情况下，此表格按花费时间最多排序。这意味着顶部的端点通常非常缓慢、请求非常频繁，或两者兼有。您可以更改排序顺序以按吞吐量、未命中率百分比、平均值大小或平均事务持续时间查看顶级事务。

要调查特定端点，请点击它以打开侧边栏显示一些样本事件和额外的持续时间指标。

## 样本列表

为了帮助您比较缓存命中（在缓存中找到值）与未命中（在缓存中找不到对应键的值）随时间的性能，Sentry会自动从**缓存**页面中显示所选时间段的两种样本分布。

这些样本事件显示为顶部平均持续时间图表中的三角形。这些样本事件也在下面的表格中列出，以及它们的span、键、事务持续时间和命中/未命中状态。

从样本列表中，您可以深入了解端点内缓存性能的特定好、平均或差示例。要查看包含span操作的详细跟踪，请点击**Span ID**。

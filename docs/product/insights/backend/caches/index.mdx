缓存可用于加速数据检索，从而提高应用程序性能。缓存临时存储数据以加快后续对该数据的访问。这允许您的应用程序从缓存内存中获取数据（如果可用），而不是必须再次下载该数据或从可能缓慢的数据层检索它。缓存可以加速像问答门户、游戏、媒体共享和社交网络等应用程序的读取密集型工作负载。

成功的缓存会导致高命中率，这意味着获取时数据存在。当获取的数据在缓存中不存在时，会发生缓存未命中。如果您启用了性能监控并且您的应用程序使用缓存，您可以使用Sentry查看缓存的性能表现。

Sentry的缓存监控提供有关缓存利用率和延迟的洞察，以便您可以改进与缓存交互的端点的性能。

从[缓存页面](https://sentry.io/orgredirect/organizations/:orgslug/insights/caches/)开始，您可以获得应用程序中对缓存进行至少一次查找的事务的概览。从那里，您可以通过点击事务并查看其样本列表来深入了解特定的缓存span操作。

## 检测

缓存监控目前支持当[cache_spans选项](/platforms/python/integrations/django/#options)设置为`True`时，[Django缓存框架](https://docs.djangoproject.com/en/5.0/topics/cache/)的[自动检测](/platform-redirect/?next=%2Ftracing%2Finstrumentation%2Fautomatic-instrumentation)。其他框架需要自定义检测。

### 自定义检测

如果可用，自定义检测按环境逐一记录，如下所列：

- [Python SDK](/platforms/python/tracing/instrumentation/custom-instrumentation/caches-module/)
- [JavaScript SDKs](/platforms/javascript/guides/node/tracing/instrumentation/custom-instrumentation/caches-module/)
- [PHP SDK](/platforms/php/tracing/instrumentation/caches-module/)
- [Java SDK](/platforms/java/tracing/instrumentation/custom-instrumentation/caches-module/)
- [Ruby SDK](/platforms/ruby/tracing/instrumentation/custom-instrumentation/caches-module/)
- [.NET SDK](/platforms/dotnet/tracing/instrumentation/custom-instrumentation/caches-module/)

要查看可以在span上设置的缓存数据，请参阅[缓存模块开发规范](https://develop.sentry.dev/sdk/performance/modules/caches/)。

消息队列使分布式架构中的异步服务到服务通信成为可能。队列非常适合使有时失败的工作更具弹性，因此是分布式应用程序的构建块。队列可以帮助处理的一些示例包括处理来自第三方API的webhook或处理定期任务（例如为用户每日计算指标）。

如果您启用了性能监控并且您的应用程序与消息队列系统交互，您可以配置Sentry来监控它们的性能和健康状况。

队列允许您监控队列消费者和生产者的性能和错误率，为您的分布式系统提供可观察性。

**队列**页面为您提供了高级概览，以便您可以看到消息被写入的位置。（您可能会看到主题名称或实际的队列名称，具体取决于消息系统。）如果您点击事务，您将看到**目标摘要**页面，该页面提供有关应用程序中特定端点的指标，这些端点要么写入，要么从目标读取。您还可以深入应用程序中表示创建消息的生产者和读取消息的消费者中的各个端点。您将看到代表应用程序处理消息的实际跟踪。

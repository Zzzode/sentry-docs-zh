**屏幕加载**页面显示了应用程序加载其屏幕所需时间的概览。它帮助您识别缓慢或回归的屏幕，并提供额外信息，以便您更好地理解影响初始显示时间（TTID）和完全显示时间（TTFD）缓慢的因素。

Sentry自动跟踪TTID，但[TTFD](/product/insights/mobile/mobile-vitals/#初始显示时间和完全显示时间)需要手动调用`reportFullyDisplayed()` API来报告屏幕已加载所有内容并完全显示。

### 最低SDK要求：

**对于Android：**

- `>=5.0`用于自动活动事务
- `>=6.10.0`用于[TTID](/platforms/android/tracing/instrumentation/automatic-instrumentation/#time-to-initial-display)
- `>=6.17.0`用于[TTFD](/platforms/android/tracing/instrumentation/automatic-instrumentation/#time-to-full-display)

**对于iOS：**

- `>=7.12.0`用于UIViewController事务
- `>=8.4.0`用于[TTID](/platforms/apple/guides/ios/tracing/instrumentation/automatic-instrumentation/#time-to-initial-display)+[TTFD](/platforms/apple/guides/ios/tracing/instrumentation/automatic-instrumentation/#time-to-full-display)

**对于Flutter：**

- `>=7.18.0`[路由检测的TTID+TTFD](/platforms/dart/guides/flutter/integrations/routing-instrumentation/#time-to-initial-display)

**对于React Native：**

- `>=5.20.0`[React导航的TTID+TTFD](/platforms/react-native/performance/instrumentation/time-to-display/)

默认情况下，**屏幕加载**页面显示您所选时间范围内屏幕计数最高的两个版本的指标。要选择不同的版本集进行比较，请使用页面顶部的"版本选择器"。

图表部分显示所选版本随时间的TTID和TTFD指标比较。

**您可能看不到任何数据的原因：**

- 您没有任何带有op `ui.load`的事务
- 您的SDK不满足最低SDK要求

在这里，您将看到按[设备类别](/concepts/search/searchable-properties/#device-classification)（高、中、低或未知）分组的TTID和TTFD值的细分，跨越您选择的版本。这将帮助您了解具有不同设备性能级别的用户如何受到影响。

要查看按特定设备类别细分的每个版本的慢TTID和TTFD事件和配置文件，请使用图表下方的"设备类别"筛选器。

在页面底部，您将看到一个在屏幕加载期间发生的span表格，按屏幕在该span中花费的时间最多排序。在这里，您将能够：

- 通过比较您选择的两个版本中每个span的平均持续时间来发现span回归。

- 通过查看平均span持续时间，找到您感兴趣的版本中最慢的span。

- 通过查看"总计数"，查看哪些span发生最频繁。（这可能帮助您识别屏幕加载期间可能发生的过度io调用或重复工作。）

**符合条件的span操作：**

- file.read
- file.write
- ui.load
- http.client
- db
- db.sql.query
- db.sql.transaction

### 影响TTID和TTFD的span：

如果span在TTID或TTFD之前结束，它们可能会影响各自的指标。它们在Sentry中被标记，可以在**屏幕摘要**页面上查看。但仅仅因为span被识别为影响TTID或TTFD，并不一定意味着它是罪魁祸首。此信息是近似值，应用于在聚合级别上查找可能有问题的span，以便您可以缩小需要调查的span范围。

## Span详细视图

![Span详细视图示例](./img/screen-loads-span-detail.png)

点击span描述会打开一个侧面板，让您查看span详细信息，包括您选择的每个版本中该span的平均持续时间和计数。

在下面的表格中，您将看到采样事件和配置文件（如果存在）的列表。通过比较两者，您将能够看到您正在查看的span的持续时间与所选两个版本中平均span的偏差程度。

Sentry会自动识别样本事件以帮助您调查性能问题。（它们显示为"平均持续时间"图表中的三角形。）为了给您提供准确的画面，在整个所选时间段内采样了比平均值快、比平均值慢和平均span持续时间的一系列样本。

您可以使用样本列表深入查看并比较给定屏幕内感兴趣的快速、平均和缓慢事件。

点击样本事件将带您进入[跟踪视图页面](/concepts/key-terms/tracing/trace-view/)的span瀑布中的查询span详细信息，或配置文件图标以查看事件的[火焰图](/product/explore/profiling/flame-charts-graphs/)。要获取不同的事件样本，请点击"尝试不同样本"按钮。

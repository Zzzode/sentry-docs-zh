## 应用程序启动

应用程序启动指标跟踪移动应用程序启动所需的时间。为此，Sentry测量[冷启动](/concepts/search/searchable-properties/events/#measurementsapp_start_cold)和[热启动](/concepts/search/searchable-properties/events/#measurementsapp_start_warm)。

冷启动和热启动的定义根据操作系统略有不同。在iOS上，Apple建议您的应用程序最多需要400毫秒来渲染第一帧。在Android上，[Google Play控制台](https://developer.android.com/topic/performance/vitals/launch-time#av)会在冷启动超过五秒或热启动超过两秒时发出警告。有关按操作系统的定义，请查看相应的SDK文档：

- [Android](/platforms/android/tracing/instrumentation/automatic-instrumentation/#app-start-instrumentation)
- [Apple](/platforms/apple/guides/ios/tracing/instrumentation/automatic-instrumentation/#app-start-tracking)
- [Flutter](/platforms/dart/guides/flutter/tracing/instrumentation/automatic-instrumentation/#app-start-instrumentation)
- [React Native](/platforms/react-native/tracing/instrumentation/automatic-instrumentation/#app-start-instrumentation)

<Alert>

Sentry的应用程序启动检测旨在尽可能全面和代表用户体验，并遵守平台供应商的指导方针。出于这个原因，Sentry报告的应用程序启动可能比其他工具中看到的要长。在[Apple应用程序启动文档](/platforms/apple/guides/ios/tracing/instrumentation/automatic-instrumentation/#app-start-tracing)和[Android应用程序启动文档](/platforms/android/tracing/instrumentation/automatic-instrumentation/#app-start-instrumentation)上阅读更多。

</Alert>

## 缓慢和冻结帧

为了跟踪用户界面的响应性，Sentry测量缓慢帧和冻结帧。两者都表示帧的渲染时间过长，导致UI显得滞后或冻结。

通常，手机或平板电脑以60或120帧每秒（fps）渲染。基于此，单个帧有16.67/8.33毫秒的时间预算来渲染。

- **缓慢帧**：使用60 fps时，缓慢帧是渲染时间超过16毫秒（Android）或16.67毫秒（iOS）的帧。
- **冻结帧**：冻结帧是渲染时间超过700毫秒的帧。

Sentry还支持更高的帧率，特别是随着120 fps显示器变得越来越流行。对于这些应用程序，Sentry检测帧率并相应调整缓慢帧计算。

在移动性能指标洞察页面上，您可以看到缓慢和冻结帧率，指示在事务生命周期内缓慢或冻结的帧的百分比。

## 帧延迟

帧延迟是用户感知的渲染帧的总延迟持续时间。例如，在60fps的刷新率下，如果两个帧渲染各需要20毫秒，则帧延迟为8毫秒（2 * (20ms - 16ms) = 8ms）。

您可以在以下平台上跟踪帧延迟：
- [Android](/platforms/android/tracing/instrumentation/perf-v2/#frames-delay)
- [Apple，作为缓慢和冻结帧的一部分](/platforms/apple/guides/ios/tracing/instrumentation/automatic-instrumentation/#slow-and-frozen-frames)
- [Flutter](/platforms/dart/guides/flutter/integrations/slow-and-frozen-frames-instrumentation/)

## 初始显示时间和完全显示时间

为了跟踪屏幕渲染其第一帧需要多长时间，以及渲染其第一帧和所有内容需要多长时间，Sentry分别测量初始显示时间和完全显示时间。

- **初始显示时间**：跟踪应用程序渲染其第一帧需要多长时间。这包括首次加载屏幕上的应用程序启动时间。它不包括第一帧绘制后延迟加载的任何内容。初始显示时间会自动测量并默认启用。

- **完全显示时间**：跟踪移动应用程序渲染其第一帧和所有内容需要多长时间。这包括第一帧后异步加载的内容，例如从网络加载内容后。完全显示时间是选择加入的，需要您手动调用API来报告屏幕已加载所有内容并完全显示。

[Web Vitals](https://web.dev/vitals/) 是Google定义的一组指标，用于衡量渲染时间、响应时间和布局偏移。每个数据点都提供有关应用程序整体 [性能](/product/insights/overview/) 的洞察。

浏览器内Sentry SDK收集web vitals信息（在支持的地方）并将该信息添加到前端 [事务](/product/insights/overview/transaction-summary/) 中。然后，这些web vitals在 [**性能 > Web Vitals** 页面](/product/insights/web-vitals/) 中汇总，以便为您提供每个页面为用户执行情况的快速概览。

![Web Vitals可视化](../../img/diagram-transaction-vitals.png)

## 核心Web Vitals

Google认为核心Web Vitals是衡量网页用户体验的最重要指标。根据2021年5月的 [Google博客文章](https://developers.google.com/search/blog/2020/11/timing-for-page-experience)，这些指标也会影响您的搜索排名。

### 最大内容绘制(LCP)

[最大内容绘制(LCP)](https://web.dev/lcp/) 衡量视口中覆盖最大像素区域的内容渲染所需的时间 - 换句话说，用户看到页面上主要内容需要多长时间。此内容可以采用文档对象模型(DOM)的任何形式，例如图像、SVG或文本块。

### 交互到下一次绘制(INP)

<Alert>
2024年3月12日，交互到下一次绘制(INP)取代了首次输入延迟(FID)作为核心Web Vital。在此之前，INP是Sentry未收集的实验指标。要开始收集INP测量值，请确保您的JavaScript SDK版本为 [7.104.0](https://github.com/getsentry/sentry-javascript/releases/tag/7.104.0) 或更高版本，并且选项 [`enableInp`](/platforms/javascript/tracing/instrumentation/automatic-instrumentation/#enableinp) 已开启（从版本 `8.0.0` 开始，`enableInp` 默认启用）。
</Alert>

[交互到下一次绘制(INP)](https://web.dev/articles/inp) 衡量从用户与页面交互（通过点击、轻触或键盘输入）到下一次绘制（屏幕上内容的渲染）发生的时间。INP旨在评估用户在采取行动后看到网站响应的速度，这对于提供流畅和响应迅速的用户体验至关重要。

### 累积布局偏移(CLS)

[累积布局偏移(CLS)](https://web.dev/cls/) 是在渲染过程中发生的每个意外元素偏移的单个布局偏移得分的总和。一个例子是尝试在尚未完成加载的页面上点击链接，由于图像渲染问题，该链接在您有机会点击之前向下移动。CLS web vital得分不基于持续时间。它代表破坏性和视觉不稳定偏移的程度。

![累积布局偏移示例](../../img/diagram-transaction-cls.png)

每个布局偏移得分使用影响和距离分数计算。影响分数是两个渲染帧之间元素影响的总可见区域。距离分数测量其相对于视口移动的距离。

```
布局偏移得分 = 影响分数 * 距离分数
```

让我们看一下上面有一个不稳定元素的示例 - 正文文本。影响分数大约是页面的50%，并将正文文本向下移动20%。布局偏移得分为0.1，即0.5*0.2的乘积。因此，CLS为0.1。

## 其他Web Vitals

这些Web Vitals通常对用户不太可见，但对核心Web Vitals的问题进行故障排除很有用。

### 首次绘制(FP)

首次绘制(FP)衡量第一个像素出现在视口中所需的时间，渲染任何与之前显示的内容不同的视觉变化。这可能采用文档对象模型(DOM)的任何形式，例如背景颜色、画布或图像。FP帮助开发人员了解页面上是否发生了任何意外渲染。

### 首次内容绘制(FCP)

[首次内容绘制(FCP)](https://web.dev/fcp/) 衡量第一个内容在视口中渲染的时间。这可能采用文档对象模型(DOM)的任何形式，例如图像、SVG或文本块。FCP经常与首次绘制(FP)重叠。FCP帮助开发人员了解用户看到页面上任何内容变化需要多长时间。

### 首字节时间(TTFB)

[首字节时间(TTFB)](https://web.dev/articles/ttfb) 衡量用户的浏览器接收页面内容的第一个字节所需的时间。TTFB帮助开发人员了解他们的缓慢是由初始响应还是由于渲染阻塞内容引起的。

### 首次输入延迟(FID)

<Alert>
截至2024年3月12日，首次输入延迟(FID)被INP取代作为核心Web Vital。虽然Sentry不再在我们的性能得分计算中包括FID，也不在 **Web Vitals** 性能页面上，但我们继续收集FID指标。您可以在 [Discover](/product/explore/discover-queries/) 中查询FID测量值。
</Alert>

[首次输入延迟(FID)](https://web.dev/fid/) 衡量用户尝试通过点击按钮、链接或任何其他自定义JavaScript控制器与视口交互时的响应时间。FID数据对于了解应用程序页面上的交互是否成功至关重要。


## 阈值

Google的"良好"、"需要改进"和"较差"阈值用于将数据点分类为相应的Web Vitals的绿色、黄色和红色。"需要改进"在Sentry中称为"一般"。

| Web Vital                                                       | 良好     | 一般     | 较差    |
| --------------------------------------------------------------- | -------- | -------- | ------- |
| [最大内容绘制](#largest-contentful-paint-lcp) (LCP) | &lt;= 2.5s  | &lt;= 4s    | > 4s    |
| [交互到下一次绘制](#interaction-to-next-paint-inp) (INP)| &lt;= 200ms | &lt;= 500ms | > 500ms |
| [累积布局偏移](#cumulative-layout-shift-cls) (CLS)   | &lt;= 0.1   | &lt;= 0.25  | > 0.25  |
| [首次绘制](#first-paint-fp) (FP)                             | &lt;= 1s    | &lt;= 3s    | > 3s    |
| [首次内容绘制](#first-contentful-paint-fcp) (FCP)     | &lt;= 1s    | &lt;= 3s    | > 3s    |
| [首字节时间](#time-to-first-byte-ttfb) (TTFB)           | &lt;= 100ms | &lt;= 200ms | > 600ms |
| [首次输入延迟](#first-input-delay-fid) (FID)               | &lt;= 100ms | &lt;= 300ms | > 300ms |

<Alert>

某些Web Vitals如FP、FCP、LCP和TTFB是相对于事务开始测量的。与 [Lighthouse](https://github.com/GoogleChrome/lighthouse) 等其他工具生成的值相比，值可能不同。

</Alert>

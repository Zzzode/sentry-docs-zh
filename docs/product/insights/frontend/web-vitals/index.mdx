Web vitals是一组衡量网页用户体验质量的指标。要了解有关这些指标的更多信息，请参阅 [Web Vitals概念](/product/insights/web-vitals/web-vitals-concepts/)。

对于您已检测的Web应用程序，Sentry通过 [支持的浏览器](web-vitals-concepts/#browser-support) 从真实用户流量中收集web vitals。这些web vitals用于为您的Web应用程序计算 [性能得分](#performance-score)。使用这些指标，我们展示具有最多 [机会](#opportunity) 来改善应用程序整体性能的页面。

## Web Vitals页面

**Web Vitals**页面位于**Insights**选项卡下，为您提供所选项目的页面加载和交互性能概览。您可以将此页面作为起点，调查影响Web应用程序的不良Web Vitals，并深入了解哪些页面对Web性能影响最大。

在左上角，[性能得分](#performance-score) 环显示应用程序的整体性能评级。环的每个组件代表单个Web Vital及其对性能得分的相对权重和影响。右侧的面积图向您展示了按Web Vitals划分的性能得分随时间的变化。

在此下方，您可以看到应用程序的p75 vital值和每个Web Vital的单独得分。这些指标可以帮助您优先考虑哪些Web Vitals最需要关注。单击Web Vital以打开该指标的更详细摘要，并查看哪些页面具有最多的 [机会](#opportunity) 进行改进。

在**Web Vitals**页面底部，可排序的表格显示应用程序页面的列表，以及它们与每个Web Vital关联的p75值。每个页面还有自己的单独性能得分。机会列显示如果将页面得分最大化到100，页面得分对应用程序整体性能得分的潜在改进。此表格上方的搜索栏允许您按路由名称筛选特定页面。

<Alert>

**Web Vitals**页面仅显示来自应用程序初始页面加载事务和交互的数据。不包括导航。

</Alert>

## 页面概览

从**Web Vitals**中的表格或Web Vital摘要面板，单击页面以打开其**页面概览**。在这里，您可以看到单个页面的Web Vitals摘要。在**页面概览**中，您可以进一步深入了解特定页面加载样本 [事件](/concepts/key-terms/tracing/trace-view/)、[交互](/product/insights/web-vitals/web-vitals-concepts/#interaction-to-next-paint-inp)、[重播](/product/explore/session-replay/) 或 [配置文件](/product/explore/profiling/)。

**页面概览**页面在页面右侧边栏中显示"页面加载"图表。页面右侧边栏中还显示一个迷你**聚合跨度瀑布图**，向您展示应用程序页面可能采用的常见跨度路径。单击"查看完整瀑布图"按钮或页面顶部的"聚合跨度"选项卡以查看完整的**聚合跨度瀑布图**。

## 性能得分

**性能得分**是从0到100的评级，总结了Web应用程序的感知性能。Web应用程序上的页面加载和交互基于其 **Web Vitals** 进行评分。每个 **Web Vital** 通过 [对数正态分布](https://www.desmos.com/calculator/o98tbeyt1t) 给出从0到100的评级，然后按页面和整个Web应用程序进行加权汇总。

**页面加载** 最多可以有4个可以在每个事件上找到的单个 **Web Vital** 组件，而交互贡献于 **INP**（交互到下一次绘制）。每个 **Web Vital** 的默认权重和得分评级阈值可以在下表中找到。

## 机会

**机会**得分是与每个页面关联的数字值，旨在让您了解哪些页面最有价值进行改进。机会得分是如果您将页面得分提高到100，应用程序整体性能得分的最大可能增加。如果您的应用程序中的页面已经得分为100，机会得分将为0，因为无法进一步优化此页面。

机会得分计算也基于页面看到的流量进行加权。高流量页面往往具有更高的机会得分。例如，得分为70且每天经历1000次页面加载的页面将比得分为50且每天仅经历10次页面加载的页面具有更高的机会得分。

### 先决条件和限制

**Web Vitals** 和 **性能得分** 由Sentry的JavaScript SDK通过 [BrowserTracing](/platforms/javascript/configuration/integrations/browsertracing/) 集成自动捕获。

您可以在 [安装文档](/platforms/javascript/) 中找到如何在Web应用程序上设置Sentry。如果您使用JavaScript框架，可能需要遵循特定框架的其他设置说明：

- [React SDK](/platforms/javascript/guides/react/)
- [Vue SDK](/platforms/javascript/guides/vue/)
- [Angular SDK](/platforms/javascript/guides/angular/)
- [Svelte SDK](/platforms/javascript/guides/svelte/)

虽然Sentry为许多前端框架提供自动工具化，但某些环境和场景可能无法完全支持。如果您无法在 **Web Vitals** 页面中找到您的页面，请检查您的页面加载是否满足以下条件：

- **性能得分** 仅支持 **Chrome**、**Firefox**、**Safari**、**Opera** 和 **Edge** 桌面和移动浏览器。
- 某些页面加载可能无法捕获Sentry支持的所有web vitals。这可能取决于客户端浏览器和/或您的网页的独特行为。**性能得分** 基于可用的web vitals计算。
  - 在 [浏览器支持](/product/insights/frontend/web-vitals/web-vitals-concepts/#browser-support) 表中找出完整的web vitals列表。

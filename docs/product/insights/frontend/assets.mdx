## 先决条件和限制

资源文件页面仅适用于启用了性能监控的前端JavaScript项目。目前仅支持JavaScript、CSS、图片以及由CSS发起的某些资源文件（如字体）。

为了获得最佳体验，我们建议为您的项目启用`BrowserTracing`集成来自动检测资源文件性能数据。以下JavaScript平台支持此功能：

- [原生JavaScript](/platforms/javascript/performance/)
- [Angular](/platforms/javascript/guides/angular/performance/)
- [Astro](/platforms/javascript/guides/astro/performance/)
- [Ember](/platforms/javascript/guides/ember/performance/)
- [Gatsby](/platforms/javascript/guides/gatsby/performance/)
- [Next.js](/platforms/javascript/guides/nextjs/performance/)
- [React](/platforms/javascript/guides/react/performance/)
- [Remix](/platforms/javascript/guides/remix/performance/)
- [Svelte](/platforms/javascript/guides/svelte/performance/)
- [SvelteKit](/platforms/javascript/guides/sveltekit/performance/)
- [Vue](/platforms/javascript/guides/vue/performance/)

### Span资格

Sentry通过查找与资源文件相关的span来提取指标。

JavaScript SDK使用浏览器的[资源计时API](https://developer.mozilla.org/zh-CN/docs/Web/API/Performance_API/Resource_timing)在`pageload`和`navigation`事务上自动生成`asset` span。

如果您正在使用<PlatformLink to="/tracing/instrumentation/automatic-instrumentation">自动检测</PlatformLink>，资源文件监控应该无需任何配置即可工作。

如果您手动检测了Sentry，您需要确保您的span符合我们的标准以获得最佳体验：

- span的`op`字段以`resource`为前缀（例如，`resource.script`或`resource.css`）。
- span的`description`包含加载资源的URL，这应该对应于[PerformanceResourceTiming名称字段](https://developer.mozilla.org/zh-CN/docs/Web/API/PerformanceEntry/name)。
- `http.response_transfer_size` span数据值设置为资源的[总传输大小](https://developer.mozilla.org/zh-CN/docs/Web/API/PerformanceResourceTiming/transferSize)。
- `http.response_content_length` span数据值设置为资源的[编码主体大小](https://developer.mozilla.org/zh-CN/docs/Web/API/PerformanceResourceTiming/encodedBodySize)。
- `http.decoded_response_content_length` span数据值设置为资源的[解码主体大小](https://developer.mozilla.org/zh-CN/docs/Web/API/PerformanceResourceTiming/decodedBodySize)。
- `resource.render_blocking_status` span数据值设置为资源的[渲染阻塞状态](https://developer.mozilla.org/zh-CN/docs/Web/API/PerformanceResourceTiming/renderBlockingStatus)。

以下是使用SDK手动创建资源span的示例代码片段。

```javascript
// https://developer.mozilla.org/zh-CN/docs/Web/API/PerformanceResourceTiming
const resources = performance.getEntriesByType("resource");
resources.forEach((entry) => {
  const startTime = msToSec(entry.startTime);
  const duration = msToSec(entry.duration);

  const span = Sentry.startInactiveSpan({
    startTimestamp: startTime,
    op: entry.initiatorType
      ? `resource.${entry.initiatorType}`
      : "resource.other",
    description: entry.name,
    data: {
      "http.response_transfer_size": entry.transferSize,
      "http.response_content_length": entry.encodedBodySize,
      "http.decoded_response_content_length": entry.decodedBodySize,
      "resource.render_blocking_status": entry.renderBlockingStatus,
    },
  });

  // 覆盖结束时间戳以匹配性能条目
  span.finish(startTime + duration);
});
```

## 资源文件页面

**资源文件**页面为您提供了所选项目中应用程序资源文件性能的快速概览。您可以将此页面作为起点来调查潜在的问题资源文件，并深入了解各种资源文件如何影响应用程序性能。

点击侧边栏"洞察"下的"资源文件"打开**资源文件**页面。或者，**性能**页面也会在"最耗时的资源文件"小组件中显示影响最大的资源文件。从那里，您可以点击"查看全部"打开**资源文件**页面。

在页面顶部，每分钟请求数（吞吐量）和平均持续时间的摘要图表为您提供了资源文件性能的高级洞察。如果您发现异常或想进一步调查某个时间范围，您可以直接在图表中点击并拖动来选择该时间范围以筛选该时间范围的数据。

下面的资源文件表格显示了分组资源文件的列表，以及它们的类型、数量（每分钟请求数）、平均持续时间、应用程序加载该资源文件所花费的总时间（花费时间）以及资源文件的[平均编码大小](https://developer.mozilla.org/zh-CN/docs/Web/API/PerformanceResourceTiming/encodedBodySize)。

默认情况下，资源文件表格按花费时间最多排序，这可以作为给定资源文件相对性能影响的代理。资源文件的花费时间是给定时间段内其所有持续时间的总和，换句话说，是其平均持续时间与每分钟请求数的乘积。这意味着顶部的资源文件通常加载非常缓慢、非常频繁，或两者兼有。

您可以点击列标题来更改表格的排序方式。按每分钟请求数排序以查看加载最频繁的资源文件，或按平均持续时间排序以查看加载最慢的资源文件。

您还可以按域、类型、所在页面以及是否已渲染阻塞来筛选资源文件。

要查看更多详细信息，请点击表格中的资源文件以打开其**资源文件摘要**页面。

### 渲染阻塞是什么意思？

渲染阻塞资源文件是指会阻止浏览器在屏幕上渲染任何内容的资源文件，直到该资源文件被浏览器完全下载和处理。

一个例子是HTML文档`<head>`中的`<script/>`。加载时，浏览器会希望在渲染内容之前完全加载此脚本，因为它假设可能需要该脚本中的某些内容来渲染。如果不是这种情况，您可以向脚本添加`defer`或`async`属性以减少或消除渲染阻塞。

Sentry使用[PerformanceResourceTiming API](https://developer.mozilla.org/zh-CN/docs/Web/API/PerformanceResourceTiming/renderBlockingStatus)中的`resource.render_blocking_status`属性捕获资源文件的渲染阻塞状态。

### 资源文件参数化

为了使Sentry能够将相似的资源文件分组在一起，Sentry对资源文件URL进行参数化，删除可能动态的元素。这有助于跟踪特定资源文件在不同版本中的性能，即使它们具有动态段（用于缓存清除或CDN）。

如果您想进一步改进分组，请考虑我们在参数化URL时使用的以下规则。这些规则可以帮助您了解如何命名URL以改进分组。

以下标记将在资源文件URL中被替换为\*

1. 版本字符串（`myfile.v3.0.js`被替换为`myfile.*.js`）
2. 超过5位的十六进制字符串（`myfile.7A9B3E.js`被替换为`myfile.*.js`）
3. UUID
4. 超过一位的整数ID（`1234.7A9B3E.js`被替换为`*.*.js`）
5. 超过25个字符的字符串
6. 整个路径，除了常见路径段如static、chunks、media等

<Alert>

资源文件参数化仍在进行中。随着这些改进的进行，您将暂时在资源文件模块中看到新旧分组的实例。

请通过GitHub Issues告诉我们任何反馈。

</Alert>

要查看分组中资源文件URL的示例，请将鼠标悬停在资源文件表格中的URL上。

## 资源文件摘要页面

要打开资源文件的**资源文件摘要**页面，请点击**资源文件**页面表格中的资源文件。

在页面顶部，显示所选时间范围的平均编码大小、平均解码大小、平均传输大小、平均持续时间和每分钟请求数。在此下方，您可以看到随时间变化的每分钟请求数、平均持续时间和平均资源文件大小的摘要图表。

在底部，您可以找到资源文件所在页面的列表，按该页面上资源文件的每分钟请求数排序。

如果您想调查特定页面，请点击它以打开侧边栏显示一些样本事件。

### 图片样本

查看图片资源文件时，资源文件摘要页面还会显示5个最大的图片样本。如果资源文件具有以下扩展名之一，Sentry会确定该span是图片：'jpg'、'jpeg'、'png'、'gif'、'svg'、'webp'、'apng'、'avif'。

如果大小信息不可用，样本将不会按任何特定顺序排序。如果图片可公开访问且启用了"启用图片"项目设置，则实际图片将在UI中渲染。

## 样本列表

点击页面以打开其样本列表。此侧面板显示资源文件在特定页面中的摘要指标（每分钟请求数、平均持续时间和花费时间）。

表格显示了样本事件列表、它们的span持续时间以及与平均值的持续时间差异。此表格按最长span持续时间排序。

Sentry会自动查找各种样本以帮助您调查性能问题。所选事件将覆盖整个所选时间范围，以及一系列持续时间（比平均值快、慢和接近）。您可以通过比较不同的样本事件来了解性能特征。

这些相同的事件也显示为上方平均持续时间图表中的三角形。

您可以通过点击底部的"尝试不同样本"按钮生成新的随机样本事件列表。

从样本列表中，您可以深入了解给定页面中给定资源文件的特定好、平均或差示例。点击事件ID以深入了解资源文件span在[跟踪视图](/concepts/key-terms/tracing/trace-view/#产品演练-跟踪详情页面)页面的span瀑布中的详细信息。

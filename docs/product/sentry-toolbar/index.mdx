开发人员在本地、暂存和生产环境中排除 Web 应用程序故障时花费大量时间。Sentry 工具栏从 [sentry.io](http://sentry.io) 提取数据，并在您拥有最多上下文理解时向您显示相关、可操作的问题：当您浏览自己的网站时。Sentry 工具栏是 Web 应用程序中的浮动小组件，通过三个不同的 **页面感知** 面板：问题、反馈和功能标志，为正在查看的特定页面提供有意义的 Sentry 洞察。

![网站上下文中的折叠工具栏](./img/toolbar-collapsed.png)

## 问题面板

问题面板向您显示您当前正在查看的页面的最高优先级前端 [问题](/product/issues/)。从 Sentry 工具栏中，您可以快速跳转到 Sentry 中的特定问题以获取更详细的信息（示例：Sentry 项目、问题标题、描述以及问题首次和最后出现的时间）并采取行动（示例：分配给团队成员或标记为已解决）。通过问题面板，您可以浏览对您的业务最重要的页面，并了解影响用户群的顶级问题。

![打开问题面板的工具栏](./img/issues-panel.png)

## 反馈面板

反馈面板向您显示您所在页面的最新用户反馈消息，以便您可以更轻松地将其置于上下文中。反馈通常描述并非严格代码抛出错误的次优体验，例如误导性 UX、损坏的链接和拼写错误。反馈面板通过从 [Sentry 的用户反馈小组件](/product/user-feedback/#user-feedback-widget) 提取反馈消息来工作。就像问题面板一样，点击特定的用户反馈将带您到 Sentry 中的完整详细用户反馈页面。

![打开反馈面板的工具栏](./img/feedback-panel.png)

## 功能标志面板

功能标志是一个强大的工具，允许您控制应用程序中功能的可见性，使您能够自信地发布、测试和实验。功能标志面板允许您快速查看和本地覆盖功能标志。您可以覆盖任何功能标志，使其在浏览器会话中为 `true` 或 `false`，以便您可以验证其行为并观察它可能对错误产生的影响。了解如何 [配置功能标志面板](/product/sentry-toolbar/setup/#implement-feature-flag-adapter) 以及 [Sentry 可以跟踪哪些数据](/product/explore/feature-flags/) 与您的功能标志系统相关。


![打开功能标志面板的工具栏](./img/feature-flags-panel.png)

## 登录流程

设置 Sentry 工具栏后，您将在页面中心看到一个"登录到 Sentry"按钮。要访问来自您的 Sentry 组织的任何信息，您必须点击此按钮并登录到 Sentry。
![工具栏登录按钮](./img/login-button.png)

点击按钮后，您将看到用于登录 Sentry 的弹出窗口。
![Sentry 登录弹出窗口](./img/login-popup.png)

如果这是您第一次 [设置 Sentry 工具栏](/product/sentry-toolbar/setup)，将有一个按钮链接到 Sentry 的工具栏设置页面以配置您的域。
![工具栏配置域按钮](./img/configure-domain.png)

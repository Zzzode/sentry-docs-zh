## 前置条件

为了让 Sentry 工具栏发挥最佳效果，请在您的应用中[启用跟踪](/platforms/javascript/tracing/)。启用跟踪后，Sentry 工具栏将能够将问题和反馈与浏览器位置中的当前 URL 关联起来。

## 1. 允许域名

由于 Sentry 工具栏将在您的应用中对用户可见，因此重要的是要考虑应在哪些环境中渲染它。请参阅 [FAQ："我应该在哪些环境中启用 Sentry 工具栏？"](/product/sentry-toolbar/faq/#in-what-environments-should-i-enable-the-dev-toolbar) 获取提示。

您需要编辑[项目设置](https://sentry.io/orgredirect/organizations/:orgslug/settings/projects/)页面，以允许工具栏连接到 Sentry。将任何生产、暂存或开发域名添加到列表中。仅将您信任并控制的域名添加到此列表中。

![Sentry 工具栏设置页面](./img/sentry-project-settings-toolbar.png)

您可能还需要将您的 Sentry 域名添加到 CSP 响应标头中的 [`frame-src`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Security-Policy/frame-src) 指令中。有关更多信息，请参阅 [FAQ："为什么工具栏卡在'正在连接到...'？"](/product/sentry-toolbar/faq/#why-is-the-toolbar-stuck-saying-connecting-to-)。

## 2. 安装

如果您正在开发基于 React 的应用程序，现在是时候将工具栏添加到您的 `package.json` 文件中了。

或者，您可以跳到下一步查找 CDN 配置说明。

```bash {tabTitle: npm}
npm install --save @sentry/toolbar
```
```bash {tabTitle: yarn}
yarn add @sentry/toolbar
```

## 3. 配置

最后，无论您使用的是 React 应用程序还是从 CDN 加载工具栏，您都需要使用 JavaScript 初始化工具栏。这将提示任何访问者登录到您的 Sentry 组织。

```javascript {tabTitle: React}
import {useSentryToolbar} from '@sentry/toolbar';

useSentryToolbar({
  // 记住要有条件地启用工具栏。
  // 这将减少没有 Sentry 登录凭据的用户的网络流量。
  enabled,

  initProps: {
    organizationSlug: 'acme',
    projectIdOrSlug: 'website',
  },
})
```
```html {tabTitle: CDN}
<html>
<head>...</head>
<body>
    ...
    <!--
    将此放在页面底部，这样它就不会
    阻塞其他关键的 JavaScript。

    记住要有条件地包含工具栏
    脚本。这将减少没有 Sentry 登录凭据的用户的网络流量。
    -->
    <script src="https://browser.sentry-cdn.com/sentry-toolbar/latest/toolbar.min.js"></script>
    <script>
    window.SentryToolbar.init({
      organizationSlug: 'acme',
      projectIdOrSlug: 'website',
    });
    </script>
</body>
</html>
```

请参阅 [FAQ："如何有条件地初始化工具栏？"](/product/sentry-toolbar/faq/#how-can-i-conditionally-initialize-the-toolbar) 获取有关为不同环境实现条件的帮助。

### 初始化配置选项

最低限度，您必须设置 `organizationSlug` 和 `projectIdOrSlug`。

完整的选项列表如下：

| 选项 | 类型 | 描述 | 默认值 |
| --- | --- | --- | --- |
| `organizationSlug` | `string` | 用户应登录到的组织。例如 'acme' | *必需值* |
| `projectIdOrSlug` | `string \| number` | 此网站/web 应用关联的项目。 | *必需值* |
| `environment (可选)` | `string \| string[] \| undefined` | 此部署的环境。用于缩小工具栏 UI 中的搜索结果范围。如果您想查看所有环境的结果，请设置为 `undefined` 或 `""` 或 `[]`。 | `undefined` |
| `placement (可选)` | `'right-edge' \| 'bottom-right-corner'` | 在屏幕上渲染工具栏的位置。 | `'right-edge'` |
| `theme (可选)` | `'system' \| 'dark' \| 'light'` | 是否使用深色或浅色模式。 | `'system'` |
| `featureFlags (可选)` | `FeatureFlagAdapter \| undefined` | 请参阅下面的[功能标志面板](/product/sentry-toolbar/setup/#feature-flag-Panel) | `undefined` |
| `sentryOrigin (可选)` | `string \| undefined` | 可以找到 Sentry 的来源。用于加载与 Sentry 的连接，并生成指向网站的链接。例如：'https://acme.sentry.io' | 'https://sentry.io' |
| `domId (可选)` | `string \| undefined` | 给予创建的包含工具栏 html 的 div 的 id。 | 'sentry-toolbar' |
| `debug (可选)` | `string \| undefined` | 要启用的调试目标的逗号分隔字符串。示例：'logging,state'。如果列表包含 'all' 或 'true'，则将启用所有目标。有效目标：'logging' 'login-success' 'settings' 'state' | `undefined` |
| `mountPoint (可选)` | `HTMLElement \| () => HTMLElement \| undefined` | 在 DOM 中挂载工具栏的位置。 | `document.body` |
| `transactionToSearchTerm (可选)` | `(transactionName: string) => string` | 将 transactionName 转换为搜索词的钩子 | 请参阅 [transactionToSearchTerm.ts](https://github.com/getsentry/sentry-toolbar/blob/main/src/lib/utils/transactionToSearchTerm.ts) |

### 额外的 React Hook 选项

React hook 支持一些其他顶级选项。默认值如下：

```javascript {tabTitle:React}
useSentryToolbar({
  initProps: {
    organizationSlug: 'acme',
    projectIdOrSlug: 'website',
  },

  // 可选：
  enabled: true,

  // 可选：要么 version 要么 cdn
  // 如果两者都设置，则 cdn 将覆盖 version
  version: 'latest',
  cdn: 'https://browser.sentry-cdn.com/sentry-toolbar/latest/toolbar.min.js',
})
```

| 选项 | 类型 | 描述 | 默认值 |
| --- | --- | --- | --- |
| `enabled` | `boolean (可选)` | 有条件地初始化工具栏。将其设置为 false 以避免在浏览器上请求工具栏代码，或安全地卸载现有的工具栏实例 | `true` |
| `version` | `string (可选)` | 从 CDN 请求特定版本的工具栏。建议使用 latest 以进行自动更新。请参阅 https://github.com/getsentry/sentry-toolbar/releases 获取可用发布标签列表。 | 'latest' |
| `cdn` | `string (可选)` | 覆盖上面的 version 字段。设置 CDN 在您想要自托管特定版本的工具栏时很有用。 | 'https://browser.sentry-cdn.com/sentry-toolbar/latest/toolbar.min.js' |


### 卸载工具栏

如果您已调用 `SentryToolbar.init({...})` 来渲染工具栏，但现在想要手动从页面中删除或卸载它，您可以调用从 `init()` 返回的清理函数。这将卸载所有注入的 HTML 和 CSS。登录凭据不会被删除，因此您可以重新插入工具栏并仍然通过身份验证。

如果您使用的是 React hook，则当 React 组件被卸载时，工具栏将被卸载。或者如果您设置 `enabled: false`。

```javascript
const unmountToolbar = window.SentryToolbar.init({ ... });

// 稍后...
unmountToolbar();
```
```javascript {tabTitle:React}
useSentryToolbar({
  enabled: false, // 工具栏将不会被挂载
  initProps: {
    ...
  },
})
```

## 功能标志面板

<Alert>
  如果您在产品内部使用功能标志，那么还要在您的 SDK 中设置[功能标志评估和更改跟踪](/platforms/javascript/feature-flags/)。
</Alert>

为了将您的功能标志平台与 Sentry 工具栏集成，您将需要一个可以从您的提供商读取标志数据的适配器。它还将存储和检索要应用于您的本地浏览器会话的覆盖列表。

有一个内置的 `OpenFeatureAdapter` 与 [open-feature/js-sdk-contrib](https://github.com/open-feature/js-sdk-contrib) 兼容。要使用它，请调用 `SentryToolbar.OpenFeatureAdapter` 实现。

```html {tabTitle: CDN}
<script>
// 定义您的提供商
const provider = new FlagdWebProvider({...});

// 将提供商设置到 OpenFeature SDK 中
OpenFeature.setProvider(provider);

window.SentryToolbar.init({
  ...
  // 将提供商设置到 OpenFeatureAdapter 中
  featureFlags: window.SentryToolbar.OpenFeatureAdapter({provider})
});
</script>
```
```typescript {tabTitle: React}
import {OpenFeature} from '@openfeature/web-sdk';
import {FlagdWebProvider} from '@openfeature/flagd-web-provider';

// 定义您的提供商
const provider = new FlagdWebProvider({...});

// 将提供商设置到 OpenFeature SDK 中
OpenFeature.setProvider(provider);

window.SentryToolbar.init({
  ...
  // 将提供商设置到 OpenFeatureAdapter 中
  featureFlags: window.SentryToolbar.OpenFeatureAdapter({provider})
});
```

您还可以通过实现 [`FeatureFlagAdapter` 接口](https://github.com/getsentry/sentry-toolbar/blob/main/src/lib/types/featureFlags.ts) 来创建自己的适配器。

适配器接口如下：
```typescript
type FlagValue = boolean | string | number | undefined;
type FlagMap = Record<string, FlagValue>;
interface FeatureFlagAdapter {
  /**
   * 所有已知标志名称及其评估值。
   */
  getFlagMap: () => Promise<FlagMap>;

  /**
   * 任何覆盖或手动设置的标志和值。
   */
  getOverrides: () => Promise<FlagMap>;

  /**
   * 手动将标志设置为特定值，覆盖评估值。
   */
  setOverride: (name: string, override: FlagValue) => void;

  /**
   * 清除此浏览器中所有覆盖的回调。
   */
  clearOverrides: () => void;

  /**
   * 深度链接到您的外部功能标志提供商，并了解更多关于
   * 此特定标志的信息。
   */
    urlTemplate?: undefined | ((name: string) => string | URL | undefined);
}
```

[MockFeatureFlagAdapter.tsx](https://github.com/getsentry/sentry-toolbar/blob/main/src/env/demo/MockFeatureFlagAdapter.tsx) 是一个可用作参考的示例适配器。

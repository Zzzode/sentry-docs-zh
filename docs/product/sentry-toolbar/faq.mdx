<Expandable permalink title="我应该在哪些环境中启用工具栏？">

由于 Sentry 工具栏将对用户可见，因此重要的是要考虑应在哪些环境中渲染它。

如果您的 Web 应用程序需要身份验证才能访问：
- 在开发和暂存环境中，始终初始化 Sentry 工具栏。
- 在生产环境中，仅当员工登录时才初始化 Sentry 工具栏。

如果您的 Web 应用程序不需要身份验证：
- 在开发和暂存环境中，始终初始化工具栏。
- 在生产环境中，不要初始化工具栏。

初始化 Sentry 工具栏允许所有开发人员和测试人员快速从他们正在查看的页面返回到 Sentry 进行进一步调试。
在生产环境中，它可以使开发人员更容易重现问题，但不应为网站的所有用户初始化 - 仅在员工/工程师等访问时初始化。

一旦您决定在哪里以及何时希望工具栏出现，您就需要将这些条件写入您的代码库。具体实现需要您根据应用程序的工作方式和团队的设置来编写。

</Expandable>

<Expandable permalink title="如何有条件地初始化工具栏？">

实现初始化工具栏的特定条件会因应用程序和使用的框架或模板库而异。

例如，如果在开发和暂存中显示工具栏的条件，如果用 JavaScript 编写，可能如下所示：

```html {tabTitle:CDN}
<script>
const env = process.env.ENVIRONMENT || 'development';
const isDev = env === 'development' || env === 'staging';

if (isDev) {
  window.SentryToolbar.init({ ... });
}
</script>
```
```javascript {tabTitle:React}
const env = process.env.ENVIRONMENT || 'development';
const isDev = env === 'development' || env === 'staging';

useSentryToolbar({
  enabled: isDev,
  initProps: {
    ...
  },
})
```
</Expandable>

<Expandable permalink title="为什么工具栏卡在"正在连接到..."？">

如果您在工具栏中看到消息"正在连接到..."，这意味着工具栏无法连接到 Sentry 服务器。这可能由于几个原因发生：

1. 检查您的浏览器控制台是否有类似 `Refused to frame 'https://my-org.sentry.io/toolbar/sentry/javascript/iframe/' because it violates the following Content Security Policy directive: "frame-src 'self'".` 的消息。这意味着您的网站不允许工具栏的 iframe 加载。要解决此问题，您可以将您的 sentry 域添加到 CSP 的 [`frame-src`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Security-Policy/frame-src) 指令中。结果将是一个响应头，显示 `Content-Security-Policy: frame-src my-org.sentry.io;`

2. Sentry 服务器已关闭或无法访问。如果发生这种情况，请检查以下内容：
    - 检查 [Sentry 状态页面](https://status.sentry.io/) 查看是否有任何问题。
    - 如果 Sentry 服务器已关闭，您可以稍后再试。
    - 如果 Sentry 服务器无法访问，您可以稍后再试。
</Expandable>

<Expandable permalink title="有计划将工具栏包含在 JavaScript SDK 中吗？">

[Sentry 工具栏](https://github.com/getsentry/sentry-toolbar)和 [JavaScript SDK](https://github.com/getsentry/sentry-javascript) 是我们有意保持分离的不同功能。

两者之间的一些差异包括：
  - 工具栏是一个 UI 产品，专注于更容易查找和采取行动处理现有数据，而 SDK 则作为收集和发送数据到服务器的基础设施。
  - 工具栏具有不同的[依赖项](https://github.com/getsentry/sentry-toolbar/blob/main/package.json)集，并使用 JavaScript SDK 不使用的不同浏览器 API。例如：工具栏将与 cookie 和本地存储等事物交互。通过保持这些代码片段分离，更容易审计 [GitHub 上的 SDK 代码](https://github.com/getsentry/sentry-javascript) 以验证它不会在最终用户的浏览器中持久化信息。
  - 设置和部署说明非常不同。SDK 最好部署在暂存和生产环境中，并且可以使用环境变量轻松配置。Sentry 工具栏需要特殊考虑才能部署到生产中，通常通过创建条件，使其仅包含在您自己 Sentry 组织的成员中。

</Expandable>

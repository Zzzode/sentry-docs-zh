# 旧版端点迁移

在 Crons 产品的早期测试版中，签到端点存在于
`sentry.io/api/0/` API 命名空间下。此命名空间不是为
摄取级别流量设计的，因此我们已将签到 API 移至使用
[Relay](/product/relay/) 来摄取签到事件。

如果您正在为 Crons 使用旧版端点，请迁移到新端点以避免问题。

<Alert>

旧版端点**已弃用**，将来会被移除，
但尚未设定移除日期。我们会在设定日期时进行沟通。

</Alert>

## 我是否在使用旧版端点？

SDK 不使用我们的旧版端点。

<Alert level="warning" title="Sentry CLI">

**v2.19.0 之前**版本的 `sentry-cli` 正在使用旧版 API 端点。[请升级](/cli/installation/#updating-and-uninstalling)到最新版本。

如果您的 CLI 配置了身份验证令牌，它将使用旧版 API 端点。请将您的 CLI 配置更新为使用项目的 DSN。有关更多信息，请参阅[配置指南](/cli/crons/#configuration)。

</Alert>

使用 HTTP 端点时，您可以这样区分它们：

```
# ❌ 旧版端点
https://sentry.io/api/0/organizations/___ORG_SLUG___/monitors/<monitor_slug/

# ❌ 没有组织上下文的旧版端点
https://sentry.io/api/0/monitors/<monitor_slug/

# ❌ 旧版签到详情端点
https://sentry.io/api/0/monitors/<monitor_slug/checkins/<check_in_id>/

# ✅ Relay 签到摄取端点
https://___ORG_INGEST_DOMAIN___/api/___PROJECT_ID___/cron/<monitor_slug>/___PUBLIC_KEY___/
```

### 有什么变化？

需要注意一些向后不兼容的更改。

- 现在只有一个签到端点。"签到详情"
  端点已被移除。

- API 端点现在支持带有 JSON 正文的 `POST`，以及现在
  支持 `GET`。您可以使用查询参数指定 `status`、
  `environment` 等。

  API 端点将始终响应 `202` 且没有正文（除非
  有错误）。

- 由于没有"详情"端点，签到的 `PUT` 方法
  也已被移除。您将始终使用 `POST` 或 `GET` 进行签到。

- 对于[重叠
  作业](/product/crons/getting-started/http/#overlapping-jobs-optional)，仍然
  可以通过查询参数或 JSON 正文指定 `check_in_id`。
  但是，API 不再响应自动生成的签到 ID。
  如果您需要稳定的签到 ID，您必须自己在
  客户端生成。

- `monitor_config` 结构略有变化。`schedule` 现在是
  根据计划类型而变化形状的对象。

  ```json
  {"type": "crontab", "value": "0 * * * *"}
  {"type": "interval", "value": "2", "unit": "hour"}
  ```

## 迁移示例

请参阅下面的代码示例，帮助您从旧版端点迁移到当前端点。

### 简单 HTTP 签到

```sh
# ❌ 旧版简单 `in_progress` -> `ok` 签到
curl -X POST \
    'https://sentry.io/api/0/organizations/___ORG_SLUG___/monitors/<monitor_slug>/checkins/' \
    --header 'Authorization: DSN ___PUBLIC_DSN___' \
    --header 'Content-Type: application/json' \
    --data-raw '{"status": "in_progress"}'

curl -X POST \
    'https://sentry.io/api/0/organizations/___ORG_SLUG___/monitors/<monitor_slug>/checkins/' \
    --header 'Authorization: DSN ___PUBLIC_DSN___' \
    --header 'Content-Type: application/json' \
    --data-raw '{"status": "ok"}'


# ✅ 新样式 `in_progress` -> `ok` 签到
SENTRY_INGEST="https://___ORG_INGEST_DOMAIN___"
SENTRY_CRONS="${SENTRY_INGEST}/api/___PROJECT_ID___/cron/<monitor_slug>/___PUBLIC_KEY___/"

curl "${SENTRY_CRONS}?status=in_progress"

curl "${SENTRY_CRONS}?status=ok"
```

### 监控更新签到

```sh
# ❌ 旧版使用签到更新监控
curl -X POST \
    'https://sentry.io/api/0/organizations/___ORG_SLUG___/monitors/<monitor_slug>/checkins/' \
    --header 'Authorization: DSN ___PUBLIC_DSN___' \
    --header 'Content-Type: application/json' \
    --data-raw '{"monitor_config": {"schedule": "0 * * * *", "schedule_type": "crontab"}, "status": "in_progress"}'

# ✅ 新样式使用签到更新监控
SENTRY_INGEST="https://___ORG_INGEST_DOMAIN___"
SENTRY_CRONS="${SENTRY_INGEST}/api/___PROJECT_ID___/cron/<monitor_slug>/___PUBLIC_KEY___/"

curl -X POST "${SENTRY_CRONS}?status=in_progress" \
    --header 'Content-Type: application/json' \
    --data-raw '{"monitor_config": {"schedule": {"type": "crontab", "value": "0 * * * *"}}, "status": "in_progress"}'
    # 👉 注意 schedule 现在是对象 ---^
```

### 重叠作业签到（指定 `check_in_id`）

```sh
# ❌ 旧版带 ID 的签到
curl -X POST \
    'https://sentry.io/api/0/organizations/___ORG_SLUG___/monitors/<monitor_slug>/checkins/' \
    --header 'Authorization: DSN ___PUBLIC_DSN___' \
    --header 'Content-Type: application/json' \
    --data-raw '{"status": "in_progress"}'
# 响应 { "id": "2bc1a871-a1b7-4577-82fc-fa6d2468aabc" }

# ✅ 新样式使用签到更新监控
CHECK_IN_ID="$(uuidgen)"
# 👉 注意您现在需要生成签到 UUID

SENTRY_INGEST="https://___ORG_INGEST_DOMAIN___"
SENTRY_CRONS="${SENTRY_INGEST}/api/___PROJECT_ID___/cron/<monitor_slug>/___PUBLIC_KEY___/"

curl "${SENTRY_CRONS}?check_in_id=${CHECK_IN_ID}&status=in_progress"
# 👉 注意不返回签到 ID
```

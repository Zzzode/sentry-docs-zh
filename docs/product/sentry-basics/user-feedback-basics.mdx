Sentry 的[用户反馈小组件](/product/user-feedback/)是一个强大的工具，用于从真实用户那里捕获定性洞察，在推出新功能或测试版功能时尤其有价值。或者，它也可以用于在内部发布期间从您的团队成员那里收集反馈。

将用户反馈视为产品开发中的核心信号，特别是对于测试版和早期采用者计划。我们建议您将反馈审查集成到规划、站会或回顾中；然后您可以使用反馈趋势来指导产品方向和路线图决策。

这个 4 步指南将引导您在应用程序上实施和检测用户反馈。

## 1. 在上下文中嵌入小组件

将小组件带到用户面前；不要让他们去找它。

将反馈小组件直接嵌入到实验性或高影响力的功能区域中。例如，在您的结账屏幕上：

![](feedback-img/img/feedback-on-checkout-page.png)

您可以采用的一种方法是在功能内放置情境化的点赞/点踩按钮，这些按钮将触发反馈表单打开。您可以使用[自定义标签](/platforms/javascript/enriching-events/tags/)跟踪用户是否为每个反馈点击了👍或👎。该标签将出现在用户反馈详情页面上，并可选择在提交反馈后出现在反馈警报本身上。

![](feedback-img/img/feedback-thumbs-up-down.png)


<details>
<summary>**查看示例 JavaScript 代码片段**</summary>

```javascript
"use client";

import * as Sentry from "@sentry/react";
import { Fragment } from "react";

export default function ThumbsUpDownButtons({source}: {source: string) {
  const feedback = Sentry.getFeedback();
  
  return (
    <Fragment>
      <strong>这有帮助吗？</strong>
      <button
        title="我喜欢这个"
        onClick={async () =>{
          const form = await feedback?.createForm({
            messagePlaceholder: '您最喜欢什么？',
            tags: {
              component: 'ThumbsUpDownButtons',
              'feedback.source': source,
              'feedback.type': 'positive',
            }
          });
          form?.appendToDom();
          form?.open();
        }}
      >
        是 👍
      </button>

      <button
        title="我不喜欢这个"
        onClick={async () =>{
          const form = await feedback?.createForm({
            messagePlaceholder: '我们如何改进？',
            tags: {
              component: 'ThumbsUpDownButtons',
              'feedback.source': source,
              'feedback.type': 'negative',
            }
          });
          form?.appendToDom();
          form?.open();
        }}
      >
        否 👎
      </button>
    </Fragment>
  );
}
```

</details>



## 2. 自定义占位符文本

个性化提示会导致更好的反馈。根据功能的上下文定制占位符文本。


以下是指导用户的占位符文本示例：

> 您对[您的功能名称]的体验如何？

> 我们如何改善您的体验？

> 我们仍在构建[您的功能名称] - 我们如何使其更好？



最好为清晰度和相关性定制您的提示；这将引导用户提供有用的输入。使用友好的语言鼓励诚实、随意的反馈。


## 3. 向反馈提交添加自定义标签以实现更好的路由

如果您在应用程序中有多个嵌入到不同功能的反馈按钮，标签对于实现更快的警报路由和为正确团队进行分类至关重要。

您可以根据小组件的放置自动应用标签。例如，使用像 `feedback.source:checkout` 这样的自定义标签。这允许在 Slack 中进行路由并在仪表板中进行过滤。我们建议您使用单个键定义和标准化标签，例如 `feedback.source`。


## 4. 设置到相关频道的警报

反馈只有在被合适的人看到时才有用。

您可以将用户反馈直接发送到相应团队的频道——无论您使用 Slack、Microsoft Teams 还是 Discord（参见[集成完整列表](/product/alerts/create-alerts/routing-alerts/#integrations)）。例如，您可以标记反馈 `feedback.source:checkout` 并始终将其发送到您在 Slack 中的 `#proj-billing` 频道。通过此工作流程，工程师几乎实时地看到反馈并承担责任。

按照以下步骤设置到相关频道的反馈警报：

1. 在 Sentry 中创建[新警报规则](https://sentry.io/alerts/new/issue/)。
2. 滚动到"设置条件"部分，并将"IF"过滤器设置为`问题的类别等于..."反馈"`和`事件的标签匹配...[您的自定义标签]`。
3. 在"THEN"过滤器中，选择`发送[您的集成]通知`，然后添加您希望将反馈发送到的频道的详细信息。您还可以添加您希望在反馈警报上看到的标签。例如，`user.email`。
4. 添加警报名称和负责人。

**→ [在此处注册 Sentry](https://sentry.io/signup/)**

## 如何充分利用 Sentry

我们建议您做三件事来充分利用 Sentry：

- **连接您的代码库**，确保您还设置了 [源代码管理集成](/organization/integrations/source-code-mgmt/) 和 [源映射](/product/sentry-basics/integrate-frontend/upload-source-maps/)（如果适用）。这将允许 Sentry 通过可读的堆栈跟踪和可疑提交将错误和性能问题直接链接到有问题的代码行。
- **设置 [警报](/product/alerts/)** 以确保您知道您关心的事情出现问题时。您可以选择通过我们的聊天集成之一（如 [Slack](/organization/integrations/notification-incidents/slack/) 或 [Discord](/organization/integrations/notification-incidents/discord/)）接收警报。查看我们的 [警报最佳实践](/product/alerts/best-practices/) 以了解如何仅获得相关通知。
- **自定义 [发现](/product/explore/discover-queries/)、[仪表板](/product/dashboards/) 和 [查询](/product/explore/discover-queries/query-builder/)** 以更好地控制当您的应用程序出现问题时您获得的洞察类型。最后，创建针对您的用例量身定制的 [自定义标签](/concepts/search/searchable-properties/#custom-tags) 以实现更精细的搜索和查询。

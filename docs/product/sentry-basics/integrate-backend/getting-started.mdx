在本教程中，您将把后端应用程序源代码导入到本地开发环境中，添加 Sentry SDK 并进行初始化。

<Alert>

如果您使用的是自己的源代码，可以跳过本教程，并选择：

- 按照[入门指南](/platforms/python/)文档中的说明操作。
- 直接转到[下一步](/guides/integrate-backend/configuration-options/)。

</Alert>

## 先决条件

1. 演示应用程序源代码需要 Python 开发环境来构建、安装和运行应用程序。确保您已具备以下条件：

   - 源代码编辑器（如 [VS-Code](https://code.visualstudio.com)）
   - [Python](https://www.python.org/)
   - [Sentry-CLI](/cli/)

2. 要开始监控应用程序中的错误，您需要在 Sentry 账户中创建一个新项目。

## 步骤 1：创建项目

按照以下步骤为示例 Django 应用程序创建一个新的 Sentry 项目：

1. 登录您的 [Sentry 组织](https://sentry.io)。

1. 从左侧导航菜单中选择**项目**以显示所有项目的列表。

1. 点击"创建项目"并根据您的应用程序进行适当配置：

    - **选择平台**：根据您要监控的代码选择项目的语言或框架。在本例中，选择 **Django**。

    - **设置警报频率**：出于本教程的目的，选择**稍后我将创建自己的警报**。

      > 如果您担心警报疲劳，请了解如何在[警报最佳实践](/product/alerts/best-practices/)中设置警报。

    - **命名项目并分配团队**：在**项目名称**字段中命名您的项目，并通过在**团队**下拉菜单中选择一个团队来分配团队（或点击**+**创建新团队）。

    - 点击**创建项目**。这将带您进入快速配置 Django SDK 指南，该指南概述了如何配置 SDK。本教程将更详细地介绍 SDK 设置。

1. 复制 DSN 密钥并妥善保存。每个项目都有一个唯一的 DSN（数据源名称）。DSN 告诉 SDK 将事件发送到哪里，以便事件与正确的项目关联。稍后您需要将 DSN 密钥粘贴到源代码中，以便本教程中生成的错误转到您的新项目。

   > 您也可以随时在**[项目] > 设置 > SDK 设置 > 客户端密钥 (DSN)**中找到项目的 DSN。

## 步骤 2：获取代码

1. 在 GitHub 上打开示例[代码仓库](https://github.com/sentry-tutorials/backend-monitoring)。

2. 点击"Fork"并选择您希望此仓库被 fork 到的目标 GitHub 账户。

3. fork 完成后，点击"Clone or download"并复制仓库 URL。您可以使用 HTTPS、SSH 或通过 GitHub CLI 执行此操作。

   ![Clone Repository](./img/fork_n_clone.png)

4. 将 fork 的仓库克隆到本地环境：

   ```bash
   > git clone <repository HTTPS or SSH url>
   ```

5. 现在示例代码已在本地可用，在您首选的代码编辑器中打开 `backend-monitoring` 项目。

## 步骤 3：为您的仓库启用提交跟踪

Sentry 可以通过建议一个[可疑提交](/product/sentry-basics/integrate-frontend/configure-scms/)来帮助您更快地解决错误，该提交可能已将错误引入到您的代码库中。这通过配置_提交跟踪_来实现。集成您的源代码管理解决方案并添加您的代码仓库是启用提交跟踪所必需的。在我们的[发布文档](/product/releases/?platform=node/suspect-commits/)中了解更多信息。

1. 打开您的 Sentry 账户并导航到**设置 > 集成**以启用 GitHub 集成并添加您的 `backend-monitoring` 仓库。有关更详细的信息，请按照[我们的 GitHub 文档](/organization/integrations/source-code-mgmt/github/)中描述的步骤操作。

## 步骤 4：安装 SDK

Sentry 通过使用应用程序运行时中的特定于平台的 SDK 来捕获数据。要使用 SDK，请在源代码中导入、初始化和配置它。

1. 通过在 `requirements.txt` 文件中定义依赖项来安装 `sentry-sdk`。SDK 文档和发布信息可在 [Sentry SDK](https://github.com/getsentry/sentry-python) GitHub 仓库中找到。

2. 打开 `settings.py` 文件（位于 `./backend-monitoring/myproject/settings.py`）。这是我们导入、初始化和配置应用程序中 Sentry SDK 的地方。


   ```python {filename: myproject/settings.py}
   import sentry_sdk

   sentry_sdk.init(
       dsn="YOUR_DSN",  # 替换为您的 DSN
       ...
   )
   ```
   > 在 `settings.py` 中将 `YOUR_DSN` 替换为您在[项目创建教程](/guides/integrate-frontend/create-new-project/)中复制的 `DSN` 密钥值。


## 步骤 5：安装依赖项并运行演示应用程序

要在您的 localhost 上构建和运行演示应用程序：

1. 打开 shell 终端并将目录更改为 `backend-monitoring` 项目根文件夹。

2. 如果您尚未安装 Python，请运行以下命令：

   ```bash
   brew install python
   ```

3. 在项目根目录中创建 Python 虚拟环境。

   ```bash
   python -m venv .venv
   ```

   > 您可以将虚拟环境命名为任何您喜欢的名称。我们将其命名为 ".venv"。

4. 要激活虚拟环境，请运行：

   ```bash
   source .venv/bin/activate
   ```

5. 安装 Sentry 的命令行工具以使用发布跟踪和 GitHub 集成来获取提交数据：

   ```bash
   python -m pip install "sentry-cli"
   ```

6. 打开项目根文件夹中包含的 `Makefile`。该文件用于模拟 CI/CD 流程。

7. 遵循 `deploy` 目标执行流程。

`Makefile` 中提到的命令将在教程的下一部分[配置选项](/product/sentry-basics/integrate-backend/configuration-options/)中详细解释。

 <Alert>
除了安装 Python 要求并运行服务器外，我们还使用 `sentry-cli` 创建新的 Sentry 发布并将提交与该发布关联。Sentry 在标记可疑提交时将查看这些提交。
</Alert>

8. 要执行 `sentry-cli` 命令，请按照这些[说明](/product/sentry-basics/integrate-frontend/upload-source-maps/#step-1-prepare-the-build-environment)获取您的 `SENTRY_AUTH_TOKEN`、`SENTRY_ORG` 和 `SENTRY_PROJECT` 环境变量的值。

   ![Makefile Config](./img/makefile_config.png)

   > `sentry-cli` 可以通过提供这些值来配置，可以通过环境变量或通过专用配置文件。在 [Sentry CLI：配置和身份验证](/cli/configuration/)中了解更多信息。

9. 运行以下命令以安装所需的 Python 库，设置 Sentry 发布，并运行 Django 服务器：

    ```bash
    make deploy
    ```

    ![Deploy & Serve](./img/run_django_server.png)

    > 请注意，终端中创建了一个新发布，并且提交与之关联。一旦部署成功完成，您将在终端中看到确认。

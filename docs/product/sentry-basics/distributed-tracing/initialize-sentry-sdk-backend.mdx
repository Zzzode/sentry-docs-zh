本节将指导您如何将示例应用程序导入到本地开发环境中，添加 Sentry SDK 并进行初始化。

如果您使用的是自己的源代码，可以跳过本节。相反：

- 选择您的[平台](/platforms/)并按照其**入门指南**将 Sentry SDK 添加到您的代码中。
- 然后，跳转到[下一步](/product/sentry-basics/distributed-tracing/generate-first-error/)。

## 1. 克隆示例应用程序

示例应用程序是一个使用 Express 和 Node 的基本后端应用程序。

1. 在 GitHub 上 fork [示例应用程序仓库](https://github.com/getsentry/tracing-tutorial-backend)。

1. 将 fork 的仓库克隆到本地环境：

   ```bash
   git clone git@github.com:<your_username>/tracing-tutorial-backend.git
   ```

1. 在您首选的代码编辑器中打开 `tracing-tutorial-backend` 项目。

## 2. 添加 Sentry Node SDK

Sentry 通过使用特定于平台的 SDK 来捕获数据，您需要将其添加到应用程序的运行时中。要使用 SDK，请在源代码中导入并配置它。此演示项目使用 [Sentry 的 Node SDK](https://github.com/getsentry/sentry-javascript/tree/master/packages/node)。

1. 导航到 `tracing-tutorial-backend` 项目文件夹，并使用 NPM 安装 Sentry Express SDK。

   ```bash {tabTitle:npm}
   npm install --save @sentry/node @sentry/profiling-node
   ```

   ```bash {tabTitle:yarn}
   yarn add @sentry/node @sentry/profiling-node
   ```

   ```bash {tabTitle:pnpm}
   pnpm add @sentry/node @sentry/profiling-node
   ```

1. 在项目根目录下创建一个名为 `instrument.js` 的文件。使用以下代码导入并初始化 SDK：

   ```javascript {filename:instrument.js}
   const Sentry = require("@sentry/node");
   const { nodeProfilingIntegration } = require("@sentry/profiling-node");

   Sentry.init({
     dsn: "sample_DSN_key",
     integrations: [nodeProfilingIntegration()],
     // 追踪
     // 捕获 100% 的事务
     tracesSampleRate: 1.0,

     // 设置性能分析的采样率 - 这是相对于 tracesSampleRate 的
     profilesSampleRate: 1.0,
   });
   ```

   重要的是要尽可能早地在应用程序生命周期中导入和初始化 SDK，这样 Sentry 就可以在整个生命周期中捕获错误。

1. 将您的 DSN 密钥添加到 Sentry SDK 配置中。

   粘贴您在[上一节](/product/sentry-basics/distributed-tracing/create-new-project/)中创建的项目中复制的 DSN 密钥值。注意：此 DSN 密钥应该与您用于配置前端项目的**不同**。您的前端和后端项目应该各自有一个唯一的 DSN。

   ```javascript {filename:instrument.js}
   Sentry.init({
     dsn: "<your_DSN_key>",
     // ...
   });
   ```

1. 保存文件。

   在 `Sentry.init()` 中设置的选项称为 SDK 的配置。唯一必需的配置选项是 DSN。但是，SDK 支持许多其他配置选项。在我们的[配置](/platforms/javascript/guides/react/configuration/)文档中了解更多信息。

   上述配置启用了 Sentry 的错误监控功能，以及其[**追踪**](/product/tracing/)和[**会话重放**](/platforms/javascript/guides/react/session-replay)功能。

1. 在 `server.js` 文件的顶部导入 `instrument.js`。然后导入 `Sentry` 并在所有控制器之后和任何其他中间件之前设置错误处理程序：

```javascript diff {filename:server.js}
+    // 重要：确保在文件顶部导入 `instrument.js`。
+    // 如果您使用的是 ECMAScript 模块 (ESM) 语法，请使用 `import "./instrument.js";`
+    require("./instrument.js");

+    // 所有其他导入在下面
+    // 如果您使用 ESM，请使用 `import * as Sentry from "@sentry/node"`
   const Sentry = require("@sentry/node");
   const express = require("express");
   const productsRoute = require('./routes/products');
   const cors = require('cors')

   const app = express();

   app.use(express.static('public'));
   app.use(cors());

   app.get('/', (req, res) => {
       res.send('<h1>Hello, Express.js Server here!</h1>');
   });

   app.get("/products/debug-sentry", (req, res) => {
     console.log("Sentry Error thrown!");
     throw new Error("My first Sentry error!");
   });

+    // 错误处理程序必须在任何其他错误中间件之前注册，并在所有控制器之后注册
+    Sentry.setupExpressErrorHandler(app);

+    // 可选的贯穿错误处理程序
+    app.use(function onError(err, req, res, next) {
+        // 错误 ID 附加到 `res.sentry` 以返回
+        // 并可选择显示给用户以获取支持。
        console.log('500 error thrown!');
        res.statusCode = 500;
        res.end(res.sentry + "\n");
    });

   app.use('/products', productsRoute);

   const port = 3001;

    app.listen(port, () => {
       console.log(`Server is running on port 3001`);
   });

```

## 3. 构建并运行示例应用程序

在 `tracing-tutorial-backend` 项目文件夹中：

1. 安装项目依赖项。

   ```bash
   npm install
   ```

1. 在开发模式下启动应用程序：

   ```bash
   node server.js
   ```

   应用程序启动后，您将在终端中看到类似于此的确认消息：

   ```bash
   Server is running on port 3001
   ```

> **故障排除提示**：如果应用程序由于语法错误或缺少依赖项/模块错误而无法启动，请确保您使用的是 Node 18+ 并重新安装依赖项。如果您使用的是 [nvm](https://github.com/nvm-sh/nvm)，请确保使用正确的节点版本 `nvm use 18`，然后 `npm install`。

1. 在浏览器中打开示例应用程序。

   示例应用程序应该在 [http://localhost:3001/](http://localhost:3001/) 或上一步中终端输出的 URL 上运行。

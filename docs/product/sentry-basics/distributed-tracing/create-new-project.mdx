为了捕获错误和其他事件，您需要在 Sentry 中设置一个项目。在本教程中，我们将为 React 前端项目创建一个项目，并为 Express 后端项目创建另一个独立的项目。

因为您可以为应用程序中使用的**每种**语言或框架创建一个 Sentry 项目（例如，您可能为 API 服务器和前端客户端分别设置不同的项目），所以单个应用程序可能在 Sentry 中关联多个项目。一旦您在 Sentry 中设置了与应用程序不同部分对应的所有项目，您就能将事件限定到组织中的特定应用程序，并将责任分配给特定用户和团队。有关更多信息，请参阅[创建项目的最佳实践](/organization/getting-started/#4-create-projects)。

## 创建前端项目

按照以下步骤为示例 React 应用程序创建一个新的 Sentry 项目：

1. 登录您的 [Sentry 组织](https://sentry.io)。

1. 从左侧导航菜单中选择**项目**。 

1. 点击"创建项目"并根据您的应用程序进行适当配置：

   - **选择平台**：根据您要监控的代码选择项目的语言或框架。在本例中，选择 **React**。

   - **设置警报频率**：出于本教程的目的，选择**在高优先级问题上提醒我**。

     > 如果您担心警报疲劳，请了解如何在[警报最佳实践](/product/alerts/best-practices/)中设置警报。

   - **命名项目并分配团队**：在**项目名称**字段中命名您的项目，并通过在**团队**下拉菜单中选择一个团队来分配团队（或点击**+**创建新团队）。

   - 点击**创建项目**。这将带您进入快速配置 React SDK 指南，该指南概述了如何配置 SDK。本教程将更详细地介绍 SDK 设置。

   - 您会注意到 Sentry React SDK 设置代码已预先填充了项目的唯一 DSN（数据源名称）。DSN 告诉 SDK 将事件发送到哪里，以便事件与正确的项目关联。稍后在本教程中，您需要将 DSN 密钥粘贴到 SDK 配置代码中。您现在可以记下它，或者您也可以随时在**[项目] > 设置 > SDK 设置 > 客户端密钥 (DSN)**中找到项目的 DSN。

1. 点击**带我去问题页面**进入新项目的**问题**页面。

## 创建后端项目

接下来，按照以下步骤为示例后端应用程序创建一个新的 Sentry 项目：

1. 登录您的 [Sentry 组织](https://sentry.io)。

1. 从左侧导航菜单中选择**项目**以显示所有项目的列表。

1. 点击"创建项目"并根据您的应用程序进行适当配置：

   - **选择平台**：在本例中，选择 **Express**。

   - **设置警报频率**：再次选择**在高优先级问题上提醒我**。

   - **命名项目并分配团队**：确保为您的后端应用程序指定一个与前端应用程序不同的项目名称。

   - 点击**创建项目**。这将带您进入快速配置 Express SDK 指南，该指南概述了如何配置 SDK。

1. 复制 DSN 密钥并妥善保存。

   > 您也可以随时在**[项目] > 设置 > SDK 设置 > 客户端密钥 (DSN)**中找到项目的 DSN。

1. 点击**带我去问题页面**进入新项目的**问题**页面。

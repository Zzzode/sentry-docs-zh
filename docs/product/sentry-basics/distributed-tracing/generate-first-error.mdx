现在示例应用程序已在您的本地环境中运行并与 Sentry SDK 集成，您已准备好生成第一个错误。请确保您已同时运行两个应用程序（分别在 http://localhost:3000 和 http://localhost:3001 上）。

您应该看到"一站式商店"网站在 http://localhost:3000 上运行。点击任何产品按钮都会触发对后端的调用以获取产品信息。只要您的 Express 服务器正在运行，您就应该看到每个产品的标题、图片和描述。

## 1. 触发错误

为了开始使用 Sentry 的错误监控功能，您首先需要一些错误。让我们添加一个明显的错误，这样在 Sentry 中很容易看到。

如果您使用的是自己的源代码，请跳过此步骤。相反，选择您的[平台](/platforms/)并按照其**入门指南**中的**验证**步骤引入错误。

1. 在 `tracing-tutorial-frontend` 仓库中，打开 `src/App.js` 并通过将传递给 `getProduct()` 函数的字符串从 `nonfat-water` 更改为 `debug-sentry` 来更新 `onClick` 处理程序：

```jsx diff {filename:App.js}
    <div className="btn-parent">
        <button className="btn" onClick={() => getProduct("clown-shoes")}>
            Clown Shoes
        </button>
    </div>
    <div className="btn-parent">
-      <button className="btn" onClick={() => getProduct("nonfat-water")}>
+      <button className="btn" onClick={() => getProduct("debug-sentry")}>
        Nonfat Water
       </button>
    </div>
```

在 `tracing-tutorial-backend` 仓库的 `server.js` 文件中有一个中间件函数处理 `/products/debug-sentry` 路由，它会抛出一个示例错误：

```javascript {filename:server.js}
app.get("/products/debug-sentry", (req, res) => {
  console.log("Sentry Error thrown!");
  throw new Error("My first Sentry error!");
});
```

1. 保存文件。

1. 返回浏览器窗口并尝试再次点击**脱脂水**按钮，您将不再看到预期的产品信息。相反，您会看到显示`500 错误`消息。

1. 打开运行 `tracing-tutorial-backend` 服务器的终端窗口。在这里，您会看到一条日志，让您知道发生了错误：

```bash
Sentry Error thrown!
```

这确认了我们刚才在前端产生的错误已在后端触发了相应的错误。

## 2. 在 Sentry 中查看错误

现在您已经触发了错误，让我们看看它在 Sentry 中的样子。

1. 在 Sentry.io 中打开[**追踪**](https://sentry.io/orgredirect/organizations/:orgslug/traces/)页面。确保为此教程选择的一个（或两个）项目已在项目下拉菜单中被选中。

2. 您应该看到一个追踪，显示 React 和 Express 图标，表明此追踪跨越了两个项目。点击左侧的蓝色追踪 ID 查看追踪视图。

3. 在[追踪视图](/concepts/key-terms/tracing/trace-view/)页面上，您可以看到此追踪中发生的每个跨度。您可以看到 React 应用程序上的错误直接触发了 Express 应用程序上的错误。

本节将指导您如何将示例应用程序导入到本地开发环境中，添加 Sentry SDK 并进行初始化。

如果您使用的是自己的源代码，可以跳过本节。相反：

- 选择您的[平台](/platforms/)并按照其**入门指南**将 Sentry SDK 添加到您的代码中。
- 然后，跳转到[下一步](/product/sentry-basics/distributed-tracing/initialize-sentry-sdk-backend/)。

## 1. 克隆示例应用程序

示例应用程序是一个使用 React 和 webpack 的基本前端应用程序。

1. 在 GitHub 上 fork [示例应用程序仓库](https://github.com/getsentry/tracing-tutorial-frontend)。

1. 将 fork 的仓库克隆到本地环境：

   ```bash
   git clone git@github.com:<your_username>/tracing-tutorial-frontend.git
   ```

1. 在您首选的代码编辑器中打开 `tracing-tutorial-frontend` 项目。

## 2. 添加 Sentry React SDK

Sentry 通过使用特定于平台的 SDK 来捕获数据，您需要将其添加到应用程序的运行时中。要使用 SDK，请在源代码中导入并配置它。此演示项目使用 [Sentry 的 React SDK](https://github.com/getsentry/sentry-javascript/tree/develop/packages/react)。

1. 使用 NPM 安装 Sentry React SDK。

   确保您在 `tracing-tutorial-frontend` 项目文件夹中。

   ```bash {tabTitle:npm}
   npm install @sentry/react --save
   ```

   ```bash {tabTitle:yarn}
   yarn add @sentry/react
   ```

   ```bash {tabTitle:pnpm}
   pnpm add @sentry/react
   ```

1. 导入并初始化 SDK。

   打开 `src/index.js` 并在最后一个 import 语句下方添加以下代码行：

   ```javascript {filename:src/index.js} {12}
   import * as Sentry from "@sentry/react";

   Sentry.init({
     dsn: "your_DSN_key",
     integrations: [
       Sentry.browserTracingIntegration(),
       Sentry.replayIntegration(),
     ],
     // 追踪
     tracesSampleRate: 1.0, //  捕获 100% 的事务
     // 设置 'tracePropagationTargets' 以控制应为哪些 URL 启用分布式追踪
     tracePropagationTargets: ["localhost", /^https:\/\/yourserver\.io\/api/],
     // 会话重放
     replaysSessionSampleRate: 0.1, // 这将采样率设置为 10%。您可能希望在开发期间将其更改为 100%，然后在生产中以较低的速率采样。
     replaysOnErrorSampleRate: 1.0, // 如果您尚未对整个会话进行采样，则在采样发生错误的会话时将采样率更改为 100%。
   });
   ```

   重要的是要尽可能早地在应用程序生命周期中导入和初始化 SDK，这样 Sentry 就可以在整个生命周期中捕获错误。

1. 将您的 DSN 密钥添加到 Sentry SDK 配置中。

   粘贴您在[上一节](/product/sentry-basics/distributed-tracing/create-new-project/)中创建的项目中复制的 DSN 密钥值。

   ```javascript {filename:src/index.js}
   Sentry.init({
     dsn: "<your_DSN_key>",
     // ...
   });
   ```

1. 保存文件。

在 `Sentry.init()` 中设置的选项称为 SDK 的配置。唯一必需的配置选项是 DSN。但是，SDK 支持许多其他配置选项。在我们的[配置](/platforms/javascript/guides/react/configuration/)文档中了解更多信息。

上述配置启用了 Sentry 的错误监控功能，以及[**追踪**](/product/tracing/)和[**会话重放**](/platforms/javascript/guides/react/session-replay)功能。请注意 `tracePropagationTargets` 选项，这是启用项目运行的任何 URL 上的追踪所必需的。由于本教程的两个项目都在 `localhost` 上运行，所以我们已经设置好了。

## 3. 构建并运行示例应用程序

在 `tracing-tutorial-frontend` 项目文件夹中：

1. 安装项目依赖项。

   ```bash
   npm install
   ```

1. 在开发模式下启动应用程序。

   ```bash
   npm start
   ```

   应用程序启动后，您将在终端中看到类似于此的确认消息：

   ```bash
   Compiled successfully!

   You can now view tracing-demo in the browser.

      Local:            http://localhost:3000
      On Your Network:  http://10.0.0.39:3000
   ```

   > **故障排除提示**：如果应用程序由于语法错误或缺少依赖项/模块错误而无法启动，请确保您使用的是 Node 18+ 并重新安装依赖项。如果您使用的是 [nvm](https://github.com/nvm-sh/nvm)，请确保使用正确的节点版本 `nvm use 18`，然后 `npm install`。

1. 在浏览器中打开示例应用程序。

   示例应用程序应该在[http://localhost:3000/](http://localhost:3000/)或上一步中终端输出的 URL 上运行。您应该看到一个示例电子商务页面；此页面上的按钮将无法正常工作，并将触发未捕获的运行时错误，直到您让后端运行起来。

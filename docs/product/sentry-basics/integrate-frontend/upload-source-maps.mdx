在创建 Web 应用程序时，大多数开发工作流程都涉及通过转译和压缩来转换您的 JavaScript 代码，使其在浏览器中更高效地运行。源映射文件作为工具（如 Sentry）的指南，将 JavaScript 的转换版本转换回您编写的代码。源映射文件可以由您的 JavaScript 构建工具生成。

示例项目使用 webpack 作为其构建工具。我们将配置它来生成源映射并检查它们是否输出。

如果您的项目有不同的配置，请使用 [Sentry 向导](/platforms/javascript/sourcemaps/#uploading-source-maps) 来配置您的源映射并跳到第 3 步。

## 1. 使用您的构建工具生成源映射文件

1. 要配置 webpack 来生成源映射，请打开 `webpack.config.js` 并在 `module.exports` 内添加以下行：

   ```javascript {filename:webpack.config.js}
   ...
   module.exports = {
      entry: "./src/index.js",
      output: {
         filename: "main.js",
         path: path.resolve(__dirname, "build"),
      },
      devtool: "source-map",
      ...
   }
   ...
   ```

   如果您使用的是示例项目，您可以取消第 11 行的注释。

1. 保存文件。

1. webpack 仅在您创建生产构建时生成源映射。为此，请使用 `Ctrl + C` 停止在开发模式下运行的应用程序，并通过运行以下命令启动生产构建：

   ```bash
   npm run build
   ```

   您应该在项目中看到一个新的 `build` 文件夹。在里面，应该有一个 `main.js` 文件和一个 `main.js.map` 文件。`main.js` 是您压缩的 JavaScript。`main.js.map` 是源映射文件。

## 2. 将源映射上传到 Sentry

1. 安装 Sentry webpack 插件

   ```bash
   npm install @sentry/webpack-plugin --save-dev
   ```

1. 要生成 Sentry 身份验证令牌并将其存储以便可以使用，请将以下内容粘贴到您的终端中：

   <OrgAuthTokenNote />

   ```bash {filename:.env}
   export SENTRY_AUTH_TOKEN=___ORG_AUTH_TOKEN___
   ```

1. 打开 `webpack.config.js` 并添加以下行以将插件配置添加到 webpack。


   ```javascript {filename:webpack.config.js}
   const { sentryWebpackPlugin } = require("@sentry/webpack-plugin");

   ...

   module.exports = {
      ...
     plugins: [
         sentryWebpackPlugin({
            org: "___ORG_SLUG___",
            project: "___PROJECT_SLUG___",
            authToken: process.env.SENTRY_AUTH_TOKEN,
         }),
      ],
      ...
   }
   ```

1. 您可以在 Sentry URL（例如，https://**organization_slug**.sentry.io/）中找到您的 `org`，或在**组织设置**页面中找到。

## 3. 触发另一个错误

1. 使用上一步中的更改创建新的生产构建：

   ```bash
   npm run build
   ```

1. 确认您的源映射已正确上传到 Sentry 并与您项目的最新发布关联。您的终端输出应包括一个`源映射上传报告`，可能如下所示：

   ```bash
   > 组织：<your_org>
   > 项目：<your_project>
   > 发布：0d40018e21151113e224f208fb934a0d29f10508
   > 分发：无
   > 上传类型：工件包
   源映射上传报告
      压缩脚本
         ~/f320b889-aa78-4850-8625-802a2ee9aca3-0.js（源映射位于 main.js.map，调试 ID f320b889-aa78-4850-8625-802a2ee9aca3）
      源映射
         ~/f320b889-aa78-4850-8625-802a2ee9aca3-0.js.map（调试 ID f320b889-aa78-4850-8625-802a2ee9aca3）
   [sentry-webpack-plugin] 信息：成功将源映射上传到 Sentry
   ```

1. 运行您的生产构建：

   ```bash
   npx serve build
   ```

1. 在浏览器中打开示例应用程序的生产构建。

   示例应用程序应该在 [http://localhost:3000/](http://localhost:3000/) 或上一步中终端输出的 URL 上运行。

1. 在浏览器中，确保开发者控制台已打开，并执行"清空缓存并硬重新加载"以确保提供更新的代码。

   ![发布已创建](./img/upload-source-maps-08.png)

1. 通过将产品添加到购物车并点击"结账"再次生成错误。

1. 转到 Sentry 中的**问题**页面。由于您添加了新的发布，即使它是与之前相同的错误，也应该创建一个新问题。

1. 点击新问题以打开其**问题详情**页面。

1. 请注意，现在问题有一个发布标签，并且错误堆栈跟踪现在已解压缩。堆栈跟踪现在包括每个堆栈帧中的文件名、方法名、行号和列号以及源代码上下文。

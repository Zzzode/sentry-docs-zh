Sentry 使用来自源代码仓库的提交元数据来帮助您更快地解决问题。这是通过在您的**问题详情**页面中建议_可疑提交_（可能已引入错误的提交）来完成的。它还允许 Sentry 显示_建议的负责人_——这些提交的作者列表——并建议将他们分配给解决问题。

现在您已经创建了发布，您可以告诉 Sentry 哪些提交与此最新版本的代码相关联；这称为_提交跟踪_。

## 1. 集成您的 GitHub 账户和仓库

使用 Sentry 的 GitHub 集成将您的源代码仓库链接到您的 Sentry 项目。这使 Sentry 能够使用提交元数据来帮助您简化问题分类。

1. 要将 GitHub 与您的 Sentry 组织集成，请按照 [GitHub 集成安装说明](/organization/integrations/source-code-mgmt/github/#install) 操作。确保为集成提供对适当仓库的访问权限——在本例中，是您 fork 的 `frontend-tutorial` 仓库。

   添加集成后，您应该被重定向到一个页面，显示您在 Sentry 中的 GitHub 仓库。验证是否已添加正确的仓库。

1. 转到"代码映射"选项卡。

1. 点击"添加代码映射"。

1. 填写请求的信息，以在您的 Sentry 项目和 `frontend-tutorial` 仓库的 `main` 或 `master` 分支之间添加代码映射。告诉 Sentry 您的组件位于 `src/` 目录中：

   ![添加代码映射](./img/configure-scms-09.png)

## 2. 设置提交跟踪

_发布_是部署到环境的代码版本。请参阅 [发布](/product/releases/) 文档了解更多信息。Sentry 使用发布来匹配您的源映射和源代码。发布还可用于启用 Sentry 的 [可疑提交](/product/issues/suspect-commits/) 功能。

Sentry webpack 插件可以在创建项目的新构建时自动配置发布信息并将其发送到 Sentry。让我们启用此功能。

1. 返回您的 `frontend-tutorial` 项目并使用 `Ctrl + C` 停止它。

1. 创建 Sentry 身份验证令牌：

   <OrgAuthTokenNote />

   ```bash {filename:.env}
   SENTRY_AUTH_TOKEN=___ORG_AUTH_TOKEN___
   ```

    稍后您需要将身份验证令牌传递给构建命令。我们建议将其存储在环境变量中。身份验证令牌应始终保持机密，因此避免将其存储在版本控制中。

1. 要开始关联提交和发布，请将以下代码行添加到 `webpack.config.js` 中的 Sentry webpack 插件配置中：


   ```javascript {filename:webpack.config.js}
      sentryWebpackPlugin({
         org: "___ORG_SLUG___",
         project: "___PROJECT_SLUG___",
         authToken: process.env.SENTRY_AUTH_TOKEN,

         // 启用自动创建发布和关联提交
         release: {
           create: true,
           setCommits: {
              auto: true,
           },
         },
      }),
   ```

1. 保存文件。

## 3. 创建新的错误和提交

现在发布已设置为跟踪提交，调整错误并创建新的提交以验证这是否有效。

1. 在 `src/components/App.js` 中，编辑第 97 行。这将确保 Sentry 识别它可以与您将创建的发布关联的新错误。

   ```javascript {filename:src/components/App.js}
   this.myCodeIsMorePerfect();
   ```

1. 保存文件。

1. 使用以下命令将所有更改添加到 git：

   ```bash
   git commit -am '添加了新错误和其他内容'
   git push
   ```

1. 创建另一个生产构建。在此过程中，webpack 插件将创建新的发布并将您的提交与该发布关联。

   ```bash
   npm run build
   ```

1. 运行您的生产构建：

   ```bash
   npx serve build
   ```

1. 通过在您的应用程序中生成另一个错误来创建 Sentry 的新问题。打开您的应用程序运行的 URL，将产品添加到购物车，然后点击"结账"。您应该很快在 Sentry 中看到新问题。

## 4. 验证可疑提交和建议的负责人

现在可疑提交和建议的负责人应该开始出现在**问题详情**页面上。Sentry 使用堆栈跟踪中观察到的文件、这些文件的作者和所有权规则来确定这些。

1. 检查您的电子邮件，了解有关新错误的警报。请注意，新的"可疑提交"部分已添加到电子邮件中。

   ![可疑提交邮件](./img/issue_email_suspect_commits.png)

1. 点击"在 Sentry 上查看"以打开**问题详情**页面。

1. 验证"可疑提交"部分现在指向可能引入错误的可疑提交。您可以点击提交链接在 GitHub 上查看实际的提交详细信息。

1. 在右侧面板中，点击"分配给"下的下拉菜单以查看建议的负责人。您会看到可疑提交的作者被列为该问题的建议负责人。

   ![建议的负责人](./img/suggested_assignees.png)

   您可以通过点击他们的电子邮件将问题分配给建议的负责人。或者，您可以选择将问题分配给项目关联的其他用户或团队。

## 5. 验证发布配置

验证发布是否已正确配置。

1. 在主面板的"标签"部分中，找到 `release` 标签并将鼠标悬停在发布 ID 链接上。您应该在弹出窗口中看到发布现在包含提交数据。

1. 点击发布 ID 链接以打开**发布详情**页面。

1. 选择"提交"选项卡。请注意，发布现在包含关联的提交列表。

## 6. 使用堆栈跟踪链接

堆栈跟踪链接允许您从 [sentry.io](https://sentry.io) 中的堆栈跟踪跳转到源代码提供程序中的相应文件。这是通过将堆栈跟踪中的文件路径与源代码提供程序中的文件路径匹配来完成的。

1. 转到上一步中生成的错误的**问题详情**页面。

1. 点击"告诉我们您的源代码在哪里"，以便 Sentry 可以验证您的代码映射。

1. 在弹出窗口中，验证或输入文件路径和正确的 GitHub URL。

1. 点击"保存"。您现在应该看到堆栈跟踪已链接到 GitHub 中的代码文件。

1. 点击"在 GitHub 中打开此行"以直接跳转到导致错误的代码行。

## 后续步骤

您现在应该已设置好 Sentry 的错误监控，源映射和可疑提交正常工作。

除了错误监控之外，Sentry 还提供其他功能来帮助您检测和修复应用程序的问题，例如：

- [会话重放](/product/explore/session-replay/)，在发生错误时查看用户会话的视频式重现。
- [洞察](/product/insights/)，跨服务跟踪应用程序性能和错误。
- [性能分析](/product/explore/profiling/)，在传统跟踪之上获得更深层次的可见性。

以及 [Codecov](https://docs.codecov.com/docs)，为您提供 CI/CD 的一部分的自动代码覆盖率报告。

本节将指导您如何将示例应用程序导入到本地开发环境中，添加 Sentry SDK 并进行初始化。

如果您使用的是自己的源代码，可以跳过本节。相反：

- 选择您的[平台](/platforms/)并按照其**入门指南**将 Sentry SDK 添加到您的代码中。
- 然后，跳转到[下一步](/guides/integrate-frontend/generate-first-error/)。

## 1. 克隆示例应用程序

示例应用程序是一个使用 React 和 webpack 的基本前端应用程序。

1. 在 GitHub 上 fork [示例应用程序仓库](https://github.com/getsentry/frontend-tutorial)。

1. 将 fork 的仓库克隆到本地环境：

   ```bash
   git clone git@github.com:<your_username>/frontend-tutorial.git
   ```

1. 在您首选的代码编辑器中打开 `frontend-tutorial` 项目。

## 2. 添加 Sentry React SDK

Sentry 通过使用特定于平台的 SDK 来捕获数据，您需要将其添加到应用程序的运行时中。要使用 SDK，请在源代码中导入并配置它。此演示项目使用 [Sentry 的 React SDK](https://github.com/getsentry/sentry-javascript/tree/develop/packages/react)。

1. 使用 NPM 安装 Sentry React SDK。

   确保您在 `frontend-tutorial` 项目文件夹中。

   ```bash {tabTitle:npm}
   npm install @sentry/react --save
   ```

   ```bash {tabTitle:yarn}
   yarn add @sentry/react
   ```

   ```bash {tabTitle:pnpm}
   pnpm add @sentry/react
   ```

1. 导入并初始化 SDK。

   打开 `src/index.js` 并在最后一个 import 语句下方添加以下代码行：

   ```javascript {filename:src/index.js}
   import * as Sentry from "@sentry/react";

   Sentry.init({
     dsn: "<your_DSN_key>",
     integrations: [Sentry.browserTracingIntegration(), Sentry.replayIntegration()],

     // 将 tracesSampleRate 设置为 1.0 以捕获 100%
     // 的事务用于性能监控。
     // 了解更多信息
     // https://docs.sentry.io/platforms/javascript/configuration/options/#traces-sample-rate
     tracesSampleRate: 1.0,

     // 为所有会话的 10% 捕获重放，
     // 加上为 100% 的出错会话捕获重放
     // 了解更多信息
     // https://docs.sentry.io/platforms/javascript/session-replay/configuration/#general-integration-configuration
     replaysSessionSampleRate: 0.1,
     replaysOnErrorSampleRate: 1.0,
     ${<Include name="code-comments/javascript/trace-sample-rate" />}
   });
   ```

   重要的是要尽可能早地在应用程序生命周期中导入和初始化 SDK，这样 Sentry 就可以在整个生命周期中捕获错误。 <Include name="platforms/configuration/options/js-config-options" />



1. 将您的 DSN 密钥添加到 Sentry SDK 配置中。

   粘贴您在[上一节](/product/sentry-basics/integrate-frontend/create-new-project/)中创建的项目中复制的 DSN 密钥值。

   ```javascript {filename:src/index.js}
   Sentry.init({
     dsn: "<your_DSN_key>",
     // ...
   });
   ```

1. 保存文件。

在 `Sentry.init()` 中设置的选项称为 SDK 的配置。唯一必需的配置选项是 DSN。但是，SDK 支持许多其他配置选项。在我们的[配置](/platforms/javascript/guides/react/configuration/)文档中了解更多信息。

上述配置启用了 Sentry 的错误监控功能，以及其[性能](/platforms/javascript/guides/react/performance)（跟踪）和[会话重放](/platforms/javascript/guides/react/session-replay)功能。

## 3. 构建并运行示例应用程序

在 `frontend-tutorial` 项目文件夹中：

1. 安装项目依赖项。

   ```bash
   npm install
   ```

1. 在开发模式下启动应用程序。

   ```bash
   npm start
   ```

   应用程序启动后，您将在终端中看到类似于此的确认消息：

   ```bash
   <i> [webpack-dev-server] 项目正在运行：
   <i> [webpack-dev-server] 回环地址：http://localhost:3000/
   ...
   webpack 5.87.0 在 1306 毫秒内成功编译
   ```

   > **故障排除提示**：如果应用程序由于语法错误或缺少依赖项/模块错误而无法启动，请确保您使用的是 Node 18+ 并重新安装依赖项。运行 `nvm use 18`，然后 `npm install`。

1. 在浏览器中打开示例应用程序。

   示例应用程序应该在 [http://localhost:3000/](http://localhost:3000/) 或上一步中终端输出的 URL 上运行。

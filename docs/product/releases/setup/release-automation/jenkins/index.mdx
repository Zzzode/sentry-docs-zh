本指南将引导您完成在Jenkins中自动化Sentry发布管理和部署通知的过程。在Jenkins中部署后，您将能够将提交与发布关联。您还能够应用源映射以在Sentry中查看原始代码。

在开始之前，请确认您的Sentry项目已正确设置以跟踪提交元数据，方法是[安装仓库集成](/product/releases/setup/release-automation/)。安装完成后，并且您已添加了仓库，请返回本指南。如果您已安装了仓库集成，您就可以开始了。

## 创建Sentry内部集成

为了使Jenkins与Sentry安全通信，您需要创建一个新的内部集成。在Sentry中，导航到**设置 > 开发者设置 > 新建内部集成**。

为您的新集成命名（例如，"Jenkins部署集成"）并指定必要的权限。在这种情况下，您需要"发布"的管理员访问权限和"组织"的读取访问权限。

有关作用域和API端点的更多详细信息，请参见[API权限](/api/permissions/)的完整文档。

![内部集成权限视图。](./img/internal-integration-permissions.png)

点击"保存"，然后记录您的令牌，您将在下一节中需要它。

## 将令牌添加到Jenkins凭据

为了在Jenkins中安全地访问您的内部集成令牌，请将其存储为[凭据](https://www.jenkins.io/doc/book/using/using-credentials/)：

1. 从Jenkins主页（经典Jenkins UI的仪表板），点击**凭据 > 系统**。
2. 在**系统**下，点击**全局凭据（无限制）**链接以访问此默认域。
3. 点击**添加凭据**。
4. 从**类型**字段中选择密钥文本。
5. 给您的凭据一个**ID**（例如，"sentry-auth-token"）并在**密钥**字段中粘贴您的内部集成令牌。
6. 点击**确定**保存凭据。

![Jenkins添加凭据视图。](./img/jenkins-add-credential.png)

## 通知Sentry部署

根据您用于构建和部署的Jenkins项目类型，您需要遵循不同的步骤：

- [管道(Jenkinsfile)](/product/releases/setup/release-automation/jenkins/#pipeline)
- [自由风格项目](/product/releases/setup/release-automation/jenkins/#freestyle-project)

有关下面发布管理概念的更多详细信息，请参见[发布管理](/cli/releases/)的完整文档。

### 管道

如果您使用Jenkins管道，您需要在部署阶段之后添加一个包含以下代码段的新阶段：

```groovy
pipeline {
		// ...
    stages {

        /// ...

        stage('通知Sentry部署') {
            environment {
                SENTRY_AUTH_TOKEN = credentials('sentry-auth-token')
                SENTRY_ORG = 'sample-organization-slug'
                SENTRY_PROJECT = 'sample-project-slug'
                SENTRY_ENVIRONMENT = 'production'
            }
            steps {
                // 安装Sentry CLI
                sh 'command -v sentry-cli || curl -sL https://sentry.io/get-cli/ | bash'

                sh '''
                    export SENTRY_RELEASE=$(sentry-cli releases propose-version)
                    sentry-cli releases new -p $SENTRY_PROJECT $SENTRY_RELEASE
                    sentry-cli releases set-commits $SENTRY_RELEASE --auto
                    sentry-cli sourcemaps upload --release $SENTRY_RELEASE /path/to/sourcemaps
                    sentry-cli releases finalize $SENTRY_RELEASE
                    sentry-cli deploys new -e $SENTRY_ENVIRONMENT
                '''
            }
        }
    }
}
```

**注意**：

- 如果您部署的项目不需要源映射，或者您已使用其他方法将源映射发送到Sentry，请省略`sourcemaps upload`行。
- 如果您无法安装仓库集成，请使用[创建发布端点](/product/releases/associate-commits/#using-the-api)发送提交元数据，或省略`set-commits`行。
- `credentials('sentry-auth-token')`指的是刚刚添加到Jenkins的凭据的ID。
- `sentry-cli releases propose-version`默认为被部署的提交的提交SHA。要将其设置为不同的版本，请将`SENTRY_RELEASE`修改为首选版本。

### 自由风格项目

如果您使用自由风格项目，您需要在部署后添加另一个构建步骤。

1. 确保您添加了一个绑定，将环境变量`SENTRY_AUTH_TOKEN`映射到上一步中添加的内部集成令牌凭据。

   ![环境变量绑定视图。](./img/jenkins-freestyle-bindings.png)

2. 在部署步骤之后添加以下构建步骤：

   ![Jenkins自由风格shell视图。](./img/jenkins-freestyle-shell.png)

   ```bash
   curl -sL https://sentry.io/get-cli/ | bash || true

   export SENTRY_ORG=sample-organization-slug
   export SENTRY_PROJECT=sample-project-slug
   export SENTRY_ENVIRONMENT=production
   export SENTRY_RELEASE=$(sentry-cli releases propose-version)

   sentry-cli releases new -p $SENTRY_PROJECT $SENTRY_RELEASE
   sentry-cli releases set-commits $SENTRY_RELEASE --auto
   sentry-cli sourcemaps upload --release $SENTRY_RELEASE path-to-sourcemaps-if-applicable
   sentry-cli releases finalize $SENTRY_RELEASE
   sentry-cli deploys new -e $SENTRY_ENVIRONMENT
   ```

   **注意**：

   - 如果您部署的项目不需要源映射，或者您已使用其他方法将源映射发送到Sentry，请省略`sourcemaps upload`行。
   - 如果您无法安装仓库集成，请使用[创建发布端点](/product/releases/associate-commits/#using-the-api)发送提交元数据，或省略`set-commits`行。
   - `sentry-cli releases propose-version`默认为被部署的提交的提交SHA。要将其设置为不同的版本，请将`SENTRY_RELEASE`修改为首选版本。

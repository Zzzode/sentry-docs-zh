[Sentry发布GitHub Action](https://github.com/marketplace/actions/sentry-release)只需一步即可在GitHub中自动化Sentry发布管理。发送Sentry发布信息后，您将能够将提交与发布关联。此外，您可以上传源映射以查看原始的、未压缩的源代码。

有关Sentry发布管理概念的更多详细信息，请参见[发布](/product/releases/)的完整文档。

## 先决条件

为了使Sentry发布GitHub Action与Sentry安全通信，您需要配置一个[组织令牌](https://sentry.io/orgredirect/organizations/:orgslug/settings/auth-tokens/)。


您还需要设置您的组织和项目slug，如果您使用的是自托管Sentry实例，请提供用于连接到Sentry的URL，通过`SENTRY_URL`。

<OrgAuthTokenNote />

```bash
SENTRY_AUTH_TOKEN=___ORG_AUTH_TOKEN___
SENTRY_ORG=___ORG_SLUG___
SENTRY_PROJECT=___PROJECT_SLUG___
# 对于自托管
# SENTRY_URL=https://my-sentry-url
```

我们建议将这些作为[加密密钥](https://docs.github.com/en/actions/security-for-github-actions/security-guides/using-secrets-in-github-actions)存储在您的仓库中。

要设置仓库密钥，点击您的仓库**设置①**，从侧边栏选择**密钥和变量②**并点击**Actions③**。点击**新建仓库密钥④**并添加密钥**`SENTRY_AUTH_TOKEN`⑤**、**`SENTRY_ORG`⑥**和**`SENTRY_PROJECT`⑦**。

![在GitHub上添加加密仓库密钥](../../../img/github_secrets.png)

## 发布

您可以通过设置`version`选项来创建新的Sentry发布。默认情况下，Sentry发布GitHub Action将使用触发GitHub工作流的GitHub sha。

```yml
- uses: actions/checkout@v4
  with:
    fetch-depth: 0

- name: 创建Sentry发布
  uses: getsentry/action-release@v3
  env:
    SENTRY_AUTH_TOKEN: ${{ secrets.SENTRY_AUTH_TOKEN }}
    SENTRY_ORG: ${{ secrets.SENTRY_ORG }}
    SENTRY_PROJECT: ${{ secrets.SENTRY_PROJECT }}
  with:
    environment: production
    version: 'v1.3.4'
```

## 源映射

如果您的应用程序是用JavaScript编写的，您可能会部署压缩的应用程序代码。为了使Sentry错误中的堆栈跟踪可读，您需要上传源映射到Sentry。

通过`sourcemaps`选项提供源映射的路径，以确保您的堆栈跟踪正确解压缩。
访问[什么是调试ID](/platforms/javascript/sourcemaps/troubleshooting_js/debug-ids/)如果您想了解更多。

```yml
- uses: actions/checkout@v4
  with:
    fetch-depth: 0

- name: 创建Sentry发布
  uses: getsentry/action-release@v3
  env:
    SENTRY_AUTH_TOKEN: ${{ secrets.SENTRY_AUTH_TOKEN }}
    SENTRY_ORG: ${{ secrets.SENTRY_ORG }}
    SENTRY_PROJECT: ${{ secrets.SENTRY_PROJECT }}
  with:
    environment: production
    sourcemaps: './dist'
```

<Alert>

  重要的是Sentry发布GitHub Actions在管道中的任何构建步骤之后但在部署之前运行，
  并部署在操作运行后构建的文件，以便Sentry可以为您正确解压缩堆栈跟踪。

</Alert>

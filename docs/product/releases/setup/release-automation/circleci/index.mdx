本指南将引导您完成在CircleCI中自动化Sentry发布管理和部署通知的过程。在CircleCI中部署后，您将能够将提交与发布关联。您还能够应用源映射以在Sentry中查看原始代码。

在开始之前，请确认您的Sentry项目已正确设置以跟踪提交元数据，方法是[安装仓库集成](/product/releases/setup/release-automation/)。安装完成后，并且您已添加了仓库，请返回本指南。如果您已安装了仓库集成，您就可以开始了。

## 创建Sentry内部集成

为了使CircleCI与Sentry安全通信，您需要创建一个新的内部集成。在Sentry中，导航到**设置 > 开发者设置 > 新建内部集成**。

为您的新集成命名（例如，"CircleCI部署集成"）并指定必要的权限。在这种情况下，您需要"发布"的管理员访问权限和"组织"的读取访问权限。

有关作用域和API端点的更多详细信息，请参见[API权限](/api/permissions/)的完整文档。

![内部集成权限视图。](./img/internal-integration-permissions.png)

点击"保存"，然后记录您的令牌，您将在下一节中需要它。

## 在CircleCI中设置环境变量

接下来，您需要几个环境变量来配置Sentry CLI：

- `SENTRY_AUTH_TOKEN` - 您的内部集成令牌
- `SENTRY_ORG` - 您的Sentry组织slug
- `SENTRY_PROJECT` - 您的Sentry项目slug

为了安全地访问您的内部集成令牌，请将其存储为[项目上的环境变量](https://circleci.com/docs/2.0/env-vars/#setting-an-environment-variable-in-a-project)：

1. 在CircleCI应用程序中，通过点击管道页面上的齿轮图标，或应用程序中其他页面上的三个点，转到您项目的设置。
2. 点击环境变量。
3. 通过点击"添加变量"按钮添加新变量，并输入`SENTRY_AUTH_TOKEN`作为名称，您的内部集成令牌作为值。

![CircleCI环境变量视图。](./img/project-settings-env-var.png)

其他变量不包含敏感信息，因此您将使用环境键在config.yml文件中将它们定义为环境变量。

## 创建发布并通知Sentry部署

为了自动化您的Sentry发布管理流程，您需要在部署步骤之后添加下面的`创建发布并通知Sentry部署`步骤，要么在新作业中，要么在现有部署作业的末尾：

```yaml
notify-sentry-deploy:
  executor:
  # 指定运行部署作业的执行器
  environment:
    SENTRY_ORG: sample-organization-slug
    SENTRY_PROJECT: sample-project-slug
    SENTRY_ENVIRONMENT: production
  steps:
    - checkout
    # ...
    - run:
        name: 创建发布并通知Sentry部署
        command: |
          curl -sL https://sentry.io/get-cli/ | bash
          export SENTRY_RELEASE=$(sentry-cli releases propose-version)
          sentry-cli releases new -p $SENTRY_PROJECT $SENTRY_RELEASE
          sentry-cli releases set-commits $SENTRY_RELEASE --auto
          sentry-cli sourcemaps upload --release $SENTRY_RELEASE path-to-sourcemaps-if-applicable
          sentry-cli releases finalize $SENTRY_RELEASE
          sentry-cli deploys new -e $SENTRY_ENVIRONMENT
```

有关上面代码段中发布管理概念的更多详细信息，请参见[发布管理](/cli/releases/)的完整文档。

**注意**：

- 如果您部署的项目不需要源映射，或者您已使用其他方法将源映射发送到Sentry，请省略`sourcemaps upload`行。
- 如果您无法安装仓库集成，请使用[创建发布端点](/product/releases/associate-commits/#using-the-api)发送提交元数据，或省略`set-commits`行。
- `sentry-cli releases propose-version`默认为被部署的提交的提交SHA。要将其设置为不同的版本，请将`SENTRY_RELEASE`修改为首选版本。

Sentry发布管理可以使用任何持续集成和交付(CI/CD)平台或自动化服务器来自动化。

## 安装仓库集成

通知Sentry发布信息并提供提交元数据最容易通过我们的仓库集成之一完成。如果您不希望Sentry连接到您的仓库，或者您使用的是不受支持的仓库提供商或VCS，我们也支持[手动流程](/product/releases/associate-commits/)来发送您的提交数据。

<Alert level="warning" title="所需权限">

要设置或配置集成，您需要是Sentry组织的所有者、经理或管理员。了解有关[Sentry中的角色](/organization/membership/)的更多信息。

如果您要链接GitHub仓库，您必须对仓库具有管理员或所有者权限，并且Sentry必须是您[GitHub账户设置](https://github.com/settings/applications)中的授权GitHub应用。

</Alert>

1. 安装适用的集成。可用的集成列在您的**组织设置 > 集成**下。

2. 点击实例旁边的"配置"按钮。

   ![](./img/releases-repo-integrations.png)

3. 在**仓库**面板中，点击"添加仓库"以添加您希望跟踪提交的任何仓库。然后，集成将发送关于推送到这些仓库的每个提交的元数据（如作者和更改的文件）到Sentry。

![](./img/releases-repo-add.png)

  <Alert>

如果您在添加集成时遇到问题，请[断开](https://sentry.io/account/settings/identities/)然后[重新连接](https://sentry.io/account/settings/social/associate/github/)您的GitHub身份。

  </Alert>

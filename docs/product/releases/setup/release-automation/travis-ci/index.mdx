本指南将引导您完成在 Travis CI 中自动化 Sentry 发布管理和部署通知的过程。在 Travis CI 中部署后，您将能够识别可能是新错误罪魁祸首的可疑提交。您还能够应用源映射以在 Sentry 中查看原始代码。

在开始之前，请确认您的 Sentry 项目已正确设置以跟踪提交元数据，方法是[安装仓库集成](/product/releases/setup/release-automation/)。安装完成后，添加您的仓库，然后返回本指南。如果您已经安装了仓库集成，就可以开始了。

## 创建 Sentry 内部集成

为了让 Travis CI 与 Sentry 安全通信，您需要创建一个新的内部集成。在 Sentry 中，导航到**设置 > 开发者设置 > 新建内部集成**。

为您的新集成命名（例如，"Travis CI 部署集成"）并指定必要的权限。在这种情况下，您需要"发布"的管理员访问权限和"组织"的读取访问权限。

有关范围和 API 端点的更多详细信息，请参阅[API 权限](/api/permissions/)的完整文档。

![内部集成权限视图。](./img/internal-integration-permissions.png)

点击"保存"，然后记录您的令牌，您将在下一节中需要它。

## 在 Travis CI 中设置环境变量

接下来，您需要几个环境变量来配置 Sentry CLI：

- `SENTRY_AUTH_TOKEN` - 您的内部集成令牌
- `SENTRY_ORG` - 您的 Sentry 组织标识符
- `SENTRY_PROJECT` - 您的 Sentry 项目标识符

要安全地访问您的内部集成令牌，请将其存储为[仓库上的环境变量](https://docs.travis-ci.com/user/environment-variables/#defining-variables-in-repository-settings)：

1. 在 Travis CI 应用程序中，通过点击"更多选项"菜单中的"设置"进入您的仓库设置。
2. 找到"环境变量"部分。
3. 添加一个新变量，名称为 `SENTRY_AUTH_TOKEN`，值为您的内部集成令牌。

![Travis CI 环境变量视图。](./img/travis-env-vars.png)

其他变量不包含敏感信息，因此您将使用 `env` 键在您的 .travis.yml 文件中将它们定义为环境变量。

## 创建发布并通知 Sentry 部署

要自动化您的 Sentry 发布管理过程，您需要在部署步骤后添加下面的`创建发布并通知 Sentry 部署`作业：

```yaml
jobs:
  include:
    # ...
    - name: 创建发布并通知 Sentry 部署
      env: SENTRY_ORG=sample-organization-slug SENTRY_PROJECT=sample-project-slug SENTRY_ENVIRONMENT=production
      script: |
        curl -sL https://sentry.io/get-cli/ | bash
        export SENTRY_RELEASE=$(sentry-cli releases propose-version)
        sentry-cli releases new -p $SENTRY_PROJECT $SENTRY_RELEASE
        sentry-cli releases set-commits $SENTRY_RELEASE --auto
        sentry-cli sourcemaps upload --release $SENTRY_RELEASE path-to-sourcemaps-if-applicable
        sentry-cli releases finalize $SENTRY_RELEASE
        sentry-cli deploys new -e $SENTRY_ENVIRONMENT
```

有关上述代码片段中发布管理概念的更多详细信息，请参阅[发布管理](/cli/releases/)的完整文档。

**注意**：

- 如果您部署的项目不需要源映射，或者您已经使用其他方法将源映射发送到 Sentry，请省略 `sourcemaps upload` 行。
- 如果您无法安装仓库集成，请使用[创建发布端点](/product/releases/associate-commits/#using-the-api)发送提交元数据，或省略 `set-commits` 行（`set-commits` 是可疑提交所必需的）。
- `sentry-cli releases propose-version` 默认为正在部署的提交的提交 SHA（推荐）。要将其设置为不同的版本，请将 `SENTRY_RELEASE` 修改为首选版本。

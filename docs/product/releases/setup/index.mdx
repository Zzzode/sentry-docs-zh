设置您的应用程序以完全向Sentry发送发布数据是一个多步骤过程，并且可以适应您组织的需求。

如果您使用我们的[发布自动化](/product/releases/setup/release-automation/)选项之一，它将自动化此过程的步骤3.2和4。如果您手动管理发布，则应遵循以下所有步骤。

## 1. 配置您的SDK

通过将您的SDK配置为提供发布标识符并启用会话跟踪，您可以监控[发布](/product/releases/)和[发布健康状况](/product/releases/health/)。查看您的特定平台的文档以[配置SDK](/platform-redirect/?next=/configuration/releases/)。

<Alert>

您必须在SDK中[配置环境](/platform-redirect/?next=/configuration/environments/)才能使用此功能。

</Alert>

## 2. 上传源映射或其他调试文件

在您的代码被处理的情况下——由webpack、Terser或C++编译器等工具打包、压缩、转译、编译或以其他方式混淆——源映射（或其他调试文件）对于Sentry以原始未转换形式在堆栈跟踪中显示源代码是必要的。在启用[可疑提交](#3-关联提交)之前，设置源映射也是必要的。查看我们的SDK文档，为您的特定平台查找源映射或其他调试文件设置说明（如果不在以下列表中）：

- [Android](/platforms/android/configuration/using-ndk/#symbolicate-stack-traces)
- [Flutter](/platforms/dart/guides/flutter/upload-debug/)
- [iOS](/platforms/apple/guides/ios/dsym/)
- [JavaScript](/platforms/javascript/sourcemaps/)
- [Node.js](/platforms/javascript/guides/node/sourcemaps/)

## 3. 关联提交

[将提交关联](/product/releases/associate-commits/)到发布允许您更容易地查明哪些更改可能对给定问题负责。要设置它们：

1. [安装仓库集成](/organization/integrations/source-code-mgmt/)，以便Sentry可以访问您的提交历史。（如果您不想使用集成，也可以通过我们的API手动上传提交数据。）
2. 配置您的发布管理脚本以[将提交与每个发布关联](/product/releases/associate-commits/)。如果您使用我们的[发布自动化](/product/releases/setup/release-automation/)选项之一，这将为您完成。

在[关联提交](/product/releases/associate-commits/)中了解更多关于跟踪提交的好处。

## 4. 通知Sentry

如果您不使用我们的[发布自动化](/product/releases/setup/release-automation/)选项之一，请在创建发布后通过发送额外请求让Sentry知道您已部署发布：

```bash
sentry-cli releases --org ORGANIZATION_SLUG deploys VERSION new -e ENVIRONMENT
```

<Alert>

确保您已正确配置[令牌](/cli/configuration/)。

</Alert>

通常，发布 = 部署，但如果您想在多个环境中部署发布，此步骤还允许您单独创建这些部署。

您也可以使用我们的[API](/api/releases/create-a-deploy/)来创建部署。

当您通知Sentry您已将发布部署到新环境时，我们可以自动向已提交到正在部署的发布的Sentry用户发送电子邮件。

如果[完成发布](/cli/releases/#finalizing-releases)尚未作为发布自动化过程的一部分包含在内，请确保这样做。有关更多信息，请查看我们的[CLI文档](/cli/releases/)。

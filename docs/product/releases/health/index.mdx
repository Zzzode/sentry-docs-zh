<Alert>

请注意，会话（如下所述）与[会话重播](/product/explore/session-replay/)（用户会话的视频式重现）不同。我们不收取发布健康状况会话的费用。

</Alert>

您可以通过观察用户采用情况、应用程序使用、[崩溃](#crashes)百分比和[会话数据](#sessions)来监控发布的健康状况。发布健康状况数据提供了关于崩溃和错误对用户体验影响的洞察，并通过发布详情、图表和过滤器揭示每个新问题的趋势。这些数据主要显示在[发布](/product/releases/)和[发布详情](/product/releases/release-details)页面上，但也反映在[sentry.io](https://sentry.io)的其他部分。

> 许多SDK在初始化SDK时自动管理[会话](#sessions)的开始和结束，但发布健康状况配置对于确保您接收到有用的数据至关重要，因此请查看我们关于[哪些SDK支持发布健康状况](/product/releases/setup/#release-health)的信息，以获取配置SDK的链接（如需要）。

一旦您配置了SDK，Sentry会将数据连接到您应用程序的特定发布和相关代码。本页面描述了一些关键的发布健康状况概念，以及您可以从每种发布健康状况数据中学到什么。

## 会话

Sentry用于监控健康状况的主要组件是会话，它代表用户和应用程序之间的交互。

Sentry区分两种会话：

- 用户模式/应用程序模式会话
- 服务器模式/请求模式会话

### 用户模式/应用程序模式会话

<ul>

<li style={{listStyleType:'none'}}>
  用户或应用程序模式会话从应用程序启动开始。
  或者，它从将已在后台运行的应用程序带回前台开始。在这种情况下，用户由一个或多个以下定义：唯一ID或UUID、用户名或电子邮件。
</li>

<br></br>

<li style={{listStyleType:'none'}}>
  此类会话随着应用程序的关闭或应用程序被发送到后台而结束。如果应用程序在后台停留少于30秒，我们不会再次启动会话。
  即使在后台也保持活动的应用程序（例如，音乐播放器）应该为后台进程手动跟踪会话。
</li>

</ul>

### 服务器模式/请求模式会话

<ul>

<li style={{listStyleType:'none'}}>
  服务器或请求模式会话大致对应于服务器设置中的HTTP请求或RPC调用。当服务器收到请求时会话开始，当服务器发送响应时终止。
</li>

<br></br>

<li style={{listStyleType:'none'}}>
  这些通常数量很高，因为每个会话对应一个请求。
</li>

</ul>

无论是应用程序模式还是会请求模式，会话都会提交到Sentry，以便您可以跟踪应用程序的使用和采用情况。当您的应用程序用户遇到崩溃、错误或异常退出时，会话将被相应标记，Sentry计算派生指标。这些指标包括指定时间范围内没有经历崩溃的用户数量等数据。

就项目运行的模式而言，Sentry能够自动检测它是否处于服务器设置中，因此应该以服务器模式/请求模式运行，或者它是否处于应用程序设置中，因此应该以用户模式/应用程序模式运行。

虽然上述会话描述涵盖了许多情况，但会话的定义可能因平台而异。例如，对于JavaScript应用程序，我们为每个页面加载和单页应用程序的每次导航更改创建会话。相比之下，后端应用程序没有像Web和移动应用程序那样明确定义的用户实体，因此对于这些应用程序，每个新的API请求代表一个新的会话。查看您的[特定平台](/platform-redirect/?next=/configuration/releases/%23sessions)文档以获取更多信息。

### 活跃会话/用户

_活跃会话_ 是在过去24小时内在应用程序中启动的会话。

_活跃用户_ 是在过去24小时内至少启动过一次应用程序的用户。

根据您的"显示"选择，**发布**页面的排序选项会改变。要按活跃会话排序发布，请在"显示"下拉菜单中选择"会话"，要按活跃用户排序，请选择"用户"。

![排序依据：活跃用户下拉菜单](./img/release_active_users.png)

## 崩溃

_崩溃_ 是应用程序出现显式未处理错误或硬崩溃的情况。您通常可以在[sentry.io](https://sentry.io)中查看捕获此事件的相应问题，并且不应包括未导致应用程序结束的错误。要在**发现**或**问题**页面中搜索崩溃事件，请按`error.unhandled:true`过滤。

### 无崩溃会话/用户

_无崩溃会话_ 数字是指定时间范围内未因应用程序崩溃而结束的会话的百分比。

_无崩溃用户_ 数字是指在指定时间段内未经历崩溃的不同用户的百分比。例如，如果用户尝试在您的应用程序上完成结账流程并且发生崩溃，这代表一个[崩溃用户](#crashed-users)。相比之下，如果用户能够成功完成结账，他们将被计为无崩溃用户。

您可以设置[崩溃率警报](/product/alerts/alert-types/#sessions-crash-rate-alerts)，以在会话或用户的无崩溃百分比低于特定阈值时通知您。

根据您的"显示"选择，**发布**页面的排序选项会改变。要按无崩溃会话排序发布，请在"显示"下拉菜单中选择"会话"，要按无崩溃用户排序，请选择"用户"。

![排序依据：无崩溃会话下拉菜单](./img/release_crash_free_sessions.png)

![排序依据：无崩溃用户下拉菜单](./img/release_crash_free_users.png)

<Alert>

请注意，崩溃数量预计不会与崩溃会话数量匹配，因为会话不受[入站过滤器](/concepts/data-management/filtering/)或[采样](/platform-redirect/?next=/configuration/sampling/)的影响。

</Alert>

### 崩溃用户

在指定时间范围内经历崩溃的用户数量。

## 发布采用

根据选择的显示选项，我们显示：

- 会话采用 - 特定发布在过去24小时内的会话数量。
- 基于用户的采用 - 特定发布在过去24小时内至少启动过一次应用程序的用户数量。

采用进度条计算为此发布中的会话/用户占项目中所有发布的百分比，在过去24小时内：

![发布页面上的采用条](./img/adoption_bar.png)

### 采用图表

<Alert>

此功能仅适用于移动或桌面设备可用的项目。

</Alert>

如果您选择配置了发布健康状况的项目，并且在所选时间段内有发布和用户/会话数据，则会显示采用图表。该图表为您提供项目中每个发布的采用百分比，随时间变化。

每个发布在图表中用一条线表示，您可以通过点击它导航到[发布详情](/product/releases/health/release-details/)。

![发布采用图表](./img/release_adoption_chart.png)

如果您没有选择项目或选择了多个项目，图表将不会显示。

### 采用阶段

_采用阶段_ 是您的发布相对于其他发布使用情况的高级概述。它每小时通过处理过去六小时的会话来计算：

![发布页面上的采用阶段标签。](./img/adoption-stage-label.png)

<Alert>

采用阶段标签仅针对移动项目显示，并且仅当您过滤到单个环境时。

</Alert>

对于每个发布，发布的会话百分比是针对同一环境和项目中的所有会话计算的。然后根据10%的阈值将其标记为以下三个阶段之一：

- 已采用 - 与所有会话相比，发布的会话百分比高于阈值。也就是说，该发布在过去六小时内被看到在所有发布中的10%或更多会话中。
- 低采用 - 与所有会话相比，发布的会话百分比低于阈值。也就是说，该发布在过去六小时内被看到在所有发布中的少于10%会话中。
- 已替换 - 发布之前被标记为"已采用"，但不再满足最低阈值。达到阈值的"已替换"发布将再次被标记为"已采用"。

## 会话状态

会话可以具有以下状态之一：

### 异常

应用程序超时、冻结或被操作系统强制退出。通常没有相应的Sentry问题，因为这是被动操作。

### 崩溃

应用程序出现显式未处理错误或硬[崩溃](#crash)。

### 错误

应用程序正常关闭，但会话经历了处理的错误。通常，您可以在sentry.io中查看任何问题/错误。

### 健康

会话正常结束，在其生命周期内没有发生错误。

会话状态可能因平台而略有不同定义，并非所有平台都使用上述所有状态。查看您的[特定平台](/platform-redirect/?next=/configuration/releases/%23sessions)文档以获取更多信息。

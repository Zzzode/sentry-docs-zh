Sentry Prevent AI 是由 Sentry 开发的生成式 AI 助手。它帮助您为未覆盖的代码生成新测试并审查您的代码更改，在合并拉取请求之前预测错误并提供改进建议。

## 开始使用

要在您的 GitHub 组织或特定仓库中启用 Sentry Prevent AI，您需要安装 [Seer by Sentry 应用](https://github.com/apps/seer-by-sentry)。这允许助手分析拉取请求、提供见解并编写测试。**您还必须启用 [Sentry Github 集成](/organization/integrations/source-code-mgmt/github/)。**

1. 在您的 GitHub 组织或特定仓库中安装 [Seer by Sentry 应用](https://github.com/apps/seer-by-sentry)。
2. 在您的 Sentry [组织设置](https://sentry.io/orgredirect/settings/:orgslug/)中启用这些必需设置：
   - `显示生成式 AI 功能`
   - `启用 PR 审查和测试生成`

如果您不是管理员，请与您的组织管理员或所有者分享 Seer by Sentry GitHub 应用集成链接以进行安装。

## Sentry Prevent AI 命令

安装应用后，Sentry Prevent AI 可以通过三种方式帮助您：

1. **错误预测** - 当您创建拉取请求并将其设置为`准备审查`时，Sentry Prevent AI 将检查您代码中的错误。如果未发现错误，您将在 PR 描述中看到 🎉 表情符号作为反应。否则，Sentry Prevent AI 将向您的 PR 添加评论。

2. **`@sentry review`** - 在 PR 评论中使用此命令，助手将审查 PR 并预测错误，以及提出建议。

3. **`@sentry generate-test`** - 在 PR 评论中使用此命令，助手将为您的 PR 生成测试。

添加评论后，助手将回复，确认请求。对于代码审查，您将看到添加到您的 PR 的评论。对于测试生成，您将看到带有指向生成测试链接的评论。

## 常见问题

- **Sentry Prevent AI 需要访问哪些数据才能使 AI 系统运行，以及哪些信息会发送给第三方 AI 提供商？**

  Sentry Prevent AI 需要访问您的拉取请求，包括 PR 元数据、仓库信息、文件名、目录结构和代码差异。只有文件名、代码差异和 PR 描述会发送给 AI 提供商进行分析。

- **Sentry Prevent AI 是否在 GitHub 之外运行，它是否在后台运行？**

Sentry Prevent AI 仅在 GitHub 上运行。它将在 PR 标记为`准备审查`时运行一次，否则仅在通过评论触发时运行。

  您可以在此处了解有关 AI 隐私和安全的更多信息[此处](/product/ai-in-sentry/ai-privacy-and-security/)。

- **为什么 `@sentry` 命令不起作用？**

  如果您尚未启用，Sentry Prevent AI 将回复一条消息，以在您的[组织设置](https://sentry.io/orgredirect/settings/:orgslug/)中启用`显示生成式 AI 功能`和`启用 PR 审查和测试生成`设置。启用后，请再次尝试 `@sentry review` 或 `@sentry generate-test` 以运行 Sentry Prevent AI。

- **Prevent AI 功能是否在欧盟区域可用？**

  目前不可用，但我们正在努力。同时，您可以创建一个新的 Sentry 组织，然后按照上述设置说明进行操作。

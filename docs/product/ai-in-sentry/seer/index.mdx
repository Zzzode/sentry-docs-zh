Seer 是 Sentry 的 AI 调试代理。它利用 Sentry 丰富的上下文（问题详情、跟踪数据、日志和性能分析）来帮助您更快地排查和修复错误以及性能问题。

## Seer 功能

Seer 提供三种主要功能来增强您的调试体验：

- [**问题修复**](#issue-fix)：一个协作代理，擅长根本原因分析和解决问题
- [**问题扫描**](#issue-scan)：自动扫描传入的问题，为您的警报提供额外上下文并自动化分类
- [**自动化**](#automation)：将问题修复和扫描结合成一个强大的工作流：实时识别可操作的问题并生成高质量的修复方案，完全自动化

### 问题修复

Seer 将代码中的所有相关上下文与 Sentry 的遥测数据相结合，提供一流的调试体验。该工作流包含三个步骤：

1. 使用 Sentry 的上下文和您的代码库对问题进行根本原因分析
2. 基于分析结果识别解决方案
3. 生成实现解决方案的代码补丁并创建 PR

### 问题扫描

Seer 能够自动分析 Sentry 摄取的问题。此初步审查用于：

- 生成"可操作性"评分，指示特定问题是否可以通过代码更改修复
- 用有关问题的有用分析增强问题警报
- 在问题详情页面添加对潜在问题的初步猜测

可操作性评分在问题反馈中通过 Seer 图标表示，指示可能是简单修复的问题。

### 自动化

您可以使用自动化来触发问题扫描和修复。您可以配置三种自动化设置：

1. **自动问题扫描**：启用 Seer 的问题扫描功能
2. **自动问题修复**：允许您自定义 Seer 采用的自动化程度。例如，将其设置为"高可操作性及以上"将自动为您所有高可操作性的问题触发问题修复。
3. **自动修复的停止点**：Seer 允许您自定义自动化的程度。默认情况下，它在解决方案步骤停止，但您可以选择让它一直进行到代码更改，甚至自动生成 PR。

## Seer 使用的数据

Seer 是一个强大的调试代理，可以访问各种数据源和工具。在调试问题时，它可能会检查：

- **问题上下文**：来自您[问题](/product/issues/)的错误消息、堆栈跟踪和事件元数据
- **跟踪数据**：分布式[跟踪](/concepts/key-terms/tracing/#whats-a-trace)和跨度信息
- **日志**：来自您应用程序的结构化[日志](/product/explore/logs/)（测试版）
- **您的代码库**：来自链接 GitHub 仓库的相关代码，支持分布式服务的多个仓库
- **性能数据**：性能分析和性能指标
- **交互反馈**：您在过程中的输入和指导

## 开始使用 Seer

要开始使用 Seer 的功能：

1. **启用生成式 AI 功能**：确保在您的[组织设置](https://sentry.io/orgredirect/organizations/:orgslug/settings/#hideAiFeatures)中启用了 AI 功能
2. **设置 GitHub 集成**：对于问题修复功能，[连接您的 GitHub 仓库](/organization/integrations/source-code-mgmt/github/)
3. **配置设置**：在项目设置中的 `Seer Automation` 下自定义 Seer 的行为。Seer 默认使用这些设置启用：
    - **自动问题扫描**：已启用
    - **自动问题修复**：为高可操作性及以上问题启用

    > **注意**：您可以在[统计和使用情况选项卡](https://sentry.io/settings/stats/)中监控您的 Seer 使用情况，并可以在此处了解有关管理 Seer 预算的更多信息[此处](/pricing/quotas/manage-seer-budget/)。
4. **开始使用**：在问题详情页面中查找 Seer 选项

## 隐私和安全

Sentry 为您的数据隐私和安全提供了强有力的保证。简要说明：

- 默认情况下，未经您的许可，Sentry 不会使用您的数据训练生成式 AI 模型。
- AI 生成的输出仅向您和您账户中的其他授权用户显示。

您可以在此处了解有关我们数据隐私实践的更多信息[安全和法律文档](/security-legal-pii/security/ai-ml-policy/#use-of-identifying-data-for-generative-ai-features)。

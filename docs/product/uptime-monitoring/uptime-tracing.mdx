Sentry 的正常运行时间监控使用[分布式跟踪](/product/tracing/#whats-distributed-tracing)来跟踪正常运行时间检查期间请求和操作的流程和时间。这有助于您通过连接相关错误和性能数据快速找到停机的根本原因。

您可以跟踪错误以识别和诊断崩溃或异常等问题。这对于所有正常运行时间警报都会自动启用。您还可以跟踪跨度以监控操作的性能和流程，例如 API 调用或数据库查询。跨度跟踪默认情况下是禁用的，但可以通过在正常运行时间警报设置中允许采样来启用。错误关注问题，而跨度关注性能。

## 正常运行时间跟踪如何工作

Sentry 正常运行时间跟踪会自动将 `sentry-trace` 标头附加到正常运行时间警报设置中配置的 URL 发出的每个传出请求。此标头传播跟踪标识符。

如果为处理传入正常运行时间检查请求的应用程序配置了 Sentry 支持的某个后端 SDK，则跟踪将继续通过您的应用程序。了解更多关于[分布式跟踪如何工作](/product/tracing/)的信息。

正常运行时间检查请求示例：

```HTTP
GET /example-uptime-endpoint HTTP/1.1
Host: sentry.io
User-Agent: SentryUptimeBot/1.0 (+http://docs.sentry.io/product/uptime-monitoring/)
Sentry-Trace: 32d4011600324838afcd666edadc1d09-8d5ca7419a02ca36
```

## 跟踪错误

错误跟踪默认情况下为正常运行时间检查启用。当检测到停机时，会创建一个[正常运行时间问题](/product/issues/issue-details/uptime-issues/)。然后您可以转到 Sentry 的[**跟踪浏览器**](/product/explore/traces/)页面查看任何相关错误。

由于正常运行时间请求不会覆盖您 SDK 的错误采样率，如果该率设置为低于 1，某些错误可能不会出现在跟踪中。

<Expandable permalink title="禁用正常运行时间错误跟踪">
  要为正常运行时间检查禁用错误跟踪，请在您的 SDK 中配置一个[发送前](/platform-redirect/?next=/configuration/filtering/)过滤器以忽略来自 Sentry 的 `User-Agent` 的错误。以下是一个示例：

```typescript {tabTitle: Node.js Express} {filename: instrument.mjs}
import * as Sentry from "@sentry/node";

Sentry.init({
  dsn: "___PUBLIC_DSN___",
  // Node.js Express 应用程序的过滤示例
  beforeSend(event) {
    const userAgent = event.request?.headers?.["user-agent"];

    // 忽略来自 SentryUptimeBot 的请求中捕获的事件
    if (userAgent && userAgent.includes("SentryUptimeBot")) {
      return null;
    }

    // 处理其他事件
    return event;
  },
});
```

</Expandable>

## 跟踪跨度

与错误跟踪不同，跨度跟踪**默认情况下**为正常运行时间检查禁用，但可以通过在正常运行时间警报设置中允许采样来启用。启用跨度跟踪可能会有所帮助，因为它提供了正常运行时间检查期间请求和操作的时间和流程的详细视图，这对于诊断超时或性能问题特别有用。

### 为跨度启用正常运行时间跟踪

要启用跨度跟踪，请使用"允许采样"选项修改您的正常运行时间警报设置。这将确保 Sentry 将采样决定推迟到您的 SDK，它将遵循您配置的跟踪采样率。

由于正常运行时间请求不会覆盖您 SDK 的错误采样率，如果该率设置为低于 1，某些错误可能不会出现在跟踪中。

![正常运行时间警报允许采样配置](./img/uptime-allow-sampling.png)

### 自定义采样

为了确保正常运行时间检查的所有跨度都被采样，即使您的 SDK 的跟踪采样率低于 1，您也可以配置一个[采样函数](/platform-redirect/?next=/configuration/sampling/)。以下是一个示例：

```typescript {tabTitle: Node.js Express} {filename: instrument.mjs}
import * as Sentry from "@sentry/node";

Sentry.init({
  dsn: "___PUBLIC_DSN___",
  // Node.js Express 应用程序的自定义跟踪器函数
  tracesSampler: ({ name, attributes, parentSampled }) => {
    const userAgent = attributes?.["http.user_agent"];

    if (
      typeof userAgent === "string" &&
      userAgent.includes("SentryUptimeBot")
    ) {
      // 对来自 SentryUptimeBot 的跨度进行 100% 采样
      return 1;
    }

    // 对其他跨度进行 50% 采样
    return 0.5;
  },
});
```

## 计费注意事项

正常运行时间检查期间捕获的错误和跨度在 Sentry 中[按常规事件计费](https://sentry.io/pricing/)。请谨慎配置采样以管理成本。

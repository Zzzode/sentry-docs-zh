Sentry 的正常运行时间监控让您轻松监控 Web 服务的可用性和可靠性。启用后，它会持续跟踪配置的 URL，提供即时警报和洞察，以快速识别停机时间并解决问题。

通过利用[分布式跟踪](/concepts/key-terms/tracing/distributed-tracing/)，Sentry 使您能够精确定位正常运行时间检查期间发生的任何错误，简化分类并加速根本原因分析。这有助于您提高 Web 服务的可靠性和正常运行时间。

## 设置

正常运行时间是[自动配置](/product/uptime-monitoring/automatic-detection/)的，作为错误数据中所有 URL 中最常遇到的主机名的新警报，确保从一开始就持续监控您最关键的主机名。

您还可以[创建正常运行时间监控警报](/product/alerts/create-alerts/)用于特定 URL。它们完全可以自定义，包括 HTTP 方法、标头和正文等请求详细信息。

## 正常运行时间检查标准

我们的正常运行时间监控系统通过以定期、预配置的间隔执行 HTTP 请求来验证您的 URL 的可用性。要使 URL 被视为正常运行，响应必须满足以下标准：

1. **成功响应（2xx 状态码）：**
   URL 必须返回 200–299 范围内的 HTTP 状态码，表示请求成功。
2. **自动处理重定向（3xx 状态码）：** Sentry 将跟踪返回 300–399 范围内 HTTP 状态码的 URL 的重定向，并验证最终目标 URL 返回成功响应。这确保重定向不会错误触发停机警报。
3. **超时设置：** 每个请求都有 10 秒的超时阈值。
   如果服务器在此期间内未响应，检查将被标记为失败，
   表示潜在的停机时间或性能问题。
4. **DNS 问题检测：** 我们的监控还包括 DNS 解析问题的检测。
   如果检测到 DNS 问题，检查将被标记为失败，
   允许您解决底层连接问题。

### 正常运行时间检查失败

正常运行时间警报持续监控配置的 URL，并定义上述标准。如果发生故障，
将创建一个新的[正常运行时间问题](/product/issues/issue-details/uptime-issues/)，包括有关失败检查和相关错误的详细信息。

为了防止由临时网络问题引起的错误警报，**只有在初始检测到停机后连续三次失败后才会生成问题**。此外，正常运行时间检查以轮询方式从各种地理位置执行。这确保每次失败的检查来自不同的区域，减少了由于本地化网络故障导致的误报可能性。

_在极少数情况下，Sentry 无法执行预定的正常运行时间检查（例如在停机期间），检查状态将被标记为"未知"。_

## 通知

要开始获取新停机问题的通知，[配置问题警报](/product/alerts/create-alerts/issue-alert-config/)并选择问题类别"中断"。然后选择您希望如何被通知（通过电子邮件、Slack 等）。

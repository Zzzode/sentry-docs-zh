## 验证防火墙配置

某些托管平台可能会阻止来自 Sentry 正常运行时间机器人的传入请求，错误地触发正常运行时间警报。我们建议验证您的防火墙配置，确保允许来自 Sentry 的传入请求。

如果您需要配置防火墙允许列表以包括 Sentry 的正常运行时间机器人，我们建议根据我们的 `User-Agent` 进行检查，因为我们的 IP 地址可能会在不通知的情况下更改。

### 用户代理

我们的正常运行时间检查请求使用以下 `User-Agent`：

```
SentryUptimeBot/1.0 (+http://docs.sentry.io/product/uptime-monitoring/)
```

### IP 地址

有关正常运行时间检查使用的 IP 地址的完整列表，请参阅 [IP 范围](/security-legal-pii/security/ip-ranges/#uptime-monitoring)。

## 验证问题警报是否匹配停机问题

正常运行时间警报会创建停机问题。如果您在检测到停机时未收到通知，请确保您已正确[配置了问题警报](/product/alerts/create-alerts/issue-alert-config/)，问题类别为"正常运行时间"。

## 概述

### 什么是采样率？

将 Sentry 添加到您的应用程序可以为您提供大量有关错误和性能的宝贵信息，否则您将无法获得这些信息。信息越多越好 - 只要它是正确的信息，并且数量合理。您可以使用采样率仅捕获一定百分比的错误和跟踪等事件。

### 为什么不捕获所有事件？

我们出于两个原因建议对事务进行采样：

1. 捕获单个跟踪涉及的开销很小，但为每个页面加载或 API 请求捕获跟踪可能会给您的系统增加不必要的负载。

1. 启用采样允许您更好地管理发送到 Sentry 的事件数量，以便您可以根据组织的需求和预算调整数量。

选择平衡数据准确性与性能和存储问题的采样率。您应该旨在收集足够的数据以获得有意义的见解，而不会使资源过载。如果不确定，请从低比率开始，随着更好地了解流量模式逐渐增加。

## 采样率选项

<Alert>
以下部分选项并非在每个 SDK 中都可用；查看我们的特定于平台的文档以获取更多信息。
</Alert>

### 错误事件

- **SampleRate** - 配置错误事件的采样率，范围为 0.0 到 1.0。默认值为 1.0，表示将发送 100% 的错误事件。如果设置为 0.1，则仅发送 10% 的错误事件。事件是随机选择的。

### 跟踪

- **tracesSampleRate** - 一个介于 `0` 和 `1` 之间的数字，控制给定事务发送到 Sentry 的百分比机会。（`0` 表示 0%，而 `1` 表示 100%。）同样适用于应用程序中创建的所有事务。必须定义此选项或 `tracesSampler` 才能启用跟踪。

- **tracesSampler** - 负责确定给定事务发送到 Sentry 的百分比机会的函数。它将自动传递有关事务及其创建上下文的信息，并且必须返回介于 0（发送机会为 0%）和 1（发送机会为 100%）之间的数字。还可以通过为不需要的事务返回 0 来用于过滤事务。必须定义此选项或 `tracesSampleRate` 才能启用跟踪。

在 Sentry 中了解有关[跟踪](/product/tracing/)的更多信息。

### 会话重放

- **replaysSessionSampleRate** - 立即开始记录并持续整个用户会话的重放的采样率。1.0 收集所有重放，0 不收集任何重放。

- **replaysOnErrorSampleRate** - 发生错误时记录的重放的采样率。这种类型的重放将记录错误前最多一分钟的事件，并继续记录直到会话结束。1.0 捕获所有出现错误的会话，0 不捕获任何会话。

在 Sentry 中了解有关[会话重放](/product/explore/session-replay/)的更多信息。

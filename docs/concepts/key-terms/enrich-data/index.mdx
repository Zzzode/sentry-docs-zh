Sentry SDK实时捕获应用程序中的错误、异常、崩溃、事务以及通常任何出错的内容。
一旦捕获，SDK会使用来自应用程序运行时的上下文信息来丰富数据，并将所有这些数据作为**事件**发送到您的Sentry账户。

虽然Sentry默认会添加大量此类上下文数据，但我们强烈建议您通过SDK添加自定义数据。多个Sentry工作流程和工具可以基于这些数据来配置和微调，因此根据您的工作方式调整它们对于从Sentry获取更多价值非常有帮助。

## 数据类型

SDK允许您设置各种类型的数据，然后将其附加到应用程序中发生的每个事件。通常，这些数据类型包括：

1. **非结构化数据** - 包括额外数据，如`面包屑`、`额外数据`和`自定义上下文`。虽然这些字段**不可搜索**，但它们用信息丰富了您的事件，这些信息将帮助您调试和解决相关问题。

2. **结构化数据** - 键值对，**已索引且可搜索**，允许您跨项目**搜索、过滤和查询所有事件**，以及帮助您调试特定问题。这些包括：

   - `自定义标签` - 一种在错误数据中反映适用于您的应用程序、开发过程或业务的关键特征的方法。这些可能包括应用程序的变体和风格、用户类型、租户ID、付款计划等。
   - `发布版本`和`环境` - 作为SDK配置选项可用的预定义数据字段。
   - `用户上下文` - 由一组预定义数据字段组成。设置其中任何一个（在适用时）允许Sentry构建用户身份并定义用户唯一性。

## 使用您的数据

添加对您的应用程序、用户和业务逻辑独特的自定义**结构化数据**可以丰富您的数据，并为每个事件提供有价值的上下文。您还可以使用相同的数据来自定义Sentry工作流程，并使它们适应您的精确需求。让我们看看这些主要工作流程：

### 警报规则

有针对性的警报是在应用程序中断时集中注意力的关键，并准确通知能够修复它的人。您向事件添加的数据越多，您获得的灵活性就越大，可以创建精确的警报。以下数据类型可以通过添加相关条件来触发警报，从而纳入您的警报规则：

- **用户上下文**：一旦某个特定问题影响了定义阈值数量的唯一用户。
- **自定义标签**：当事件包含与您的定义值匹配的自定义标签值时。
- **环境**：当事件在特定环境中发生时。

这些在问题和指标警报中都很有用。例如：

1. **问题警报**：以下规则将在**某个问题**具有_错误或致命_级别，源自我的_EMEA_发布版本在我的_prod_环境中，在_一小时内_影响超过_50个企业_最终用户时触发Slack消息：
   ![示例规则的问题警报配置。](./img/002.png)

2. **指标警报**：以下规则将在_一小时内_超过_50个企业_最终用户受到_警告_级别错误（**来自多个问题**）的影响，源自我的_EMEA_发布版本在我的_prod_环境中时触发Slack消息：

### 过滤、搜索和排序问题

在**问题**页面上，自定义数据允许您：

- 使用环境过滤器下拉菜单根据应用程序的`环境`过滤问题。
- 按（唯一）受影响`用户`的数量排序问题。
- 在问题搜索栏中通过您的`自定义标签`、`用户上下文`属性和`发布版本`ID搜索问题。

### 发现

发现是一个强大的查询引擎，它建立在您的数据之上，允许您揭示整个系统运行状况中的模式、异常和更深入的见解。考虑到这一点，您向事件添加的（自定义）数据越多，您在构建查询时获得的灵活性就越大，这些查询通过_标签_解决您的环境、发布版本、自定义开发和业务特征，以及受这些事件影响的最终用户。

### 所有权规则

所有权规则允许您通过定义项目设置中的规则或使用代码所有者功能，将电子邮件通知定位到项目团队成员并自动将问题分配给他们。所有权规则可以基于错误的来源`URL`、错误堆栈跟踪中文件的`路径`或错误的`标签`值来定义。

![问题所有者标签规则](./img/005.png)

代码所有者功能允许您导入GitHub或GitLab的CODEOWNERS文件，然后我们根据这些文件路径分配问题。

您将能够通过精确定位通知给正确的人员并自动将新问题分配给其适当所有者来减少噪音。您可以在**[项目] > 设置 > 所有权规则**中配置所有权规则。在[所有权规则](/product/issues/ownership-rules/)中了解更多信息。

### 发布工作流程

在SDK上配置发布版本ID会将应用程序运行时中发生的每个错误与代码的特定发布版本相关联。这样，您就知道错误何时被引入到您的源代码中。它还解锁了这些强大的功能和工作流程：

1. **提交跟踪** - 有了发布版本，Sentry可以跟踪哪些提交参与了发布以及进行这些提交的作者。
2. **使用发布版本解决** - 当您将问题标记为在特定发布版本中已解决时，在先前发布版本中发生的该问题的新事件将被忽略，并且不会触发任何通知。
3. **回归通知** - 如果您将问题标记为在发布版本中已解决，而新事件仍然在该相同发布版本或任何未来发布版本中发生，Sentry将认为这是一个回归并会通知您。
4. **发布健康** - Sentry可以通过观察用户采用率、无崩溃用户和会话等各种指标来监控您的发布版本的[健康](/product/releases/health/)。（并非所有发布健康功能都适用于支持发布健康的所有SDK。）

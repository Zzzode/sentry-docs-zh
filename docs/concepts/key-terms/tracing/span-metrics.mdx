跨度指标使您能够将用户定义的属性附加到跨度，这可能包括应用程序指标或应用程序追踪中的重要调试上下文。这种方法通过将属性直接连接到生成它们的操作来提供上下文丰富的性能监控。

这些属性允许您使用代表各种类型测量数据的属性来丰富追踪跨度：

- 性能属性（内存使用、处理时间、延迟）
- 业务指标（交易价值、用户参与率）
- 技术指标（队列深度、缓存命中率）
- 调试上下文（输入参数、进程状态）

通过将属性直接附加到跨度，您可以创建执行路径及其相关性能数据的统一视图。

```javascript
// 为数据库跨度添加性能指标
// 带有动态属性的简单数据库查询
Sentry.startSpan(
  {
    name: "数据库查询",
    op: "db.query",
  },
  (span) => {
    // 执行查询并将结果添加到跨度
    const result = executeQuery("SELECT * FROM users WHERE active = true");

    // 使用结果数据设置属性
    span.setAttribute("db.rows_returned", result.length);
    span.setAttribute("db.execution_time_ms", result.executionTime);

    return result;
  }
);
```

## 配置跨度属性的好处

### 上下文可观察性

跨度属性提供将执行流程与开发人员关心的特定指标或性能测量连接起来的上下文数据。在调查问题时，您可以在单个视图中查看发生了什么（追踪）和性能特征。

### 统一遥测

通过将这些属性集成到追踪中，您可以维护单个遥测管道，而不是为追踪和指标管理单独的系统，从而实现简化的检测和更高效的监控。

```javascript
// 为支付处理跨度添加业务上下文
Sentry.startSpan(
  {
    name: "处理支付",
    op: "payment.process",
    attributes: {
      "payment.amount": 99.99,
      "payment.currency": "USD",
      "payment.method": "credit_card",
      "customer.type": "returning",
    },
  },
  async () => {
    // 支付处理实现
  }
);
```

### 加速故障排除

当出现性能问题时，这些属性提供必要的上下文来快速识别瓶颈。例如，在调查缓慢的结账流程时，您可以立即看到哪个特定组件（支付网关、数据库查询、第三方API）导致了延迟。

### 技术-业务关联

跨度属性通过将响应时间或错误率等技术性能数据直接连接到转化率或收入等业务指标来实现技术性能数据与业务结果之间的关联。

## 实现方法

有两种主要方法可以在跨度上实现属性：

### 1. 增强现有跨度

使用附加属性增强自动创建或手动定义的跨度：

```javascript
// 为现有的文件上传跨度添加指标
const span = Sentry.getActiveSpan();
if (span) {
  // 用户上下文
  span.setAttribute("user.subscription_tier", "premium");

  // 单个操作中的多个指标
  span.setAttributes({
    "memory.heap_used": 1024000,
    "processing.total_steps": 5,
  });
}
```

### 2. 创建专用指标跨度

专门为分组相关属性或指标创建跨度，特别适用于复杂操作：

```javascript
// 创建用于监控外部API使用的跨度
Sentry.startSpan(
  {
    name: "第三方API使用",
    op: "external.api",
    attributes: {
      // 性能指标
      "api.response_time_ms": 245,

      // 上下文数据
      "feature.using_api": "image_recognition",
      "user.plan": "enterprise",
    },
  },
  async () => {
    // API调用实现
  }
);
```

## 实现用例

以下示例演示了如何将属性应用于常见应用程序场景：

### 用户界面性能

监控与用户体验相关的客户端性能指标：

```javascript
// UI性能监控
Sentry.startSpan(
  {
    name: "页面交互",
    op: "ui.interaction",
    attributes: {
      "ui.first_input_delay_ms": 24,
      "ui.time_to_interactive_ms": 320,
      "ui.frames_dropped": 0,
      "user.device_type": "mobile",
      "feature.being_used": "image_carousel",
    },
  },
  async () => {
    // UI交互处理
  }
);
```

### 数据库操作监控

跟踪数据库性能特征及其对应用程序行为的影响：

```javascript
// 数据库查询监控
Sentry.startSpan(
  {
    name: "产品搜索查询",
    op: "db.query",
    attributes: {
      "db.query_type": "SELECT",
      "db.table": "products",
      "db.execution_time_ms": 145,
      "db.rows_returned": 87,
      "db.index_used": "product_category_idx",
      "business.search_term": "winter jacket",
      "business.search_filters_applied": 3,
    },
  },
  async () => {
    // 数据库查询执行
  }
);
```

### 文件处理操作

监控应用程序堆栈中的文件处理操作：

```javascript
// 文件处理监控
Sentry.startSpan(
  {
    name: "处理上传的图像",
    op: "file.process",
    attributes: {
      // 技术指标
      "file.size_bytes": 2500000,
      "file.type": "image/jpeg",

      // 处理指标
      "processing.steps_completed": ["virus_scan", "resize", "compress"],
      "processing.total_time_ms": 850,

      // 上下文数据
      "feature.using_upload": "user_avatar",
      "subscription.allows_hd": true,
    },
  },
  async () => {
    // 图像处理实现
  }
);
```

### 外部服务集成

监控第三方服务交互的性能和可靠性：

```javascript
// 支付网关监控
Sentry.startSpan(
  {
    name: "支付网关",
    op: "payment.gateway",
    attributes: {
      // 性能指标
      "gateway.response_time_ms": 980,
      "gateway.retry_count": 0,

      // 交易数据
      "order.total_amount": 159.99,
      "order.items_count": 3,

      // 服务指标
      "gateway.fee_amount": 4.5,
      "gateway.fee_percent": 0.029,
    },
  },
  async () => {
    // 支付网关集成
  }
);
```

## 实施指南

### 最佳实践

#### 指标选择和设计

选择提供可操作洞察的指标进行调试或性能监控。考虑哪些数据点将有助于诊断问题或对您的应用程序做出明智决策。

#### 命名约定

遵循`category.metric_name`格式维护一致的命名模式，以确保指标在您的组织中是可发现和可理解的。

#### 数据类型选择

为您的指标选择适当的数据类型：

- 测量值的数值（`response_time_ms`: 250）
- 状态指标的布尔值（`cache.hit`: true）
- 分类数据的字符串值（`user.subscription`: 'premium'）
- 多值数据的数组（`processing.steps_completed`: ['download', 'process', 'upload']）

#### 性能考虑

注意收集指标的性能影响，特别是对于高容量操作：

- 考虑为高频率操作实施采样
- 优先考虑具有最高分析价值的指标
- 避免冗余或密切相关的指标

## 从独立指标迁移

如果您当前正在使用Sentry的独立指标产品，迁移到跨度属性具有几个优势：

- **增强的上下文**：属性直接连接到生成它们的操作
- **实现效率**：追踪和指标的单一一致API
- **改进的关联**：属性、追踪和错误之间的直接关联

迁移过程：

1. 识别您当前的指标检测点
2. 在您的应用程序中找到相应的跨度
3. 将您的指标转移到跨度属性
4. 更新任何仪表板或警报以引用新的指标源

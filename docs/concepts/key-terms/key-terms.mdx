以下部分术语在[关键功能](#关键功能)部分中具有相同名称（大写）的相应功能。

## 关键术语

- **警报** - 警报通过在满足某些警报规则条件时向您发送通知，让您实时了解代码中的问题。有多种类型的警报可用，具有可自定义的阈值和集成。

- **附件** - 存储与错误事件相关的附加文件，例如配置或日志文件。

- **数据** - 您发送到 Sentry 的任何内容。这包括事件（错误或事务）、附件和事件元数据。

- **分布式跟踪** - 当跟踪包括多个服务中的工作时，它被称为[分布式跟踪](/concepts/key-terms/tracing/distributed-tracing/)，因为跟踪分布在这些服务中。

- **DSN** - 数据源名称。DSN 告诉 Sentry SDK 将事件发送到哪里，以便事件与正确的项目关联。创建项目时，Sentry 会自动为您分配一个 DSN。在我们的完整 [DSN 文档](/product/sentry-basics/dsn-explainer/)中了解更多信息。

- **环境** - `environment` 是您可以添加到 SDK 的 Sentry 支持的标签，旨在指代代码部署的命名约定，例如 _开发_、_测试_、_暂存_ 或 _生产_。环境可帮助您更好地过滤问题、事务等。在我们的完整[环境文档](/product/sentry-basics/environments/)中了解更多信息。

- **错误** - 什么算作错误因平台而异，但总的来说，如果有看起来像异常的东西，它可以作为 Sentry 中的错误捕获。Sentry 自动捕获错误、未捕获的异常和未处理的拒绝，以及取决于平台的其他类型的错误。

- **事件** - 错误或事务。

- **问题** - 问题是相似错误或性能问题的分组。每个事件都有一组称为其指纹的特征，Sentry 使用这些特征对它们进行分组。例如，当错误事件由代码的相同部分触发时，Sentry 将它们分组在一起。将事件分组到问题中可以让您了解问题发生的频率以及它影响了多少用户。

- **性能监控** - 性能监控是跟踪应用程序性能并测量指标的行为，例如发送了多少事务以及给定事务的所有实例的平均响应时间。为此，Sentry 捕获由事务和[跨度](/product/sentry-basics/tracing/distributed-tracing/#traces-transactions-and-spans)组成的[分布式跟踪](/product/sentry-basics/tracing/distributed-tracing/)，以测量单个服务以及这些服务内的操作。

- **项目** - 项目代表您在 Sentry 中的服务或应用程序。您可以为应用程序中使用的特定语言或框架创建项目。例如，您可能为 API 服务器和前端客户端设置了单独的项目。有关更多信息，请查看我们[创建项目的最佳实践](/organization/getting-started/#4-create-projects)。项目允许您将事件与组织中的不同应用程序相关联，并将责任和所有权分配给组织内的特定用户和团队。

- **发布** - 发布是部署到环境的代码版本。当您通知 Sentry 发布时，您可以识别与之相关的新问题和回归，确认问题是否在下一个发布中得到解决，并应用源映射。在完整的[发布文档](/product/releases/)中了解更多信息。

- **发布健康** - 发布健康数据提供了关于崩溃和错误对用户影响的洞察，因为它与用户的体验相关，并揭示了每个新问题的趋势。在完整的[发布健康文档](/product/releases/health/)中了解更多信息。

- **sentry.io** - Sentry 的 SaaS 客户用户界面，其中由我们的 SDK 捕获的事件数据被可视化。（对于自托管用户，用户界面位于您公司的内部域上。）

- **Sentry SDK** - Sentry 的特定于编程语言/框架的应用程序监控库。当您将我们的 SDK 之一添加到您的应用程序时，来自您的应用程序的事件数据被捕获并发送到 Sentry，以便我们可以为您提供错误和性能报告。

- **跨度** - [跨度](/concepts/key-terms/tracing/distributed-tracing/#traces-transactions-and-spans)是跟踪组成的基本单元。多个父-子跨度组在 Sentry 中构成一个跟踪并共享一个 trace_id，这可以帮助您更快地找到问题的根本原因。

- **团队** - 团队与您的 Sentry 项目相关联，其成员作为组接收问题通知。了解有关[设置团队](/organization/getting-started/#2-set-up-teams)的更多信息。

- **跟踪** - [跟踪](/concepts/key-terms/tracing/)是您想要测量或跟踪的整个操作的记录 - 如页面加载、用户在您的应用程序中完成某些操作的实例或后端中的 cron 作业。

- **事务** - 事务表示服务被调用的单个实例，以支持您想要测量或跟踪的操作，如页面加载、页面导航或异步任务。事务事件按事务名称分组。

## 关键功能

这些关键功能中的每一个在 [sentry.io](https://sentry.io) 应用程序中都有自己的页面（或一组页面）。

- **问题** - 显示有关应用程序中分组问题的信息。从这里，您可以转到**问题详细信息**页面，以更详细地查看任何问题。在完整的[问题文档](/product/issues/)中了解更多信息。

- **项目** - 按团队列出您是成员的项目，并为您提供项目的高级概述。从这里，您可以转到每个项目的**项目详细信息**页面，以更详细地查看。在完整的[项目文档](/product/projects/)中了解更多信息。

- **探索** - Sentry 提供了多种搜索和调查应用程序数据的方法 - 包括应用程序的跟踪、指标、配置文件、重放和事件 - 以获取您需要的信息，以便更快、更轻松地调试问题。使用 Discover 页面跨环境查询事件，可视化结果，并解锁对整个系统健康状况的洞察。在完整的[探索文档](/product/explore/)中了解更多信息。

- **洞察** - Sentry 提供洞察，帮助您识别服务级别的机会，以改善应用程序性能的各个方面，包括请求、资产、缓存、队列等。位于"洞察"标题下的模块提供了该项目在该类别中表现如何的概述，以及查看示例事件及其跟踪以帮助您诊断潜在问题的能力。

- **洞察概览** - [sentry.io](http://sentry.io) 中的主要视图，您可以在其中搜索或浏览事务数据。该页面显示可视化事务或趋势的图表，以及一个表格，您可以在其中查看相关事务并深入了解有关它们的更多信息。在完整的[性能监控文档](/product/insights/overview/)中了解更多信息。

- **警报** - 您可以在其中创建新的警报规则并管理现有警报规则的地方。在完整的[警报文档](/product/alerts/)中了解更多信息。

- **仪表板** - 通过允许您浏览多个项目的错误和性能数据，为您提供应用程序健康状况的广泛概述。仪表板由一个或多个小部件组成，每个小部件可视化一个或多个**探索**或**问题**查询。在完整的[仪表板文档](/product/dashboards/)中了解更多信息。

- **发布** - 提供每个发布版本的高级视图、关联的项目、每个发布的采用阶段、每个提交的作者以及发布健康数据，包括无崩溃用户的百分比和无崩溃会话的百分比。您可以直接导航到**发布**页面，或从**问题详细信息**页面通过选择"最后看到"下列出的发布 ID 导航。在完整的[发布文档](/product/releases/)中了解更多信息。

Sentry可以直接通过[OpenTelemetry Protocol](https://opentelemetry.io/docs/specs/otel/protocol/)接收[OpenTelemetry](https://opentelemetry.io)追踪数据。如果您已有现有的OpenTelemetry追踪检测，您可以配置OpenTelemetry导出器直接将追踪数据发送到Sentry。Sentry的OTLP接收端点目前正在开发中，有一些已知限制：

- 不支持跨度事件（Span events）。所有跨度事件在接收过程中都会被丢弃。
- 部分支持跨度链接（Span links）。我们接收并显示跨度链接，但它们不能被搜索、过滤或聚合。链接显示在[追踪视图](/concepts/key-terms/tracing/trace-view/)中。
- 部分支持数组属性。我们接收并显示数组属性，但它们不能被搜索、过滤或聚合。数组属性显示在[追踪视图](/concepts/key-terms/tracing/trace-view/)中。
- Sentry不支持接收OTLP指标或OTLP日志。

配置OpenTelemetry导出器的最简单方法是使用环境变量。您需要配置追踪端点URL以及认证头。在您应用程序运行的服务器上设置这些变量。

```bash {filename: .env}
export OTEL_EXPORTER_OTLP_TRACES_ENDPOINT="___OTLP_TRACES_URL___"
export OTEL_EXPORTER_OTLP_TRACES_HEADERS="x-sentry-auth=sentry sentry_key=___PUBLIC_KEY___"
```

或者，您可以直接在应用程序代码中配置OpenTelemetry导出器。以下是使用OpenTelemetry Node SDK的示例：

```typescript {filename: app.ts}
import { NodeSDK } from "@opentelemetry/sdk-node";
import { OTLPTraceExporter } from "@opentelemetry/exporter-trace-otlp-http";

const sdk = new NodeSDK({
  traceExporter: new OTLPTraceExporter({
    url: "___OTLP_TRACES_URL___",
    headers: {
      "x-sentry-auth": "sentry sentry_key=___PUBLIC_KEY___",
    },
  }),
});

sdk.start();
```

您可以在Sentry项目设置中找到Sentry的OTLP端点和公钥的值。

1. 在Sentry中转到[设置 > 项目](https://sentry.io/orgredirect/organizations/:orgslug/settings/projects/)页面。
2. 从列表中选择一个项目。
3. 在此项目下的"SDK设置"标题下转到"客户端密钥（DSN）"子页面。

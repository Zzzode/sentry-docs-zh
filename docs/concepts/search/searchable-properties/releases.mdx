在**发布**中，您只能按以下属性搜索：

- [`release`](#release)
- [`release.build`](#releasebuild)
- [`release.package`](#releasepackage)
- [`release.stage`](#releasestage)
- [`release.version`](#releaseversion)

但是，这些属性也可以在 [sentry.io](https://sentry.io) 上面列出的其他页面中搜索（在[事件属性](#event-properties)下）。

## 可搜索属性

以下是可用于发布搜索的键和标记列表。

<DefinitionList>

### `release`

发布是部署到环境的代码版本。您可以使用发布版本的精确匹配创建标记，或 `release:latest` 选择最近的发布。[了解更多](/product/releases/usage/sorting-filtering/#latest-release)。

- **类型：** 字符串

### `release.build`

标识应用迭代的数字。例如，iOS 上的 `CFBundleVersion` 或 Android 上的 `versionCode`。[了解更多](/product/releases/usage/sorting-filtering/)。

- **类型：** 数字

### `release.package`

项目/应用的唯一标识符。例如，iOS 上的 `CFBundleIdentifier` 或 Android 上的 `packageName`。[了解更多](/product/releases/usage/sorting-filtering/)。

- **类型：** 字符串

### `release.stage`

发布相对于其他发布的使用情况。值可以是 `adopted`、`low` 或 `replaced`。[了解更多](/product/releases/health/#adoption-stages)。

- **类型：** 字符串

### `release.version`

名称的较短版本；没有包的名称或哈希的短版本。[了解更多](/product/releases/usage/sorting-filtering/)。

- **类型：** 字符串

</DefinitionList>

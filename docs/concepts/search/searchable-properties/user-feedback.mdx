[用户反馈](/product/user-feedback/) 允许您的用户创建错误报告，以便他们可以立即让您了解隐藏的问题。每个报告都会自动包含相关的重放、标记和错误，使修复问题变得非常简单。

您可以在**用户反馈**页面中按用户反馈属性进行搜索。

## 可搜索属性

以下是可用于用户反馈搜索的键和标记列表。

<DefinitionList>

### `assigned`

返回分配给定义用户或团队的用户反馈提交。值可以是用户 ID（您的电子邮件地址）、`me` 表示您自己、`none` 表示无受让人、`my_teams` 或 `#team-name` 表示您所属的团队的。

- **类型：** 团队或组织用户

### `browser.name`

浏览器的名称。

- **类型：** 字符串

### `device.brand`

设备品牌。

- **类型：** 字符串

### `device.family`

设备系列。通常是跨代型号名称的共同部分。例如，iPhone、Samsung Galaxy。

- **类型：** 字符串

### `device.model_id`

内部硬件版本以精确识别设备。

- **类型：** 不适用

### `device.name`

设备名称。

- **类型：** 字符串

### `dist`

区分同一版本应用程序的构建或部署变体。例如，dist 可以是 Xcode 构建的构建号或 Android 构建的版本代码。

- **类型：** 字符串

### `environment`

事件首次出现的环境。

- **类型：** 字符串

### `id`

反馈 ID。

- **类型：** UUID

### `is`

用户反馈提交的属性。值可以是：`unresolved`、`resolved`、`assigned`、`unassigned`、`linked` 或 `unlinked`。`linked` 和 `unlinked` 值根据它们是否链接到外部问题跟踪器返回用户反馈提交。

- **类型：** 状态

### `level`

用户反馈提交的严重性。如果反馈是通过崩溃报告模态框传入的，则级别与最终用户遇到的错误相关联。对于来自用户反馈小组件的反馈，级别为 `info`。

- **类型：** 字符串

### `os.name`

操作系统的名称。

- **类型：** 字符串

### `sdk.name`

发送事件的 Sentry SDK 的名称。

- **类型：** 字符串

### `sdk.version`

发送事件的 Sentry SDK 的版本。

- **类型：** 字符串

### `timestamp`

事务的完成时间戳。返回与匹配日期时间的事件。

- **类型：** 日期时间

### `transaction`

错误或事务名称标识符。

- **类型：** 字符串

### `url`

触发反馈的页面的 URL。

- **类型：** 字符串

### `user.email`

用户的电子邮件地址仅在 Sentry 从 `Sentry.setUser` SDK 选项中作为标记接收时才存在。此属性不适用于在用户反馈小组件中输入的电子邮件，这些电子邮件不可搜索。

- **类型：** 字符串

### `user.id`

用户的应用程序特定内部标识符。

- **类型：** 字符串

### `user.ip`

用户的 IP 地址。Sentry 使用 IP 地址作为未验证用户的唯一标识符。

- **类型：** 字符串

### `user.username`

用户名，通常比 `user.id` 更好的标签。

- **类型：** 字符串

</DefinitionList>

事件是使用 Sentry SDK 捕获的底层事件数据——即错误和事务。

您可以在以下 [sentry.io](https://sentry.io) 页面中按事件属性进行搜索：

- 发现 - 在查询构建器中（取决于数据集选择）
- 仪表板 - 在小组件构建器内（取决于数据集选择）
- 性能 - 仅在事务摘要中
- 问题 - 如下列表中所示
- 警报 - 创建指标警报时

<Alert>

请注意，在警报中，只有有限数量的属性可用于[过滤事务事件](/product/alerts/create-alerts/metric-alert-config/#tags--properties)。

</Alert>

在**问题**页面中搜索事件属性时，搜索将返回具有_一个或多个匹配事件_的任何问题，这些事件符合提供的事件属性过滤器。

## 可搜索属性

以下是可用于事件搜索的键和标记列表。

<DefinitionList>

### `apdex(threshold)`

返回具有您输入的 [Apdex 分数](/product/insights/overview/metrics/#apdex) 的结果。值必须在 `0` 和 `1` 之间。较高的 apdex 值表示较高的用户满意度。

- **类型：** 数字

### `app.in_foreground`

指示应用是在前台还是后台。值为 `1/0` 或 `true/false`

- **类型：** 布尔值

### `avg(field)`

返回与所选字段匹配的平均值的结果。该字段可以是数字或持续时间。通常与比较运算符一起使用。

- **类型：** 匹配字段

### `count_if(column,operator,value)`

返回满足传递给函数参数条件的匹配计数结果。

- **类型：** 数字

### `count_miserable(field,threshold)`

返回与字段的唯一实例计数匹配的结果，这些实例高于痛苦阈值。

- **类型：** 数字

### `count_unique(field)`

返回与输入字段的唯一实例计数匹配的结果。

- **类型：** 数字

### `count_web_vitals(vital,threshold)`

返回与 Web Vitals 质量阈值（`poor`、`meh`、`good` 或 `any`）匹配的计数结果。

- **类型：** 数字

### `count()`

返回与匹配计数的结果。（与问题搜索中的 `timesSeen` 相同。）不需要参数。

- **类型：** 数字

### `culprit`

已弃用

- **类型：** 字符串

### `device.arch`

CPU 架构

- **类型：** 字符串

### `device.battery_level`

如果设备有电池，这可以是定义电池电量的浮点值（范围 0-100）。

- **类型：** 字符串

### `device.brand`

设备品牌

- **类型：** 字符串

### `device.charging`

设备是否正在充电。不是布尔值。

- **类型：** 字符串

### `device.class`

客户端设备的估计性能等级，估计为 `high`、`medium` 或 `low`。有关更多详细信息，请参见下面的[设备分类](#device-classification)部分。

- **类型：** 字符串

### `device.family`

设备系列。通常是跨代型号名称的共同部分。例如，iPhone、Samsung Galaxy。

- **类型：** 字符串

### `device.locale`

已弃用

- **类型：** 字符串

### `device.name`

设备详细信息

- **类型：** 字符串

### `device.online`

设备是否在线。字符串，可以是 `True` 或 `False`。

- **类型：** 字符串

### `device.orientation`

描述设备的方向，可以是 `portrait` 或 `landscape`。

- **类型：** 字符串

### `device.screen_density`

设备屏幕密度（像素）。

- **类型：** 字符串

### `device.screen_dpi`

设备屏幕每英寸点数。

- **类型：** 字符串

### `device.screen_height_pixels`

设备屏幕高度（像素）。

- **类型：** 字符串

### `device.screen_width_pixels`

设备屏幕宽度（像素）。

- **类型：** 字符串

### `device.simulator`

指示此设备是模拟器还是真实设备。字符串，可以是 `True` 或 `False`。

- **类型：** 字符串

### `device.uuid`

已弃用

- **类型：** UUID

### `dist`

区分同一版本应用程序的构建或部署变体。例如，dist 可以是 Xcode 构建的构建号或 Android 构建的版本代码。

- **类型：** 字符串

### `environment`

指您的代码部署命名约定。例如，_development_、_testing_、_staging_ 等。[了解更多](/product/sentry-basics/environments/)。在 [sentry.io](https://sentry.io) 的某些页面中，您可以使用下拉菜单按环境过滤。

- **类型：** 字符串

### `epm()`

返回与匹配事件每分钟计数的结果。不需要参数。

- **类型：** 数字

### `eps()`

返回与匹配事件每秒计数的结果。不需要参数。

- **类型：** 数字

### `error.handled`

指示用户是否已处理异常——例如，使用 try...catch。如果所有堆栈跟踪都处理错误，则错误被视为已处理。值为 `1/0` 或 `true/false`

- **类型：** 布尔值

### `error.main_thread`

指示错误是否发生在主线程上。值为 `1/0` 或 `true/false`

- **类型：** 布尔值

### `error.mechanism`

描述创建此异常的机制的对象。

- **类型：** 数组

### `error.type`

异常类型。例如，`ValueError`。

- **类型：** 数组

### `error.unhandled`

`error.handled` 的反义。

- **类型：** 布尔值

### `error.value`

导致或显示错误的字段的原始值。

- **类型：** 数组

### `event.type`

事件类型（transaction、error、default、csp 等）。事务类型在**问题**中不可用。

- **类型：** 字符串

### `failure_count()`

返回具有与失败列表中的 `transaction.status` 值匹配的事件计数的结果。值可以是：`ok`、`cancelled`、`unknown`。不需要参数。

- **类型：** 数字

### `failure_rate()`

返回与失败事务的匹配率的结果——即 `failure_count()` 除以 `count()`（总计数）。不需要参数。

- **类型：** 数字

### `geo.city`

城市全名

- **类型：** 字符串

### `geo.country_code`

ISO 3166-1 国家代码

- **类型：** 字符串

### `geo.region`

国家全名

- **类型：** 字符串

### `has`

返回具有定义的标记或字段的结果，但不包括该标记或字段的值。例如，输入 `has:user` 会找到具有 `user` 标记的事件。

- **类型：** 错误

### `http.method`

创建事件的[请求](https://develop.sentry.dev/sdk/data-model/event-payloads/request/)的 HTTP 方法。

- **类型：** 字符串

### `http.referer`

标识请求资源的网页。

- **类型：** 字符串

### `http.status_code`

HTTP 状态代码，指示响应是否成功。例如，`200` 或 `404`。

- **类型：** 字符串

### `http.url`

导致错误的请求的完整 URL，但不包含任何参数

- **类型：** 字符串

### `id`

事件 id。在**问题**中，仅使用 ID 值而不使用 `id` 键。

- **类型：** UUID

### `issue`

短问题代码，例如 `SENTRY-ABC`。

- **类型：** 字符串

### `last_seen()`

事件最后出现的日期时间。等同于 `max(timestamp)`。不需要参数。

- **类型：** 日期时间

### `level`

事件的严重性（如：fatal、error、warning）。对于事务始终设置为 info。

- **类型：** 字符串

### `location`

错误发生的位置。

- **类型：** 字符串

### `max(numeric field)`

返回与输入字段的匹配最大值的结果。

- **类型：** 匹配字段

### `measurements.app_start_cold`

[冷启动](/product/insights/mobile/mobile-vitals/app-starts/)指应用在重启或更新后首次启动。应用不在内存中且不存在进程。

- **类型：** 持续时间

### `measurements.app_start_warm`

[温启动](/product/insights/mobile/mobile-vitals/app-starts/)指应用已经至少启动过一次且部分在内存中。例如，用户退出您的应用，但随后重新启动它。进程可能继续运行，但应用必须从头开始重新创建活动。

- **类型：** 持续时间

### `measurements.cls`

[累积布局偏移 (CLS)](/product/insights/web-vitals/web-vitals-concepts/#cumulative-layout-shift-cls) 是在渲染过程中每个意外元素偏移的单个布局偏移分数的总和。

- **类型：** 数字

### `measurements.fcp`

[首次内容绘制 (FCP)](/product/insights/web-vitals/web-vitals-concepts/#first-contentful-paint-fcp) 测量第一个内容在视口中渲染的时间。

- **类型：** 持续时间

### `measurements.fid`

[首次输入延迟 (FID)](/product/insights/web-vitals/web-vitals-concepts/#first-input-delay-fid) 测量用户尝试与视口交互时的响应时间。

- **类型：** 持续时间

### `measurements.fp`

[首次绘制 (FP)](/product/insights/web-vitals/web-vitals-concepts/#first-paint-fp) 测量第一个像素出现在视口中所需的时间，渲染与之前显示的任何视觉变化。

- **类型：** 持续时间

### `measurements.frames_frozen`

[慢速和冻结帧](/product/insights/mobile/mobile-vitals/#slow-and-frozen-frames) 测量应用的响应性。

- **类型：** 数字

### `measurements.frames_frozen_rate`

返回与冻结帧的匹配率的结果。即 `measurements.frames_frozen` 除以 `measurements.frames_total`。

- **类型：** 数字

### `measurements.frames_slow`

[慢速和冻结帧](/product/insights/mobile/mobile-vitals/#slow-and-frozen-frames) 测量应用的响应性。

- **类型：** 数字

### `measurements.frames_slow_rate`

返回与慢速帧的匹配率的结果。即 `measurements.frames_slow` 除以 `measurements.frames_total`。

- **类型：** 数字

### `measurements.frames_total`

返回与总帧数的匹配计数的结果。

- **类型：** 数字

### `measurements.lcp`

[最大内容绘制 (LCP)](/product/insights/web-vitals/web-vitals-concepts/#largest-contentful-paint-lcp) 测量最大内容出现在视口中的渲染时间。

- **类型：** 持续时间

### `measurements.stall_count`

[停顿](/platforms/react-native/tracing/instrumentation/automatic-instrumentation/#stall-tracking) 是指 JavaScript 事件循环完成所需时间比预期长。仅适用于 React Native。

- **类型：** 数字

### `measurements.stall_longest_time`

[最长停顿时间](/platforms/react-native/tracing/instrumentation/automatic-instrumentation/#stall-tracking) 是最长事件循环停顿的时间（以毫秒为单位）。仅适用于 React Native。

- **类型：** 持续时间

### `measurements.stall_percentage`

停顿百分比等于 `stall_total_time` 除以 `transaction.duration`。仅适用于 React Native。

- **类型：** 数字

### `measurements.stall_total_time`

[总停顿时间](/platforms/react-native/tracing/instrumentation/automatic-instrumentation/#stall-tracking) 是所有停顿的总时间（以毫秒为单位）。仅适用于 React Native。

- **类型：** 持续时间

### `measurements.ttfb`

[首字节时间 (TTFB)](/product/insights/web-vitals/web-vitals-concepts/#time-to-first-byte-ttfb) 测量用户浏览器接收页面内容第一个字节所需的时间。

- **类型：** 持续时间

### `measurements.ttfb.requesttime`

请求开始到响应开始之间的时间（参见[图表](https://web.dev/ttfb/#what-is-ttfb)）。

- **类型：** 持续时间

### `message`

返回与匹配消息的错误或与匹配事务名称的事务。也匹配包含提供值的任何消息。搜索 `message:undefined` 将匹配消息为 `undefined is not an object` 的事件。原始文本搜索（没有 `message` 键的搜索）也针对此字段进行检查。对于错误，消息可以是元素的连接，因此搜索可能包括意外结果。

- **类型：** 字符串

### `min(numeric field)`

返回与输入字段的匹配最小值的结果。

- **类型：** 匹配字段

### `os.build`

操作系统的内部构建版本。

- **类型：** 字符串

### `os.kernel_version`

独立的内核版本字符串。这通常是 `uname` 系统调用的完整输出。

- **类型：** 字符串

### `os.distribution_name`

Linux 发行版名称。这映射到 [`/etc/os-release/`](https://www.freedesktop.org/software/systemd/man/latest/os-release.html) 中的 `ID`。

- **类型：** 字符串

### `os.distribution_version`

Linux 发行版版本。这映射到 [`/etc/os-release/`](https://www.freedesktop.org/software/systemd/man/latest/os-release.html) 中的 `VERSION_ID`。

- **类型：** 字符串

### `percentile(field,level)`

返回与字段的近似百分位数到级别的结果。级别可以在 `0` 和 `1` 之间。例如，如果您想找到事务持续时间的第 50 个百分位数，您将输入 `percentile(transaction.duration, 0.5)`。

- **类型：** 数字

### `platform`

平台名称。这默认为 `other`，仅适用于此列表中的平台：
`as3`、`c`、`cfml`、`cocoa`、`csharp`、`elixir`、`go`、`groovy`、`haskell`、`java`、`javascript`、`native`、`node`、`objc`、`other`、`perl`、`php`、`python`、`ruby`

- **类型：** 字符串

### `project`

项目名称。在 [sentry.io](https://sentry.io) 的某些页面中，您还可以使用下拉菜单按项目过滤。

- **类型：** 字符串

### `project.id`

项目的 id。

- **类型：** 数字

### `pXY(duration field)`

返回与字段的近似百分位数的结果。将 "XY" 替换为 50、75、95、99 或 100。例如，如果您想找到事务持续时间的第 50 个百分位数，您将输入 `p50(transaction.duration)`。

- **类型：** 数字

### `release`

发布是部署到环境的代码版本。您可以使用发布版本的精确匹配创建标记，或 `release:latest` 选择最近的发布。[了解更多](/product/releases/usage/sorting-filtering/#latest-release)。

- **类型：** 字符串

### `release.build`

标识应用迭代的数字。例如，iOS 上的 `CFBundleVersion` 或 Android 上的 `versionCode`。[了解更多](/product/releases/usage/sorting-filtering/)。

- **类型：** 数字

### `release.package`

项目/应用的唯一标识符。例如，iOS 上的 `CFBundleIdentifier` 或 Android 上的 `packageName`。[了解更多](/product/releases/usage/sorting-filtering/)。

- **类型：** 字符串

### `release.stage`

发布相对于其他发布的使用情况。值可以是 `adopted`、`low` 或 `replaced`。[了解更多](/product/releases/health/#adoption-stages)。

- **类型：** 字符串

### `release.version`

名称的较短版本；没有包的名称或哈希的短版本。[了解更多](/product/releases/usage/sorting-filtering/)。

- **类型：** 字符串

### `sdk.name`

发送事件的 Sentry SDK 的名称。

- **类型：** 字符串

### `sdk.version`

发送事件的 Sentry SDK 的版本。

- **类型：** 字符串

### `spans.browser`

基于跨度操作的事务的累积浏览器时间。

- **类型：** 持续时间

### `spans.db`

基于跨度操作的事务的累积数据库时间。

- **类型：** 持续时间

### `spans.http`

基于跨度操作的事务的累积 HTTP 时间。

- **类型：** 持续时间

### `spans.resource`

基于跨度操作的事务的累积资源时间。

- **类型：** 持续时间

### `spans.ui`

基于跨度操作的事务的累积 UI 时间。

- **类型：** 持续时间

### `stack.abs_path`

源文件的绝对路径。在事件中，这是一个数组；在问题中，这是一个单个值。

- **类型：** 数组，单个值

### `stack.colno`

调用的列号，从 1 开始。

- **类型：** 数组

### `stack.filename`

相对于项目根目录的源文件路径。在事件中，这是一个数组。在问题中，这是一个单个值。

- **类型：** 数组，单个值

### `stack.function`

被调用函数的名称。在事件中，这是一个数组。在问题中，这是一个单个值。

- **类型：** 数组，单个值

### `stack.in_app`

指示帧是否与堆栈跟踪中相关代码的执行相关。例如，可能为您的应用框架提供支持的框架的 Web 服务器的帧可能不相关。但是，一旦您开始处理代码，对框架库的调用可能是相关的。值可以是 `1`（真）或 `0`（假）。

- **类型：** 数组

### `stack.lineno`

调用的行号，从 1 开始。

- **类型：** 数组

### `stack.module`

平台特定的模块路径。例如，`sentry.interfaces.Stacktrace`。在事件中，这是一个数组。在问题中，这是一个单个值。

- **类型：** 数组，单个值

### `stack.package`

帧所在的"包"。根据平台的不同，这可能是不同的东西。对于 C#，它可以是程序集的名称。对于本机代码，它可以是动态库的路径或其他东西。在事件中，这是一个数组。在问题中，这是一个单个值。

- **类型：** 数组，单个值

### `sum(numeric field)`

返回与输入字段的匹配总值的结果。

- **类型：** 匹配字段

### `timestamp`

事务的完成时间戳。返回与匹配日期时间的事件。

- **类型：** 日期时间

### `timestamp.to_day`

向下舍入到最近一天的时间戳。

- **类型：** 日期时间

### `timestamp.to_hour`

向下舍入到最近一小时的时间戳。

- **类型：** 日期时间

### `title`

错误的标题或事务名称。

- **类型：** 字符串

### `trace`

跟踪表示您想要测量或跟踪的整个操作的记录——如页面加载，使用 Sentry SDK 生成的 UUID 进行搜索。

- **类型：** UUID

### `trace.parent_span`

当前事务的父级的跨度 ID。如果事务是根，则为 null。

- **类型：** UUID

### `trace.span`

事件中根事务的根跨度的跨度 ID。

- **类型：** UUID

### `transaction`

对于[事务](/product/insights/overview/transaction-summary/#what-is-a-transaction)，事务的名称。对于错误，关联事务的名称。

- **类型：** 字符串

### `transaction.duration`

事务的持续时间（以毫秒为单位）。

- **类型：** 持续时间

### `transaction.op`

标识[操作类型](https://develop.sentry.dev/sdk/performance/span-operations/)的短代码，跨度正在测量。

- **类型：** 字符串

### `transaction.status`

描述跨度/事务的状态。查看我们的[事务有效负载文档](https://develop.sentry.dev/sdk/data-model/event-payloads/transaction/)了解所有可能的状态。

- **类型：** 字符串

### `unreal.crash_type`

[Unreal 崩溃上下文类型](https://docs.unrealengine.com/4.27/en-US/API/Runtime/Core/GenericPlatform/ECrashContextType/)

- **类型：** 字符串

### `user_misery(number)`

返回具有定义的用户痛苦值的事务。[用户痛苦](/product/insights/overview/metrics/#user-misery) 是一个用户加权的性能指标，计算感到沮丧的唯一用户数量；"沮丧" 以响应时间是满意响应时间阈值的四倍（以毫秒为单位）来衡量。它突出显示对用户影响最大的事务。

- **类型：** 数字

### `user.display`

按顺序，第一个可用的用户字段：电子邮件，然后是用户名，ID，然后是 IP 地址。

- **类型：** 字符串

### `user.email`

用户名的替代或补充。Sentry 了解电子邮件地址，因此可以显示 Gravatars 等功能并解锁消息传递功能。

- **类型：** 字符串

### `user.id`

用户的应用程序特定内部标识符。

- **类型：** 字符串

### `user.ip`

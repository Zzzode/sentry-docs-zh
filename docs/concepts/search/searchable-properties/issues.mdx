[问题](/product/issues/) 是一个或多个错误事件的聚合。可搜索的问题属性包括状态、分配、聚合计数和年龄。如果您已设置[特性标志的评估跟踪](/product/issues/issue-details/feature-flags/#evaluation-tracking)，您可以搜索具有错误事件的问题，其中特性标志评估值为 `true` 或 `false`。您可以在**问题**页面和**仪表板**的小组件构建器中按问题属性进行搜索，具体取决于您的数据集选择。

## 可搜索属性

以下是可用于问题搜索的键和标记列表。

<Alert>
如果您使用 Sentry [API](/api/)，则只需要对**日期时间**和**相对时间**可搜索属性类型使用查询语法。
</Alert>

<DefinitionList>

### `age`

返回自值定义的时间以来创建的问题。语法类似于 Unix find 命令。支持的后缀：`m - 分钟`、`h - 小时`、`d - 天`、`w - 周`。例如，`age:-24h` 返回最近 24 小时内的新问题，而 `age:+12h` 返回超过 12 小时的问题。输入 `age:+12h age:-24h` 将返回 12 到 24 小时前创建的问题。

- **类型：** 相对时间

### `app.in_foreground`

指示应用是在前台还是后台。值为 `1/0` 或 `true/false`

- **类型：** 布尔值

### `assigned`

返回分配给定义用户或团队的问题。值可以是用户 ID（您的电子邮件地址）、`me` 表示您自己、`none` 表示无受让人、`my_teams` 或 `#team-name` 表示您所属的团队的。

- **类型：** 团队或组织用户

### `assigned_or_suggested`

返回分配给或建议分配给定义用户或团队的问题。建议的受让人通过匹配[所有权规则](/product/issues/ownership-rules/)和[可疑提交](/product/issues/suspect-commits/)找到。值可以是用户 ID（您的电子邮件地址）、`me` 表示您自己、`none` 表示无受让人/建议、`my_teams` 或 `#team-name` 表示您所属的团队的。

- **类型：** 团队或组织用户

### `bookmarks`

返回由定义用户添加书签的问题。值可以是您的用户 ID（您的电子邮件地址）或 `me` 表示您自己。

- **类型：** 团队或组织用户

### `device.arch`

CPU 架构

- **类型：** 字符串

### `device.brand`

设备品牌

- **类型：** 字符串

### `device.family`

设备系列。通常是跨代型号名称的共同部分。例如，iPhone、Samsung Galaxy。

- **类型：** 字符串

### `device.locale`

已弃用

- **类型：** 字符串

### `device.model_id`

内部硬件版本以精确识别设备。

- **类型：** 不适用

### `device.orientation`

描述设备的方向，可以是 `portrait` 或 `landscape`。

- **类型：** 字符串

### `device.screen_density`

设备屏幕密度（像素）。

- **类型：** 字符串

### `device.screen_dpi`

设备屏幕每英寸点数。

- **类型：** 字符串

### `device.screen_height_pixels`

设备屏幕高度（像素）。

- **类型：** 字符串

### `device.screen_width_pixels`

设备屏幕宽度（像素）。

- **类型：** 字符串

### `device.uuid`

已弃用

- **类型：** UUID

### `dist`

区分同一版本应用程序的构建或部署变体。例如，dist 可以是 Xcode 构建的构建号或 Android 构建的版本代码。

- **类型：** 字符串

### `error.handled`

指示用户是否已处理异常——例如，使用 try...catch。如果所有堆栈跟踪都处理错误，则错误被视为已处理。值为 `1/0` 或 `true/false`

- **类型：** 布尔值

### `error.main_thread`

指示错误是否发生在主线程上。值为 `1/0` 或 `true/false`

- **类型：** 布尔值

### `error.mechanism`

描述创建此异常的机制的对象。

- **类型：** 数组

### `error.type`

异常类型。例如，`ValueError`。

- **类型：** 数组

### `error.unhandled`

`error.handled` 的反义。

- **类型：** 布尔值

### `error.value`

导致或显示错误的字段的原始值。

- **类型：** 数组

### `event.timestamp`

返回与匹配日期时间的问题。

- **类型：** 日期时间

### `event.type`

事件类型（transaction、error、default、csp 等）。事务类型在**问题**中不可用。

- **类型：** 字符串

### `firstRelease`

返回在给定发布中首次出现的问题。可以是发布版本的精确匹配，或 `first-release:latest` 选择最近的发布。

- **类型：** 日期时间

### `firstSeen`

返回与首次出现匹配的问题。语法与 `age` 相同。

- **类型：** 日期时间

### `flags`

对于设置为 `true` 或 `false` 的[特性标志评估](/product/issues/issue-details/feature-flags/#evaluation-tracking)，特性标志的名称。例如，搜索键为 `my_flag` 且值为 `true` 的标志的语法是 `flags["my_flag"]:true`。

- **类型：** 布尔值

### `geo.city`

城市全名

- **类型：** 字符串

### `geo.country_code`

ISO 3166-1 国家代码

- **类型：** 字符串

### `geo.region`

国家全名

- **类型：** 字符串

### `has`

返回具有定义的标记或字段的结果，但不包括该标记或字段的值。例如，输入 `has:user` 会找到具有 `user` 标记的事件。

- **类型：** 错误

### `http.method`

创建事件的[请求](https://develop.sentry.dev/sdk/data-model/event-payloads/request/)的 HTTP 方法。

- **类型：** 字符串

### `http.referer`

标识请求资源的网页。

- **类型：** 字符串

### `http.status_code`

HTTP 状态代码，指示响应是否成功。例如，`200` 或 `404`。

- **类型：** 字符串

### `http.url`

导致错误的请求的完整 URL，但不包含任何参数

- **类型：** 字符串

### `id`

事件或重放 id。在**问题**中，仅使用 ID 值而不使用 `id` 键。

- **类型：** UUID

### `is`

问题的属性。值可以是：`unresolved`、`resolved`、`archived`、`assigned`、`unassigned`、`for_review`、`linked` 或 `unlinked`。`for_review` 值过滤未解决的问题，仅显示最近未解决或尚未标记为"已审核"的新问题。`linked` 和 `unlinked` 值根据它们是否链接到外部问题跟踪器返回问题。

- **类型：** 状态

### `issue`

短问题代码，例如 `SENTRY-ABC`。

- **类型：** 字符串

### `issue.category`

问题的类别。例如：`error`、`performance`、`replay` 和 `cron`。

- **类型：** 字符串

### `issue.type`

问题的具体类型。例如 `issue.type:performance_n_plus_one_db_queries` 返回 n 加 1 数据库查询性能问题。

- **类型：** 字符串

### `lastSeen`

事件最后出现的日期时间。例如，`lastSeen:+30d` 返回 30 天前或更早看到的问题；`lastSeen:-2d` 返回过去两天内看到的问题。这与 `age` 类似。

- **类型：** 日期时间

### `level`

事件的严重性（如：fatal、error、warning）。对于事务始终设置为 info。

- **类型：** 字符串

### `location`

错误发生的位置。

- **类型：** 字符串

### `message`

返回与匹配消息的错误或与匹配事务名称的事务。也匹配包含提供值的任何消息。搜索 `message:undefined` 将匹配消息为 `undefined is not an object` 的事件。原始文本搜索（没有 `message` 键的搜索）也针对此字段进行检查。对于错误，消息可以是元素的连接，因此搜索可能包括意外结果。

- **类型：** 字符串

### `os.build`

操作系统的内部构建版本。

- **类型：** 字符串

### `os.kernel_version`

独立的内核版本字符串。这通常是 `uname` 系统调用的完整输出。

- **类型：** 字符串

### `os.distribution_name`

Linux 发行版名称。这映射到 [`/etc/os-release/`](https://www.freedesktop.org/software/systemd/man/latest/os-release.html) 中的 `ID`。

- **类型：** 字符串

### `os.distribution_version`

Linux 发行版版本。这映射到 [`/etc/os-release/`](https://www.freedesktop.org/software/systemd/man/latest/os-release.html) 中的 `VERSION_ID`。

- **类型：** 字符串

### `platform.name`

平台名称

- **类型：** 字符串

### `project`

项目名称。在 [sentry.io](https://sentry.io) 的某些页面中，您还可以使用下拉菜单按项目过滤。

- **类型：** 字符串

### `project.id`

项目的 id。

- **类型：** 数字

### `release`

发布是部署到环境的代码版本。您可以使用发布版本的精确匹配创建标记，或 `release:latest` 选择最近的发布。[了解更多](/product/releases/usage/sorting-filtering/#latest-release)。

- **类型：** 字符串

### `release.build`

标识应用迭代的数字。例如，iOS 上的 `CFBundleVersion` 或 Android 上的 `versionCode`。[了解更多](/product/releases/usage/sorting-filtering/)。

- **类型：** 数字

### `release.package`

项目/应用的唯一标识符。例如，iOS 上的 `CFBundleIdentifier` 或 Android 上的 `packageName`。[了解更多](/product/releases/usage/sorting-filtering/)。

- **类型：** 字符串

### `release.stage`

发布相对于其他发布的使用情况。值可以是 `adopted`、`low` 或 `replaced`。[了解更多](/product/releases/health/#adoption-stages)。

- **类型：** 字符串

### `release.version`

名称的较短版本；没有包的名称或哈希的短版本。[了解更多](/product/releases/usage/sorting-filtering/)。

- **类型：** 字符串

### `sdk.name`

发送事件的 Sentry SDK 的名称。

- **类型：** 字符串

### `sdk.version`

发送事件的 Sentry SDK 的版本。

- **类型：** 字符串

### `stack.abs_path`

源文件的绝对路径。在事件中，这是一个数组；在问题中，这是一个单个值。

- **类型：** 数组，单个值

### `stack.filename`

相对于项目根目录的源文件路径。在事件中，这是一个数组。在问题中，这是一个单个值。

- **类型：** 数组，单个值

### `stack.function`

被调用函数的名称。在事件中，这是一个数组。在问题中，这是一个单个值。

- **类型：** 数组，单个值

### `stack.module`

平台特定的模块路径。例如，`sentry.interfaces.Stacktrace`。在事件中，这是一个数组。在问题中，这是一个单个值。

- **类型：** 数组，单个值

### `stack.package`

帧所在的"包"。根据平台的不同，这可能是不同的东西。对于 C#，它可以是程序集的名称。对于本机代码，它可以是动态库的路径或其他东西。在事件中，这是一个数组。在问题中，这是一个单个值。

- **类型：** 数组，单个值

### `timesSeen`

返回与匹配计数的结果。（与事件中的 `count()` 相同。）

- **类型：** 数字

### `timestamp`

事务的完成时间戳。返回与匹配日期时间的事件。

- **类型：** 日期时间

### `title`

错误的标题或事务名称。

- **类型：** 字符串

### `trace`

跟踪表示您想要测量或跟踪的整个操作的记录——如页面加载，使用 Sentry SDK 生成的 UUID 进行搜索。

- **类型：** UUID

### `transaction`

对于[事务](/product/insights/overview/transaction-summary/#what-is-a-transaction)，事务的名称。对于错误，关联事务的名称。

- **类型：** 字符串

### `unreal.crash_type`

[Unreal 崩溃上下文类型](https://docs.unrealengine.com/4.27/en-US/API/Runtime/Core/GenericPlatform/ECrashContextType/)

- **类型：** 字符串

### `user.email`

用户名的替代或补充。Sentry 了解电子邮件地址，因此可以显示 Gravatars 等功能并解锁消息传递功能。

- **类型：** 字符串

### `user.id`

用户的应用程序特定内部标识符。

- **类型：** 字符串

### `user.ip`

用户的 IP 地址。Sentry 使用 IP 地址作为未验证用户的唯一标识符。

- **类型：** 字符串

### `user.username`

用户名，通常比 `user.id` 更好的标签。

- **类型：** 字符串

</DefinitionList>

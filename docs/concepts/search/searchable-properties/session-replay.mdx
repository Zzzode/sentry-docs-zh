[会话重放](/product/explore/session-replay/) 提供了用户在网站或 Web 应用上交互的视频式重现。所有用户交互，包括页面访问、鼠标移动、点击和滚动，都会被捕获，帮助开发人员将已知问题与用户在 UI 中体验它的方式联系起来。

您可以在**重放**页面中按会话重放属性进行搜索。

## 可搜索属性

以下是可用于会话重放搜索的键和标记列表。

<DefinitionList>

### `activity`

重放活动基于错误数量、UI 事件数量和重放持续时间来计算。它表示为从 `1` 到 `10` 的数字。

- **类型：** 数字

### `browser.name`

用户 Web 浏览器的名称。例如，`Chrome`、`Firefox` 或 `Safari`。

- **类型：** 字符串

### `browser.version`

浏览器的版本字符串。

- **类型：** 字符串

### `click.alt`

被点击元素的 `alt`。例如，`"a good dog"` 会匹配元素 `<img src="/lassie.jpeg" alt="a good dog" />`。

- **类型：** 字符串

### `click.class`

被点击元素的 `class`。不需要前导 `.`。例如，`btn-primary` 会匹配元素 `<a class="btn btn-primary">Save</a>`。

- **类型：** 字符串

### `click.component_name`

被点击的前端组件的名称。例如，`MyAwesomeComponent` 会匹配 React 组件 `<MyAwesomeComponent>`。

注意：此属性要求您的项目已配置 [React 组件名称捕获](/platforms/javascript/guides/react/features/component-names)。

- **类型：** 字符串

### `click.id`

被点击元素的 `id`。不需要前导 `#`。例如，`reset-password` 会匹配元素 `<a id="reset-password">Reset</a>`。

- **类型：** 字符串

### `click.label`

被点击元素的 `aria-label`。例如，`Expand` 会匹配元素 `<button aria-label="Expand"><img src="/icons/expand.png"/></button>`。

- **类型：** 字符串

### `click.role`

被点击元素的 `role`。例如，`button` 会匹配 `<button>Save</button>` 和 `<a role="button">Submit</a>`。

- **类型：** 字符串

### `click.selector`

使用 CSS 选择器语法的子集标识的元素。例如，`#section-1` 或 `span.active` 或 `span[role=button]` 或 `.active[role=button]` 都会匹配元素 `<span id="section-1" class="active" role="button"/>`。注意，不支持 CSS 组合器、伪选择器和除 `=` 外的属性选择器。

- **类型：** 字符串

### `click.tag`

被点击元素的标签名称。例如，`input` 会匹配 `<input name="username" />`。

- **类型：** 字符串

### `click.testid`

被点击元素的 `data-testid` 或 `data-test-id`。例如，`user-name` 会匹配元素 `<a data-testid="user-name">User Name</a>`。

- **类型：** 字符串

### `click.textContent`

被点击元素的直接 `textContent`。例如，`Save` 会匹配 `<button>Save</button>` 但不会匹配 `<button><h1>Save</h1></button>`。

- **类型：** 字符串

### `click.title`

被点击元素的 `title`。例如，`Save this comment` 会匹配元素 `<a title="Save this comment" class="btn btn-primary">Save</a>`。

- **类型：** 字符串

### `count_dead_clicks`

重放中[无效点击](/product/explore/session-replay/replay-page-and-filters/#:~:text=Dead%20Clicks%3A%20User%20clicks%20on%20a%20and%20button%20tags%20that%20do%20not%20result%20in%20any%20page%20activity%20after%207%20seconds%20(i.e.%20no%20HTML%20was%20added%2C%20removed%2C%20or%20updated%3B%20no%20visual%20changes%20were%20observed%20in%20the%20page).)的数量。

- **类型：** 数字

### `count_errors`

重放中的错误数量。

- **类型：** 数字

### `count_rage_clicks`

重放中[愤怒点击](/product/explore/session-replay/replay-page-and-filters/#:~:text=Rage%20Clicks%3A%20Five%20or%20more%20clicks%20on%20a%20dead%20element%20(it%20exhibits%20no%20page%20activity%20after%207%20seconds.)%20Rage%20clicks%20are%20a%20subset%20of%20dead%20clicks)的数量。

- **类型：** 数字

### `count_segments`

重放中的段数。更多段表示随时间推移的更多活动。

- **类型：** 数字

### `count_traces`

重放中的跟踪数量。

- **类型：** 数字

### `count_urls`

用户在重放录制期间访问的 URL 数量。

- **类型：** 数字

### `dead.selector`

类似于 `click.selector` 搜索属性，但仅查询[无效点击](/product/explore/session-replay/replay-page-and-filters/#:~:text=Dead%20Clicks%3A%20User%20clicks%20on%20a%20and%20button%20tags%20that%20do%20not%20result%20in%20any%20page%20activity%20after%207%20seconds%20(i.e.%20no%20HTML%20was%20added%2C%20removed%2C%20or%20updated%3B%20no%20visual%20changes%20were%20observed%20in%20the%20page).)。使用 CSS 选择器语法的子集标识的元素。例如，`#section-1` 或 `span.active` 或 `span[role=button]` 或 `.active[role=button]` 都会匹配元素 `<span id="section-1" class="active" role="button"/>`。注意，不支持 CSS 组合器、伪选择器和除 `=` 外的属性选择器。

- **类型：** 字符串

### `device.brand`

设备品牌。

- **类型：** 字符串

### `device.family`

设备系列。通常是跨代型号名称的共同部分。例如，iPhone、Samsung Galaxy。

- **类型：** 字符串

### `device.model_id`

内部硬件版本以精确识别设备。

- **类型：** 不适用

### `device.name`

设备详细信息。

- **类型：** 字符串

### `dist`

区分同一版本应用程序的构建或部署变体。例如，dist 可以是 Xcode 构建的构建号或 Android 构建的版本代码。

- **类型：** 字符串

### `duration`

重放的持续时间（以秒为单位）。

- **类型：** 数字

### `error_ids`

重放中发生的错误事件 ID。

- **类型：** 数组

### `id`

事件或重放 id。在**问题**中，仅使用 ID 值而不使用 `id` 键。

- **类型：** UUID

### `is_archived`

重放是否已存档。

- **类型：** 布尔值

### `level`

事件的严重性（如：fatal、error、warning）。对于事务始终设置为 info。

- **类型：** 字符串

### `os.name`

操作系统的名称。例如，`Windows`、`Mac OS X` 或 `Linux`。

- **类型：** 字符串

### `os.version`

操作系统的版本号。

- **类型：** 字符串

### `platform`

平台名称。这仅是[有效平台](https://github.com/getsentry/relay/blob/9d0d2dc54233d46043a369e1dacf7db900953c14/relay-general/src/protocol/constants.rs#L1-L21)的指标属性，默认为 `other`。

- **类型：** 字符串

### `project_id`

项目的 id。

- **类型：** 字符串

### `rage.selector`

类似于 `click.selector` 搜索属性，但仅查询[愤怒点击](/product/explore/session-replay/replay-page-and-filters/#:~:text=Rage%20Clicks%3A%20Five%20or%20more%20clicks%20on%20a%20dead%20element%20(it%20exhibits%20no%20page%20activity%20after%207%20seconds.)%20Rage%20clicks%20are%20a%20subset%20of%20dead%20clicks)。使用 CSS 选择器语法的子集标识的元素。例如，`#section-1` 或 `span.active` 或 `span[role=button]` 或 `.active[role=button]` 都会匹配元素 `<span id="section-1" class="active" role="button"/>`。注意，不支持 CSS 组合器、伪选择器和除 `=` 外的属性选择器。

- **类型：** 字符串

### `release`

发布是部署到环境的代码版本。您可以使用发布版本的精确匹配创建标记，或 `release:latest` 选择最近的发布。[了解更多](/product/releases/usage/sorting-filtering/#latest-release)。

- **类型：** 字符串

### `replay_type`

触发重放的原因。例如，当 `replaysSessionSampleRate` 生效时为 `session`，或当 `replaysOnErrorSampleRate` 被采样而不是 `session` 时为 `buffer`。在[会话重放文档](/platforms/javascript/session-replay/understanding-sessions/)中了解更多。

- **类型：** 字符串

### `screen`

用户在重放期间访问的特定屏幕或视图。您还可以使用 `screens` 并传入字符串数组一次搜索多个屏幕。
别名：`url`

### `sdk.name`

发送事件的 Sentry SDK 的名称。

- **类型：** 字符串

### `sdk.version`

发送事件的 Sentry SDK 的版本。

- **类型：** 字符串

### `seen_by_me`

您是否已看过此重放。
别名：`viewed_by_me`

- **类型：** 布尔值

### `trace`

跟踪表示您想要测量或跟踪的整个操作的记录——如页面加载，使用 Sentry SDK 生成的 UUID 进行搜索。

- **类型：** UUID

### `url`

用户在重放期间访问的特定 URL。您还可以使用 `urls` 并传入字符串数组一次搜索多个 URL。
别名：`screen`

- **类型：** 字符串

### `user.email`

用户名的替代或补充。Sentry 了解电子邮件地址，因此可以显示 Gravatars 等功能并解锁消息传递功能。

- **类型：** 字符串

### `user.geo.city`

用户地理位置的城市全名。

- **类型：** 字符串

### `user.geo.country_code`

用户地理位置的国家。国家代码基于 [ISO 3166-1](https://www.iso.org/iso-3166-country-codes.html)。

- **类型：** 字符串

### `user.geo.region`

用户地理位置的国家全名。

- **类型：** 字符串

### `user.geo.subdivision`

用户地理位置的行政区划全名。（例如，州、省等）。

- **类型：** 字符串

### `user.id`

用户的应用程序特定内部标识符。

- **类型：** 字符串

### `user.ip`

用户的 IP 地址。Sentry 使用 IP 地址作为未验证用户的唯一标识符。

- **类型：** 字符串

### `user.username`

用户名，通常比 `user.id` 更好的标签。

- **类型：** 字符串

</DefinitionList>

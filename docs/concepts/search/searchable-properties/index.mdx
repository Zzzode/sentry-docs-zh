Sentry的搜索为您提供保留关键字，如`is`、`user`、`server`和`browser`，您可以使用这些关键字来搜索问题、事件和回放的属性（以及发布的特殊情况）。您还可以创建[自定义标签](#自定义标签)来进行搜索。本页面提供如何使用这些属性的指导，并链接到相应的可搜索属性列表。

## 搜索属性

Sentry的可搜索属性分为以下五个类别：

- [问题属性](/concepts/search/searchable-properties/issues/)
- [事件属性](/concepts/search/searchable-properties/events/)
- [跨度属性](/concepts/search/searchable-properties/spans/)
- [会话回放属性](/concepts/search/searchable-properties/session-replay/)
- [用户反馈属性](/concepts/search/searchable-properties/user-feedback/)
- [发布属性](/concepts/search/searchable-properties/releases/)

## 搜索提示

搜索词应该自动完成，当它们不自动完成时，意味着您有一个无效的数据集。如果您输入无效的搜索词并按Return或Enter键，会显示错误消息。例如，在**发现**中，如果您在"结果"表列中没有包含分组函数，则无法使用分组函数（键中包含括号的属性，例如`epm()`）进行搜索。

当您使用作为函数的属性进行搜索时，如果键包含参数，您仍然需要添加过滤器或值以拥有完整的搜索token。例如，函数`count_unique(field)`将您选择的任何字段作为参数，然后您添加一个数字作为要过滤的值。因此，典型的搜索可能如下所示：`count_unique(user):>=20`。

其他需要注意的事项：

- 持续时间或数字类型的搜索属性通常与[比较运算符](/concepts/search/#comparison-operators)一起使用，而不仅仅是冒号（`:`）来查找精确匹配，因为精确匹配不太可能存在。
- 带有"不接受参数"标记的属性仍然需要过滤器或值，但不在括号内接受参数。例如，`epm():>12`。
- 搜索函数中的默认字段/参数可以更新。例如，可以为`count_if(column,operator,value)`更新默认参数`column`、`operator`和`value`。

## 自定义标签

此外，您可以使用您指定的任何标签作为token。标签是分配给事件的各种键/值对，您可以在以后使用它们作为细分或快速访问来查找相关事件。

大多数SDK通常通过<PlatformLink to="/enriching-events/tags/">配置作用域</PlatformLink>来支持配置标签。

标签的几个常见用途包括：

- 服务器的主机名
- 您平台的版本（例如，iOS 5.0）
- 用户的语言

## 设备分类

`device.class`为开发人员提供了一种简单的方法，通过单个可搜索属性了解事件客户端设备的性能级别。这对于服务于大量移动设备范围的项目特别有用，在这种情况下，开发人员通常不得不解析iOS和Android上大量的规格和型号。

`device.class`的可能值为`high`、`medium`和`low`，表示设备的估计性能级别。这是一个基于以下规格计算的属性：

| iOS的`device.class` | `model`                             |
| ---------------------- | ----------------------------------- |
| high                   | iPhone 12系列或更高          |
| medium                 | iPhone 7系列到iPhone 11系列 |
| low                    | iPhone 6系列或更低            |

| Android的`device.class` | `processor_count` | `processor_frequency` | `memory_size` |
| -------------------------- | ----------------- | --------------------- | ------------- |
| high                       | >= 8              | >= 2500 MHz           | >= 6 GiB      |
| medium                     | >= 8              | >= 2000 MHz           | >= 4 GiB      |
| low                        | < 8               | < 2000 MHz            | < 4 GiB       |

这些分类基于对当今市场上可用移动设备的分析，并可能随着市场发展而变化。

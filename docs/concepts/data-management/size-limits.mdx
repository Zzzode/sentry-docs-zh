Sentry 对事件、附件、请求和各种事件字段施加大小限制。

## 行为

以下描述了 Sentry 如何处理大小限制：

- 超出有效负载大小限制的事件、附件和请求将立即被丢弃，并显示 `413 有效负载过大` 错误。
- Sentry 允许压缩内容编码，并在解压缩前后应用单独的限制。
- 压缩后超过 200KB 或解压缩后超过 1MB 的事件将被拒绝。
- 压缩后超过 20MB 或解压缩后超过 100MB（所有文件合计）的 minidump 上传也将被拒绝。
- 超出单个大小限制的事件字段将被修剪和截断。
- 由于大小限制而被拒绝的事件数量计入[使用统计](/product/stats/#usage-stats)中的_已丢弃_类别。

## 建议

为了避免使用您的附件配额（基于大小而不是发送的附件数据实例数量）或完全丢弃事件，请考虑限制传递给 Sentry API 的值的大小。例如，如果您的应用程序将应用程序状态或请求正文附加到 Sentry 事件，请先截断它们。

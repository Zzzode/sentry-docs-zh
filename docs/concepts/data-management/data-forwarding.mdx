Sentry 提供了将处理过的错误事件转发到某些第三方提供商的能力，例如 [Segment](https://segment.com) 和 [Amazon SQS](https://aws.amazon.com/sqs/)。

当您可能希望更深入地分析异常或授权其他团队（如商业智能功能）时，这通常很有用。

通过导航到**[项目] » 设置 » 数据转发**来配置数据转发，然后为给定集成提供所需的信息。

只有错误事件会被转发。不支持事务事件的转发。

## Amazon SQS

与 Amazon SQS 的集成使得将异常快速轻松地传回您自己的系统变得简单。

Amazon 的有效负载与我们的标准 API 事件负载相同，并将随着时间的推移而发展。有关此数据格式的更多详细信息，请参阅我们的 [API 文档](/api/events/retrieve-an-event-for-a-project/)。

## Segment

Segment 集成将在您的数据管道中生成_错误捕获_事件。这些事件**仅**为 `error` 事件捕获，并且仅在用户上下文中存在 ID 时捕获。事件的一般形状看起来大致如下：

```json
{
  "userId": "1",
  "event": "错误捕获",
  "properties": {
    "environment": "production",
    "eventId": "002c8bbde8324dae9f12a0b96f5b1e51",
    "exceptionType": "ValueError",
    "release": "a2def1",
    "transaction": "/api/0/users/{user}/",
    "userAgent": "Mozilla/5.0 (X11; Linux x86_64; rv:54.0) Gecko/20100101 Firefox/54.0",
    "page": {
      "url": "https://sentry.io/api/0/users/{user}/",
      "method": "GET",
      "search": "",
      "referer": "https://sentry.io/"
    }
  },
  "timestamp": "2017-05-20T15:29:06Z"
}
```

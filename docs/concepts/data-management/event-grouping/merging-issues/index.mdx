如果您有看起来相似但尚未自动分组的问题，并且您希望减少噪音，您可以通过合并它们来实现。

在此示例中，合并前有两个非常相似但尚未自动分组的问题。一个问题在过去 24 小时内有 39 个事件，另一个有 76 个：
![合并前](./img/before-merge.png)

将两个问题合并为一个后，该单个问题的事件计数 115 反映了它们已被分组在一起：
![合并后](./img/after-merge.png)

您还可以从单个问题内部执行此操作。点击"类似问题"选项卡，选择您要合并的所有问题，然后点击"合并"：
![类似问题](./img/similar-issues.png)

如果您想取消合并问题，请转到**问题详细信息**页面并点击"合并问题"选项卡。您将看到已合并到该问题中的指纹。选择适当的复选框并点击"取消合并"。

需要注意的是，我们不会从您合并问题的方式推断任何新的分组规则。未来事件将通过与现在合并的单个问题中相同的条件添加到合并的问题集中。

如果 Sentry 创建的问题多于您可以合并的问题，或问题太少，您可以尝试更新您的[指纹规则](/concepts/data-management/event-grouping/fingerprint-rules/)或[堆栈跟踪规则](/concepts/data-management/event-grouping/stack-trace-rules/)。

Sentry除了提供自托管解决方案外，还提供云托管的软件即服务（SaaS）解决方案，两者在功能上是相同的。然而，许多客户发现自托管的Sentry很快就会变得维护、扩展和支持成本高昂，使得我们的SaaS产品成为更好且成本更低的选择。为了促进从自托管到SaaS的迁移，我们提供了一个自助式流程，称为"迁移"（relocation）。

## 迁移到云端

如果您的组织最初使用的是自托管的Sentry，您已经在配置和定制您的Sentry组织方面投入了时间和精力。如果您决定迁移到SaaS账户，您的工作不会白费，因为您不需要从头开始。您可以将大部分已配置的内容迁移到您的新账户中。

为了使这种迁移尽可能无缝，Sentry提供了一个工作流程，允许您在迁移到SaaS时保留大部分账户设置和配置。您所需要的只是对自托管安装的owner访问权限，以及通过终端访问它的方式。

### 您将保留的内容

所有配置、团队和项目结构以及大部分元数据都将被迁移。具体来说，对于您[选择迁移的每个组织](#2-选择要迁移的组织)，这包括：

- 所有组织设置
- 用户账户和所有组织成员选项
- 项目及其设置
- 团队及其设置
- 告警规则
- 定时任务（Crons）
- 仪表板
- 启用的数据清理器
- 仓库
- 最近和保存的搜索

### 会遗留的内容

与特定问题、事件、堆栈跟踪或文件（如回放和源映射）相关的任何数据都不会被转移。此外，出于安全原因，您需要重新连接和重新验证您的所有集成，您的用户需要在迁移后重新连接他们的2FA并创建新的API令牌。

以下是一些不会被迁移的项目的非详尽列表：

- 集成，包括迁移项目依赖集成的部分（例如：如果告警规则触发Slack通知，该触发器将被省略）
- Webhooks
- 问题（Issues）
- 事件（Events）
- 回放和/或其他[事件附件](/platforms/javascript/enriching-events/attachments/)
- 源映射、[调试信息文件](/cli/dif/)和/或[调试ID](/platforms/javascript/sourcemaps/troubleshooting_js/debug-ids/)
- 发布信息，包括通过特定拉取请求对外部仓库的任何引用
- 部署信息
- 自定义头像

需要注意的一点是，虽然我们无法转移历史事件数据，但新的传入事件将在您[更新DSN](#5-更新您的dsn)后立即填充您迁移组织中的项目。所有迁移的组织成员将在迁移完成后自动通过电子邮件收到通知和重新邀请。

### 版本支持窗口

自助迁移仅可能从两个最新的次要自托管版本进行。也就是说，如果最新的自托管版本是24.2.3，您的自托管安装必须至少为24.1.0才能成功迁移。

如果您想从更早的版本迁移，我们要求您先将自托管安装[更新](https://develop.sentry.dev/self-hosted/releases/)到最新版本。如果由于某种原因无法做到这一点，请[联系支持](https://sentry.io/contact/enterprise/)获取更多指导。

### 数据安全

在迁移过程中，您的数据将端到端加密。唯一会通过网络传输的文件将是一个加密的压缩包，我们永远不会以明文形式存储它，并将在迁移开始后30天内删除。

## 如何迁移

### 1. 使用促销代码注册

在开始迁移过程之前，请在Sentry上[注册](https://sentry.io/signup/relocate/)一个新账户。如果您已有Sentry账户，请登录并导航到[迁移着陆页](https://sentry.io/relocation/)。无论哪种情况，您都应该看到如下页面：

![迁移开始页面](./img/relocation-landing-page.png)

**要获得100美元积分，请务必在此页面使用促销代码`relocate100`。**

### 2. 选择要迁移的组织

您的自托管安装可能有多个组织，每个组织都由唯一的slug标识。您需要决定要保留哪些组织slug。您始终可以在迁移后通过转到**组织设置**页面来更改它们。

![在设置页面上更改组织slug](./img/org-slug.png)

### 3. 遵循应用内迁移说明

其余的应用内迁移说明将指导您完成如何导出自托管实例可迁移数据的具体细节。这将需要对您运行自托管实例的机器进行终端访问。以下是快速概述：

- 您将被要求将我们在SaaS后端生成的公钥保存到您的自托管机器上
- 然后您将使用此密钥执行一个命令，该命令将生成一个包含您所有数据的加密压缩包
- 最后，您将被提示上传此压缩包

大多数迁移在一小时内完成，尽管某些大型或复杂的情况可能需要更长时间。如果您在24小时内没有收到我们的更新，请[联系支持](https://sentry.io/contact/enterprise/)获取进一步帮助。

### 4. 选择计划

迁移完成后，您可以选择最适合您需求的[定价计划](https://sentry.io/pricing/)。请注意，计划定价根据您的[事件量](/product/accounts/quotas/)而有所不同。

如果您预期会有更高的量，或者您对我们的企业功能感兴趣（例如优先支持和SLA，以及单租户选项），请[联系我们](https://sentry.io/contact/enterprise/)了解我们的**企业计划**。

### 5. 更新您的DSN

虽然您的项目数据已完全迁移，但所有项目的DSN都将已更改，因为它们现在位于https://sentry.io，而不是您之前的自托管域名。这意味着您需要更新您在每个相关项目的每个SDK中使用的[DSN](/product/sentry-basics/dsn-explainer/)。完成此步骤后，您应该开始像以前一样看到问题和事件（尽管希望不会太多！）。

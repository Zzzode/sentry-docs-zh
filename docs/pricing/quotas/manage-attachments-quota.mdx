将所有错误附件发送到 Sentry 可能会过快地消耗您的配额，因此我们提供了控制存储的附件 _类型_ 和 _数量_ 的工具。

这些技巧将帮助您管理附件配额，并按从最简单（或最省时）到最具挑战性（或可能最耗时）的顺序排列。前三个可以在 [sentry.io](https://sentry.io) 中完成。最后一个将在您的 SDK 中完成。

## 开始之前：检查您的配额使用情况

您可以在 **统计** 的"使用情况统计"选项卡中查看您的聚合附件。这些信息将帮助您回答关于传入附件分解或哪些项目正在消耗您的配额的关键问题。这些问题的答案可以帮助您确定需要在何处对 SDK 过滤器和配置进行进一步微调。

此页面对您组织的所有成员都可访问，因此您 Sentry 组织中的所有者可以与直接负责给定项目的开发人员共享此页面。此外，您可以返回此页面评估您所做的更改是否产生了预期效果。

### 如何查看传入附件的分解？

[使用情况统计](/product/stats/#usage-stats)选项卡显示整个组织最多90天内 Sentry 接收的数据总量的详细信息。该页面将事件（按项目）分为三类：_已接受_、_已丢弃_ 或 _已过滤_。只有已接受的附件会影响您的配额：

![使用情况统计页面概览](./img/usage-stats-attachments.png)

### 我最忙的项目是什么？

**统计** 的"使用情况统计"选项卡中的"项目"表按项目分解您的数据，以便您可以看到哪些项目正在消耗您的配额。点击表中项目名称旁边的设置图标将打开项目设置页面，您可以在其中管理其入站过滤器和速率限制

## 调整配额

<Alert>

只有您 Sentry 组织的账单或所有者成员才能更新配额。

</Alert>

一旦您的附件量接近或已超过配额，具有"所有者"组织权限级别的队友将收到[通知](/product/alerts/notifications/#quota-notifications)电子邮件。然后您可以选择[增加](#increasing-quotas)或[减少](#decreasing-quotas)您的配额。

### 增加配额

如果您因为超出配额而丢弃了您想要保留的附件，请通过在计费期间的任何时间增加您的[预留容量或按量付费](/pricing/#pricing-how-it-works)预算来添加到您的配额。

当您超出配额阈值时，服务器将响应429 HTTP状态码，该状态码向SDK和客户端传达停止发送事件及其关联附件的信号。此状态码附带一个`Retry-After`标头，指示此速率限制处于活动状态的时间。但是，客户端不应重试事件，而应丢弃事件直到速率限制到期，以防止队列积压。请注意，由于事件摄取和速率限制是异步发生的，429 HTTP状态码总是略有延迟。

要增加您的配额，请转到 **设置 > 订阅** 并点击"管理订阅"按钮以访问您的订阅选项。当您增加配额时，更改会立即生效。

如果您使用的是开发者计划并希望增加配额，则需要升级到团队或商业计划。在这些计划中，您可以预付更多附件量并按需购买按量付费预算。在我们的[定价页面](https://sentry.io/pricing/)上了解 Sentry 的计划。

<Alert>

如果您在计费期间从团队升级到商业，您的按需定价会追溯更改。

</Alert>

### 预留容量

如果您需要的附件数据量稳步增加，您可能需要增加预留容量或量。预留容量比按量付费预算便宜，因为您需要预付费用。在我们的定价文档中了解更多关于[预留容量](/pricing/#pricing-how-it-works)的信息。

如果您认为您对更多事件的需求是暂时的，则不应增加预留容量，因为[减少预留容量与您的计费周期相关](#decreasing-quotas)。

### 按量付费预算

如果您需要临时增加附件配额，我们建议您添加或增加按量付费预算。这非常适合像推出新版本应用程序这样的情况，您预计当月会收到更多数据。在我们的定价模型中了解更多信息[此处](/pricing/#pricing-how-it-works)。

### 减少配额

计划降级和预留容量减少在您的计费期结束时处理，剩余容量不能退款。例如，如果您的月度计费期从每月5日开始，而您在6月20日减少了预留容量，则此更改将在7月4日处理。从7月5日开始的计费期将反映您的新预留容量。

<Alert level="warning">

如果您有年度计费期，计划降级和预留容量减少将在您的**下一个计费年度**开始时生效。

</Alert>

对按量付费预算的更改通常会立即生效，并保证在24小时内生效。但是，您不能将按量付费预算减少到低于您在当前期间已消耗的金额。

要减少您的配额，请转到 **设置 > 订阅** 并点击"管理订阅"。当您到达"查看并确认"步骤时，将显示这些更改生效的日期。

<Alert>

我们强烈建议您在计费期的最后一天**之前**进行订阅更改。根据您的时区，在某些情况下，在计费期的最后一天所做的更改要到下一个计费期才会生效。

</Alert>

## 速率限制

如果您已启用崩溃报告的存储，您可以为每个问题存储的最大崩溃报告数量设置限制。要设置这些限制，请转到 **设置 > 安全与隐私** 并在"存储原生崩溃报告"下拉菜单中进行选择。此设置适用于您的整个组织。

## 入站数据过滤器

SDK 配置需要更改源代码并取决于您的下一次部署，但服务器端过滤器可以在 **[项目] > 设置 > 入站过滤器** 的"数据过滤器"部分中为每个项目轻松配置。

虽然您不能直接为附件设置过滤器，但当您为错误事件设置入站过滤器时，其关联的附件会被间接过滤。在[管理您的错误配额](/pricing/quotas/manage-event-stream-guide/)中了解有关为错误事件设置数据过滤器的更多信息。

## 附件大小

为了避免用完您的附件配额（这是基于大小而不是发送的附件数量），或完全避免附件被丢弃，请考虑限制传递到 Sentry API 的值的大小。例如，如果您的应用程序将应用程序状态或请求主体附加到 Sentry 事件，请先截断它们。

有关更多信息，请参阅我们的[大小限制](/concepts/data-management/size-limits)指南。

将所有应用程序错误发送到 Sentry 可以确保您在代码中发生错误时实时收到通知。但是，通过基本设置，您可能会过快地消耗错误配额，并触发比您可以使用更多的通知。Sentry 提供了控制被监控的错误的 _类型_ 和 _数量_ 的工具。这些工具允许您：

1. 接收更具可操作性和意义的错误事件。
2. 为实际破坏代码的错误保留实时通知。
3. 管理您的错误配额。

应用适当的过滤器、SDK 配置和速率限制是一个迭代且持续的过程，但这些技巧将向您展示如何充分利用您的错误事件。这些技巧从最简单或最省时的到最具挑战性或可能最耗时的进行排序。选项 1-5 都是您可以在 [sentry.io](https://sentry.io) 中做的事情，而其余的则是您将在 SDK 中要做的事情。

## 开始之前：检查您的配额使用情况

您可以在 **统计** 的"使用情况统计"选项卡中查看您的聚合事件。这些信息将帮助您回答关于传入事件分解或哪些项目正在消耗您的配额的关键问题。这些问题的答案可以帮助您确定需要在何处对 SDK 过滤器和配置进行进一步微调。

此页面对您组织的所有成员都可访问，因此您 Sentry 组织中的所有者可以与直接负责给定项目的开发人员共享此页面。此外，您可以返回此页面检查您所做的更改是否产生了预期效果。

## 宽限期

所有账户都可以使用一次性宽限期。这意味着如果账户耗尽了错误事件量，并且之前从未触发过宽限期，我们将以每小时最多10,000个事件的最大速率继续接受事件三天。如果您选择不增加事件量，Sentry 将在当前计费月的剩余时间内停止接受事件。但是，如果您增加量，宽限期内接受的错误事件将计为消耗量。

虽然此宽限期不依赖于[峰值保护功能](#7-spike-protection)，但您可能会收到已进入宽限期的通知以及峰值通知。

## 峰值保护

峰值保护保护用户免受事件量激增的影响。通常，峰值是由异常引起的，有可能迅速耗尽您可用的事件量。您的基线消耗用于计算阈值，如果事件量超过此阈值，则会触发峰值保护。当触发峰值保护时，事件开始被丢弃，确保您不会因超额量而被收费。

## 调整配额

<Alert>

只有您 Sentry 组织的账单或所有者成员才能更新配额。

</Alert>

一旦您的事件量接近或已超过配额，具有"所有者"组织权限级别的队友将收到[通知](/product/alerts/notifications/#quota-notifications)电子邮件。然后您可以选择[增加](#increasing-quotas)或[减少](#decreasing-quotas)您的配额。

如果这是您第一次超出错误配额并且您使用的是付费计划，您将进入一次性宽限期。通过阅读此[帮助文章](https://sentry.zendesk.com/hc/en-us/articles/23983502355099-What-happens-when-I-run-out-of-event-capacity-and-a-grace-period-is-triggered)了解更多信息。

### 增加配额

如果您因为超出配额而丢弃了您想要保留的事件，请通过在计费期间的任何时间增加您的[预留](#reserved-volume)或[按量付费](/pricing/#pricing-how-it-works)配额来添加到您的配额。

当您超出配额阈值时，服务器将响应429 HTTP状态码，该状态码向SDK和客户端传达停止发送事件的信号。此状态码附带一个`Retry-After`标头，指示此速率限制处于活动状态的时间。但是，客户端不应重试事件，而应丢弃事件直到速率限制到期，以防止队列积压。请注意，由于事件摄取和速率限制是异步发生的，429 HTTP状态码总是略有延迟。

要增加您的配额，请转到 **设置 > 订阅** 并点击"管理订阅"按钮以访问您的订阅选项。当您增加配额时，更改会立即生效。

如果您使用的是开发者计划并希望增加配额，则需要升级到团队或商业计划。在这些计划中，您可以预付更多事件量并按需购买按量付费量。在我们的[定价页面](https://sentry.io/pricing/)上了解 Sentry 的计划。

<Alert>

如果您在计费期间从团队升级到商业，您的按量付费定价会追溯更改。

</Alert>

### 预留容量

如果您需要的事件数量稳步增加，您可能需要增加预留容量或量。预留容量比按量付费量便宜，因为您需要预付费用。它还允许您提前选择您想要可用的事件数量，而不仅仅是设置任意的按量付费预算。在我们的定价文档中了解更多关于[预留容量](/pricing/#prepaid-reserved-capacity)的信息。

如果您认为您对更多事件的需求是暂时的，则不应增加预留容量，因为[减少预留容量与您的计费周期相关](#decreasing-quotas)。

### 按量付费预算

如果您需要临时增加事件配额，我们建议您添加或增加按量付费预算。这非常适合像推出新版本应用程序这样的情况，您预计当月会有更多事件。在我们的定价文档中了解更多关于[按量付费预算](/pricing/#pricing-how-it-works)的信息。

### 减少配额

计划降级和预留容量减少在您的计费期结束时处理，剩余容量不能退款。例如，如果您的月度计费期从每月5日开始，而您在6月20日减少了预留容量，则此更改将在7月4日处理。从7月5日开始的计费期将反映您的新预留容量。

<Alert level="warning">

如果您有年度计费期，计划降级和预留容量减少将在您的**下一个计费年度**开始时生效。

</Alert>

对按量付费量的更改通常会立即生效，并保证在24小时内生效。但是，您不能将按量付费预算减少到低于您在当前期间已消耗的金额。

要减少您的配额，请转到 **设置 > 订阅** 并点击"管理订阅"。当您到达"查看并确认"步骤时，将显示这些更改生效的日期。

<Alert>

我们强烈建议您在计费期的最后一天**之前**进行订阅更改。根据您的时区，在某些情况下，在计费期的最后一天所做的更改要到下一个计费期才会生效。

</Alert>

## 速率限制

<Include name="feature-available-for-plan-business.mdx" />

速率限制允许您设置项目密钥在时间段内将接受的最大错误事件量。例如，如果您有一个在生产中生成大量噪音的项目，速率限制允许您设置最大数据量，例如"每分钟500个事件"。此外，您可以为同一项目创建第二个密钥用于暂存环境，该环境不受限制，确保您的 QA 过程仍然不受影响。

在 **设置 > 项目 > [项目] > SDK 设置 > 客户端密钥 (DSN)** 中，点击"配置"，您可以为每个项目创建多个 DSN 密钥，并为每个密钥分配不同的（或无）速率限制。这将允许您根据发布、环境等动态分配具有不同阈值的密钥。

![每个 DSN 密钥的速率限制](./img/manage-event-stream-11.png)

设置限制后，由于速率限制而丢弃的事件会生成429错误代码。

虽然速率限制对于管理您的月度事件配额非常有用，但请记住，一旦超过定义的阈值，后续事件将被丢弃。因此，您不应不断达到速率限制；相反，它应该作为旨在保护您免受意外峰值的天花板。

### 设置有用的速率限制

设置项目速率限制的一个好方法是根据您的平均流量计算预期事件量。以下是如何做到这一点：

1. 转到 **设置 > 项目 > [项目] > SDK 设置 > 客户端密钥 (DSN)** 并通过点击"配置"打开项目 DSN 密钥配置。
1. 在"过去30天的密钥使用情况"图表中，查找最高点或最大日速率。在下面的示例中，过去一个月的最大日速率小于34K：
   ![计算速率限制](./img/manage-event-stream-14-new.png)
   {/*<!-- 图像注释：Sentry, airflow, Santry, app-frontend -->*/}
1. 根据速率，选择**每日**最大值或上限。在此示例中，我们计算的每日最大值约为35K，约为**每小时**1458个事件，或约**每分钟**24个事件。
1. 设置每日、每小时或基于分钟的速率限制。我们建议使用基于分钟的速率，以避免随机事件峰值可能耗尽您的每日或每小时配额并使您长时间没有事件数据的情况。

您应该定期返回并检查图表，查看由于速率限制而丢弃的事件数量，如果需要，请重新访问您的设置。

![重新访问速率限制](./img/manage-event-stream-15-new.png)

## 审查事件重复

以下规则适用于错误事件重复和您的配额：

- 如果您之前已解决某个问题，则新的错误事件将计入您的配额，因为它可能代表代码中的回归。
- 如果您选择忽略具有相同指纹的错误事件的通知，则新事件将计入您的配额，因为事件仍在发生。
- 如果您["删除并丢弃"](#delete--discard)某个问题，则**未来**具有相同指纹的错误事件将不计入您的配额。

### 删除并丢弃

<Include name="feature-available-for-plan-business.mdx" />

如果您有一个无法或不愿解决的无关的、重复出现的问题，您可以从**问题详情**页面通过点击"归档"来将其归档（临时或永久）。这将过滤掉未来的匹配事件。

![归档问题](./img/manage-event-stream-06-new.png)

您可以在**设置 > 项目 > [项目] > 处理 > 入站过滤器**中的"已丢弃问题"选项卡中找到已删除和已丢弃问题的列表。从这里，您可以取消丢弃任何这些问题以接收未来事件。

您始终可以通过在**问题**页面的搜索栏中添加 `is: archived` 来查看已归档的问题，并可以从那里取消归档它们。

![已归档问题列表](./img/manage-event-stream-07-new.png)

一旦您确定了一组已丢弃的问题，返回您的 SDK 配置并将相关错误添加到您的 `beforeSend` 客户端过滤中可能是有意义的。

## 入站数据过滤器

虽然 SDK 配置需要更改源代码并取决于您的下一次部署，但服务器端过滤器可以在 **设置 > 项目 > [项目] > 处理 > 入站过滤器** 的"数据过滤器"部分中为每个项目轻松配置。

应用后，您可以使用"入站数据过滤器"页面顶部提供的图表跟踪过滤的事件（数量和原因）。

![内置入站过滤器](./img/manage-event-stream-03.png)

Sentry 提供了几种在服务器端过滤所有事件和附件的方法，这些都是在检查潜在速率限制之前应用的。入站过滤器包括：

- 常见浏览器扩展错误
- 来自 `localhost` 的事件
- 已知旧版浏览器错误
- 已知网络爬虫
- 错误消息
- 代码的特定发布版本
- 某些 IP 地址

上述前四个选项是您只需打开或关闭的设置。设置其他的解释如下。

在这些检查处理完毕后，未根据这些过滤器丢弃的事件和附件会计入您的配额。它们被接受进入 Sentry，在那里它们被持久化和存储。

### IP 过滤器

如果您有恶意客户端，Sentry 支持阻止 IP 发送数据。导航到 **[项目] > 设置 > 项目 > [项目] > 入站过滤器** 以在"IP 地址"字段中添加 IP 地址（或子网）。

### 按发布版本过滤

<Include name="feature-available-for-plan-business.mdx" />

如果您发现导致过多噪音的问题发布版本，Sentry 支持忽略来自该发布版本的所有事件和附件。导航到 **设置 > 项目 > [项目] > 入站过滤器**，然后将发布版本添加到"发布版本"字段。

### 按错误消息过滤

<Include name="feature-available-for-plan-business.mdx" />

Sentry 还支持过滤掉特定或某种类型的错误。导航到 **设置 > 项目 > [项目] > 入站过滤器**，然后将错误消息添加到"错误消息"字段。

为确保您向入站过滤器设置添加了正确的消息，请检查问题中事件的 JSON。按错误消息过滤设置与文件末尾附近的 `title` 字段中找到的数据匹配。

## SDK 采样率

如果为 SDK 定义了采样率，则 SDK 评估是否应将此事件作为事件的代表性部分发送。为每个 SDK [记录](/platform-redirect/?next=/configuration/sampling/)设置采样率。

SDK 采样率不是动态的；更改它需要重新部署。此外，请记住，设置 SDK 采样率会限制对事件源的可见性。为您的项目设置速率限制可能更适合您的需求。

## SDK 过滤：beforeSend

所有 Sentry SDK 都支持 `beforeSend` 回调方法。实现后，当 SDK 捕获错误事件时，在将其发送到您的 Sentry 账户之前调用该方法。它以事件对象作为参数接收，因此您可以使用它来根据事件上可用的自定义逻辑和数据（如 _标签_、_环境_、_发布版本_、_错误属性_ 等）修改事件数据或完全丢弃事件（通过返回 `null`）。请注意，只有错误和消息事件通过 `beforeSend`。事务事件有一个单独的方法 `beforeSendTransaction`，尽管并非所有 SDK 都支持它。在[过滤事件](/platform-redirect/?next=/configuration/filtering/)中了解有关这两种方法的更多信息。

## SDK 配置

Sentry SDK 有几个配置选项可用于过滤不需要的错误，使其不会离开应用程序的运行时。其中许多选项是特定于平台的，因此请确保在我们的平台文档中的[配置](/platform-redirect/?next=/configuration/)下查找您的选项。以下是 JavaScript 和其他 SDK 的一些配置示例。

### JavaScript

JavaScript SDK 包括多个集成：您可以配置、启用或禁用的功能插件。在 [JavaScript SDK 集成](/platforms/javascript/configuration/integrations/)中了解更多信息。几个集成允许您配置您希望 Sentry 监控的事件类型；其中两个，InboundFilters 和 GlobalHandlers，解释如下：

#### InboundFilters

集成默认启用，并为 SDK 添加以下配置选项：

- `allowUrls`：可能引发可接受异常的域，以正则表达式模式格式表示。
- `denyUrls`：匹配应阻止发送事件的错误 URL 的字符串或正则表达式模式列表。在 SDK 上配置 `allowUrls` 和 `denyUrls` 都可用于阻止 `allowUrls` 中列出的域的子域。
- `ignoreErrors`：指示 SDK 如果与列出的任何错误消息或正则表达式匹配，则永远不要向 Sentry 发送错误。SDK 将尝试匹配消息，如果没有消息，则匹配包含异常类型和值的字符串，格式如下：`ExceptionType: value`。
- `ignoreTransactions`：指示 SDK 如果与列出的任何事务名称或正则表达式匹配，则永远不要向 Sentry 发送事务。

要了解更多信息并查看代码示例，请查看：

- [过滤](/platforms/javascript/configuration/filtering/)
- [丰富事件：添加上下文](/platforms/javascript/enriching-events/context/)

#### GlobalHandlers

此集成附加全局处理程序以捕获未捕获的异常 (`onerror`) 和未处理的拒绝 (`onunhandledrejection`)。两个处理程序默认启用，但可以通过配置禁用。在 [GlobalHandlers 集成](/platforms/javascript/configuration/integrations/globalhandlers/)中了解更多信息。

查看 [BrowserApiErrors](/platforms/javascript/configuration/integrations/browserapierrors/) 和 [ReportingObserver](/platforms/javascript/configuration/integrations/reportingobserver/) 集成的其他配置选项。

### 其他 SDK

**Java** - Sentry 的 [Java SDK](/platforms/java/) 提供了与常见 Java 日志记录库的集成。配置允许您设置日志记录阈值，确定捕获日志消息作为面包屑的最小级别。它还提供了一个设置来配置捕获事件的最小日志级别。

**PHP** - `error_types` 配置选项允许您设置您希望 Sentry 监控的错误类型。在 PHP [基本选项](/platforms/php/configuration/options/#common-options)中了解更多信息。

**Ruby** - `excluded_exceptions` 配置选项允许您设置您希望抑制的异常类型。在 [Ruby 配置文档](/platforms/ruby/configuration/options/#optional-settings)中了解更多信息

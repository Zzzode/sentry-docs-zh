<Alert>
[日志](/product/explore/logs/)目前在团队、商业和开发者计划中可用。这些计划包括一定量的日志数据处理和存储。一旦超出包含的配额，您将根据按量付费（PAYG）预算被收费。
</Alert>

## 关键术语

- **日志配额**：您的订阅为处理和存储来自应用程序的日志数据分配的初始配额。

- **按量付费（PAYG）预算**：您设置的用于支付超出包含配额的额外日志处理的预算。

- **日志条目**：从您的应用程序发送到 Sentry 的单个日志消息，包括时间戳、日志级别和自定义属性等元数据。

- **日志处理**：在 Sentry 的基础设施中摄取、解析、索引和存储日志数据。

## 开始之前：检查您的日志使用情况

您可以在统计和使用情况页面上查看您的日志使用情况，以了解按时间窗口和项目划分的日志条目分解情况。这可能有助于您确定需要在何处进一步微调使用情况。

此页面对您组织的所有成员都可访问，因此您 Sentry 组织中的所有者将能够与直接负责给定项目的开发人员共享它。您还能够使用此页面来评估您的更改是否按预期工作。

### 如何查看传入日志的分解？

统计和使用情况页面还显示整个组织最多90天内 Sentry 接收的日志数据总量的详细信息。该页面按项目将事件分为三类：已接受、已丢弃或已过滤。只有已接受的日志会影响您的配额：

![统计和使用情况页面示例](./img/usage-stats-errors.png)

### 哪些项目正在消耗我的日志支出？

统计页面上的"项目"表按项目分解您的数据，以便您可以识别消耗最多配额的项目。这些可以按总日志、已接受日志、已过滤日志等进行排序。

## 调整 PAYG 预算

只有您 Sentry 组织的账单或所有者级别的成员才能更新预算。

一旦您的日志配额接近或已超过其包含量，具有"所有者"组织权限的队友将开始接收[通知](/product/alerts/notifications/#quota-notifications)电子邮件。然后他们将能够选择增加或减少 PAYG 预算以解锁额外使用。

### 增加日志预算

如果您因为超出日志配额而无法发送新日志，您可以在计费期间的任何时间通过增加 PAYG 预算来添加到您的预算。这非常适合推出新版本应用程序或需要增加日志记录以进行调试的情况。要添加 PAYG 预算，请在您的[订阅页面](https://sentry.io/orgredirect/organizations/:orgslug/settings/billing)中设置月度最大共享预算。

[了解更多关于设置您的 PAYG 预算的信息](/pricing/#pricing-how-it-works)。

## 管理支出

### SDK 配置

对于各种 SDK，有能力在发送到 Sentry 之前过滤掉日志，因此您可以避免发送您不关心的日志。这是减少日志配额使用的最有效方法；以下是几个示例：

#### JavaScript SDK

```js
Sentry.init({
  dsn: "___PUBLIC_DSN___",
  enableLogs: true,
  beforeSendLog: (log) => {
    if (log.level === "info") {
      // 过滤掉所有 info 日志
      return null;
    }
    return log;
  },
});
```

#### Python SDK

```python
import sentry_sdk
from sentry_sdk.types import Log, Hint
from typing import Optional

def before_log(log: Log, _hint: Hint) -> Optional[Log]:
    # 过滤掉所有 info 级别日志
    if log["severity_text"] == "info":
        return None
    return log

sentry_sdk.init(
    dsn="___PUBLIC_DSN___",
    "enable_logs": True,
    "before_send_log": before_log,
)
```

### 日志过滤

您还可以使用[入站过滤器](/concepts/data-management/filtering/#logs-filtering)在服务器级别过滤日志。虽然这不会阻止数据发送到 Sentry，但它确实会阻止过滤的日志计入您的配额。

以下入站过滤器适用于日志：

- **日志消息** - 基于日志消息匹配过滤日志
- **发布版本** - 过滤来自特定发布版本的日志

要设置日志消息过滤：

1. 导航到 **[项目] > 项目设置 > 入站过滤器**
2. 添加模式以过滤掉特定日志消息
3. 使用通配符进行灵活匹配，例如，`*连接超时*`，以过滤所有包含"连接超时"的日志

### 减少日志配额使用的最佳实践

1. **设置适当的日志级别**：仅记录生产所需的级别，例如，WARNING 及以上。
2. **使用结构化日志记录**：在日志属性中包含相关上下文，而不是冗长的消息
4. **在源头过滤**：使用 `beforeSendLog` 回调在发送前过滤日志
5. **监控使用模式**：定期检查您的使用情况统计，以识别日志消耗高的项目
6. **设置警报**：在接近配额限制时配置通知

### 监控您的日志使用情况

通过以下方式跟踪您的日志配额使用情况：

- 定期检查[使用情况统计](/product/stats/#usage-stats)页面
- 设置[配额通知](/product/alerts/notifications/#quota-notifications)
- 在统计分解中查看项目特定使用情况
- 监控过滤更改随时间的影响

通过实施这些策略，您可以有效地管理日志配额，同时保持对应用程序行为所需的可见性。

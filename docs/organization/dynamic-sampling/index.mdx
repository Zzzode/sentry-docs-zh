## 概述

生成大量事件量的项目可能会快速消耗您的计费配额。这可能会造成不平衡，高容量项目获得更多可见性，为监控其他项目留下不足的配额。这就是动态采样的用武之地。

动态采样采用先进的采样技术来保留您发送到 Sentry 的数据的代表性样本。其算法旨在优先保留对调试和分析有价值的样本，同时减少常见或冗余数据的保留。
<Alert>
动态采样仅适用于 span 和事务，不适用于错误。
</Alert>

根据您组织的计划类型，有两种可用的动态采样风格：
- **带有采样优先级的动态采样** - 适用于选定的团队、商业和企业计划\
  通过分析传入流量模式，动态采样能够根据实际流量量和接受事务的内容自动调整其采样率。有关更多详细信息，请查看 [动态采样优先级](#动态采样优先级) 部分。
- **带有采样优先级和自定义采样率的动态采样** - 适用于选定的企业计划\
  直接从 UI 配置和调整存储 span 的采样率，无需修改您的 SDK。这使得您可以进行即时更新，而无需等待代码冻结、应用商店批准或重新部署。此外，通过分析传入流量模式，动态采样能够根据接受的 span 内容优先处理数据。有关更多详细信息，请查看 [自定义采样率配置](#自定义采样率配置) 部分。

## 自定义采样率配置
<Alert>
自定义采样率配置自 2024 年 6 月起在选定的企业计划中可用。
</Alert>
在本节中，您将了解如何在您的组织中使用动态采样。动态采样根据所需的采样控制提供两种模式：

- **默认模式** 允许您设置组织级目标采样率，并自动调整项目级采样以匹配组织级采样率。
- **高级模式** 通过允许您手动在项目级别设置采样率，为您提供更精细的控制。

![动态采样设置概述](./img/mode-and-target-sample-rate.png)

### 默认模式

要在默认模式下使用动态采样，您必须为整个组织指定一个目标采样率，该采样率决定应在所有项目中存储多少传入流量。然后，动态采样会自动将此采样率分配到您的项目中。采样率会持续调整，以考虑事件量随时间的变化。

**执行以下调整：**

- 低容量项目获得更高的采样率，确保不会错过重要数据。
- 高容量项目具有降低的采样率，以平衡整体存储。
- 根据最近的流量模式持续调整采样率，使用短滑动窗口来适应季节性变化，如假期、周末和夜间流量。

动态采样还优先处理每个项目分配的采样率中最有价值的数据。您可以在 [动态采样优先级](#动态采样优先级) 部分中了解更多信息。这个动态系统确保您的采样适应不断变化的流量，并在项目之间保持平衡。

### 高级模式

如果您想要更精细的控制，您可以通过启用高级模式为特定项目设置单独的采样率来自定义数据保留。

在这里，您可以为组织中的每个项目配置固定的采样率。此值将指定应为各自项目存储的传入流量的百分比。采样率可以在 0% 到 100% 之间，如果未调整，则默认为 100%。

![高级动态采样设置](./img/advanced.png)

当您编辑各个项目的采样率时，还将计算您组织的估计整体采样率。这是基于每个项目流量量的加权平均值。请记住，此值是估计值，可能会随着流量量的波动而变化。

要一次性调整所有项目的采样率，请通过转到 [动态采样设置页面](https://sentry.io/orgredirect/organizations/:orgslug/settings/dynamic-sampling/) 来编辑您组织的整体采样率。这将按比例更新每个项目的采样率以满足您选择的目标。最大速率为 100%，调整将在项目之间平衡速率。您将能够在保存之前查看和完成更新的采样率。确保点击"应用更改"按钮以保存新的采样率。

![在高级模式下一次性更改所有项目](./img/advanced-org-level.png)

无论是否激活高级模式，[动态采样优先级](#动态采样优先级) 都将在每个项目中应用，提升某些 span 并降低其他 span 的优先级。

当您停用高级模式时，重新校准过程将立即开始，每个项目的采样率将重置为计算值。一旦您停用了高级模式，就无法恢复手动配置的项目采样率。

### 项目预览

[项目预览](https://sentry.io/orgredirect/organizations/:orgslug/settings/dynamic-sampling/) 表在动态采样设置页面上显示您选择的项目可能如何受到您选择的采样率的影响。预览根据过去 24 小时或过去 30 天的流量显示您项目的动态分配率。

![动态采样模式和目标采样率设置](./img/project-sample-rate-preview.png)

由于项目率持续变化，我们建议您选择 24 小时预览，以获得当前采样率的更准确视图，用于实时分析。我们建议 30 天预览用于查看计费周期中的平均采样率，用于长期规划。

## 分布式跟踪的采样率

由于 [分布式跟踪](/product/tracing/#whats-distributed-tracing) 可以在一个项目中发起，并具有跨多个其他项目的 span，因此重要的是要注意，Sentry **始终应用跟踪发起项目的采样率** 到该跟踪的每个项目。

例如，如果跟踪在您的前端应用程序中开始，您在那里设置的采样率将应用于同一跟踪的一部分的已连接后端应用程序。如果跟踪在后端应用程序中开始，同样的规则也适用。

如果您导航到 [项目预览](https://sentry.io/orgredirect/organizations/:orgslug/settings/dynamic-sampling/) 表，您将看到发送的 span 数量，位于跟踪开始的原始项目旁边。这包括连接到原始项目的其他项目中传播跟踪的任何 span。这使您可以直接查看您配置的采样率将影响多少 span。要查看按项目所属的 span 计数细分，请点击预览中每个项目左侧的展开图标。

![显示动态采样对分布式跟踪影响的图表](./img/client-to-backend-accumulated-sampling-effect.png)

## SDK 与动态采样

SDK 采样发生在应用程序级别，在事件发送到 Sentry 之前。它通过仅发送设定的百分比来过滤事件，减少数据量和检测开销。这有助于直接在源头管理高事件量。

另一方面，动态采样发生在事件发送到 Sentry 后的服务器级别。采样率根据 Sentry 接收到的事件量动态调整，无需对应用程序进行任何更改。

SDK 采样率通常只能在部署应用程序的新版本时更改，除非实现了自定义机制，使得可以动态更改采样率。但是使用动态采样，可以为组织中的每个项目动态配置采样率，而无需对应用程序进行任何更改。

此外，当高级模式未激活时，动态采样持续监控每个项目的流量，并调整每个项目的采样率，以优先处理最相关的数据。

### 何时使用动态采样与 SDK 采样

动态采样的最大优势是其灵活性。它根据 Sentry 接收到的事件量调整采样率。这使其成为具有高低容量项目的组织的绝佳选择，因为它通过根据每个项目的事件量定制采样率来帮助最大化您的配额。

动态采样可以根据其内容优先处理 span，比 SDK 提供更多洞察。例如，它可以专注于来自最新版本或低容量项目的 span，让您更好地了解应用程序最重要的部分。

SDK 采样在 SDK 配置中使用静态速率。它非常适合具有一致事件量的组织，这些组织希望限制发送到 Sentry 的事件百分比。这有助于减少 span 以降低开销、节省带宽（特别是对于移动应用程序）或减少接受的 span 数量。

### 决定您的 SDK 采样率

为了让 Sentry 获得最完整、最准确的应用程序健康状况，我们建议您向我们发送 100% 的事件。您可以通过将 [tracesSampleRate](/platform-redirect/?next=/performance/) 设置为 1.0 来实现这一点。如果您的组织不可行，请设置较低的值，或使用 [tracesSampler](/platform-redirect/?next=/performance/) 根据上下文数据选择性地过滤事件。请注意，在计算吞吐量时，计量基于接收的事件，而不是存储的事件。

<Alert>
  对 SDK 采样配置的更改可能会影响您的配额。如果您
  增加 SDK 采样率，您还将增加
  发送到 Sentry 的事件量。这可能需要您调整配额。
</Alert>

您可以同时使用动态采样和 SDK 采样，以进一步优化特定用例的事件量。例如，您可以设置较低的 SDK 采样率，以确保仅将一定百分比的事件发送到 Sentry，然后使用动态采样根据 Sentry 接收到的事件量调整采样率。

### 如何更改动态采样优先级

要在 Sentry 中更改项目的动态采样优先级，请转到 **设置 > 项目**，选择您感兴趣的项目，然后在"处理"部分中向下滚动到"性能"，并使用切换开关在"动态采样优先级"下进行任何更新。

### 自动优先级

Sentry 根据以下规则自动优先处理或降低优先级：

#### 最新发布

最新版本是动态采样优先级，因为当您创建新版本时，我们假设您希望在早期采用阶段获得更多可见性，从而提高您在版本获得关注时捕获新问题的能力。
由于您可能希望在不同环境中部署相同版本，我们还将考虑您的环境并优先处理这些跟踪。

#### 开发环境

由于开发环境通常与生产环境相比生成少量事件，我们在测试阶段优先处理它们，以便您可以获得有意义的洞察。如果环境名称匹配以下 glob 模式之一，我们认为环境是开发环境：

- `*debug*`
- `*dev*`
- `*local*`
- `*qa*`
- `*test*`

#### 低容量项目

<Alert>
  您将无法禁用低容量项目动态采样优先级
  因为它是在组织范围内设置的，而不是在每个项目的基础上设置的。
</Alert>

您组织中的一些项目与其他项目相比容量非常高。这意味着如果我们平等对待所有项目，低容量项目可能会被淹没。为了确保无论项目规模如何，您都有足够的样本，我们优先处理低容量项目。

#### 低容量事务

span 可以按其在项目中的事务名称进行逻辑分组。这意味着具有相同事务名称的 span 很可能表示相同的操作或代码路径，而具有不同事务名称的 span 通常表示不同的操作。
我们优先处理低容量事务的原因与我们优先处理低容量项目的原因相同：这样它们就不会被高容量事务淹没。我们希望确保低容量代码路径以足够的数量进行采样，以生成应用程序的完整视图。

#### 健康检查

我们降低健康检查类型事务的优先级，因为虽然它们对于检查应用程序的稳定性很重要，但它们除了与它们关联的任务外没有任何价值。如果事务名称匹配以下 glob 模式之一，我们认为事务是健康检查：

- `*healthcheck*`
- `*heartbeat*`
- `*/health`
- `*/healthy`
- `*/healthz`
- `*/live`
- `*/livez`
- `*/ready`
- `*/readyz`
- `*/ping`

### 获取样本

我们的自动动态采样优先级以通用方式很好地收集基线样本，因为它们优先保留在任何给定时间对任何客户有价值的数据。但是，在某些情况下，例如调查特定问题时，非常特定的数据会暂时变得更加重要。在这种情况下，自动采样优先级可能无法提供足够的样本，因此您可能需要创建调查规则。

调查规则是可以通过 Sentry UI 创建的采样规则，用于临时增加特定场景（例如，具有特定标签值的事件）收集的事件数量。当可用样本太少（少于 5 个）时，可以创建这些规则以保留更多样本。创建后，规则将保持活动状态最多 48 小时或直到收集 100 个样本。当收集足够的事件时，您将通过电子邮件收到通知。

调查采样规则仅适用于事务，不保证完整跟踪将被采样。可以同时存在的最大采样规则数量为 50。

#### 创建新规则

当您在 [发现](/product/explore/discover-queries/) 或 [事务摘要](/product/performance/transaction-summary/) 中查询事务并收到少于 5 个样本时，您可以点击"获取样本"为相应查询的事务创建新的调查规则并保留更多样本。

![发现中的调查模式](./img/investigation-bias-discover.png)

<Alert>

此功能仅适用于事务。因此，在发现中，您必须在查询中指定 `event.type:transaction` 才能访问该功能，而在性能中，事件类型是隐式的。

</Alert>

点击此按钮将为查询创建调查规则。规则成功创建后，将出现通知，按钮将被替换为显示调查正在进行以及其活动时间的消息。

![规则成功创建](./img/investigation-bias-rule-created.png)

如果用户稍后执行相同的搜索，并且调查仍处于活动状态，通知将指示已有活动规则正在为查询收集数据。

也可以从性能中的事务摘要页面创建规则。

![事务摘要中的调查模式](./img/investigation-bias-tx-summary.png)

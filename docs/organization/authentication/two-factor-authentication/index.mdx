为了增强安全性，您可以要求您的组织成员使用双重身份验证 (2FA) 登录 Sentry。

## 前言

需要 Sentry 所有者权限才能在您的组织中强制执行双重身份验证。

1. 在您可以要求组织成员使用 2FA 之前，您必须 [为您的用户账户启用双重身份验证](https://sentry.io/settings/account/security/)。
2. 我们建议通知您的组织成员，并要求他们在要求双重身份验证之前为其账户设置 2FA。您可以在组织的成员页面上查看哪些成员已注册。

   ![](./img/require-2fa-members.png)

## 设置要求 2FA

当您要求双重身份验证时，未注册 2FA 的成员将从您的组织中删除。他们将失去对您的组织项目和通知的访问权限，并将收到一封设置 2FA 的电子邮件。他们可以在从您的组织中删除后的三个月内恢复其访问权限和设置。

1. 转到 **组织设置** >> 常规设置。
2. 在安全和隐私下，切换 **要求双重身份验证** 开关。

   ![](./img/require-2fa-switch.png)

3. 点击 **确认**。

## 查看已删除的成员

要查看因 2FA 不合规而从您的组织中删除的成员，您可以查看组织的审计日志。

1. 转到 **组织设置** >> 审计日志。

   ![](./img/require-2fa-audit-log.png)

2. 您可以过滤操作 **member.pending** 以查看已删除的组织成员，并过滤操作 **org.edit** 以查看 require_2fa 组织设置何时更改。

## 要求 2FA 和 SSO

目前，要求 2FA 不适用于单点登录 (SSO)，因此您需要使用身份提供商要求双重身份验证。

## 重新加入组织的步骤

<Alert>
  要恢复您的访问权限和以前设置，您需要在
  从组织中删除后的三个月内设置 2FA。
</Alert>

如果您因 2FA 不合规而从组织中删除，请按照以下步骤重新加入组织并恢复您的访问权限和设置：

1. 在您的收件箱中查找来自 Sentry 的标题为 **强制性：启用双重身份验证** 的电子邮件。

   ![](./img/require-2fa-email.png)

2. 点击 **启用双重身份验证** 按钮。
3. 如果您找不到电子邮件，请要求组织所有者或经理转到 **组织设置** >> 成员页面并重新发送您的邀请。

   ![](./img/require-2fa-resend-invite.png)

4. 点击电子邮件后，系统将提示您在重新加入组织之前 **设置双重身份验证**。

   ![](./img/require-2fa-rejoin.png)

5. 这将带您到您的 **账户设置** >> [安全](https://sentry.io/settings/account/security/) 页面，您需要设置至少一种双重身份验证形式。

   ![](./img/require-2fa-add-auth.png)

6. 注册 2FA 后，请记住将恢复代码保存在安全的地方，并考虑添加备用电话号码。
7. 然后使用左侧边栏导航到您的组织。

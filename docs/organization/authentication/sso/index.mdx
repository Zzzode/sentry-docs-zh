单点登录（或 SSO）允许您通过第三方提供商管理整个组织的成员资格。

## 前言

在您真正开始启用 SSO 之前，请记住，一旦激活，所有现有用户都需要链接他们的账户，然后才能继续使用 Sentry。因此，我们建议在非高峰时间与您的团队协调。话虽如此，链接账户非常快，所以我们不认为这是一个真正的障碍。

## 入门

说完这些，前往您的组织主页。如果您有 SSO 访问权限，您会在侧边栏中看到一个"Auth"链接。首先点击它，然后继续点击您希望配置的提供商旁边的"配置"链接。

此外，一旦 SSO 成功配置，我们将自动向每个预先存在的成员发送一封包含链接账户说明的电子邮件。即使他们没有点击链接，下次他们尝试访问组织中的任何页面时，我们也会要求他们链接他们的账户（与您刚刚经历的相同的身份验证流程）。

## 默认成员资格

通过 SSO 创建新账户的每个成员都将被授予具有成员角色的全局组织访问权限。这意味着他们可以访问任何团队的事件，但他们将无法创建新项目或管理当前项目。

## 安全性

我们的 SSO 实现优先考虑安全性。我们积极监控链接的账户，如果有任何合理迹象表明账户的访问可能已被撤销，我们将禁用它们。通常这对您来说是透明的，但如果提供商以意外方式运行，您可能会遇到更频繁的重新授权请求。

会话持续时间为 [Django 的默认会话长度](https://docs.djangoproject.com/en/1.11/topics/http/sessions/#using-cookie-based-sessions)，即两周。

## 提供商

### Google 商业应用

<Include name="feature-available-for-plan-trial-team.mdx" />

启用 Google 集成将要求您针对 Google Apps 账户进行身份验证。完成后，成员资格将仅限于给定 Apps 域的成员（即 `sentry.io`）。

<Alert>

如果您需要允许跨多个域的用户访问您的组织或更改您的 Google 域，请联系我们的支持团队。

</Alert>

### GitHub 组织

<Include name="feature-available-for-plan-trial-team.mdx" />

GitHub 集成将针对所有组织进行身份验证，完成后会提示您选择希望限制访问的组织。

### SAML2 身份提供商

<Include name="feature-available-for-plan-business-no-trial.mdx" />

Sentry 提供 [基于 SAML2 的身份验证](https://en.wikipedia.org/wiki/SAML_2.0)，可以使用通用 SAML2 提供商手动配置，或使用下面列出的特定提供商，这些提供商为该身份提供商提供特定默认值：

- [AuthO](/organization/authentication/sso/#auth0)
- [Azure Active Directory](/organization/authentication/sso/#azure-active-directory)
- [Okta](/organization/authentication/sso/#okta)
- [OneLogin](/organization/authentication/sso/#onelogin)
- [Rippling](/organization/authentication/sso/#rippling)
- [Jumpcloud](/organization/authentication/sso/#jumpcloud)

Sentry 支持以下 SAML 服务：

- 身份和服务提供商发起的 SSO
- 身份提供商发起的 SLO（单点注销）

Sentry 的断言消费者服务使用 HTTP-POST 绑定。

</Alert>

#### Auth0

在您的 Auth0 仪表板中，在 SSO 集成页面下找到 Sentry 应用并将其添加到您的组织。

作为 Auth0 SSO 配置的一部分，您必须向 Sentry 提供 Auth0 身份提供商元数据。此 URL 在 Sentry SSO 集成的教程选项卡中可用。

您可以参考 [Auth0 文档](https://auth0.com/docs/integrations/sso/sentry) 获取更详细的设置说明。

#### Azure Active Directory

在您的 Azure AD 仪表板中，在企业应用程序下找到 Sentry 应用并将其添加到您的组织。

您可以参考 [我们的文档](/organization/authentication/sso/azure-sso/) 获取更详细的设置说明。

#### Okta

在您的 Okta 管理员仪表板中，在 Okta 应用程序网络中找到 Sentry 应用并将其添加到您的组织。

作为 Okta SSO 配置的一部分，您必须向 Sentry 提供 Okta 身份提供商元数据。此 URL 可以位于登录方法 SAML2 设置面板下，查找可以右键单击并复制链接地址的'身份提供商元数据'链接。

您可以参考 [我们的文档](/organization/authentication/sso/okta-sso/) 获取更详细的设置说明。

#### OneLogin

在您的 OneLogin 仪表板中，在应用目录中找到 Sentry 应用并将其添加到您的组织。

作为 OneLogin SSO 配置的一部分，您必须向 Sentry 提供 OneLogin 身份提供商颁发者 URL。此 URL 特定于您的 OneLogin 账户，可以在 Sentry OneLogin 应用程序配置页面的"SSO"选项卡上找到。

您可以参考 [OneLogin 文档](https://onelogin.service-now.com/support?id=kb_article&sys_id=1d199dcfdbdc5700d5505eea4b96196a) 获取更详细的设置说明。

#### Rippling

在您的 Rippling 管理员仪表板中，在建议的应用列表中找到 Sentry 应用并选择它。

当提示输入 Rippling 元数据 URL 时，将其复制到 Sentry Rippling 提供商配置中。您必须完成 Rippling 应用程序配置，然后才能完成 Sentry 提供商配置。

#### Jumpcloud

在您的 Jumploud 管理员门户中：

1. 导航到 **用户身份验证 > SSO**。
1. 添加新的 SSO 应用程序并搜索 Sentry。
1. 点击"配置"。
1. 在 SSO 菜单下，为您的租户更新 SP 实体 ID 和 ACS URL。它们应采用 [此处](/organization/authentication/sso/saml2/#register-sentry-with-idp) 描述的格式。
1. 点击"保存"。

确保在尝试这些后续步骤之前，您已在 Jumpcloud 中为自己分配了 Sentry 应用。

1. 在 [sentry.io](https://sentry.io) 中，导航到您组织中的 **设置 > 身份验证**。
1. 点击 Jumpcloud 提供商上的"配置"。
1. 转到注册页面中的"XML"。
1. 通过点击"导出元数据"，在 Jumpcloud Sentry SSO 应用的"SSO"选项卡下下载您的 Jumpcloud 元数据。
1. 将您的 XML 元数据粘贴到文本字段中，然后点击"解析元数据"。
1. 在"映射身份提供商"页面上，如果您为 Sentry 应用保留了 Jumpcloud 属性的默认值，请在各自的字段中填写 'uniqueID'、'email'、'firstname' 和 'lastname'。从"服务提供商属性名称"列复制这些属性，而不是"JumpCloud 属性名称"列。

现在，Sentry 应该已配置为 Jumpcloud SSO。

#### 自定义 SAML2 集成

对于上面未列出的其他 SAML2 SSO 提供商，Sentry 为 [基于 SAML2 的身份验证](/organization/authentication/sso/saml2/) 提供通用连接器，可以手动配置。

## SCIM 配置

<Include name="feature-available-for-plan-business-no-trial.mdx" />

<Alert level="warning">

如果您更改组织 slug，您需要在输入 SCIM 配置值的地方进行相同的更新。

</Alert>

跨域身份管理系统 ([SCIM](http://www.simplecloud.info/)) 是由身份提供商和应用程序实现的标准，用于促进自动身份管理。Sentry 支持该规范的子集，用于配置组织成员和团队。请查看相关文档以了解您的用例：

- [Okta SCIM 设置](/organization/authentication/sso/okta-sso/okta-scim/)
- [Azure AD SCIM 设置](/organization/authentication/sso/azure-sso/#scim-integration)
- 如果您的提供商未在此处列出，SCIM 可能会受支持，因为它是一个通用标准。如果您遇到问题，请联系我们的支持团队。

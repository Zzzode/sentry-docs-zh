Sentry 为 [基于 SAML2 的身份验证](https://en.wikipedia.org/wiki/Security_Assertion_Markup_Language) 提供通用身份验证提供商，允许 Sentry 组织的所有者手动配置任何启用 SAML2 的 IdP 系统。下面记录了一般集成步骤。

# 设置

<Alert level="warning">

如果您更改组织 slug，您需要在输入 SAML 配置值的步骤中进行相同的更新。

</Alert>

Sentry 支持以下 SAML 服务：

- 身份和服务提供商发起的 SSO
- 身份提供商发起的 SLO（单点注销）

## 向 IdP 注册 Sentry

在将 Sentry 连接到身份提供商 (IdP) 之前，首先将 Sentry 注册为 IdP 端的应用程序非常重要。Sentry 的 SAML 端点如下，其中 `YOUR_ORG_SLUG` 替换为您的组织 slug：

<table class="table">
  <tbody valign="top">
    <tr>
      <th>ACS:</th>
      <td>
        <code class="docutils literal">
          https://sentry.io/saml/acs/YOUR_ORG_SLUG/
        </code>
      </td>
    </tr>
    <tr>
      <th>SLS:</th>
      <td>
        <code class="docutils literal">
          https://sentry.io/saml/sls/YOUR_ORG_SLUG/
        </code>
      </td>
    </tr>
    <tr>
      <th>Metadata:</th>
      <td>
        <code class="docutils literal">
          https://sentry.io/saml/metadata/YOUR_ORG_SLUG/
        </code>
      </td>
    </tr>
  </tbody>
</table>

**这三样东西是什么？**

- `ACS` 表示 _断言消费者服务_。它用于根据您的 IdP 与其集成的服务提供商之间制定的规则建立会话。请注意，Sentry 的 ACS 端点使用 HTTP-POST 绑定。
- `SLS` 代表 _单点注销服务_，用于处理来自 IdP 的注销请求。
- `Metadata`，在某些系统中也称为 `entityID`，是指 IdP 或 SP 的配置数据。在这种情况下，Sentry 中的元数据端点是指您的 Sentry 组织在服务提供商端的元数据。

在 IdP 端设置这些值时，重要的是要记住 Sentry 不需要提供签名证书即可使集成正常工作。

## 向 Sentry 注册 IdP

有三种不同的方法可以将您的 IdP 注册到 Sentry：元数据、XML 和数据字段。每种方法都在下面分解，并将产生相同的最终结果。

### 使用元数据 URL

此方法只需要 IdP 平台提供的元数据 URL。提供后，Sentry 将自动获取它需要的元数据。

![](./img/saml2-metadata-url.png)

### 使用提供商 XML

要使此方法起作用，管理员需要提供 IdP 生成的元数据文件的内容。内容直接粘贴到文本字段中后，Sentry 将完成其余工作。请注意，Sentry 不需要签名证书。

![](./img/saml2-provider-xml.png)

以下是元数据 XML 内容的外观示例。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<md:EntityDescriptor xmlns:md="urn:oasis:names:tc:SAML:2.0:metadata" entityID="http://www.okta.com/SENTRY_APP_ID">
  <md:IDPSSODescriptor WantAuthnRequestsSigned="false" protocolSupportEnumeration="urn:oasis:names:tc:SAML:2.0:protocol">
    <md:KeyDescriptor use="signing">
      <ds:KeyInfo xmlns:ds="http://www.w3.org/2000/09/xmldsig#">
        <ds:X509Data>
          <ds:X509Certificate>...</ds:X509Certificate>
        </ds:X509Data>
      </ds:KeyInfo>
    </md:KeyDescriptor>
    <md:NameIDFormat>urn:oasis:names:tc:SAML:1.1:nameid-format:unspecified</md:NameIDFormat>
    <md:NameIDFormat>urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress</md:NameIDFormat>
    <md:SingleSignOnService Binding="urn:oasis:names:tc:SAML:2.0:bindings:HTTP-POST" Location="https://YOUR_OKTA_ORG.okta.com/app/sentry/SENTRY_APP_ID/sso/saml" />
    <md:SingleSignOnService Binding="urn:oasis:names:tc:SAML:2.0:bindings:HTTP-Redirect" Location="https://YOUR_OKTA_ORG.okta.com/app/sentry/SENTRY_APP_ID/sso/saml" />
  </md:IDPSSODescriptor>
</md:EntityDescriptor>
```

### 使用提供商数据

此注册方法最复杂，需要匹配来自 IdP 的数据字段。完成此操作的最简单方法是在元数据文件中查找值，例如上面提供的值。

![](./img/saml2-provider-data.png)

基于上面的 XML 文件示例，要插入的字段值如下：

- `EntityID`
- `X509Certificate`
- `SingleSignOnService`

在此示例中，`SingleLogoutService` 未由 IdP 提供，并且在 Sentry 中被视为可选值。

## 映射 IdP 属性

<Alert title="关于字段名称的说明">

元数据字段名称可能因提供商而异。例如，Microsoft Azure AD 将这些元数据字段称为 **声明**，而 Okta 将它们称为 **属性**。同样，一个平台可能使用 **user.email**，而另一个供应商使用 **emailaddress**。

</Alert>

在这里，Sentry 成员的字段值需要与 IdP 中成员的相应值匹配。基本必需字段是 IdP 的用户 ID 和电子邮件地址，但 Sentry 也可以从中选择性地提取名字和姓氏值。

![](./img/saml2-map-attributes.png)

## 确认成功注册

SAML 集成流程完成后，相关身份验证页面将反映成功集成的状态。从这里，您可以向在集成之前存在的任何现有成员发送提醒，他们将收到链接其账户的电子邮件提示。

![](./img/saml2-success.png)

由于 Sentry 使用即时 (JIT) 配置，新成员在其首次使用 SAML SSO 登录尝试期间会自动在 Sentry 中注册。这些账户将根据 Sentry 中组织设置中的默认成员资格角色分配其成员资格类型。

## SCIM 配置

SCIM 可用于 SAML2 IdP 的自动用户配置、取消配置和团队分配。[为 SCIM 配置 Sentry](/organization/authentication/sso/okta-sso/okta-scim/#sentry-configuration)，然后按照 IdP 的文档进行进一步设置。在 [SCIM API 文档](/api/scim/) 中了解有关 Sentry SCIM API 端点的更多信息。

## 常见问题

#### 将新成员添加到启用 SAML2 的 Sentry 组织的过程是什么？

Sentry 利用成员账户的即时配置；在身份提供商中创建的任何新成员将在该成员首次尝试登录 Sentry 时在 Sentry 中自动创建账户。

#### 在启用/强制执行 SAML 之前存在的 Sentry 账户会发生什么？

现有成员将收到一封电子邮件，通知他们新的 SAML 身份验证选项，一旦启用（无论是否强制执行），他们将能够将其 IdP 系统中的账户与其 Sentry 成员资格链接。

#### 如果成员不再存在于身份提供商中，Sentry 是否会取消配置成员？

目前，Sentry 的 SAML2 集成不会自动取消配置非活动用户账户。
相反，成员仍保留在 Sentry 中，但无法登录，因为他们无法再访问 IdP 平台进行登录。目前，非活动成员账户需要由 Sentry 中的经理或所有者手动删除。

#### 尝试使用 IdP 设置 SAML2 SSO 失败，并显示消息"提供商未返回有效用户身份"。这里发生了什么？

这里问题的核心是，不同的 IdP 平台（Okta、Azure AD 等）使用不同的术语和约定来表示集成工作所必需的字段。因此，可能会将不正确的值映射到 Sentry，导致 SSO 失败并显示此错误消息。

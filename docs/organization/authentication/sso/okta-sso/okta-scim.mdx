如果您更改组织 slug，您还需要在 Sentry 的"组织设置"的"身份验证"部分中更新它。

</Alert>

### 支持的功能

- 创建用户
- 停用用户
- 推送组
- 导入组
- 配置组织级角色
- 更新用户属性（仅限组织级角色）

### 要求

Okta SCIM 配置需要：

- 订阅 Sentry 商业计划或更高版本。
- 为 Okta 配置 SAML SSO [如此处所述](/organization/authentication/sso/okta-sso/)，或使用 Sentry Okta 应用程序的 Okta 登录选项卡配置 SAML。
- **在 Okta 的登录应用程序选项卡中选择电子邮件作为应用程序用户名格式**。
  ![Okta 用户名配置](./img/scim-username-config.png)

### Sentry 配置

1. 登录 [sentry.io](https://sentry.io)。选择 **设置 > 身份验证**
   ![Sentry Okta SCIM 身份验证页面](./img/scim-settings-auth.png)
1. 在常规设置下选择"启用 SCIM"，然后"保存设置"
   ![Sentry 启用 SCIM 说明](./img/scim-enable-sentry.png)
   Sentry 将显示"SCIM 信息"，其中包含您的身份验证令牌和 SCIM 基本 URL。
   ![SCIM 凭据字段](./img/scim-sentry-fields.png)

### Okta 配置

1. 使用您的管理员账户登录到您的 Okta 组织。从管理员控制台侧边栏，选择应用程序 > 应用程序，然后选择现有的 Sentry 应用程序。

2. 选择"配置"选项卡，然后"配置 API 集成"。
   ![Okta 配置 SCIM API](./img/scim-okta-configure.png)

3. 选择"启用 API 集成"，然后输入来自身份验证设置页面的 SCIM URL 作为基本 URL 字段。

4. 对于 API 令牌，请从身份验证设置页面复制身份验证令牌值。

5. 选择"测试 API 凭据"。您应该会看到"应用程序验证成功"消息出现。

6. 选择"保存"以被引导到 SCIM 应用程序设置。

7. 在配置页面上，选择"到应用程序"，然后"编辑"：
   ![Okta 保存 SCIM 设置](./img/scim-okta-edit.png)

8. 启用"创建用户"和"停用用户"，然后点击"保存"按钮。
   ![Okta 创建和停用用户](./img/okta-user-sync.png)

一旦进行了这些更改，新分配的用户将收到邀请电子邮件。如果用户被取消分配，他们将从 Sentry 中的组织中删除。

您可以使用"推送组"来同步和分配 Okta 中的组；它们将反映在 Sentry 团队中。

<Alert>

如果您使用 Okta 来分配组织成员资格，您将无法通过 Sentry 进行成员资格更改，并且需要继续使用 Okta。要删除这些用户，请在 Okta 中取消配置它们。

</Alert>

![Okta 配置用户无法离开](./img/okta-provisioned-unable-to-leave.png)

### 配置组织级角色

以下是如何为 Okta 组分配组织级角色的方法：

1.  向您的 Okta 应用程序配置文件添加新的自定义属性

    1. 导航到 Okta 中的应用程序设置

    2. 在"配置"选项卡下，选择"转到配置文件编辑器"
       ![Okta 配置页面](./img/okta-go-to-profile.png)

    3. 选择"+ 添加属性"
       ![Okta 添加属性](./img/okta-add-attribute.png)

    4. 使用以下设置填写表单（对于下面未列出的任何设置，您可以设置您想要的任何值。）：

       - 数据类型：`string`
       - 显示名称：`组织角色`
       - 变量名称：`sentryOrgRole`
       - 外部名称：`sentryOrgRole`
       - 外部命名空间：`urn:ietf:params:scim:schemas:core:2.0:User`
       - 属性类型：`组`

       ![Okta 自定义属性](./img/okta-custom-attribute-values.png)

2.  将组分配给您的 okta 应用程序
    ![Okta 分配组](./img/okta-assign-group.png)

3.  在表单中，输入组织级角色的字符串
    ![Okta 设置角色](./img/okta-set-group-attribute.png)

- 如果 `sentryOrgField` 字段留空，组成员将被配置为默认组织级角色。可以在 Sentry 中配置此默认角色，在设置 -> 组织 -> 身份验证下。否则，角色必须是以下之一：
  - 管理员
  - 经理
  - 计费
  - 成员
- 无效的角色名称将阻止组成员被配置。要重试，您需要先删除该组。
- 出于安全原因，无法通过 SCIM 配置"所有者"角色。但是，您 <i>可以</i> 取消配置在 Sentry 中具有"所有者"角色但未通过 SCIM 配置的用户。
  - 对于具有自定义角色的自托管用户，这扩展到任何具有 `org:admin` 权限的角色

<Alert>

如果用户是多个 Okta 组的成员，则仅使用第一个组设置的属性。

</Alert>

<Alert>

如果您使用 Okta 来分配组织级角色，您将无法通过 Sentry 进行成员资格更改，并且需要继续使用 Okta。要更改用户的组织级角色，您需要取消配置，然后使用新角色重新配置它们。

</Alert>

使用 Okta 分配角色的用户将只能通过 Okta 进行成员资格更改。

![Okta 角色受限角色选择](./img/okta-role-restricted-sentry-role-select.png)

### 更新用户属性

目前，Sentry 仅支持更新组织级角色的用户属性的能力。一旦您向 Okta 应用程序配置文件添加了 `sentryOrgRole` 属性，您就可以编辑分配给应用程序的用户的属性。

![Okta 编辑属性](./img/okta-edit-sentryOrgRole-attribute.png)

用户在 Sentry 中的角色将反映其身份验证设置中的组织角色。这意味着如果您将用户的属性更改为空白，其组织级角色将从 Sentry 中删除。

## 故障排除

### 我在 Okta 中配置用户时收到"找不到匹配用户"错误。

确保在 Okta 的 Sentry 应用程序的"配置"选项卡中启用了 `创建用户` 和 `停用用户` [（请参阅 Okta 配置第 7 步）](#okta-配置)。

### 我将 Sentry 分配给一个组，但没有创建团队。

在 Okta 中将 Sentry 分配给组与将 Sentry 分配给该组的每个成员相同。从功能上讲，这
在 Sentry 中配置每个组成员，但不创建团队。要创建团队，请从 Okta 中 Sentry 应用程序的 `推送组` 选项卡将组推送到 Sentry。

### 我将一个组推送到 Sentry，但团队为空。

通过 Okta 推送组告诉 Sentry 创建一个与 Okta 组同名的新团队。只有那些已被配置为该组成员的用户才会被添加到 Sentry 团队中。确保 Okta 中的组包含适当的用户，并且它们已被配置。

### 我将一个组推送到 Sentry，但收到"找不到用户"错误。

如果可能，隔离导致错误的用户。然后从 Sentry 中删除用户，并使用 Okta 重新配置它们。

### 我无法使用 Sentry 应用程序删除其成员资格由身份提供商管理的成员。

当团队成员资格由身份提供商（如 Okta）管理时，只能通过该身份提供商更新或撤销。

### 我无法使用 Sentry 应用程序添加或删除团队成员。

如果您使用 Okta 将成员分配给团队，您将无法通过 Sentry 进行成员资格更改，并且需要继续使用 Okta。

### 我使用 Okta 从 Sentry 中删除了用户，但他们仍然显示在我的 Sentry 组织中。

确保在 Okta 的 Sentry 应用程序的"配置"选项卡中启用了 `停用用户`。重新配置用户，然后再次删除它们。

### 如何删除我通过 Okta 设置的用户的组织级角色？

将用户的角色属性更改为空白。

## 已知问题

- 目前不支持导入用户功能，因为 Sentry 的 SCIM API 目前不支持用户 `name` 属性字段：`firstName` 和 `lastName`。为了兼容性，它们作为 `N/A` 值返回。
- 如果为任何给定用户将 `active` 字段设置为 `false`，其组织成员记录将被删除。
- 资源唯一支持的过滤操作是 `eq`。
- 配置新团队时，Sentry 将规范化团队 `displayName` 并将大写转换为小写，并将空格转换为破折号。
- GET /Groups 端点无法在组中返回超过 10000 个成员，请参阅 [SCIM API 文档](/api/scim/) 了解更多信息。

## 评估跟踪

Sentry 可以跟踪应用程序中发生的标志评估。标志评估将显示在问题详情页面的"功能标志"部分中，以表格形式呈现，其中"可疑"标志预测会以黄色突出显示。通过阅读[问题详情页面文档](/product/issues/issue-details/#feature-flags)了解如何在 Sentry UI 中与功能标志洞察进行交互。

### 设置评估跟踪

要设置评估跟踪，请访问我们支持的某个语言页面：
* [JavaScript](/platforms/javascript/configuration/integrations/statsig/)
* [Python](/platforms/python/integrations/statsig/)

## 变更跟踪

Sentry 可以跟踪功能标志定义的更改并报告可疑的功能标志编辑。

### 设置变更跟踪

启用变更跟踪是一个五步过程。要开始，请在新标签页中访问[功能标志设置页面](https://sentry.io/orgredirect/organizations/:orgslug/settings/feature-flags/change-tracking/)。然后按照下面列出的步骤操作。

1. **点击"添加新提供商"按钮。**
    - 每种提供商类型可以注册一个 webhook 密钥。
    - 在下拉菜单中选择 Statsig，该下拉菜单显示"选择提供商"。
2. **注册 webhook URL。**
    ![Statsig Webhook 模态框](./img/statsig-webhook-modal.png)
    - 转到您的 Statsig 控制台并导航到 `/integrations/` 页面，可以通过点击左侧边栏导航中的设置，然后在"产品配置"标题下点击"集成"找到该页面。
    - 选择 Webhook 选项，这将打开一个模态框。
    - 复制 Sentry 设置中提供的 webhook URL 并将其粘贴到模态框的"Webhook URL"输入框中。
3. **设置签名密钥。**
    - 复制 URL 下方的 Webhook 签名密钥，并将其粘贴到 Sentry 设置中"密钥"旁边的输入框中。
    - 请妥善保管此字符串，并将其记在某处以备将来需要时使用。当您重新打开 webhook 模态框时，Statsig 仍会向您显示密钥。请记住，如果您点击"生成新密钥"，您将需要再次将密钥添加到 Sentry 设置中。
4. **可选：事件过滤器。**
   - 我们目前仅支持来自此 webhook 的变更跟踪事件。为避免发送不必要的事件，请点击 Statsig 模态框右上角的"事件过滤"，然后勾选"配置更改"。点击保存以确认。
   - 如果您只想跟踪来自 Statsig 开发、暂存和/或生产环境的更改，请在事件过滤中的"配置更改" > "环境"下勾选相应的环境。点击保存以确认。
   - 与环境类似，您还可以按"标签"、"目标应用"或"团队"过滤配置更改。
   ![Statsig 事件过滤器](./img/statsig-event-filters.png)

5. **保存您的 webhook。**
    - 通过点击 Sentry 设置中的"添加提供商"来保存密钥。
    - 通过点击 Statsig 中的"确认"来保存 webhook。

保存后，Sentry 现在将接受并验证发送到您组织的功能标志 webhook 端点的所有入站 hook。

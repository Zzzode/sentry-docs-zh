## 评估跟踪

Sentry 可以跟踪应用程序中发生的标志评估。标志评估将显示在问题详情页面的"功能标志"部分中，以表格形式呈现，其中"可疑"标志预测会以黄色突出显示。通过阅读[问题详情页面文档](/product/issues/issue-details/#feature-flags)了解如何在 Sentry UI 中与功能标志洞察进行交互。

### 设置评估跟踪

要设置评估跟踪，请访问[探索页面](/product/explore/feature-flags/)并选择您选择的语言和 SDK。不使用支持的 SDK？没关系。我们支持您现有系统的通用解决方案。

要设置通用评估跟踪，请访问我们支持的某个语言页面：
* [JavaScript](/platforms/javascript/configuration/integrations/featureflags/)
* [Python](/platforms/python/feature-flags/#generic-api)

## 变更跟踪

Sentry 可以跟踪功能标志定义的更改并报告可疑的功能标志编辑。

对于通用情况，您需要编写一个符合我们 API 的 webhook。您可以在下面找到有关如何执行此操作的说明。

### 设置变更跟踪

启用变更跟踪是一个四步过程。要开始，请在新标签页中访问[功能标志设置页面](https://sentry.io/orgredirect/organizations/:orgslug/settings/feature-flags/change-tracking/)。然后按照下面列出的步骤操作。

1. **点击"添加新提供商"按钮。**
    - 每种提供商类型可以注册一个 webhook 密钥。
    - 在下拉菜单中选择 Generic，该下拉菜单显示"选择提供商"。
2. **保存 webhook URL。**
    - 从 Sentry 设置中复制 webhook URL 并将其保存以供步骤 4 使用。
3. **设置签名密钥。**
    - 在您的功能标记系统的 UI 中，为此 webhook 查找或创建"签名密钥"。如果您正在编写自己的 webhook，密钥长度必须为 10-64 个字符（我们建议至少 32 个）。
    - 复制签名密钥并将其粘贴到 Sentry 设置中"密钥"旁边的输入框中。
    - 通过点击 Sentry 设置中的"添加提供商"来保存密钥。
4. **编写您自己的 webhook。**
    - 配置您的系统，在功能标志定义更改时向 webhook URL 发送 POST 请求。
    - 使用步骤 3 中的签名密钥对您的 webhook 负载进行签名。
    - 有关更多详细信息，请参见 [#api-documentation](/organization/integrations/feature-flag/generic/#api-documentation)。

### API 文档

本节记录了我们的身份验证过程以及资源的字段和结构。

#### 身份验证

身份验证使用"签名密钥"执行。"签名密钥"必须用于在您的功能标记系统中对 webhook 负载进行签名。使用您的密钥对负载进行签名会产生"签名"。Sentry 需要此签名，以便我们可以将我们的签名函数的结果与您的进行比较。双方必须使用相同的签名函数。Sentry 使用 webhook 负载的 HMAC SHA256 十六进制摘要。在 Python 中，此函数如下所示：

```python
def hmac_sha256_hex_digest(secret: str, message: bytes):
    return hmac.new(secret.encode(), message, hashlib.sha256).hexdigest()
```

此函数的结果必须作为标头发送到 Sentry：`X-Sentry-Signature signature_result`。

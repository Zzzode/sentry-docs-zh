此集成需要为您希望使用它的每个项目进行设置。它由 [Sentry 社区](https://open.sentry.io/)维护和支持。

### 注册附加组件

Sentry 为 Heroku 提供了原生附加组件。虽然此附加组件不是必需的，但它将允许您在 Heroku 内维护合并计费，以及轻松配置您的 Sentry 凭据。

要将 Sentry 添加到现有 Heroku 应用，请前往 [Sentry Heroku 附加组件](https://elements.heroku.com/addons/sentry) 页面。

完成后，您将能够通过您的配置确认 Sentry 的凭据是否可用：

```bash
heroku config:get SENTRY_DSN
```

<Alert>

如果您不使用附加组件，您仍然可以绑定 `SENTRY_DSN` 环境变量，SDK 将自动获取它。

</Alert>

### 安装 SDK

无论您是否使用附加组件，您仍然需要按照我们的标准平台特定说明[安装 SDK](/platforms)。

### 配置发布

无论您使用附加组件还是自己配置 Sentry，您还可能希望授予对您的 dyno 元数据的访问权限，这将允许 Sentry 自动获取您的 dyno 正在运行的发布的 git SHA。

为此，启用 `runtime-dyno-metadata` 功能：

```bash
heroku labs:enable runtime-dyno-metadata -a <应用名称>
```

这将公开 `HEROKU_SLUG_COMMIT` 环境变量，大多数 Sentry SDK 将自动检测并使用它进行配置。

接下来，您需要添加您的存储库并设置应用 webhook：

1.  首先将您的存储库连接到您的 Sentry 组织，以便我们自动检索您的提交数据。在**设置 > 集成**中找到您的存储库集成（例如 GitHub、GitLab、Bitbucket）。从配置选项卡中，点击"配置"。

    ![配置 Heroku 集成](./img/heroku-configure-repo-integration.png)

    在配置页面上，点击"添加存储库"，然后选择您要使用的存储库。

    ![添加存储库](./img/heroku-repo-add.png)

2.  在**设置 > 集成**中找到 Heroku 集成，点击"添加到项目"，然后选择您要与 Heroku 一起使用的项目。

    ![Heroku 集成详情页面](./img/heroku-integration-detail.png)

3.  在 Heroku 集成配置中，指定要与您的 Sentry 项目关联的存储库和部署环境。

    ![Heroku 项目配置](./img/heroku-project-config.png)

4.  导航到您项目的发布设置，并将应用 webhook Heroku cli 命令复制到您的 Heroku 配置中。

    ![Heroku 发布配置](./img/heroku-config.png)

5.  运行 Heroku cli 命令后，您将看到一个 Webhooks 签名密钥。复制并粘贴此密钥到 Sentry 中的"Webhook 密钥"字段。

您将开始获得丰富的提交信息和每次新发布的部署电子邮件，以及跟踪问题在哪个发布中被看到。

## 发布和源映射集成

<Alert level="warning">

如果您更改了组织标识符，您需要更新此集成的配置。在我们的[故障排除指南](/organization/integrations/troubleshooting)中了解更多信息。

</Alert>

连接您的 Sentry 和 Vercel 项目，以自动通知 Sentry 每次部署并上传 Next.js 应用程序的源映射。
要了解有关在 Next.js 应用中使用 Sentry 的更多信息，请查看 [Next.js SDK](/platforms/javascript/guides/nextjs/)。

### 安装

<Alert>

需要 Sentry 所有者、管理员或管理员权限才能安装此集成。

</Alert>

1. 访问 [https://vercel.com/integrations/sentry](https://vercel.com/integrations/sentry)

2. 点击"添加集成"并按照设置选择 Vercel 范围、Vercel 项目并查看应用权限。

   ![显示 Vercel 安装的 Sentry 模态框](./img/vercel_install.png)

   <Alert title="重要" level="warning">
     安装创建了一个内部集成来生成用于构建发布的身份验证令牌
     如果 Vercel 内部集成被删除，
     您的 Vercel 集成将停止工作。
   </Alert>

   ![显示 Vercel 内部集成的 Sentry 模态框](./img/vercel_internal_integration.png)

### 配置

使用 Vercel [链接项目](#项目链接)以上传源映射并通知 Sentry 发布部署。

#### 项目链接

1. 当安装程序提示时，选择要链接在一起的 Sentry 项目和 Vercel 项目。

   ![显示将 Sentry 项目链接到 Vercel 项目的 Sentry 模态框](./img/vercel_link_project.png)

   - 此步骤将在选定的 Vercel 项目中生成环境变量。设置这些变量意味着您可以跳过 Sentry webpack 插件设置中的 [CLI 配置步骤](https://github.com/getsentry/sentry-webpack-plugin#cli-configuration)。您可以在 Vercel 的**项目设置 > 常规 > 环境变量**中看到这些变量。

   ![显示将 Sentry 项目链接到 Vercel 项目的 Sentry 模态框](./img/vercel_env_vars.png)

   - `SENTRY_ORG` - 您的 Sentry 组织名称
   - `SENTRY_PROJECT` - 您链接的 Sentry 项目名称
   - `SENTRY_AUTH_TOKEN` - 安装时创建的 Vercel 内部集成的身份验证令牌
   - `NEXT_PUBLIC_SENTRY_DSN` - 链接的 Sentry 项目的 DSN，用于 Next.js SDK

2. 重新部署您的 Vercel 项目以触发发布。

### 使用

- 如果您尚未这样做，[使用 Sentry 检测您的代码](/platforms/javascript/)。
- 确保您已[安装存储库集成](/product/releases/setup/release-automation/)并添加了相关存储库。
- 将 Sentry bundler 插件添加到您的 bundler 配置中（[webpack 插件](https://www.npmjs.com/package/@sentry/webpack-plugin)、[Vite 插件](https://www.npmjs.com/package/@sentry/vite-plugin)、[Esbuild 插件](https://www.npmjs.com/package/@sentry/esbuild-plugin)、[Rollup 插件](https://www.npmjs.com/package/@sentry/rollup-plugin)）。如果您使用的是 Sentry 的 Next.js 或 SvelteKit SDK，这已经为您完成了。
- 如果您已经有一个与 Sentry 集成的 Vercel 项目，请确保您链接的 Sentry 项目是您已经用于报告错误的项目。

### 卸载

1. 您可以从 Vercel 或 Sentry 卸载集成。要在 Sentry 中执行此操作，请导航到**设置 > 集成 > Vercel > 配置**，点击"卸载"并确认。

2. 通过导航到**设置 > 开发者设置**并点击"Vercel 内部集成"旁边的垃圾桶图标来删除创建的内部集成。系统将提示您通过在模态框中提供的字符串来确认删除。输入字符串并点击"确认"以完成删除：

   ![显示卸载 Vercel 集成的 Sentry 模态框](./img/vercel_delete_internal_integration.png)

### 故障排除

#### 获取失败

![获取失败错误消息](./img/vercel_failed_to_fetch.png)

此问题通常发生在您在设置期间有广告拦截器阻止 Vercel 和 Sentry 之间的对话时。要解决此问题，请禁用您的广告拦截器并再次完成安装流程。

## Vercel 市场

Vercel 市场集成允许现有 Vercel 用户通过一键工作流程加入 Sentry。此设置专为**新 Sentry 用户**设计，并在 Vercel 平台内统一计费。

现有 Sentry 组织无法使用 Vercel 市场集成。

当您使用 Vercel 市场配置 Sentry 时，您将能够在设置过程中设置新 Sentry 组织的名称（在 Vercel 中称为"安装"）和项目（在 Vercel 中称为资源/产品）。

### 计费设置

使用 Vercel 市场原生集成时，用户可以在 Vercel 平台内或直接通过 Sentry 修改其组织的计费信息。信用卡设置只能在 Vercel 内修改。

订阅设置只能在 Sentry 内修改。在设置期间，组织可以在开发者、团队或商业计划之间进行选择。可以在初始设置完成后设置按需付费预算。

### 用户访问

Vercel 用户将使用 Vercel 内的"在 Sentry 中打开"按钮对 Sentry 进行单点登录访问，并能够在 Vercel 或 Sentry 中创建新项目。

用户仍然能够直接使用在设置过程中配置的登录凭据登录到他们的 Sentry 组织，而无需使用 Vercel 单点登录。对于非社交登录（Google、Github 等）用户，Sentry 将提示创建密码。

### 自动配置的环境变量

对于配置的每个项目，将在 Vercel 部署中设置以下环境变量：

- **SENTRY_PROJECT**
- **SENTRY_AUTH_TOKEN**
- **NEXT_PUBLIC_SENTRY_DSN**
- **SENTRY_ORG**

### 集成删除

如果在 Vercel 内删除集成，Sentry 组织**将不会**被删除。删除将永久删除 Vercel 账户和 Sentry 账户之间的连接，并且是**不可逆的**。

保留 Sentry 组织允许团队仍然从 Sentry 内访问历史数据，即使他们不再发送监控数据。

如果删除了原生集成，团队仍然能够利用本文档顶部描述的非原生 Vercel 集成。一次只能激活一个集成。

更多信息，请参见[介绍 Vercel 市场博客文章](https://vercel.com/blog/introducing-the-vercel-marketplace)。

## 安装

<Alert>

需要Sentry所有者、经理或管理员权限，以及Bitbucket管理员权限才能安装此集成。

</Alert>

1. 导航到**设置 > 集成 > Bitbucket**。

![安装BitBucket集成](./img/bitbucket-install-pg.png)

2. 如果您已安装旧版Bitbucket集成，请使用Bitbucket旁边的按钮**升级**。如果您没有安装旧版Bitbucket集成，请使用按钮**安装**。

3. 在结果模态框中，点击"添加安装"。

4. 应该会弹出一个Bitbucket安装窗口。选择您想要授予Sentry访问权限的Bitbucket账户，然后点击"授予访问权限"。
5. 在Sentry中您的新Bitbucket实例上，点击"配置"。

   ![配置BitBucket集成](./img/bitbucket-installed.png)

6. 添加您想要从中收集提交数据的任何仓库。

   ![添加仓库](./img/bitbucket-add-repo.png)

Bitbucket现在应该为您的Sentry组织下的所有项目启用。

### Bitbucket Server

#### 生成RSA公钥/私钥对

要生成RSA公钥/私钥对，请在终端窗口中逐个运行以下命令。

```
openssl genrsa -out bitbucket_privatekey.pem 1024
openssl req -newkey rsa:1024 -x509 -key bitbucket_privatekey.pem -out bitbucket_publickey.cer -days 365
openssl pkcs8 -topk8 -nocrypt -in bitbucket_privatekey.pem -out bitbucket_privatekey.pcks8
openssl x509 -pubkey -noout -in bitbucket_publickey.cer  > bitbucket_publickey.pem
```

#### 将您的Bitbucket Server应用程序与Sentry连接

注意：确认[Sentry的IP范围](/security-legal-pii/security/ip-ranges/)是允许的。

1. 在Sentry中，导航到**组织设置** > **集成**。
2. 在Bitbucket Server旁边，点击"安装"。
3. 在结果模态框中，点击"添加安装"。
4. 在结果窗口中，输入您的Bitbucket Server实例的基本URL、您的使用者密钥和您的私钥。点击"提交"。然后，按提示完成OAuth流程。
5. 在Sentry中，您将看到一个新的Bitbucket Server实例出现在集成页面上。

Bitbucket现在应该为您的Sentry组织下的所有项目授权。

## 配置

使用Bitbucket进行[提交跟踪](#提交跟踪)，[问题管理](#问题管理)，以及[在提交中解决](#在提交中解决)。

### 提交跟踪

提交跟踪允许您精确定位有问题的提交。发现哪些提交可能导致问题，并允许您的团队通过在提交消息中引用问题编号来解决这些问题。

有关更多详细信息，请参阅[提交跟踪和发布](/product/releases/setup/)的完整文档。

### 问题管理

<Alert>

此功能仅支持旧版Bitbucket实例。有关更多详细信息，请参见[Bitbucket支持](https://support.atlassian.com/bitbucket-cloud/docs/use-the-issue-tracker/)。

</Alert>

问题跟踪允许您从Sentry内部创建Bitbucket问题，并将Sentry问题链接到现有的Bitbucket问题。

一旦您导航到特定问题，您会在右侧面板中找到**链接问题**部分。在这里，您将能够创建或链接Bitbucket问题。

![将BitBucket问题链接到Sentry问题](./img/bitbucket-link-issue.png)

### 在提交中解决

一旦您正在发送提交数据，您就可以通过包含`fixes <SENTRY-SHORT-ID>`在提交消息中开始解决问题。例如，提交消息可能如下所示：

```
防止用户上的空查询

修复MYAPP-317
```

当Sentry看到这一点时，我们将自动用对提交的引用注释匹配的问题，稍后，当该提交是发布的一部分时，我们将问题标记为已解决。提交必须与发布相关联。否则，如果提交被压缩，Sentry将不知道何时发布了提交，问题可能永远不会被标记为回归。

### 堆栈跟踪链接

堆栈跟踪链接将您从Sentry堆栈跟踪中的文件带到源代码中的同一文件。如果您在Sentry中设置了提交跟踪，我们可以带您到源代码的确切版本（使用与事件相关联的提交）。否则，我们将链接到源代码的当前状态（使用默认分支）。

1. 导航到**设置 > 集成 > Bitbucket > 配置**。

1. 点击您的Bitbucket实例旁边的"配置"（齿轮图标）。

1. 点击"代码映射"选项卡。

1. 为您想要进行堆栈跟踪链接的每个项目设置代码映射。要创建新的代码映射，请点击"添加映射"。

1. 填写表单，然后点击"保存更改"。每个表单字段描述如下：

   - **项目**（必需）：这是Sentry项目。
   - **仓库**（必需）：这是与上述Sentry项目关联的Bitbucket项目。如果您每个Sentry项目使用多个Bitbucket项目，您将需要多个代码映射。
   - **分支**（必需）：这是您的代码的默认分支，如果您没有设置提交跟踪，我们将回退到此分支。
   - **堆栈跟踪根目录**和**源代码根目录**（可选）：有关确定这些值的信息，请参见下文。

#### 堆栈跟踪根目录和源代码根目录

<Alert>

以下信息仅对使用传统文件路径的平台有效。具有包名的平台需要额外步骤。对于**Java**，请阅读[源代码上下文页面](/platforms/java/source-context#setting-up-code-mappings)中的更多信息。

</Alert>

首先，导航到您希望映射的堆栈跟踪。找到一个**应用程序内**框架，该框架在框架右侧用气泡表示。文件名将显示为框架标题左侧的第一段文本。在此示例中，它是`src/main.py`。

<Include name="common-imgs/code-mappings-stacktrace" />

对于某些原生平台，堆栈跟踪看起来会有所不同。在这种情况下，您可以通过将鼠标悬停在文件名上来找到绝对路径。

![突出显示在UI中查找原生堆栈跟踪文件路径的位置](./img/code-mappings-stacktrace-native.png)

如果您不确定，可以通过点击事件标题中的`{}`按钮查看事件JSON。在框架的`filename`或`abs_path`中找到文本。

<Include name="common-imgs/code-mappings-event-json" />

接下来，在您的源代码提供程序（例如GitHub）中找到堆栈跟踪中看到的文件。在此示例中，路径是`flask/src/main.py`（`empower`被忽略，因为它是仓库的名称）。

<Include name="common-imgs/code-mappings-source-file" />

将堆栈跟踪中的文件路径与源代码库中找到的路径进行比较。在此示例中，堆栈跟踪中的`src/`文件夹与源代码中的`flask/src`文件夹匹配。使用该信息，将**堆栈跟踪根目录**设置为`src/`，将**源代码根目录**设置为`flask/src/`。这告诉Sentry在搜索源代码时将所有以`src/`开头的文件路径替换为`flask/src/`。

为获得最佳结果，我们建议尽可能始终为**堆栈跟踪根目录**提供非空值。

## 故障排除

如果您在设置Sentry与本地集成时遇到问题，请验证以下内容：

- 提供的安装URL是完全限定域名（FQDN），可在互联网上解析。
- Sentry用于发出出站请求的IP地址是[允许的](/security-legal-pii/security/ip-ranges/)。
- Sentry对您安装URL的访问不受路径限制。

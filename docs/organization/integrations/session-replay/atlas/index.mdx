Atlas 是一体化客户支持软件，旨在以简单的时间轴视图显示您的客户体验。轻松将 Atlas 会话链接到 Sentry 问题，并将 Sentry 问题链接到现有的 Atlas 会话。

此集成由创建它的公司维护和支持。更多详细信息，请查看我们的[集成平台文档](/organization/integrations/integration-platform/)。

## 安装和配置

<Alert>

需要 Sentry 所有者、管理员或管理员权限才能安装此集成。

Atlas **不**适用于自托管 Sentry。

</Alert>

1. 在 Atlas 中导航到**应用配置 > Sentry**

![从 Atlas 安装 Sentry 集成的流程](./img/atlas_install.png)

2. 按照说明从 Sentry 内部安装 Atlas 应用

## 会话链接

会话链接允许您将 Sentry 问题与 Atlas 中的会话关联，以便您的支持团队在协助客户时可以查看它们。

如果 Sentry 已识别用户，则会自动尝试会话链接。但是，如果有多个同时会话，则需要消除歧义。因此，最佳实践是始终从 Sentry 识别 Atlas 会话。

这可以通过首先从 Atlas 获取会话 ID 来完成，方法是从客户端调用以下内容：

```javascript
apiAxios.interceptors.request.use((config) => {
  config.headers["X-Atlas-Session-Id"] =
    window.Atlas?.recording?.getSessionId();
  return config;
});
```

您可以通过客户端 SDK 将其发送到 Sentry：

```javascript
Sentry.addEventProcessor((event) => {
  event.tags = {
    ...(event.tags || {}),
    ["atlasSupport.sessionId"]: window.Atlas?.recording?.getSessionId(),
  };
  return event;
});
```

或从服务器 SDK：

```python
sentry_sdk.set_tag("atlasSupport.sessionId", atlas_session_id)
```

以及在会话录制中：

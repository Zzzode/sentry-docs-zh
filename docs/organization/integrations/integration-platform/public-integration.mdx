公共集成面向 Sentry 用户的"大众"。这些集成最初以未发布状态用于开发目的，之后可以提交以供审核发布。更多信息，请查看此页面的[发布部分](#publication)。

下面各节中的代码示例展示了一个潜在的用例，涉及一个 Flask 应用程序接收来自 Sentry 的新问题 webhook，调用 Sentry API 获取有关问题的更多数据，并将其推送到 Pushover 作为桌面/移动通知生成器。

## 安装

用户将可以选择在 [sentry.io](https://sentry.io) 的**集成**页面上安装您的集成。如果您的集成仍处于未发布状态，唯一能够看到或安装它的 Sentry 组织将是创建该集成的组织。发布后，点击集成名称将允许用户查看集成的描述以及如果用户选择安装将授予的权限。

![用户可以安装的集成列表。](./img/integration-directory.png)

## OAuth 流程

安装后，如果您的用户已批准所有权限，Sentry 将生成一个授权代码和安装 ID。此信息通过 `installation.created` webhook 发送到您在配置中指定的 webhook URL。

但是，如果您的集成配置了重定向 URL，集成会将用户的浏览器重定向到配置的 URL，并在查询参数中包含授权代码和安装 ID。

通过实现重定向 URL 端点 `/setup` 开始构建您的集成 - 通常您在此处交换授权代码以获取允许您向 Sentry 进行 API 调用的令牌：

```python
import requests
from flask import redirect, request

@app.route('/setup', methods=['GET'])
def setup():
    code = request.args.get('code')
    install_id = request.args.get('installationId')

    url = u'https://sentry.io/api/0/sentry-app-installations/{}/authorizations/'
    url = url.format(install_id)

    payload = {
        'grant_type': 'authorization_code',
        'code': code,
        'client_id': 'your-client-id',
        'client_secret': 'your-client-secret',
    }

    resp = requests.post(url, json=payload)
    data = resp.json()

    token = data['token']
    refresh_token = data['refreshToken']
    # ... 安全地将 install_id、token 和 refresh_token 存储在数据库中 ...

    return redirect('https://sentry.io/settings/')
```

所有公共集成都可以通过固定外部 URL 安装：`https://sentry.io/sentry-apps/<your-integration-slug>/external-install/`，您可以将其暴露给用户。查看 `Linear` 的示例：https://sentry.io/sentry-apps/linear/external-install/。请注意，如果用户有多个组织，系统会要求他们从下拉菜单中选择要安装的组织。

## 认证令牌

公共集成在成功 [OAuth 安装](#oauth-process) 后生成[认证令牌](/organization/integrations/integration-platform/#auth-tokens)。
这些令牌每八小时自动过期一次，这意味着必须手动刷新它们。

### 刷新令牌

在初始安装中，您需要安装 webhook 请求或重定向 URL 中提供给您的授权代码，以及集成的客户端 ID 和客户端密钥：

```python
url = u'https://sentry.io/api/0/sentry-app-installations/{}/authorizations/'
url = url.format(install_id)

payload = {
    'grant_type': 'authorization_code',
    'code': code,
    'client_id': 'your-client-id',
    'client_secret': 'your-client-secret',
}
```

令牌在八小时后过期，因此您需要相应地刷新令牌：

```python
def refresh_token(install_id):
    url = u'https://sentry.io/api/0/sentry-app-installations/{}/authorizations/'
    url = url.format(install_id)

    refresh_token = retrieve_refresh_token_from_db(install_id)

    payload = {
        'grant_type': 'refresh_token',
        'refresh_token': refresh_token,
        'client_id': 'your-client-id',
        'client_secret': 'your-client-secret',
    }

    resp = requests.post(url, json=payload)
    data = resp.json()

    new_token = data['token']
    new_refresh_token = data['refreshToken']
    # ... 安全地在数据库中更新 token 和 refresh_token ...

    return new_token
```

对于初始授权代码交换和后续令牌刷新，您可以期望返回的数据如下：

```json

  "state": null,

```

## 验证安装

通常，如果您配置了重定向 URL，您的端会发生工作以"完成"安装。如果是这种情况，我们建议为您的集成启用"验证安装"选项。启用后，您需要发送一个请求将安装标记为正式"已安装"：

```python
requests.put(
    u'https://sentry.io/api/0/sentry-app-installations/{}/'.format(install_id),
    json={'status': 'installed'},
)
```

## 卸载

当用户卸载您的集成时，您将收到发送到您的 webhook URL 的 `installation.deleted` 请求。更多信息，请查看完整的 [Webhooks 文档](/organization/integrations/integration-platform/webhooks/)。

## 授权来源

如果您允许发出请求的页面来源，则可以从浏览器使用认证令牌。在"授权的 JavaScript 来源"字段中，添加每个来源（例如 docs.sentry.io），用换行符分隔。您不需要在来源中包含协议（http 或 https）。目前，您不能使用任何通配符（例如 `*.sentry.io`），因此如果您有多个子域，则需要单独添加它们。

## 图像要求

公共集成需要上传徽标图像。要求如下：

- 徽标大小必须在 256 × 256px 和 1024 × 1024px 之间
- 文件格式必须为 PNG
- 背景必须透明（除非徽标占据整个空间）

如果集成具有 [UI 组件](/organization/integrations/integration-platform/ui-components)，您还必须上传一个遵循这些要求的图标：

- 图标大小必须在 256 × 256px 和 1024 × 1024px 之间
- 文件必须是 RGBA 格式的 PNG
- 所有像素必须为黑色或透明

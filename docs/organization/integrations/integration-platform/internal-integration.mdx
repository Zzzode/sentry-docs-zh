内部集成适用于您组织的独特自定义集成。它们也可以简单到组织范围的令牌。无论您是仅使用 API 还是结合使用所有集成平台功能，内部集成都用于单个 Sentry 组织内。

内部集成不需要 OAuth 流程。创建后立即收到组织范围的认证令牌：

![显示公共和内部集成之间流程差异的图表。](./img/authentication-flow.png)

有关如何构建内部集成的示例，请参见[我们的轮询问题分配集成](https://blog.sentry.io/2019/11/21/customize-your-sentry-workflow-a-sample-internal-integration)（或直接跳到[GitHub 上的代码](https://github.com/getsentry/sentry-round-robin)）。

## 安装

创建内部集成将自动在您的组织上安装它。

## 认证令牌

内部集成在配置时自动生成[认证令牌](/organization/integrations/integration-platform/#auth-tokens)。如果您需要多个令牌，或需要获取新令牌，您可以转到 **设置 > 开发者设置 > [您的内部集成]** 并执行此操作。
每个内部集成最多可以有 20 个令牌。
这些令牌不会自动过期，但您可以根据需要手动撤销它们。

![显示复制令牌、创建新令牌、撤销令牌以及它们在 UTC 中创建时间能力的图像。](./img/tokens.png)

## 图像要求

内部集成不需要上传徽标或图标，但如果您愿意，可以上传它们。要求如下：

- 徽标大小必须在 256 × 256px 和 1024 × 1024px 之间
- 文件格式必须为 PNG
- 背景必须透明（除非徽标占据整个空间）

如果集成具有 [UI 组件](/organization/integrations/integration-platform/ui-components)，您还可以上传遵循这些要求的图标：

- 图标大小必须在 256 × 256px 和 1024 × 1024px 之间
- 文件必须是 RGBA 格式的 PNG
- 所有像素必须是黑色或透明

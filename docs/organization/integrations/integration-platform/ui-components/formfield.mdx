[问题链接](/organization/integrations/integration-platform/ui-components/issue-link)和[警报规则操作](/organization/integrations/integration-platform/ui-components/alert-rule-action)UI 组件可以使用 `FormField` 组件在其各自的模态框中显示属性。这些组件可以根据下面的方案指定输入来渲染自定义表单。它们的行为类似于其 HTML 等效项。

## 文本

### 架构

```json {filename:schema.json}
{
  "type": "text",
  "label": <String>,
  "name": <String>,
  "default": <String>,
}
```

### 属性

- `label` -（必需）要为表单字段渲染的标签文本
- `name` -（必需）要在字段的 `name` 属性中使用的值
- `default` - 要预填充的默认值。选项包括 `issue.title` 和 `issue.description`。仅用于[问题链接](/organization/integrations/integration-platform/ui-components/issue-link)组件。
  - `issue.title` - Sentry 问题的标题
  - `issue.description` - Sentry 问题的描述

## 文本区域

### 架构

```json {filename:schema.json}
{
  "type": "textarea",
  "label": <String>,
  "name": <String>,
  "default": <String>,
}
```

### 属性

- `label` -（必需）要为表单字段渲染的标签文本
- `name` -（必需）要在字段的 `name` 属性中使用的值
- `default` - 要预填充的默认值，选项包括 `issue.title` 和 `issue.description`。仅用于[问题链接](/organization/integrations/integration-platform/ui-components/issue-link)组件。
  - `issue.title` - Sentry 问题的标题
  - `issue.description` - Sentry 问题的描述

## 选择

### 架构

```json {filename:schema.json}
{
  "type": "select",
  "label": <String>,
  "name": <String>,
  "uri": <URI>,
  "async": <Boolean>,
  "options": <Array<Array<String, String>>>,
  "depends_on": <Array<String>>,
  "skip_load_on_open": <Boolean>
}
```

### 属性

- `label` -（必需）要为表单字段渲染的标签文本
- `name` -（必需）要在字段的 `name` 属性中使用的值
- `uri` -（如果开发者不提供 `options` 则为必需）从中检索值的 URI。查看我们的 [URI 指南](/organization/integrations/integration-platform/ui-components/#uri-guidelines)文档以获取格式帮助。
- `async` -（可选）仅在存在 `uri` 时使用。如果为 true（默认），将在用户键入时查询 URI 以获取自动完成建议（参见下面的响应格式）。如果为 false，将最初查询 URI 一次以检索所有可能的选项。此请求 _必须_ 成功，响应 _必须_ 以 Sentry 期望的格式返回数据，否则整个组件将不会渲染。
- `options` -（如果开发者不提供 `uri` 则为必需）格式为 `[["value1", "label1"], ["value2", "label2"]]` 的静态选项列表
- `depends_on` -（可选）如果字段值依赖于另一个字段的值，则当依赖字段设置时将发出请求以加载这些选项。
- `skip_load_on_open`（可选）如果为 true，此字段在页面加载时不会加载选项。相反，它将等待用户开始键入后再获取选项。

### URI 请求格式

发送到 `uri` 以加载选项的请求是 `GET` 请求。参数编码到查询参数中，如下所示：

```json
{
  "installationId": <String>,
  "projectSlug": <String>,
  "query": <String>,
  "dependentData": <String>,
}
```

- `installationId` - 与请求关联的安装的 ID
- `projectSlug` - Sentry 中项目的 slug
- `query` - 用户正在键入的搜索词
- `dependentData` - 如果该字段设置了 `depends_on`，则为依赖数据的 JSON 编码字符串

### URI 响应格式

当指定 `uri` 时，来自 `uri` 的响应必须是 JSON 并匹配以下格式：

```json
[
  {
    "label": <String>,
    "value": <String>,
    "default": <Boolean>,
  },
  ...
]
```

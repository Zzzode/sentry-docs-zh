Sentry 集成平台为开发者提供了通过基于 JSON 的架构在 [sentry.io](https://sentry.io) 的特定部分中定义要渲染的组件的能力。

您可以定义以下组件：

- [问题链接](/organization/integrations/integration-platform/ui-components/issue-link)（使用 [表单字段](/organization/integrations/integration-platform/ui-components/formfield) 组件）
- [警报规则操作](/organization/integrations/integration-platform/ui-components/alert-rule-action)（使用 [表单字段](/organization/integrations/integration-platform/ui-components/formfield) 组件）
- [堆栈跟踪链接](/organization/integrations/integration-platform/ui-components/stacktrace-link)

## 架构

UI 组件在集成详情的架构部分中指定。此架构的必需形式是：

```json {filename: schema.json}
{
  "elements": [
    // 集成要使用的组件架构
    // 例如 { "type": "alert-rule-action", ... }
  ]
}
```

### URI 指南

架构元素中的 `uri` 键描述了将接收来自该 UI 组件请求的 URI 路径。Sentry 会将架构的 `uri` 字段中的路径附加到集成的 _webhook URL_ 的主机上。
例如，如果 webhook URL 是 `https://example.com/webhook/`，而您的架构的 `uri` 属性是 `/alerts/`，那么集成将接收对 `https://example.com/alerts/` 的请求。

## 错误处理

组件渲染要么 100% 工作，要么什么都不显示。为了保护集成免受我们无法控制的错误导致的外观混乱，如果第三方组件渲染的任何部分失败，将不会渲染任何内容。

如果问题链接组件中存在错误，链接将被禁用。

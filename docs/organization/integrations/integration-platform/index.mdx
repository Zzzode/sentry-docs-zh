Sentry 的集成平台提供了一种方式，让外部服务可以使用 [webhooks](/organization/integrations/integration-platform/webhooks/)、[UI 组件](/organization/integrations/integration-platform/ui-components/)和 [REST API](/api) 与 Sentry 交互。使用此平台的集成是 Sentry 中的一流参与者。

## 示例应用

如果您是集成平台的新手，从示例项目开始可能会有所帮助。为了帮助您，我们构建了一个[示例应用](https://github.com/getsentry/integration-platform-example)，您可以将其用作起点。它演示了您的应用程序可用的不同功能和数据：

![为展示集成平台功能而构建的演示看板应用用户界面。](./img/reference-implementation-frontend.png)

它是一个看板应用，使用了 Sentry 集成可用的许多功能，并提供 Python 和 TypeScript 的起始代码。它还包括设置第一个集成的分步指南，以及文档和测试建议。通常，该应用应在几分钟内安全地使用实时 Sentry 数据。

## 创建集成

在 [sentry.io](https://sentry.io) 中，导航到 **设置 > 开发者设置**。从这里，您可以选择创建[内部](/organization/integrations/integration-platform/internal-integration)或[公共](/organization/integrations/integration-platform/public-integration)集成。内部集成只能由您的组织使用，而公共集成可以发布并可供所有 Sentry 用户使用。

![允许开发者选择公共或内部集成的设置页面。](./img/dev-settings.png)

您的集成有几个配置选项，下面简要描述。每个部分都提供了指向更详细信息的链接。

## Webhooks

Webhooks 允许您的服务根据您的选择获取有关特定资源的请求。有关更多信息，请查看[完整的 Webhooks 文档](/organization/integrations/integration-platform/webhooks/)。

为了接收 webhook 事件，您必须在创建集成时指定 webhook URL。指定 webhook URL 后，您将能够打开"警报规则操作"并创建发送通知到您的集成的问题和指标警报。

## UI 组件

Sentry 集成平台能够通过简单、声明式的语法向 [sentry.io](https://sentry.io) 本身添加丰富的 UI 组件。有关更多信息，请查看[完整的 UI 组件文档](/organization/integrations/integration-platform/ui-components/)。

## 警报

您可以通过启用"警报规则操作"切换，使任何集成在[问题警报](/organization/integrations/integration-platform/webhooks/issue-alerts)和[指标警报](/organization/integrations/integration-platform/webhooks/metric-alerts)中作为操作可用。然后，集成将在创建或更新警报规则时的操作部分中显示为服务。下面的交互式演示展示了如何设置可以接收 Sentry 警报的集成。

<Arcade src="https://demo.arcade.software/v7uhzmdV6Q5PDzoVPAE6?embed&show_copy_link=true" />

为了让您的服务接收警报规则的 webhooks，您必须在规则中将`通过 <您的集成> 发送通知`作为操作。设置完成后，您将开始接收触发警报的 webhook 请求。有关请求和有效负载的更多信息，请查看[完整的 Webhooks 文档](/organization/integrations/integration-platform/webhooks/)。如果您希望用户提供更多特定于应用的信息来处理这些 webhooks（例如受让人、团队、通知频道），请考虑实现[警报规则操作 UI 组件](/organization/integrations/integration-platform/ui-components/alert-rule-action)。

## 权限

权限指定您的服务需要 Sentry 资源的访问级别。范围直接应用于颁发给应用的任何 API 令牌。对于公共集成，Sentry 将在安装时提示用户批准这些权限。有关更多信息，请查看[完整的权限文档](/api/permissions/)。

![允许开发者设置他们需要用户哪些权限的表单。](./img/permissions.png)

您无法创建权限大于您自己用户设置的集成或更改现有集成。权限按组织范围，因此为问题和事件、项目或团队添加范围将启用对整个组织的问题和事件、项目或团队的访问。

## 认证令牌

认证令牌使用认证头传递，并用于使用 API 以用户账户身份进行身份验证。[公共集成](/organization/integrations/integration-platform/public-integration)需要 OAuth 流来获取令牌。[内部集成](/organization/integrations/integration-platform/internal-integration)在安装后自动生成令牌。有关更多信息，请查看[完整的认证文档](/api/auth/)。

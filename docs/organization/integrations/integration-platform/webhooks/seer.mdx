已订阅 Seer webhooks 的 Sentry 集成可以接收有关 Seer [问题修复](/product/ai-in-sentry/seer/issue-fix/)流程的通知。这些 webhooks 允许您跟踪自动化根本原因分析、解决方案生成和代码修复的进度。

## Sentry-Hook-Resource 标头

`'Sentry-Hook-Resource': 'seer'`

## Webhook 类型

Seer webhooks 支持七种不同的事件类型，跟踪 AI 驱动问题解决的完整生命周期：

### 根本原因分析
- `seer.root_cause_started` - 当根本原因分析开始时触发
- `seer.root_cause_completed` - 当根本原因分析完成并带有结果时触发

### 解决方案生成
- `seer.solution_started` - 当解决方案生成开始时触发
- `seer.solution_completed` - 当解决方案已生成时触发

### 代码生成
- `seer.coding_started` - 当代码生成/修复开始时触发
- `seer.coding_completed` - 当代码更改准备就绪时触发
- `seer.pr_created` - 当拉取请求被创建时触发

## 通用属性

所有 Seer webhooks 共享这些通用属性：

### action

- 类型：字符串
- 描述：发生的特定 Seer 事件（例如，`root_cause_started`、`solution_completed`）

### data['run_id']

- 类型：整数
- 描述：此 Seer 分析运行的唯一标识符

### data['group_id']

- 类型：整数
- 描述：正在分析的 Sentry 问题 ID

## 用例

Seer webhooks 使您能够：

1. **跟踪进度**：实时监控 Seer 问题修复的状态
2. **构建通知**：在修复准备就绪时发送自定义通知
3. **审计跟踪**：维护所有 Seer 修复问题的日志
4. **自定义集成**：构建响应 Seer 分析结果的自定义工具

## 最佳实践

- 存储 `run_id` 以关联相关的 webhook 事件
- 使用 `group_id` 将 webhooks 链接回原始 Sentry 问题

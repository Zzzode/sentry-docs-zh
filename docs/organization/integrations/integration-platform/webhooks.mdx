Webhooks 是两个应用程序或服务通过网络进行通信的一种方式，允许一个应用程序向另一个应用程序发送自动通知或数据更新，几乎是实时的。

该概念基于 HTTP 回调，其中 HTTP POST 请求被发送到特定 URL（您技术栈中的 HTTP 端点），并带有有效负载。Sentry 使用 JSON webhooks 将错误通知发送到您的系统或服务。

## 标头

Sentry webhooks 支持各种功能，由下面描述的四个基本 HTTP 标头组成：

```json
{
  "Content-Type": "application/json",
  "Request-ID": "<request_uuid>",
  "Sentry-Hook-Resource": "<resource>",
  "Sentry-Hook-Timestamp": "<timestamp>",
  "Sentry-Hook-Signature": "<generated_signature>"
}
```

`Content-Type` 标头标识有效负载的媒体类型为 JSON 格式。`Request-ID` 标头为跟踪和调试特定事件提供唯一标识符。

### `Sentry-Hook-Resource`

此标头让您知道下面列表中的哪个资源触发了操作：

- `installation`
- `event_alert`
- `issue`
- `metric_alert`
- `error`
- `comment`
- `seer`

### `Sentry-Hook-Signature`

此标头表示由您的 _客户端密钥_ 生成的加密哈希。其主要目的是确保请求是真实的，并且来自 Sentry 服务器。

**验证签名**

下面的代码片段让您可以使用事件有效负载验证签名。

```javascript
const crypto = require("crypto");

function verifySignature(request, secret = "") {
  const hmac = crypto.createHmac("sha256", secret);
  hmac.update(JSON.stringify(request.body), "utf8");
  const digest = hmac.digest("hex");
  return digest === request.headers["sentry-hook-signature"];
}
```

```python
import hashlib
import hmac
import json

expected_digest = request.headers.get('sentry-hook-signature')  # 如果标头缺失则返回 None
body = json.dumps(request.body)

digest = hmac.new(
    key=client_secret.encode('utf-8'),
    msg=body,
    digestmod=hashlib.sha256,
).hexdigest()

if not expected_digest:  # 签名缺失
    raise UnauthorizedError

if not hmac.compare_digest(digest, expected_digest):
    raise UnauthorizedError
```

## 请求结构

所有 webhook 请求都有一些共同元素。

`action`

与标头中的[资源](#sentry-hook-resource)对应的操作。例如，如果资源是 `issue`，操作可能是 `created`。

`installation`

一个带有安装 `uuid` 的对象，以便您可以将 webhook 请求映射到适当的安装。

`data`

data 对象包含有关[资源](#sentry-hook-resource)的信息，内容将根据 webhook 的类型而有所不同。此有效负载可能能够通过 [UI 组件](/organization/integrations/integration-platform/ui-components)进行自定义。

`actor`

actor 是触发 webhook 的人（如果有）。如果 Sentry 中的用户触发了操作，actor 就是用户。如果 Sentry 集成本身触发操作，actor 就是集成。如果操作在 Sentry 中自动触发，actor 就是"Sentry"。

```python
# 示例情况：

# 用户安装 Sentry 集成
"actor": {
    "type": "user",
    "id": <user-id>,
    "name": <user-name>,
}

# Sentry 集成发出请求分配问题
"actor": {
    "type": "application",
    "id": <sentry-app-uuid>,
    "name": <sentry-app-name>,
}

# Sentry (sentry.io) 自动解决问题
"actor": {
    "type": "application",
    "id": "sentry",
    "name": "Sentry",
}
```

## 响应

Webhooks 应在 1 秒内响应。否则，响应被视为超时。

## 开发和测试 Webhooks

如果您想在开始开发之前测试 webhook 配置并查看有效负载，提供指定 URL 供您接收 HTTP 有效负载并检查 JSON 事件有效负载的 HTTP 通配符服务会派上用场。查看相关事件有效负载后，您可以开始开发。

为了使测试和调试 webhooks 更快更容易，您可以创建本地隧道，将来自 Sentry 的传入 webhook 请求获取到您的本地机器。有各种工具可以让您选择端口或地址（如 3000 或 8080）进行隧道传输，然后为您提供将请求转发到本地服务器的临时公共 URL。

<Alert>

Splunk集成已弃用，无法在新项目中安装。

</Alert>

使用[数据转发](/concepts/data-management/data-forwarding/)功能将Splunk连接到Sentry。

<Alert>

我们仅支持Splunk Cloud方案。我们不支持Splunk Enterprise方案。有关您的Splunk安装的详细信息，请查看[Splunk文档](https://dev.splunk.com/)。

</Alert>

此集成需要为您希望使用它的每个项目进行设置。它由[Sentry社区](https://open.sentry.io/)维护和支持。

## 安装和配置

<Alert>

需要Sentry所有者、管理员或管理员权限才能安装此集成。

</Alert>

导航到**设置 > 集成 > Splunk**

![安装Splunk集成](./img/splunk.png)

### 启用HEC

首先，您需要首先启用HTTP事件收集器：

在**设置**下，选择**数据输入**：

![Splunk设置](./img/splunk-settings.png)

在本地输入下选择**HTTP事件收集器**：

![Splunk数据输入](./img/splunk-data-inputs.png)

在您的HEC设置下，点击"全局设置"：

![Splunk HEC输入](./img/splunk-hec-inputs.png)

将**所有令牌**更改为**已启用**，并记下HTTP端口号（默认为`8088`）：

![Splunk全局设置](./img/splunk-hec-global-settings.png)

<Alert>

如果您在特权环境中运行Splunk，您可能需要公开HEC端口。

</Alert>

### 创建Sentry输入

在HTTP事件收集器下，通过点击"新令牌"创建新的Sentry输入：

![Splunk Sentry输入](./img/splunk-new-http-input.png)

输入名称（例如`Sentry`），然后点击"下一步"：

![在Splunk中输入名称](./img/splunk-new-input-name.png)

选择您希望使其可访问的索引（例如`main`），然后点击"审查"：

![在Splunk中选择索引](./img/splunk-new-input-index.png)

系统将提示您审查输入详细信息。点击"提交"继续：

![提交前审查表单](./img/splunk-new-input-review.png)

输入现已创建，您应该会看到**令牌值**：

![令牌已在Splunk中成功创建](./img/splunk-new-input-final.png)

### 启用Splunk转发

要启用Splunk转发，您需要以下内容：

- 您的实例URL（见下面的说明）
- Sentry HEC令牌值

在Sentry中，导航到您要转发事件的项目，然后点击"项目设置"。

### 数据转发

在**[项目] > 设置 > 数据转发**中配置[数据转发](/concepts/data-management/data-forwarding/)，并为给定集成提供所需信息。

导航到**数据转发**后，启用Splunk集成：

![数据转发设置页面](./img/splunk-data-forwarding-setting.png)

您的实例URL将根据您使用的Splunk服务类型而有所不同。如果您使用的是自助服务Splunk Cloud，实例URL将是运行HTTP事件收集器（HEC）的Splunk实例的URL：

```
https://<host>:8088
```

对于所有其他Splunk Cloud方案，您将使用`http-inputs`前缀：

```
https://http-inputs-<host>:8088
```

如果您在防火墙后使用Splunk，您需要填写适当的主机。

填写完所需字段后，点击**保存更改**：

![Splunk设置表单](./img/splunk-data-forwarding-setting-complete.png)

我们现在将开始将所有新事件转发到您的Splunk实例。

<Alert>

Sentry将在内部限制发送到您的Splunk实例的最大事件数量为每秒1000个。

</Alert>

![显示Sentry数据的Splunk页面](./img/splunk-search-sentry.png)

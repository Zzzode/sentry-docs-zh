## 安装

<Alert>

需要 Sentry 所有者、管理员或管理员权限以及 Jira 管理员权限才能安装此集成。这包括以下权限：`read`、`write`、`act_as_user` 和 `access_email_addresses`。

</Alert>

1. 导航到**设置 > 集成 > Jira**。

![安装 Jira 集成](./img/jira-overview.png)

2. 通过 [Jira 市场](https://marketplace.atlassian.com/apps/1219432/sentry-for-jira?hosting=cloud&tab=overview) 安装 Sentry 应用。

3. 如果您启用了广告拦截器，请现在禁用它，否则您将无法完成第 4 步。

4. 选择您希望与 Jira 一起使用的 Sentry 组织。然后点击呈现的链接返回 Sentry。

5. 在 Sentry 中，您将在集成页面上看到一个新的 Jira 实例出现。

   ![Jira 集成目录](./img/jira-install.png)

Jira 现在应该已授权用于您 Sentry 组织下的所有项目。

### Jira Server/Data Center

<Alert>

截至 2024 年 2 月，Jira 已停止对 Jira Server 的支持。因此，我们已停止维护集成。您仍然可以将其与后续产品 Jira Data Center 以相同方式使用。

</Alert>

Sentry 的 Jira Server 集成是使用 [应用链接](https://confluence.atlassian.com/adminjiraserver/link-to-other-applications-938846918.html) 构建的，应该也能够与 Jira Data Center 集成。

#### 生成 RSA 公钥/私钥对

要生成 RSA 公钥/私钥对，请在您的终端窗口中逐个运行以下命令。

```bash
openssl genrsa -out jira_privatekey.pem 1024
openssl req -newkey rsa:1024 -x509 -key jira_privatekey.pem -out jira_publickey.cer -days 365
openssl pkcs8 -topk8 -nocrypt -in jira_privatekey.pem -out jira_privatekey.pcks8
openssl x509 -pubkey -noout -in jira_publickey.cer  > jira_publickey.pem
```

#### 将您的 Jira Server 应用与 Sentry 连接

<Alert>

确认 [Sentry 的 IP 范围](/security-legal-pii/security/ip-ranges/) 已允许。

</Alert>

1. 导航到**组织设置 > 集成**。
2. 在 Jira Server 旁边，点击"安装"。
3. 在结果模态框中，点击"添加安装"。
4. 在结果窗口中，输入您的 Jira Server 实例的基本 URL、您的消费者密钥和您的私钥。点击"提交"。然后，按提示完成 OAuth 流程。
5. 在 Sentry 中，您将在集成页面上看到一个新的 Jira Server 实例出现。

Jira 现在应该已授权用于您 Sentry 组织下的所有项目。

### 自托管 Sentry + Jira 集成

要使用自托管 Sentry 实例安装 Jira 集成，请遵循我们的 [develop-docs](https://develop.sentry.dev/integrations/jira/)，跳转到"安装本地 Jira 应用"部分。

## 配置

使用 Jira 利用[问题管理](#问题管理)、[问题同步](#问题同步)并接收有关问题状态更改的[通知](#问题通知)。此外，添加[忽略字段](#忽略字段)以在问题创建表单中隐藏指定字段。

### 问题管理

问题跟踪允许您从 Sentry 内创建 Jira 问题，并将 Sentry 问题链接到现有的 Jira 问题。

<Alert title="注意">
  <p>
    手动问题管理适用于团队、商业和
    企业计划的组织。
  </p>
  <p>
    自动问题管理适用于商业和企业计划的组织。
  </p>
</Alert>

问题管理可以通过两种方式配置 - 自动或手动。

#### 自动

要自动配置问题管理，请创建[**问题警报**](/product/alerts-notifications/issue-alerts/)。在选择[**操作**](/product/alerts/create-alerts/issue-alert-config/#then-conditions-actions)时，选择**创建新的 Jira 问题**。

![创建 Jira 工单警报规则操作](./img/jira-ticket-rule.png)

当警报被触发时将自动创建 Jira 问题。

#### 手动

要手动配置问题管理，一旦您导航到特定的 Sentry 问题，您将在右侧面板中找到**链接问题**部分。

![从 Sentry 问题链接 Jira 问题](./img/jira-link-issue.png)

在这里，您将能够创建或链接 Jira 问题。

![创建 Jira 问题表单](./img/jira-create-issue.png)

### 问题同步

为了最小化问题跟踪中的重复，您可以将 Sentry 问题的评论、分配人和状态更新同步到 Jira 工单，反之亦然。当您将问题委派给分配人或更新 Jira 上的状态时，更新也将填充到 Sentry 中。当您在 Sentry 中解决问题时，问题状态将自动在 Jira 中更新。当 Jira 工单被标记为完成时，您可以配置 Sentry 中相应问题的解决方法。

<Alert title="注意">
  问题同步适用于团队、商业和
  企业计划的组织。
</Alert>

要配置问题同步，请导航到**组织设置** > **集成**，然后点击您的 Jira Server 实例旁边的"配置"。在以下页面上，您可以配置您希望如何在 Sentry 和 Jira 之间同步信息。

![将 Jira 状态同步到 Sentry](./img/jira-sync.png)

<Alert title="注意">

如果您在设置 Jira Server 集成期间或之后遇到 4xx 或 5xx 错误，请参见此[故障排除部分](#接收-jira-server-集成的-4xx5xx-错误)。

</Alert>

### 问题通知

Sentry 中的警报通知可以路由到许多支持的集成，但默认情况下是针对电子邮件的。您需要配置项目的[**警报规则**](/product/alerts-notifications/alerts/)以正确将通知路由到特定集成。

### 忽略字段

忽略字段是在问题创建表单中隐藏的 Jira 字段。要添加忽略字段，请导航到**设置 > 集成**并点击您的 Jira 实例旁边的"配置"。然后向下滚动到"忽略字段"并输入 Jira 字段 ID 的逗号分隔列表。

字段 ID 与问题创建表单上的标签不同。如果您需要帮助查找字段的 ID，请查看 Atlassian 文档[此处](https://confluence.atlassian.com/jirakb/how-to-find-id-for-custom-field-s-744522503.html)。

## 故障排除

### 自托管 Sentry 安装问题

如果您在设置自托管 Sentry 时遇到问题，请验证以下内容：

- 提供的安装 URL 是完全限定域名 (FQDN)，可在互联网上解析。
- Sentry 用于发出出站请求的 IP 地址是[允许的](/security-legal-pii/security/ip-ranges/)。
- Sentry 对您的安装 URL 的访问不受路径限制。

### SaaS sentry.io 安装上的"出现问题"消息与 JIRA

![出现问题错误消息](./img/jira-something-went-wrong.png)

此问题通常发生在您在设置期间有广告拦截器阻止 JIRA 和 Sentry 之间的对话时。要解决此问题，请禁用您的广告拦截器并再次完成安装流程。

### 无法使用自定义必填字段创建问题

我们不支持 Jira 集成的自定义必填字段。我们唯一支持的必填字段是我们预先填充的字段。如果可能，您可以在 Jira 中编辑您的必填字段，或者作为解决方法，您可以先在 Jira 中创建工单，然后在 Sentry 中链接它。

### 问题警报未创建 Jira 问题

当您的警报选择的优先级选项对于给定项目不存在时，会出现此问题。如果您的项目配置包含"优先级"字段，下拉菜单将显示**所有**项目中的每个优先级。确保您的警报保存的优先级在您选择的项目上可用。

### 接收 Jira Server 集成的 4xx/5xx 错误

`400` 错误通常是因为 Jira 实例无法公开访问。确保您的实例是可路由的（也就是说，我们的服务器可以到达它！）。如果不是，您需要将您的 Jira 实例放在互联网上，以便我们可以访问它。

对于 `500` 错误，不幸的是原因可能各不相同。此错误意味着我们无法从我们的服务器连接到您的服务器。例如，也许您的代理配置不正确，或者未设置为允许我们的 IP。如果您需要允许我们的 IP，您可以在[此处](/security-legal-pii/security/ip-ranges/)找到我们的 IP 范围列表。

### 使用报告人字段时无法找到特定用户

确保用户已添加到 Jira 中的相关项目，并具有[可分配用户权限](https://confluence.atlassian.com/jiracoreserver/permissions-overview-939937977.html)。
